Artifact: Action Control Document (ENG 313)
The document that describes the actions and links them to their related issues or to other events requiring action in order to support determination, launching, and tracking of an action.
Purpose

The purpose of the Action Control Document is to support determination, launching, tracking, and effectiveness review, if any, for an action.

Description
Main Description

An action is any unanticipated work that must be assigned and tracked by the project manager.

The Action Control Document describes the actions and links them to their related issues or to other events requiring action tracking. Information in this work product should not duplicate information recorded elsewhere, such as in an Issue Document.

Brief Outline

The Action Control Document contains these components:

  • Action summary
  • Action description
  • Effectiveness review plan
  • Action history

Key Considerations

The action is opened, assigned, and launched.

It is then followed up until completion.

When the action is completed:

  • If no effectiveness review is required, the status is changed to “closed.”
  • If effectiveness review is required, the status is changed to “completed.” Once the effectiveness has been reviewed, the status is changed to “closed.”
Tailoring
Impact of not having

The consequence of not using the Action Control Document is:

  • Lack of clarity about the contents of actions
  • Difficulty in assessing progress and effectiveness of actions.
Reasons for not needing

The Action Control Document might not be necessary for simple actions, where actions are determined and launched simultaneously and usually involve only one team member. In such a case the action is simply recorded in the Action Log, or other documents such as meeting minutes or status reports.
