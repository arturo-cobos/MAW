Artifact: Subcontractor Status Report (ENG 357)
A report prepared by the subcontractor which addresses the status of the sub-project for which the subcontractor is responsible. The report is provided to the project manager. The format is identical to that of the project status report.
Domains: Project Management
Work Product Kinds: Record
Purpose
The purpose of the Subcontractor Status Report is to provide the responsible IBM project manager with an accurate assessment of the current state of the subcontractor's project or sub-project. The responsible IBM project manager is typically the top-level project manager, but, depending on the importance of the sub-project to the overall project, the project manager may delegate responsibility to another project manager within the Organizational Breakdown Structure (OBS).

Description
Main DescriptionThe Subcontractor Status Report is a scheduled report that is created by the subcontractor and addresses the sub-project for which the subcontractor is responsible. The report provides information about the progress that the subcontractor has made during the reporting period.
Brief Outline

The basic format and frequency of the Subcontractor Status Report should be defined in the supplier Agreement.

The Subcontractor Status Report contains the following components:

  • Reporting period
  • Project status summary
  • Resources
  • Costs
  • Earned Value
  • Project accomplishments
  • Planned accomplishments for next period
  • Quality status
  • Issues
  • Risks
  • Changes
  • Compliance Incidents

Key Considerations

The Subcontractor Status Report is written by the subcontractor's project manager, according to format and frequency specified in the agreement and the subcontractor's project procedures description, and then reviewed by the responsible project manager from the delivery organization.

The accuracy and completeness each Subcontractor Status Report is validated at progress review meeting held between the project managers of the two parties shortly after the report is distributed.

Tailoring
Impact of not having

Lack of current and accurate information about the status of subcontractor's sub-project can mean that the project manager does not discover that the subcontractor is going to fail to meet their obligations until it is too late to take effective corrective action.

Reasons for not needing

It is rare that projects with subcontractors do not require Subcontractor Status Reports. Examples of projects that may not need a Subcontractor Status Report include:

  • Projects that are of short (less than one month) duration
  • Projects below a cost threshold
