Artifact: Sub-Project Team Status Report (ENG 356)
A report created by the project manager or team leader for each sub-project or team and provided to the top-level project manager. The format is the same as that of the Project Status Report.
Domains: Project Management
Work Product Kinds: Record
Purpose

The purpose of the Sub-Project/Team Status Report is to alert the next-level project manager to any variances or problems so that corrective actions can be initiated. It is also used by the next level project manager to create his or her own Sub-Project/Team Status Report. Where the next level manager is the top-level project manager, it is used to create the overall Project Status Report that is presented to the line management and the sponsor.

Description
Main Description

A Sub-Project/Team Status Report is a scheduled report that is created by the project manager or team leader for each sub-project or team within the project organizational unit that he or she manages. The Sub-Project/Team Status Report provides information about the progress that the sub-project or team has made relative to its Project Management Schedule during the reporting period.

The Sub-Project/Team Status Report provides an accurate and objective view of the status of each sub-project or team within the responsible project organizational unit. It includes an account of the work completed vs. work planned, costs expended vs. planned, a forecast of the work remaining, and issues and risks which may jeopardize successful completion of the sub-project.

The format and content of the Sub-Project/Team Status Report should be reflected in the Project Procedures Description.

Brief Outline

The Sub-Project/Team Status Report is a text-based document including the following information:

  • Reporting period
  • Sub-project status summary
  • Sub-project resource summary
  • Sub-project cost summary
  • Sub-project earned value
  • Sub-project accomplishments
  • Planned accomplishments for next period
  • Quality status
  • Issues
  • Risks
  • Changes
  • Actions

Key Considerations

The Sub-Project/Team Status Report is created by the team leader or sub-project project manager. The sub-project manager/team leader follows the procedures and any business unit policies as described in the Project Management System Summary or the Project Procedures Description in preparing the report. The report is created by aggregating Individual Status Reports into the Sub-Project/Team Status Report. Augmentation with graphics is helpful in showing progress or potential problems.

Sub-Project/Team Status Reports are validated by the project manager and team status reports are validated by the sub-project or the project manager.

Tailoring
Impact of not having

Lack of current and accurate information about each sub-project makes it difficult for the next-level project manager to make informed decisions and take appropriate corrective actions.

In addition, lack of Sub-Project/Team Status Reports can lead to inaccurate reporting about the state of the overall project to the sponsoring organization and the line management.

Reasons for not needing

The policies of the delivery organization and those of the project manager typically define when Sub-Project/Team Status Reports are not required. Situations in which Sub-Project Status Reports are not required include:

  • Projects with no sub-projects
  • Projects with sub-projects containing a small number of resources
