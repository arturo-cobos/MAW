Artifact: Event Log (ENG 402)
The list of events that, in the project manager’s opinion, are important to remember for later reference or analysis.

This artifact serves as a summary that groups/consolidates the different PM logs into a single artifact for use on small/agile projects.
Domains: Project Management
Work Product Kinds: Record
Purpose

The Event Log is intended to:

  • Shed light on situations involving project objectives, and therefore help the project manager to deal with them.
  • Provide background for issue identification and resolution.
  • Facilitate assessment of “lessons learned” by providing historical data.
  • Help project managers at any level demonstrate how they carry out their responsibilities when confronting day-to-day project situations.

The project manager uses this artifact in place of multiple artifacts to capture, plan, staff, track, and close various events which supports the agility of the team by reducing the number of work products and the level of detail within those work products while maintaining an audit trail.

Description
Main Description

The Event Log is a list of events that, in the project manager’s opinion, are important to remember for later reference or analysis.

It is not intended to provide a record of events that are documented in other work products, such as a Compliance Incident Document. However, events described in the Event Log might, upon further examination, reveal an issue, a compliance incident, or other type of event.

The Event Log provides one view of the history of the project. It can be an important input to daily project management decisions and is especially useful at the end of the project as input to the “lessons learned” meeting.

This artifact combines several project management logs used for exceptions, events, and staffing, into a single document.

Exceptions and other events are collected and managed in this artifact to resolve exceptions and handle events with minimum impact to the project.

Key Considerations

Events are recorded in chronological sequence on a day-to-day basis.

Verification that the Event Log is relevant and up-to-date should be done periodically. Select widely known events or facts and verify that they have been recorded.

Use Event (RAI) Log - for creating an Event Log comprising of Risk, Issue, Action logs with measurement details.

Tailoring
Impact of not having

When events are not recorded:

  • Part of the factual data useful for handling issues will be missing.
  • Assessment of “lessons learned” may become very subjective.
  • Incorrect judgments might be made about the management of the project because of a lack of information about certain events.
Reasons for not needingIn a simple project, the project manager might choose to use only the Project Status Report to record the types of events usually listed in the Event Log. When all facts are reported through project status reports, the Event Log is not necessary. However, this is true only on simple projects.
