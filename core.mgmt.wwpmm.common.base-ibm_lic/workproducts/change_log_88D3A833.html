Artifact: Change Log (ENG 317)
The list of all changes, the owner, due date, and status.
Domains: Project Management
Work Product Kinds: Record
Purpose

The purpose of the Change Log is to provide summary information about all the unforeseen changes that the project is managing. For example, a Change Log could be used as an agenda for a change control board meeting or to provide input for a status report.

Description
Main Description

A Change Log is a list containing some of the information about all or some of the Change Request.

Brief OutlineThe Change Log contains one entry for each Change Request that is selected for inclusion. The content of each entry depends on the specific purpose for which the log is prepared. The following description should be regarded as indicative rather than definitive.

Key Considerations

A Change Log is built by extracting some of the information from some or all of the Change Requests.  This can be done manually but is more easily achieved when the Change Requests are stored in a database. It is usual to produce several different Change Logs for different purposes.
