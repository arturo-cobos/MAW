Artifact: Project Charter (ENG 341)
A document issued by or created on behalf of the sponsor to describe the project’s objectives.
Domains: Project Management
Work Product Kinds: Requirement
Purpose

The purpose of the Project Charter is to document:

  • The reasons for undertaking the project.
  • The project objectives and constraints.
  • Directions concerning the solution.
  • Who are the main stakeholders.

Description
Main Description

The Project Charter formalizes the request from a sponsor for responding to a business need.

The Project Charter is usually a short document that refers to some other more detailed documents such as a “new offering request” or a “request for proposal”. It may also be used to support a request for additional scope within a program.

Notice that the Project Charter is a special work product that is created by the sponsor.

Notation

The Project Charter has one single component. Note that the content must be interpreted according to the context.

Content Comment
Project identification  
Project / Sub-project  
Project / Sub-project manager  
Business need and environment Description of the problem to be solved, and of the business areas under consideration. (Market segments, competitive analysis, sponsor activities etc.)
Project objectives and constraints Statement of the vision to resolve the business need.

Objectives including target dates.

Constraints: particular resources to involve, budget etc.

Other projects that are impacted.

Solution background Elements available concerning the solution.

Elements of solution expected by the sponsor from the delivery organization.

Target domain where the solution is to be deployed / distributed: organizational entities, market segment(s) addressed etc.

Project stakeholders Who are the stakeholders:
  • In sponsor's organization
  • In delivery organization
  • In other IBM organizations
  • In other companies or business entities.

Key Considerations

The Project Charter is provided by the sponsor to the project team.

Tailoring
Impact of not having

Lack of a Project Charter leads to:

  • Unclear need that is likely to give unstable directions to the project.
  • Possible ambiguity about the project objectives.
  • Possible ambiguity concerning the solution.
  • Lack of identifying an important stakeholder for the project.
Reasons for not needing

The Project Charter is needed on every project. If it is not available as an input, it should be developed and agreed to with the sponsor in the early stage of the Defining activities.
