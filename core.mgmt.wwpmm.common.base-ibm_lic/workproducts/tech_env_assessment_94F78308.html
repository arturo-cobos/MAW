Artifact: Technical Environment Assessment (ENG 407)
A document which contains information about the effectiveness of the technical environment and plan for improvement.
Domains: Environment
Work Product Kinds: Record
Purpose

The Technical Environment Assessment is designed to:

  • Record technical environment problems
  • Plan for improvement.

It ensures that an ineffective technical environment does not hinder the progress of the project and threaten its success.

Description
Main Description

The Technical Environment Assessment contains information about the effectiveness of the technical environment and a plan for improvement, if needed.

The technical environment typically includes:

  • An office suite; spreadsheet, word processor, presentation, and/or drawing tools.
  • Office communications; mail and access to shared office suite files.
  • Project management tools that provide:
    • Management and operational scheduling
    • Time tracking
    • Cost tracking
    • Risk management
    • Project control book (PCB)
    • Access to business systems
  • Tools that support technical activities in all project phases:
    • Conception
    • Development
    • Qualification
    • Deployment
  • Features of the project management and the technical tool set that support the configuration management of the technical work products.
Brief Outline

The Technical Environment Assessment contains these components:

  • Assessment profile
  • Assessment area
  • Plan for improvement
  • Follow-up assessment

Key Considerations

A Technical Environment Assessment might be done multiple times during the project, depending on the individual project. It might have been planned ahead of time to coincide with specific points in the project, such as one month into start-up, or it might be done to formally investigate excessive complaints or issues regarding the technical environment.

The assessment is conducted with the project team and is based on their experience and perceptions. If there are substantiating statistics or records, include them, but collecting and evaluating such data is not usually within the scope of project management.

During the assessment, the team should define the problem from their point of view, investigate the source of the problem, and try to identify the area of responsibility for further definition and resolution. Many times problems with the technical environment must be assigned to people outside the project to ascertain an accurate technical solution.

The assessment looks at how well the implemented technical environment is meeting the requirements in the Technical Environment Plan.  But sometimes the environment is performing as planned, however the original plan was not adequate and additional measures are required.

Tailoring
Impact of not having

When the Technical Environment Assessment is not established, there is the possibility that:

  • Resources will be wasted or run short
  • Team members will not be able to work as efficiently as possible
  • Costs will be poorly controlled.
Reasons for not needingThe Technical Environment Assessment is not needed when the technical environment is very simple; for example, a few workstations whose users collaborate using only e-mail.
