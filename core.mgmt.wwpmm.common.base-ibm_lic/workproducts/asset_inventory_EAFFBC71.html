Description
Main Description

The Asset Inventory lists resources assigned to project team members.

The project environment typically includes:

  • Materiel items such as:
    • Office furniture
    • Communications equipment, such as a phone and FAX machine.
  • Technical environment components such as:
    • An office suite; spreadsheet, word processor, presentation, and/or drawing tools.
    • Office communications; mail and access to shared office suite files.
    • Project management tools that provide:
      • Management and operational scheduling
      • Time tracking
      • Cost tracking
      • Risk management
      • Project control book (PCB)
      • Access to business systems
    • Tools that support technical activities in all project phases:
      • Conception
      • Development
      • Qualification
      • Launching or deployment
  • Features of the project management and the technical tool set that support the configuration management of the technical work products.

Key Considerations

The Asset Inventory is updated throughout the project. A review of the asset inventory, verifying that the assets listed can be accounted for, might be required, depending on the project.

Written proofs of return might be required when assets are released to their owners and, if so, this supporting documentation should accompany the Asset Inventory.

The Asset Inventory should be up-to-date.
Tailoring
Impact of not having

When the Asset Inventory is not established, there is a possibility that:

  • Resources will be wasted or run short
  • Costs will be poorly controlled
  • It will not be possible to collect assets when people leave and return them to their owners
  • Planning of maintenance activities will be difficult.
Reasons for not needing

This work product would not be established by project management when the project environment is managed by a function other than project management.
