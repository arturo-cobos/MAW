Artifact: Project Evaluation Report (ENG 344)
A document, typically created at the end of the project or a major phase, that highlights key points to be gleaned from the project, including lessons learned, intellectual capital, and reusable materials.
Domains: Quality Management
Work Product Kinds: Record
Purpose
The purpose of the Project Evaluation Report is to enrich the enterprise's know-how and intellectual capital. A Project Evaluation Report is the basis for organizational learning. It provides a foundation of knowledge from which the enterprise's project processes, procedures, and performance are improved over time.

Description
Main DescriptionThe Project Evaluation Report provides a concise summary of the project and the lessons learned during its execution, and identifies potential reusable materials.
Brief Outline

The Project Evaluation Report contains these components:

  • Project summary
  • Lessons learned
    • This component describes what worked particularly well and what should be improved.
  • Intellectual capital
  • Close out report

Key Considerations

The Project Evaluation Report is typically created at the end of the project by the overall project manager. It may however be created at any time during the execution phase. For projects of long duration, it is recommended that the Project Evaluation Report be created at the end of each phase of the project or conclusion of major milestone. It generally applies to the overall delivery project but may be created at any level in the project organizational breakdown structure.

The Project Evaluation Report is created by soliciting feedback from various project stakeholder groups, such as project team members, sponsoring organization, suppliers, or delivery organization management. Review findings and project documentation can also be useful source of input.

  • To provide the most value, it is important to plan at the outset of the project what you want to learn from the project and the re-useable materials that will be created
  • Workshops are effective in soliciting feedback from various project stakeholder groups
  • Just saying "This ... didn't work well" is of little value. The value comes from suggesting ways to improve

The Project Evaluation Report is typically reviewed by the line of business management or systems assurance prior to being placed in the delivery organization's repository of Project Evaluation Reports. Note that this complies with ISO requirements.
Tailoring
Impact of not having

Lack of a Project Evaluation Report could result in not capitalizing on our past experiences. Without a Project Evaluation Report, we run the danger of continually repeating our past mistakes and not improving performance.

Reasons for not needing

The policies of the delivery organization typically define when a Project Evaluation Report is not required.

Examples of projects that may not need a Project Evaluation Report include:

  • Projects of short duration
  • Projects which are based on standard offerings
  • Projects below a threshold cost
