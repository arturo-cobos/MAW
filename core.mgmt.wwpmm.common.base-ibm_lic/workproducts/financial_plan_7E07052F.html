Description
Main Description

The Financial Plan is a set of one or more two dimensional tables that show the estimated and budgeted amounts project expenditure and revenue by expenditure type and accounting period.

The plan may be presented at several levels of detail, depending both on the stage of the project at which it is produced and on the intended audience, as well as the level of detail required for the financial measurement baseline (FMB).

Although a Financial Plan could be developed for each project organizational unit, typically one is produced for the project as a whole.

Brief Outline

The structure of the Financial Plan includes the following components

  • Validated estimates
    • By phase and expenditure type
    • Project estimates by expenditure type
  • Budget
    • Project budget by phase
    • Project budget
  • Appendix A
    • Itemized estimates by expenditure type
  • Appendix B
    • Consolidated project estimates by phase

Key Considerations

The Financial Plan is created after all the other project management plans and is expanded at intervals throughout the project, typically during Starting and at the end of each phase.

All the costs that the project will incur are identified.

  • People costs, both in-house and contractors, are derived from the Human Resource Plan and/or the Staff List.
  • Third-party supplier costs are either in the appropriate agreements or must be estimated.
  • Any risk contingency reserve and/or insurance is detailed in the Risk Management Plan.
  • Costs associated with setting up the technical environment are obtained from the Technical Environment Plan.
  • Other costs are identified through experience.

The items of cost are analyzed by type and period and entered into one or more worksheets.

The template for this work product also includes notes on how to use the template for financial tracking purposes, such as tracking actuals, calculating forecasts, and reconciling variances. The project manager usually has the responsibility to contain any variances within the budget.

The accuracy and completeness of the Financial Plan is verified by inspection and by ensuring its consistency with the other project management plans.

Tailoring
Impact of not having

A fixed price project cannot be priced correctly without a Financial Plan.

Without a Financial Plan, the project manager cannot control project costs.

Reasons for not needing

In some internal projects, the project manager is not responsible for managing expenditure and so does not need a Financial Plan.
