Main Description

Starting from the outline strategy developed earlier, develop the plan into a detailed framework adequate to ensure the system management related nonfunctional requirements can be met. Make a list of necessary hardware platforms and software environments that are necessary to implement functionality of the systems. Utilize the systems management framework design (SMFD) matrices for collection of all such information.

Also, refine the overall plan for distributing software developed earlier, defining precisely what will be distributed where.

  • Correlate systems management functions to the deployment units, non-functional requirements, architecture models, and software distribution plan.

Key Considerations

Systems management evaluation will typically require a database to catalog all the information, so that it can be sorted and reportes generated.  Manual processing of all the data for a large enterprise with many elements can be unmanageable.
