Task: Deploy Solution
For new development project, deploy to production may only occur after several iterations of development, unless the business can accept frequent change and an advanced continuous integration environment exists.
Purpose
To manage and control the introduction and execution of change into an information technology environment.

      Main Description
      The execution of deployment tasks, whether completed manually or via automation, applies to changes for:  software, hardware, control mechanisms, configurations, environment, facilities, databases, business applications, and so on.  Change requests can be initiated from any other process, but the change "package" is provided by the team.  All change activity - assignments, scheduling, approval, distribution, synchronization, installation, monitoring, and activation - is addressed by the tasks defined in this process.
