Task: Develop Test Management Plans
Creation of a high level over-arching test plan for all testing aspects of the project.
Purpose
To define a high level test plan that covers all aspects of testing anticipated for this project.
Relationships
RolesPrimary Performer:
  • Test Specialist
Additional Performers:
  • Business Analyst
  • Stakeholder
InputsMandatory:
  • Agile Project Definition and Charter
  • Project Management System Summary
  • Use Case
  • Use-Case Model
  • User Story
    		Optional:
    • None
    Outputs
    • Master Test Plan
      Main Description

      Based on the information and document gathered, develop and document the test plan.

      The test plan includes the "what" (scope and objectives), the "why" (purpose), the "when" (key milestones), and the "who" (organizational roles and responsibilities).

      The test plan is developed in a consultative fashion involving all affected groups. The affected groups may include the following:

      • Client and end users
      • Client test teams
      • Development team and internal test teams
      • Account managed operations group
      • Business analysts and system designers
      • Any other group that has an assigned responsibility within the test strategy or test plan

      In developing the test plan, the acceptance criteria are considered so that they are addressed by the test plan.
