Roadmap: How to Adopt Solution Template for Custom Service Component Dev(R4SC)
Main Description

Today's client solutions are hybrid in nature composed of multiple solution types. Solution types composing the overall solution varies from client engagement to engagement. Hence, prescriptive tasks, roles, work products and guidances for each of the solution types are not included as part of a base delivery process in SOMA. SOMA base delivery processes includes only common set of activities, tasks, roles, work products and guidance applicable across all solution types. But we require an ability to select and incorporate additional activities, roles, work products needed to support a set of solution types in a specific client solution into the base delivery process during method adoption.

SOMA has introduced the notion of Solution Template that enables method architecture to facilitate method ddoption to support hybrid solutions. Solution Template externalizes concerns – tasks, work products, roles, guidance – for a specific solution types from the base method/delivery process. Solution template modularizes the method content and externalizes the variations from the commanility in the delivery process.

" Solution Templates are context-specific pluggable extensions to a base WBS (or Delivery Process) that are non-unitary (does not appear in one place in the delivery process in which it is plugged in)."

Solution Template has two parts

  • Method content (activities, tasks, roles, work products) specific to solution type – externalizing from the base delivery process
  • Placeholders in the base delivery process for various parts of the solution templates

A method exponent assembles the delivery process and solution templates depending on the project needs during method adoption. During method adoption, the method exponent will executes the following steps:

  • Select the base delivery process
  • Select a set of solution templates for corresponding to each of solution types involved in the overall client solution
  • Assemble and tailor

In the latest release of SOMA, we have defined three delivery processes with placeholders for different solution templates define in them. Three SOMA delivery processes are published:

  1. SOMA for Application Development (SOMA for AD)
  2. SOMA Complete
  3. SOMA Foundation

Next release: Additional delivery processes such as SOMA Agile is in a draft state and will be released in future release of SOMA. Current practitioner or projects needs can be fulfilled by contacting the SOMA team (arsanjan@us.ibm.com) for the beta version of SOMA Agile.

  3. Solution Template for Custom Service Component Development - based on Recipe for Service Component (R4SC)
  4. Solution Template for Legacy Integration and Transformation
