Task: Perform the Preliminary Design Review
This task describes how to conduct the preliminary design review meeting.
Disciplines: System Engineering
Purpose
To conduct the Preliminary Design Review meeting to validate the detailed requirements, the proposed architecture to meet the requirements and the test strategy/architecture to validate the solution is built in accordance with design and requirements.
Relationships
RolesPrimary Performer:
  • Systems Engineering Professional
Additional Performers:
InputsMandatory:
  • None
Optional:
  • Requirements Traceability and Verification Matrix
  • System/Component Architecture
  • System Test Architecture
    Outputs
    • Preliminary Design Review Results
      Main Description

      See the ‘Conducting a Preliminary Design Review’ techniques paper for more specific details.

      Steps
      Establish the architecture and component baselines.
      Review system and component architectures.
      Verify system and component architectures against allocated system requirements.
      Review test architecture (software and hardware).
      Review and approve the allocated component requirements and business scenarios.
      Establish traceability from system requirements to component requirements.
      Identify/review technical risks and mitigation plans.
      Identify/review technical dependencies.
