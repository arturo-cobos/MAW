Task: Perform the System Requirements Review
Establish the system technical baseline and review the customer baseline.
Disciplines: System Engineering
Purpose
The purpose of the SRR (System Requirements Review) meeting is to establish a technical system baseline by gaining agreement between IGS and the customers that the system requirements fully meet the customer requirements.
Relationships
RolesPrimary Performer:
  • Systems Engineering Professional
Additional Performers:
  • Application Architect
InputsMandatory:
  • None
Optional:
  • Requirements Traceability and Verification Matrix
  • System/Component Architecture
  • System Requirements Specification
    Outputs
    • System Requirements Review Results
      Main Description

      See the ‘Conducting a System Requirements Review’ technique paper for more specific details.

      Steps
      Establish the system technical baseline and review the customer baseline.
      Review the customer business processes and requirements.
      Review the system requirements - both functional and nonfunctional.
      Review the system architecture.
      Review requirements acceptance criteria.
      Review traceability from customer requirements to system requirements.
      Review technical performance measures (TPMs).
      Identify technical risks and review mitigation plans.
      Identify technical dependencies and establish technical management plans.
