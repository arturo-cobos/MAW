Task: Perform the Production Readiness Review
This task describes how to conduct the Production Readiness Review meeting.
Disciplines: System Engineering
Purpose
The objective of the PRR is to verify that the release is ready to be installed into production.
Relationships
RolesPrimary Performer:
  • Project Manager
Additional Performers:
InputsMandatory:
  • None
Optional:
  • Data Migration Plan
  • Deployment Plan
  • Performance Model
  • Requirements Traceability and Verification Matrix
  • Test Findings
  • Test Log
    Outputs
    • Production Readiness Review Results
      Main Description

      See the ‘Conducting a Production Readiness Review’ techniques paper for more specific details with regard to the review template, the attendees, and the materials to be reviewed.

      Steps
      Verify the release content.
      Verify the test results.
      Verify the expected System Availability / Performance / Response times.
      Verify the Move To Production Plan.
      Verify the Data Migration Plan.
      Verify the Deployment Plan.
