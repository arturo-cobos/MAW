Task: Perform the Test Readiness Review
This task describes how to review environment before proceeding with user acceptance testing.
Disciplines: System Engineering
Purpose
The purpose is to review the Development Test results and the readiness of the acceptance test plans and environment before proceeding with User Acceptance Testing.
Relationships
RolesPrimary Performer:
  • Test Architect
Additional Performers:
InputsMandatory:
  • None
Optional:
  • Acceptance Test Plan
  • Project Quality Plan
  • Requirements Traceability and Verification Matrix
  • Test Environment Configuration
  • Test Execution Plan
  • Test Findings
  • Test Log
  • Test Plan
  • Test Specification
    Outputs
    • Test Readiness Review Results
      Main Description
      See the ‘Conducting a Test Readiness Review’ technique paper for more specific details with regard to the review template, the attendees, and the materials to be reviewed.
      Steps
      Establish a Test Baseline.
      Verify that test entrance criteria has been met for each application to be tested
      Verify test environment readiness
      Verify test data readiness
      Verify Test Team readiness
      Verify that the defect estimates are valid and that actuals are being tracked
      Verify that the Quality Model in the Quality Management Plan has valid defect estimates and that actual vs estimated defects are being tracked and analyzed.
      Obtain customer and Project Manager approval to start test execution
