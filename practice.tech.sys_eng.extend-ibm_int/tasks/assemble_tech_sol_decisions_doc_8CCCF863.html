Task: Assemble Technical Solutions Decisions Document
The Technical Solution Decisions document records important decisions about any aspect of the Technical Solution including the structure of the system, the provision and allocation of function, the contextual fitness of the system and adherence to standards.
Disciplines: System Engineering
Purpose
This task defines the process of creating the Technical Solutions Decisions Document.
Relationships
RolesPrimary Performer:
  • Infrastructure Architect
Additional Performers:

  • None

  • Atomic Characteristics
  • Candidate Asset List
  • Current Solution Evaluation
  • Decision Model
  • Reference Architecture Fit Gap Analysis
  • Service Level Characteristic Model
  • Technology Scan
  • Viability Assessment
    Outputs
    • Technical Solutions Decisions
      Main Description

      The document is created as part of the system baseline and reviewed at the System Requirements Review.
