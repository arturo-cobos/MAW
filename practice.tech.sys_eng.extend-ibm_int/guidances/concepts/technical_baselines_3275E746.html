Concept: Technical Baselines
Overview of the objectives, deliverables and reviews for the six System Engineering baselines.
Relationships
Related Elements
  • How to adopt the Systems Engineering practice
Main Description

Customer Baseline

  • Objectives:
    • Convey a clear understanding of the business scope, objectives, and requirements that pertain to this project
    • Understand the business acceptance / success criteria for the project
    • Identify "as is" business processes that will be affected (if available)
  • Deliverables Created:
    • Business Requirements Specification
  • Review:
    • Business Requirements Review

System Baseline

  • Objectives:
    • Establish the system technical baseline and review the customer baseline
    • Review changes to the customer
    • Review the system requirements - both functional and nonfunctional
    • Review the system architecture
    • Review requirements acceptance criteria
  • Deliverables Created:
    • System Requirements Specification
    • System / Component Architecture (at system level)
    • Technical Solution Decisions
    • Requirements Traceability and Verification Matrix (with system information)
  • Review:
    • System Requirements Review

Component Baseline

  • Objectives:
    • Establish the architecture and component baselines
    • Review system and component architectures
    • Verify system and component architectures against allocated system requirements
    • Review test architecture (software and hardware)
    • Review and approve the allocated component requirements and business scenarios
  • Deliverables Created/Updated:
    • Component Requirements Specification
    • System / Component Architecture (at component level)
    • System Test Architecture
    • Technical Solution Decisions
    • Requirements Traceability and Verification Matrix (with component information)
  • Review:
    • Preliminary Design Review

Design Baseline

  • Objectives:
    • Establish component design baseline
    • Review the components' design for completeness
    • Review the components' test plans
    • Verify that the end-to-end test strategy validates system requirements
    • Verify production infrastructure capacity supports the system requirements
    • Establish traceability from component requirements to component design
  • Deliverables Created:
    • All deliverables are created by the development teams, the delivery teams, and/or the test teams
  • Review:
    • Critical Design Review

Test Baseline

  • Objectives:
    • Create a Test Baseline
    • Verify that test entrance criteria has been met for each application & document exceptions
    • Verify test environment & data readiness
    • Verify Test Team readiness
  • Deliverables Created:
    • All deliverables are created by the project manager, the development teams, the delivery teams, and/or the test teams
  • Review:
    • Test Readiness Review

Production Baseline

  • Objectives:
    • Verify that the release is ready to be installed into production
  • Deliverables Created:
    • All deliverables are created by the project manager and/or the delivery teams
  • Review:
    • Production Readiness Review

Deliverables:

  • Business Requirements Specification
  • Component Requirements Specification
  • Systems Engineering Management Plan
  • System/Component Architectures
  • System Requirements Specification
  • System Test Architecture
  • Technical Solution Decisions
