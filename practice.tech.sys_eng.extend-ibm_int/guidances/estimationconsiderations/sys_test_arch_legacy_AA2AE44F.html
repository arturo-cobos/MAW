Main Description

The estimated time to assemble or package the System Test Architecture deliverable varies for all projects.

As defined in the deliverable template, several work products can be used to create the deliverable. The work products used for an individual project will depend on the project size, project complexity, project type (new development, legacy system enhancement, maintenance, etc.

The packaging time varies based on the many project factors, including:

  • The number of work products to be incorporated within the deliverable for a particular project (as defined above).
  • The complexity of the system testing to be performed on the project
  • The complexity of the work products being incorporated within the deliverable
    • The number of components
    • The complexity of the components. 

An estimated packaging time for the deliverable based on the factors above the estimated packaging time for the deliverable is less than 10 hours. This estimate is based on the assumption that the System Test Architecture document is developed after the System/Component Architecture DCD.
