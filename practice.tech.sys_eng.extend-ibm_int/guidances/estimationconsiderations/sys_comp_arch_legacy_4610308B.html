Main Description

The estimated time to assemble or package the System/Component Architecture deliverable varies for all projects. The deliverable is created with system-level information. This information is reviewed and baselined at the System Requirements Review. The deliverable is then updated with component level information, which is reviewed and baselined at the Preliminary Design Review.

As defined in the deliverable template, many possible work products can be used to create the deliverable. The work products used for an individual project will depend on the project size, project complexity, project type (new development, legacy system enhancement, maintenance, etc) and will be defined at the beginning of the project (either in the projectâ€™s Method Adoption Workshop or via a different project scope definition process).

The packaging time varies based on the many project factors, including:

  • The number of work products to be incorporated within the deliverable for a particular project (as defined above). 
  • The complexity of the work products being incorporated within the deliverable
    •  The number of components
    •  The complexity of the components. 

An estimated packaging time for the deliverable based on the factors above the estimated packaging time for the deliverable is between 16 and 60 hours. This time includes the time to assemble both the system-level architecture and the component-level architecture. Normally, the time to assemble the component-level architecture will be approximately twice as long as the time to assemble the system-level architecture.
