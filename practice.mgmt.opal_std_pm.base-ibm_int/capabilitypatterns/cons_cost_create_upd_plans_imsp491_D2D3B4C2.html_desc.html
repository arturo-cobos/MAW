Activity: Consolidate costs, create or update plans (IMSP491)
This activity will identify a set of initial project plans to be outlined or built, to describe the project management system for the project and the work approach to deliver the solution. Ensure that all project costs are accounted for, create the financial plan and the business case.
DescriptionWork Breakdown StructureTeam AllocationWork Product Usage
Purpose
To identify, create and update the project plans that are needed for the project. Ensure that all project costs are accounted for and an appropriate financial plan is in place, such as having an appropriate income schedule or payment terms that provide for the recovery of costs over time. Facilitate and confirm the client's and IBM's understanding of the complete business case.

Description

This activity will develop the plans to a sufficient level of detail to make reliable estimates and limit the risks to a manageable level.   It will also focus on consolidating the costs and related assumptions into the Financial Plan (which may be part of the Project Management System Summary.

This activity will define, create or update in total or in part the following project plans:

  • Project management system requirements
  • Project quality
  • Project data management
  • Configuration Management
  • Communications management
  • Financial
  • Communications management
  • Life sciences organization validation
  • Deployment
  • Tasks for maintenance and support infrastructure
  • Human resource
  • Technical environment
  • Work environment details
  • Project Management System Summary
  • Risk assessment
  • Product integration
  • Work breakdown structure
  • Project management schedule
  • Other plans

Other plans may include:

  • Facilities plan (if required)
  • Relocation plan (if required)
  • Conversion plan
  • Migration plan (if required)
  • Training Plan and Tracker
  • Installation and delivery plan
  • Measurement plan
  • Release plan
  • Client data security and privacy plan (if required)
  • Application recovery plan
  • Backup register
  • Disaster recovery plan
  • Storage and media control register

The Client Data Security and Privacy Plan is required if one or both of the following statements are true:

  • The client has security and privacy requirements
  • The project is dealing with Personal Information or Sensitive Personal Information

Note: Personal Information is any information that identifies or can reasonably be used to identify, contact, or locate the individual to whom such information pertains.
Sensitive Personal Information is any information that could be misused to significantly harm an individual in a financial, employment or social way.

  • For further information on Personal Information and Sensitive Personal Information as they apply in IBM, refer to the "Guidance on 'Personal Information' and 'Sensitive Personal Information'" available on the "Data Security & Privacy" intranet site.
