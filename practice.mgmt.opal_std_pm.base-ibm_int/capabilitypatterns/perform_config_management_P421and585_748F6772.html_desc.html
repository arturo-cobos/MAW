Activity: Perform configuration management (IMSP421 and IMSP585)
This activity establishes and maintains the integrity of work products that are subject to change by the project.
DescriptionWork Breakdown StructureTeam AllocationWork Product Usage
Purpose
To maintain the integrity of the internal and external work products that are subject to change by the project, ensuring that any version of a work product  placed under Configuration Management (CM) control can be recreated at any time to define and organize the elements of a system.
Relationships
Parent Activities
  • common_activities
Description

This activity includes the following:

  • Perform configuration management
    • Control configuration items
    • Build and release change
    • Record baseline status changes
    • Revise configuration management plan
    • Perform continuing baseline audits
