Task: Integrate project organizational unit work plans (next phase)
This task updates milestone with dependency details and reviews the project management schedule, in view of the dependencies.
Purpose

This task ensures that the project management schedule is not affected due to dependencies or due to the lack of  proper synchronization between project organizational units (POUs).

      Main Description

      This task is based on the WWPMM work pattern P3: Integrate project organizational unit work plans, which is used here in the context of confirming plans for the next project phase, after having assessed both the business value (for the sponsor) provided by completed phases, and the business value expected from phases, projects, and services, either not started or under execution.

      Each potential project organizational unit (POU) or performing organization should have received a project definition where an overall high level schedule was outlined. They should have assessed the feasibility of their involvement, and designed their own plans for that. Also, conditional agreements should have been signed with planned suppliers.

      This task confirms or adjusts the high level overall schedule, ensures that appropriate slack exists for dependencies between POUs, and may adjust the POU's scope of work and milestones to ensure a proper synchronization between POUs and eliminate possible bottlenecks that may affect shared resources.

      The key outputs of this task are the milestone list and the project management schedule across all the OBS levels (for all the POUs), based on the schedules from each of the POUs.

      The task includes the following:

      • Update milestone list for the dependencies
      • Reconcile work plans and confirm the overall project
