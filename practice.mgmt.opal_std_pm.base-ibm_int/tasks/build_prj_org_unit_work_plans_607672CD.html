Task: Build project organizational unit work plans (next phase)
This task will refine the work units that are identified in the project definition and make sure that the workload and timeframe are consistent, and the resources will be available.
Purpose

The purpose is to ensure that the expectations from each of the project organizational units (POU), as described in the project definition, are realistic.

      Main Description

      This task is based on the WWPMM work pattern P1: Build project organizational unit work plans, which is used here in the context of confirming plans for the next project phase, after having assessed both the business value (for the sponsor) provided by completed phases, and the business value expected from phases, projects, and services, either not started or under execution.

      It consists of having each of the performing organizations that will be part of the project, develop the plans describing the means that will be necessary to carry out their part of the project (or a project phase, or sub-project).

      This task will be launched as many times as the number of project organizational units (POUs) that have been identified for the project organization in the project definition for the next phase(s).

      Note that the goal here is to let each POU verify that what is expected from them, as described in the project definition, is realistic. In other words, this activity consists of refining, in terms of activities, tasks, and intermediate work products, the work units that are identified in the project definition, to such a level that reliable estimates can be obtained, and to make sure that the work load and time frame are consistent, and that the resources will be available.

      The key outputs of this task are updated plans at the POU level including: the deliverable definition(s), the product breakdown structure (PBS), the work product list, the team structure in the organizational breakdown structure (OBS), the human resource plan, the work breakdown structure (WBS) and the high-level project management schedule for the POU.

      All these plans will be expanded and dependency management aspects will be addressed when starting the new phase of the project.

      The task includes the following:

      • Refine deliverable definitions and phase WBS
      • Review phase estimates and planned resources
