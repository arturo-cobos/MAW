Task: Launch project, understand agreement and planning elements (IMSP391)
This task will conduct the planning for project launch, ensure that the agreement and related information from prior phases of this project is understood.
Purpose
To plan the project launch and to ensure that work done in previous phases of the project is used as the basis for this phase, providing an appropriate understanding of the project.

        Main Description

        Reference IGSDF - Using GBS CRM on Engagements for links to your local GBS CRM Playbook for business rules on whether a formal Project Launch needs to be conducted.

        This task is based on the WWPMM work pattern S1: Initiate phase.

        This task is part of activity that is the first to be launched when starting the execution of a project (or of a new major technical phase or sub-project), and needs a first set of project team members (at least the project manager) who will plan the activities necessary to get the project up and running.

        This task is aligned to the Delivery Excellence initiative Project Launch.  Project Launch is the process of establishing the necessary support, relationships, agreements and procedures designed to manage the successful execution of a project. 

        For guidance in assessing the known information about the project at this early stage, review the outcome of the seven keys based assessment, performed at the time of project launch.  This template is used to identify project risks and concerns identified during solution design that require resolution.

        The key outputs of this task are the outcome of the seven keys based assessment, performed at the time of project launch, Milestone List and Meeting Documentation.

        This task includes the following:

        1. Review the notification related to the project launch.  This letter marks the move from the sales phase to the delivery phase. 
        2. Understand agreement and other definition and planning elements
          • Conduct a review meeting with the project team and other stakeholders A kickoff meeting is conducted with the Project Quality Analyst (PQA) and other stakeholders (like Risk Management Team) to understand the status of certain basic parameters like Work Authorization, Involvement of a Transition phase in the project, Project Work Classification, Initiation into Quality etc.
        3. Develop an understanding of the project solution and how the project is to be managed by reviewing the agreement, definition and planning materials, and delivery organization guidance for projects. Review the outcome of the seven keys based assessment, performed at the time of project launch list of questions aimed at identifying project risks and concerns within the first 10 days.  Discuss any issues identified with stakeholders or the engagement partner.  Include appropriate findings in the project status report.
