Task: Handle in-scope request (IMSP460)
This task will respond to a request for new work that is in the scope of the sponsor agreement.
Purpose
To ensure that work is performed appropriately, either as a request that is in scope of the agreement, or is determined to be out-of-scope, such as a change request.

      Main Description

      This task is based on the WWPMM work pattern M8: Handle in-scope request.

      This task is launched when, in conformance with the sponsor Agreement, work will be performed on the request (within agreed boundaries) as documented by the sponsor or their delegate(s).

      If it is verified that the request is "in scope", then it does not add work, scope, or cost to what has been committed. It triggers an occurrence of work within a predefined activity that is usually defined in the Work Breakdown Structure (WBS) at a higher level (for example, under an application maintenance agreement, requests for periodic minor modifications might be in the scope of the sponsor Agreement, but a request for a whole new function, would likely not be in scope). Typically, the work items that fall under this sort of "in scope" work, are meant to be launched on demand by the sponsor.

      Thus, this task consists of the following:

      • Analyze the request to determine if it conforms with the scope defined in the Agreement for the related activity.
      • Obtain clarification if needed.
      • Determine the feasibility with appropriate technical staff.
      • Estimate the resources needed, update the work plans and assign the work packages.
      • Document the activities to be performed as an expansion of the Work Breakdown Structure.

      The key outputs of this task are new or updated Deliverable Definition(s), Work Breakdown Structure, and Operational Schedule(s).
