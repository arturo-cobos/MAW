Main Description

Architectural patterns are ready-made blueprints that solve recurring architectural problems. An Architectural pattern expresses a fundamental organization of elements within a software system. It provides a set of predefined subsystems or other fundamental building blocks. It specifies their responsibilities, and includes rules and guidelines for organizing the relationship between them [BUSCHMANN96]. In comparison to design patterns, architectural patterns are larger in scale.

Examples of architectural patterns include:
• Layers
• Three-tier
• Peer-to-peer
• Service-oriented architecture
• Model-view-controller
