Main Description

This artifact provides:

  • An inventory of an enterprise's installed applications, data, computing and network infrastructure, and the current security architecture indicating the existing or planned security zones, secure channels, or network security zones and trust levels.
  • The current status of all of the non-technical, management aspects of the overall technical environment, including:
    • The IT/IS/IT security organization(s), roles, and responsibilities
    • The existing IT strategy, policies, and plans (including skills acquisition and sourcing options)
    • Security policies, standards, and processes
    • Development policies and practices
    • System management policies and practices


These details about the technical environment are summarized into a form that can be readily used as input to many future architecture, design, and infrastructure delivery and management activities, along with observations on current known IT/IS deficiencies, concerns, and user satisfaction with IT.


It should be noted that this artifact is NOT intended for documenting the results of an assessment of the current technical environment. The artifact can be restricted in scope (for example, to the part of the technical environment affected by the introduction of a new package or system).

Key Considerations

The following general guidance is recommended:

  • Aim to gather all of the inputs and information needed for a single project at the same time, regardless of the artifacts that will be defined. Plan the information gathering as a joint exercise between all interested parties, and agree on the tools and techniques to be used, especially as some information gathering activities will cross organizational boundaries.
  • When gathering environment information, go beyond the classical information and determine if any limiting environmental situations exists (such as noisy plant floors, extra bright areas where it is hard to see a screen, or pervasive devices with small, text-only screens).
  • Verify the IT Environment by review with the client sponsor and appropriate senior executives. It may also be appropriate to hold a peer and quality assurance review (a peer who has accessibility skills may be necessary in complex IT situations).
