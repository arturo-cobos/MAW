        Main Description
        Adjust the goals for the cycle about to be performed and refine them to the level of detail required.

        The goals for the programming cycle may previously have been set in fairly general terms. As the work is about to start, it is usually necessary to be more specific about exactly what is expected, both in terms of the functionality and performance of the code and in terms of the design documentation and the level of unit testing. These specific goals are agreed between the team leader responsible for the programming cycle and the manager or architect who is overseeing the work.
