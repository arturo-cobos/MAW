Task: Evaluate Build Cycle Results
Perform a review of the accomplishments of the completed Build iteration and compare to the goals set out for this increment. Adjust future Build iteration increments accordingly.
Disciplines: Technical Management
Purpose
To evaluate the development and testing results of the completed build cycle, the adjust and refine its goals accordingly. Also, to evaluate performance against estimates and adjust estimates for the build cycles accordingly.

      Main Description
      Adjust the goals for the build cycle and refine them to the level of the iterations contained within it.

      In detailing the requirements for the current programming cycle new insights may have been gained which may cause adjustments in the goals. Also, priorities may have shifted since the time of any previous cycles, which have to be factored in.

      The goals of the individual build cycles are defined.

      Performance times for this build cycle are compared to estimates developed prior to the start of the cycle. New estimating guidelines are developed as a result of this comparison.

      The build resulting from this cycle should be reviewed with the project sponsor in order to keep him appraised of project progress, and obtain feedback on client satisfaction.
