Task: Define Goals for the Build Cycles
Identify the number of Build Cycles anticipated for the upcoming release and set the goals for each
Disciplines: Technical Management
Purpose
To set the specific goals for this release's build cycles, and secondarily to insure reusable assets to be harvested from this release receive appropriate treatment and attention during the programming process.

        Main Description

        Establish specific goals for each iteration of the build cycle. Obtain agreement between the project manager and the client sponsor to set expectations for the application function at each build review point. Include the team leaders, architects, and application developers to insure the goals are realistic and achievable.

        The goals and objectives of the build cycle iterations need to be supportive of the plan for the current release. The end result of all build cycle iterations should be a viable release candidate for deployment to the solution's production environment.

        From the Asset Harvesting List in the Candidate Asset List, identify those components contained within this release which have been selected to be harvested as reusable assets. For these components, ensure that appropriately focused treatment is given to their specification, testing, and quality assurance aspects to provide for the fulfillment of all the objectives required for their future harvesting and reuse.
