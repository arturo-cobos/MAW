Artifact: Estimation Checkpoint Report (ART 0534)
This artifact measures the effort, cost, and schedule variance experienced on a project at each periodic estimation checkpoint. It also records the current estimate to completion during project delivery at each checkpoint.
Domains: Project Management
Work Product Kinds: Record
Purpose

This artifact has two primary purposes:

  1. To manage risk within the project during delivery by measuring the variance of the project over time from the baseline estimate as the project progresses.

    Although risk mitigation activities are addressed and built into the project effort and schedule, it is useful to have an indication of how effective they are as the project progresses. Frequently the risk mitigation tasks are tracked from the perspective of being complete, not from the perspective of how effective they were in reducing the risk.

    The Estimation Checkpoint Report calculates and charts the variance over time for you based on a minimal set of inputs.

  3. To document the results of following the estimation process by recording the values of the estimate at each governance point. The estimate itself is maintained using your own or standard templates, tools, or models.

Description
Main Description

Periodic checkpoints during project delivery for estimation governance are required, where the status and content of the estimate to completion and the variance from the baseline are reviewed and approved. Governance confirms adherence to the procedure by performing the reviews and documenting the estimate details.

The Estimation Checkpoint Report records the values of the estimate including variance at each governance point within the estimate lifecycle. Each governance point is associated with a review of the estimate. The variance is a measure of the uncertainties and risks inherent in the baseline estimate and the estimate to completion as the project progresses. As the project progresses, the variance measured by the report will fall.

Brief Outline

The Estimation Checkpoint Report records:

  • Basic project information, such as the project name, client name, and project dates, category, business sector, and industry
  • The current baseline estimate of effort, cost, and schedule for the project as of the checkpoint date
  • The current estimate to complete effort, cost, and schedule for the project as of the checkpoint date
  • The current actual expended effort, cost, and schedule as of the checkpoint date

The Report calculates the current change in variance of the project from the baseline measured from the project start date to the current checkpoint date.

Key Considerations

The estimate variance as shown on the charts in the Estimation Checkpoint Report is expected to fall month to month as the project progresses due to successfully mitigating risks and completing work. If the estimate variance as measured in the report is not falling as the project progresses, find out why.

The current estimate at completion as shown on the charts is expected to track closely to the estimate baseline. If the estimate at completion is not tracking to the baseline, this is an early indication that some underlying assumption may have changed and you should find out why.

Tailoring
Impact of not having

Consequences of not having this work product are:

  • Absence of confirmation that risks that impact effort, cost and schedule are being successfully mitigated within the project
  • Absence of confirmation that risk mitigation tasks are effective in reducing risk
Reasons for not needingThis work product is highly recommended for use in documenting the progress of the project against the estimate baseline for the project or work request.
