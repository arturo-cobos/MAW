Task: Define deliverables components
This task will identify and document the components, breakdown of components, or intermediate work products, for each work product to be delivered, determine the layout of activities necessary to create them, and make a crude estimate of the workload and duration for performing those activities.
Purpose
To identify and document the deliverables and components, and the related layout of activities necessary to create them, so the project can be planned and started with a clear understanding of these items.

      Main Description

      This task consists in having each of the performing organizations that are proposed to be part of the project, develop the plans describing the means that will be necessary to carry out their part of the project (or a project phase, or sub-project).

      Break down the operational deliverables to a level that allows you to recognize:

      • An existing component/asset from which the product breakdown structure element will easily be derived.
      • A component that can be bought off the shelf or subcontracted.
      • A component for which a technical delivery process and related estimating technique are known.

      A Method Adoption Workshop (MAW) is conducted, when appropriate.  During the MAW, a review of the business requirements is done to aid in the method selection process.  Method content is available on MethodWeb for review and download to the tailoring tool (i.e. RMC).  A basic work breakdown structure (WBS) is created during this tailoring process and is used as a basis to build the project plan.

      Enhance the Work breakdown structure (WBS) by describing the activities that are necessary to build, integrate, and rework the work products to be delivered by the project organizational unit.

      Note that the goal here is to let each POU verify that what is expected from them, as described in the Project Definition, is realistic. In other terms, this activity consists of refining, in terms of activities, tasks, and intermediate work products, the work units that are identified in the Project Definition, to such a level that reliable estimates can be obtained, and to make sure that the work load and time frame are consistent, and that the resources will be available.

      This task includes the following:

      • Identify the components, breakdown of components, or intermediate work products, for each work product to be delivered to the sponsor or to another project organizational unit. This component breakdown should reach a level that the components are recognized as belonging to a technical work pattern and related estimating technique are known, or the components are identifiable as reusable assets.
      • Document the components in the Product Breakdown Structure (PBS).
      • Determine the layout of activities necessary to create those components or intermediate work products in an orderly manner, and make a crude estimate of the workload and duration for performing those activities. A method adoption workshop (MAW) can assist for some technical work products, but likely will not provide all the information required.
      • Document the Work Product List and the initial Work Breakdown Structure (WBS) for the POU.
      • Document the process, methods and tools described in the proposal for planned usage and used in delivery on the project across all delivery sites in the PM&T Profile.

      The key outputs of this task are the Deliverable Definition, Work Breakdown Structure, Product Breakdown Structure and Work Product List. In some situations, a Project Defined Process will also be built.
