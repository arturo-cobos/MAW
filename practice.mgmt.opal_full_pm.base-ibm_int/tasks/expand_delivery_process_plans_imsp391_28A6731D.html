Task: Expand delivery process plans (IMSP391)
This task will expand other plans identified in the delivery process for the project.
Purpose

To expand other plans identified in the delivery process for the project. The details of this work may be planned and tracked elsewhere in the work breakdown structure outside the project start tasks.

        Main Description

        Expand other plans identified in the delivery process for the project, The details of this work may be planned and tracked elsewhere in the work breakdown structure outside the project start tasks.

        Depending on delivery process, and characteristics of the project (size, complexity) some work products may be recommended or required by some organizations. Refer to your organization reference document or the delivery process instructions for guidance.

        Examples may include: master test plan, Facilities plan, Relocation plan, Conversion plan,  Migration plan,  deployment plan, Installation and delivery plan, Measurement plan, release plan.

        These delivery process plans should normally have been outlined during Project Planning in Solution Design, as they obviously have an impact on costs and schedules, and their activities need to be included in the price.

        A release plan(s) defines the timing and contents of each solution release to the client.

         Refer to the Release Management Guideline for additional information on release planning.
