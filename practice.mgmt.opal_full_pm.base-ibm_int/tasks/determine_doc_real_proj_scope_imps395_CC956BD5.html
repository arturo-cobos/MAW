Task: Determine and document real project scope boundaries (IMSP395)
Based on the target solution description and the overall approach, document the boundaries of the project in terms of functionality, deliverables, steps of the overall approach to be covered, and organizations affected by the project.
Purpose
To document project scope in the Project Definition and in the Deliverable Definition(s).
Relationships
RolesPrimary Performer:
  • Project Manager
Additional Performers:
InputsMandatory:
  • High-Level Solution Plans
  • Project Charter
  • Project Definition
    		Optional:
    • Service Request
      Outputs
      • Deliverable Definition
      • Project Definition
        Main Description
        Document the boundaries of the project being formulated, in terms of sponsor's business areas, locations, IT infrastructure and information systems affected by the project, and in terms of functions, level of service, and deliverables.
