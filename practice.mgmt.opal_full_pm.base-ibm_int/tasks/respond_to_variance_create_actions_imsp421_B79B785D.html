Task: Respond to variance and create actions if necessary (IMSP421)
Using the data collected for the project and the resulting updated plans and schedules, this task will identify variances, update the project status, and create actions, as necessary.
Purpose
To evaluate the project's ability to complete the work on time with the resources currently available or planned, and keep the progress variances within limits that will not put the project at risk.

        Main Description

        The key outputs of this task are the Project Status Report and the Action Log as necessary.  Additional outputs may include the Operational Schedule, the Project Management Schedule or the Staff Schedule.

        Note 1:  Action Control Document may be used if the project  manager decides to log one action at a time.

        Note 2: There are portions of the steps below that are designed to support organizations pursuing CMMI levels 4 and 5. These sections are indicated by gray shading, and are optional for organizations not using CMMI as a basis for process improvement or organizations targeting a lower maturity level.  However, such organizations may find it helpful to implement these aspects.

        Steps
        Identify effort and schedule variances

        Identify effort and schedule variances and also identify deviations involving risk identification and handling.

        • Review operational and project management schedule (s) to:
          • Assess progress and variances of work performed by comparing planned and actual dates
          • Identify the tasks and milestones that have started and completed on schedule
          • Identify overdue tasks and milestones
          • Calculate variances by comparing the percent complete with the planned percent complete in the appropriate project schedule
        Update the project status report

        Update the following in the project status report:

        • Quality Assurance Reviews - planned vs. actual dates and review ratings in the Reviews section
        • Data Security and Privacy activities as outlined in the Data Security & Privacy task plan (for example ID revalidation, QEV, CBN, etc)
        • Process Assurance Reviews - planned vs. actual number of process assurance reviews in the Reviews section
        • Corrective and Preventative Action Requests (CPARs) -for example, the cost-corrective action items or schedule-corrective action items, if any, in the Action Items section
        • Project Change Requests - number of Project Change Requests for each category, effort/price impact for the total in each category, in the Quality Status section 
        • Issues - all new or outstanding issues and status to see that actions on issues are being addressed by the commit date, in the Quality Status section
        • Risks - newly identified project risks, status of each risk and the containment\mitigation strategy, in the Quality Status section 
        • Dependencies - identified project dependencies, status of corresponding activities against plan, effects of early or late completion of activities, potential problems and dependency cost, in the Quality Status section
        • Baseline Audits - planned vs. actual number of baseline audits, in the Reviews section
        • Work Product Inspections (WPIs) - planned vs. actual number of work product inspections , in the Reviews section
        • Testing - testing activities and status, in the Testing status section
        • Deliverables accepted - list of deliverables accepted, in the Project accomplishments section
        • Root Cause Analysis - status of Root Cause Analysis activities done in the project, in the Root Cause Analysis Status section
        • Process Improvement - improvement information, in the Root Cause Analysis Status section

         Refer to the Project Monitoring and Control Guideline for additional information on the data collected for the project.

         Refer to the External Links for the GBS Guidelines and Standards for Data Security and Privacy web site for the DS&P task plan template to be integrated into the overall project task plan.

        Measure and analyze the quality

        Measure and analyze the quality of the project's work products.

        Organizational approved statistical techniques are to be followed while analyzing the project's actual performance against the project's quantitative quality goals.

         Refer to the organization level measurement program for details of the organization's measurement program, such as approved statistical techniques and organizational quality goals.

         Refer to the project quality plan for details of the project's quality goals, which are based on the organizational quality goals.

        Assess the quality

        Assess the quality of the project's deliverables, comparing to the products' quantitative quality goals on an event-driven basis.

        Monitor the conditions of satisfaction on a regular basis to ensure the identified project acceptance criteria are being met.

        Assess financial data

        Assess financial data by determining the following Earned Value Indicators using labor costs and high-level activities.

        For activities that have either been completed or are not yet started, the calculation of earned value is simple and objective. For activities in progress, some form of estimation must be used. It is important that extreme caution be exercised in using and interpreting estimates.
        Assess any additional effort, cost and schedule data

        Assess any additional effort, cost and schedule data as defined in the associated project plans.

         Refer to the Project Monitoring and Control Checklist for additional information on project monitoring and control data.
        Update the project status report as appropriate

        Update the project status report as appropriate, checking for accuracy and consistency. The project status report includes, but is not limited to, the following information:

        • Schedules and reviews
        • Status: Project, Quality, Testing, Root Cause Analysis, Data Security and Privacy
        • Resources: Budgeted and Actual
        • Costs: Budgeted and Actual
        • Earned Value Indicators
        • Accomplishments: Current and Future
        • Action items
        • Status of the activities for quantitative process management

        In the event of any resource transfer, resignation or any such change in the project organization structure, ensure that the activity handover form will be used for capturing activities and responsibilities to be handed over.

        Respond to variances and create actions

        Respond to variances and create actions, if necessary.

        Examine results of variances to project plan (financial and schedule), schedule slippage, recent changes to the plan, risk occurrences, or other events or circumstances that might require re-planning.

        Compare the earned value indicators to any thresholds that are defined in the organization's version of the organizational reference document (ORD).

         If the earned value indicator is outside the defined thresholds in the organization's version of the organizational reference document (ORD), then perform a root cause analysis in accordance with Root Cause Analysis Procedure (IMSP525).

        Examine the project's quantitative measures of quality to determine root cause

        Examine the project's quantitative measures of product and process quality to determine root cause(s).

         Refer to the Organizational Measurement and Analysis Guideline.
        Determine the response approach

        Determine the response approach and any needed actions to get the project back on track.

        Evaluate actions to determine whether or not they conform to the defined scope of the project.
