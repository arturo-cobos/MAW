Task: Confirm the validity of the planning prerequisites (IMSP391)
This task will confirm the validity of the initial scope, requirements, work breakdown structure (WBS), work product list, estimates, classification and size attributes, measurements, risks, assumptions and dependencies.
Purpose

To confirm the validity of the initial scope, requirements, work breakdown structure (WBS), work product list, estimates, classification and size attributes, measurements, risks, assumptions and dependencies.

Relationships
RolesPrimary Performer:

      Outputs
      • Estimation Report
      • Work Breakdown Structure (WBS)
        Main Description

        Confirm validity of the initial scope, requirements, work breakdown structure (WBS),
        work product list, estimates, classification and size attributes, measurements, risks, assumptions and dependencies.

        • Ascertains if any changes in scope, costs assumptions or risks has occurred. If so, reworks existing estimates and updates the estimation report.
        • Identifies and resolves any difference in cost versus budget (addressing the situation where the project budget may have been cut, but the project scope remains the same).
        Key Considerations

        Reference OPAL Procedures:

        • Project Start (Project Launch) Procedure (IMSP391)
        • Project Definition Procedure (IMSP395) and the Project Planning Procedure (IMSP491) for information regarding the definition of scope, requirements, WBS, estimates, assumptions and dependencies.
        • Project Tailoring Procedure (IMSP150) and the project defined process, for prior project tailoring decision.
        • Continuous Risk Management Procedure (IMSP600) for information regarding risk planning and mitigation.
        • Work Classification Procedure (IMSP454) for information about classifying work.
        • Quality Planning Procedure (IMSP230) and the organization level measurement program for information on measurements.
