Task: Assess actual efforts and progress data (IMSP421)
This task will collect project data, assess the actual effort and the project's progress, and update the actual progress in the appropriate plans and schedules.
Purpose
To validate and update a project's actual progress, to support the capacity to evaluate the project's ability to complete the work on time with the resources currently available or planned, and keep the progress variances within limits that will not put the project at risk.

        Main Description

        The key outputs of this task are the Milestone List,Operational Schedule, Project Management Schedule and Staff Schedule, adjusted as necessary.  Additional outputs may include updates to the Project Quality Plan, Project Data Management Plan, and the Technical Environment Plan, which may be part of the Project Management System Summary.

        This task includes the following:

        • Collect project data
        • Update actual progress within plans and schedules
        • Post measurement data to the appropriate metric repositories
        Steps
        Collect project data

        Collect project data, and assess actual effort and project's progress. This activity is launched on a short term, periodic basis (weekly, semimonthly or monthly), depending on the nature of what the project is delivering, in order to control project progress and to monitor the project's ability to meet the due dates within the financial targets.

        Collect and validate following project data:

        1. Human resource availability information, such as scheduled holidays, sick days, entry and exit dates
        2. Work environment details, and update the technical environment plan accordingly
        3. Team members progress data from the individual status report. Update the staff, operational, project management schedule(s)
        4. Supplier data from subcontractor status report(s). Update the project management schedule and operational schedule, as appropriate, for milestone and task data
        5. Cost and revenue data for accuracy, consistency and reasonability, ensuring that all appropriate charges have been entered into the organizational approved business system correctly
        6. Appropriate information on all other project data required by the project management system summary, in accordance with the organization's version of their organization reference document (ORD), as appropriate.  Examples include:
        • Number of change requests raised, accepted and closed
        • Configuration baseline audits planned and performed
        • Number of issues raised and closed by category and duration of issues
        • Status on key decisions and actions
        • Status on requirements
        • Number of defects for each status and severity
        • Peer review effort, results and defects
        • Test effort, results (findings) and defects
        • Quality assurance reviews
        • Process assurance reviews
        • Resource responsibility changes
        • Skill requirements
        • Dependency milestones and commitment dates
        • Status on project data management plan

          Refer to the Project Monitoring and Control Guideline for additional information on collecting and validating project data.

        7. The measurement data used to support the project's quantitative measures of product and process quality as defined in the project quality plan

        Update actual progress
        Update actual progress within the project management schedule and the project management system summary as appropriate, checking for accuracy and completeness.
        Post measurement data
        Post measurement data to the appropriate metric repositories, which are defined in the project quality plan, ensuring accuracy and completeness.
