Task: Ensure new project is registered and material transferred (IMSP391)
This task will ensure the project was registered and that all relevant material from prior defining and planning activities is transferred to the project.
Purpose

To ensure that the Opportunity Owner has worked with Finance to register the project and a billing code has been established. The proposal team passes over work done in previous phases of the project that will be used as the basis for this phase that will provide an appropriate understanding of the project.

        Main Description

        This task is dependent on the GBS Customer Relationship Management (CRM) Process. Reference IGSDF - Using GBS CRM on Engagements for links to your local GBS CRM Playbook for specific guidance for registering a contract in your region.

        This task is part of the activity that is the first to be launched when starting the execution of a project.
        This task includes the following:

        • Ensure the agreement is signed and all necessary approvals (including client's) are complete.
        • Ensure the new project is registered and the billing code is obtained.
        • Document the notification related to the project launch instructing the project manager of record, for a contract, that the project start (project launch) procedure is to be executed.  
          • Ensure all relevant material is transferred from any prior team to the project team. Note: This is particularly important where solution design and solution delivery are performed by different people.
        Key Considerations

        Note: Application Management Services (AMS) contracts may follow a variation of the process:

        •  Refer to the organization's communications management plan (Organizational) for P3 CTR and to the Process, Performance and People Critical Thread Reviews (P3 CTR) procedure (IMSP435), to understand how a project is registered to support the P3 CTR process
        • Note: The project registration is usually done using local tools. Please refer to your organization's organizational reference document for more information about project registration and controls applicable to your organization
        • Refer Solution Design to Solution Delivery Transition Verification to ensure all relevant materials created during the Opportunity and Solution Design phases are turned over to the Delivery Project Manager of Record
