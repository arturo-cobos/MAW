Task: Define the project management system (IMSP391 and IMSP230)
This task will define a detailed project management system as the basis for managing the project. This includes defining or expanding upon various plans, and documenting project specific procedures and standards to be used for the project.
Purpose
This task will ensure the appropriate project management system, including plans, procedures and standards, is in place to implement and support the project successfully.

        Main Description

        This task is based on the WWPMM work pattern S2: Implement project management system.

        Normally, the key features of the project management system to be implemented at project startup have been defined when planning the project, particularly when these features were expected to have an impact on the project costs and commitments.

        However, these plans are typically only "outlined", being developed to a level which was sufficient to prepare the Agreement, but that is not sufficiently detailed to control project execution. It is now time to "expand" these plans to a level of detail relevant for project execution control, by doing the following:

        • Define the project management system requirements in the Project Management System Summary in light of the project risks that have been identified, and confirm the procedures that should be enforced to facilitate project control, communications and exceptions handling
        • Review the sponsor agreement, deliverable definitions, product breakdown structure, the outline of the project quality plan, and the Client Data Security and Privacy plan that were produced when shaping the project.
        • Perform project tailoring to confirm the procedures which should be enforced to facilitate such things as project control, communications, and exceptions handling.  Use the project defined process to record tailoring and the project procedures description to record any project-specific procedures, and reference any applicable standards in the project quality plan.
        • Setup a Project Control Book to file project management work products including the project management system summary, plans, procedures and records.
