Task: Establish and analyze issue statistics (IMSP620)
Review the issue management documentation and calculate the necessary statistics to determine variances and probable cause.
Purpose
To review the issue management documentation to calculate the necessary statistics, and access and analyze variances and probable cause.
Relationships
RolesPrimary Performer:
  • Project Manager
Additional Performers:
InputsMandatory:
  • Issue Document
  • Issue Log
    		Optional:
    • None
    Outputs
    • Issue Document
    • Issue Log
      Main Description

      Review the appropriate issue management documentation.

      Calculate the necessary statistics.

      Examples of calculated statistics include the following:

      • Total number of issues raised
      • Number of open issues
      • Number of new issues raised during the reporting period
      • Total number of resolved issues
      • Number of issues that were resolved during the reporting period
      • Number of recurrent issues
      • Combinations of status criterion (such as pending, in progress, or closed), with issue type, timeframe, and/or priority

      Assess numbers, statistics, and trends by comparing the following:

      • Numbers and statistics with figures typically obtained in similar situations; for example, the same size and type of project
      • Numbers and statistics with those from the previous reporting period

       Refer to the Organizational Measurement and Analysis Procedure (IMSP473) for information about managing organizational measurements and analysis activities.

      Analyze the variances to determine the probable cause. 

      Identify new issues and/or risks based on the significance of the numbers, statistics, and trends.
