Task: Refine Deliverable Definitions and phase WBS
Disciplines: Work Plan Management
Purpose
For each project organizational unit to review the components of the solution for which they are responsible and break down the associated components to such a level that they can be estimated using known techniques or can be identified as existing reusable assets.
Relationships
RolesPrimary Performer:
  • Project Manager
Additional Performers:
InputsMandatory:
  • Deliverable Definition
  • Product Breakdown Structure (PBS)
  • Project Definition
  • Work Product List
    		Optional:
    • None
    Outputs
    • Deliverable Definition
    • Product Breakdown Structure (PBS)
    • Work Breakdown Structure (WBS)
    • Work Product List
      Main Description
      Review the project scope that is assigned to the project organizational unit (POU) and identify the work products that the POU plans to deliver to either the sponsor or another POU.

      Clarify the acceptance or completion criteria for these work products and determine the quality characteristics and quality criteria that will contribute in meeting these acceptance criteria. Document them in the Deliverable Definition(s).

      Identify the components, breakdown of components, or intermediate work products, to be produced. This breakdown should reach such a level that components can be estimated (a technical work pattern and estimating technique are known), or are reusable assets. Document this breakdown in the Product Breakdown Structure (PBS).

      Determine the activities necessary to create those components or intermediate work products, as far as possible by conducting a method adoption workshop (MAW), and estimate the workload and duration for those activities.

      Document in the Work Product List and in the Work Breakdown Structure (WBS) for the POU.
