Task: Start the deployment of organization's standard processes (IMSP491)
This task will start the deployment of organization's standard processes in the project. This is based on the work classification determined.
Purpose
The purpose of this task is to start the deployment of organization's standard processes in the project.

      Main Description
      Start the deployment of organization's standard processes in the project, based on the work classification determined.

      Initiate and maintain contact with the EPG or the deployment group designated by EPG, and the PQA, to coordinate the process deployment activities.

      The following steps can be performed for deploying process asset library in the project:

      • Determine the applicability of various components of process asset library.
      •  Perform further detailed project tailoring in accordance with the Project Tailoring Procedure (IMSP150), and tailors the process asset library components for the project, as required.
        • Evaluate the process tailoring options. When a project performs tailoring which is over and above the guidance in the Project Tailoring Guideline, the EPG and\or the deployment group needs to be involved in the review and approval of the same.
        • Document the tailoring results in the project defined process.
        •   Create project procedure description according to the Quality Planning Procedure (IMSP230)
        • Determine the tools requirements for the project.
        • Determine the measurement requirements for the project and document it in the project quality plan.
            Refer to the Quality Planning Procedure (IMSP230) for more details.
      • Determine the project team's process asset library skill gaps and coordinates with the deployment group to plan for the required training.
        Update the human resource plan with the training needs identified for the project and mentor the team on asset library processes and assets.

      Report the process asset library deployment progress to the deployment group in coordination with the PQA.

       Refer to Process Assurance Review Procedure (IMSP232) for details on reporting the project's deployment progress.
