Task: Analyze request for compliance with Agreements (IMSP460)
Disciplines: Work Plan Management
Purpose
To analyze an "in scope" request for service, determine if it is in scope in accordance with the Agreement, and plan to perform the requested service.
Relationships
RolesPrimary Performer:
  • Project Manager
Additional Performers:
InputsMandatory:
  • In-Scope Request
    		Optional:
    • Deliverable Definition
      Outputs
      • Deliverable Definition
        Main Description
        Check that the request is complete and is in scope (within agreed boundaries) relative to the nature of the activity the request refers to.

        Determine the workload and level of priority of the request.

        According to the workload and level of priority of the request, direct the request to the project organizational unit team responsible for providing the requested service.

        Also check that new workload for a given period of time is consistent with the commitments in the sponsor Agreement.

        Determine the contribution of subcontractors (if necessary) in carrying out the request and confirm that it can be managed within the scope of existing supplier Agreements.

        If the work package cannot be scheduled without affecting some commitments, raise an issue.
