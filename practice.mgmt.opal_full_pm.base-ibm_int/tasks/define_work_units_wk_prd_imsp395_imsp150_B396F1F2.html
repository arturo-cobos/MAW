Task: Define work units and work products (IMSP395 and IMSP150)
Disciplines: Project Definition
Purpose
To expand the project approach to include enough detail that can then be used to create a high-level Work Breakdown Structure (WBS) and other work products.
Relationships
RolesPrimary Performer:
  • Project Manager
Additional Performers:
InputsMandatory:
  • Product Breakdown Structure (PBS)
  • Project Defined Process
  • Project Definition
    		Optional:
    • None
    Outputs
    • Project Definition
    • Work Breakdown Structure (WBS)
    • Work Product List
      Main Description
      Expand the work units -or candidates for sub projects that were previously outlined when defining the project approach. Expand them to such a level that:
      • broad estimates (effort and duration) can be done,
      • major dependencies are highlighted,
      • critical skills and resources are determined.


      Organize these work units into a high level schedule -or Planning framework- with intermediate milestones, and identify the work products related to these milestones. Document this Planning framework in the Project Definition and create the initial Work Product List.
