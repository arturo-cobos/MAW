Task: Develop the initial project quality plan (IMSP491)
This task builds the initial project quality plan for the project.
Purpose
To develop the initial project quality plan for the project.
Relationships
RolesPrimary Performer:
  • Project Manager
Additional Performers:
InputsMandatory:
  • Project Defined Process
  • Project Definition
  • Project Procedures Description
    		Optional:
    • None
    Outputs
    • Project Procedures Description
    • Project Quality Plan
      Main Description

      The plan describes the strategy that has been put in place to achieve the planned level of quality for projects and all work products they produce such as:

      • Plan to ensure that the deliverables will meet the acceptance criteria specified in the Statement of Work or Work Request within the project cost schedule
      • Plan to ensure that quality assurance, process assurance, and work product inspection reviews are performed according to the project's defined procedures
      • Plan to ensure that testing activities are performed according to the project's defined procedure
      • Ensure implementation of project procedures and evaluate their effectiveness
      • Ensure overall compliance to the project management practices and procedures


      Note: Peer review of Design, such as those produced by the Technical Solution Procedure (IMSP595) and the Product Integration Procedure (IMSP590). Gold/Silver Code for e-business applications (for e-business only) is also an example of work products which is considered mandatory for peer reviews.

      The planning of all Quality Management activities, including quality assurance, process assurance, work product inspections, defect management, risk management, root cause analysis, subcontractor management and testing activities within all projects.

      Refer to the Quality Planning Procedure (IMSP230) for more details.
