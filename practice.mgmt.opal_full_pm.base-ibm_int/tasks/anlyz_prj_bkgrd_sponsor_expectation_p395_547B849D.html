Task: Analyze project background and sponsor's expectations (IMSP395)
Disciplines: Project Definition
Purpose
To gain knowledge about the business needs that is driving the project and to understand the sponsor's expectations for the project.
Relationships
RolesPrimary Performer:
  • Project Manager
Additional Performers:
  • Opportunity Owner
  • Relationship Owner
  • Technical Solution Architect
InputsMandatory:
  • Account Profile
  • Project Charter
    		Optional:
    • Service Request
      Outputs
      • Conditions of Satisfaction
      • Operational Schedule
      • Project background analyzed
      • Project Charter
      • Project Control Book Initialized
      • Project objectives analyzed
        Main Description

        Gain understanding and reformulate as necessary:

        • The business context within which the project being defined is being set up.
        • The business problem that must be solved and any constraints that restrict potential solutions.
        • The sponsor's view of the target solution and of the overall approach to implement it.

        Review the Project Charter (or other documents such as service request if in an outsourcing environment) and then clarify the following details with the client:

        • The client's expectations in term of:
          • The time requirements
          • The estimate or quote due date
          • The required delivery and implementation dates for the deliverables
          • Any risks that the client perceives
          • Any questions and issues that the client has

        Gain an appropriate understanding of the requirements to enable the organization to develop an appropriate response, utilizing the Requirements Development and Management Procedure (IMSP576) Identify and document the organization and client acceptance criteria (for deliverables and work products) based on the client requirements and the work request.

        Identify the project's stakeholders as appropriate.

        Clarify business controls, including Sarbanes-Oxley (SOX) compliance and data security & privacy needs.

        Build the operational schedule which documents the list of work items each member of the proposal team has to achieve, along with the effort or timeline.
