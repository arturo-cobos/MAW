Task: Track and report status (IMSP525)
This task will update the project status with the tracked data for the corrective action(s).
Purpose

To track and report the status for the root cause analysis.

Relationships
RolesPrimary Performer:
  • Project Manager
Additional Performers:
InputsMandatory:
  • Project Evaluation Report
  • Project Status Report
  • Situation Analysis Documentation
    		Optional:
    • None
    Outputs
    • Project Status Report
      Main Description

      Update the project status report with the approved corrective action(s) providing situation and root cause analysis measures.

      Provide and report the results and effectiveness of the actions, using the situation analysis documentation and action control document. 

      Track and report the improvement information. 

      Steps
      Update the project status report

      Update the project status report, with the approved corrective action item(s), agreed to owner(s) and commit dates for resolution.

      Analyze root cause analysis measures

      Analyze situation (root cause) analysis measures, such as the following:

      • Number of situation (root cause) removed
      • Cost and Effort involved in the root cause analysis and resolution activities
      Update the project status report

      Update the project status report with the appropriate situation (root cause) analysis measures.

      Track and report status

      Track and report status of situation (root cause) analysis , using the situation (root cause) analysis and corresponding action items in the project status report.

      Track and report the results

      Track and report the results and effectiveness of the actions, using the situation analysis documentation and action control document.

      • Refer to the Action Management Procedure (IMSP605) for information about raising and managing actions.
      Track and report the improvement information

      Track and report the improvement information, from the situation analysis documentation or a project evaluation report created in a Project Completion Workshop.

      • Refer to the Project Close Procedure (IMSP240) for information about conducting a Project Completion Workshop and creating a project evaluation report.
      Place the measurements in the metrics repository

      Place the measurements associated with the situation (root cause) analysis in the organization's metrics repository.

      Key Considerations

      Reference OPAL Procedures:

      • Root Cause Analysis Procedure (IMSP525)
