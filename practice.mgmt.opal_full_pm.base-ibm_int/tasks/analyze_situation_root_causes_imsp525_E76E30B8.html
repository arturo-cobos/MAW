Task: Analyze situation root causes and determine actions (IMSP525)
Disciplines: Event Management
Purpose
To analyze root causes, determine corrective and/or curative actions, document the results of the analysis and share conclusions with project stakeholders.
Relationships
RolesPrimary Performer:
  • Project Manager
Additional Performers:
InputsMandatory:
  • Situation Analysis Documentation
    		Optional:
    • None
    Outputs
    • Action Log
    • Meeting Documentation
    • Project Status Report
    • Root Cause Analysis Record
    • Situation Analysis Documentation
      Main Description
      Interview selected team members to obtain their view of the situation, its causes and what should be improved. Analyze all relevant project documentation and correspondence to investigate the sequence of events which lead to the situation and determine the root causes of the issues.

      Prioritize the issues and determine corrective actions to resolve the key issue(s) and return the project to a sound position. Also determine curative actions to avoid recurrence. Discuss the conclusions with the project stakeholders and team members that are concerned, and finalize the plans to return the project to a sound position. Raise Change Requests if any elements in the project baseline are affected or impacted by the corrective and/or curative actions.

      Produce and distribute the Situation Analysis Documentation (usually confidential).
