Work Product Descriptor (Artifact): Change Request
A request to change some document or aspect of the project that has been placed under change control. An accepted change request may result in one or more change orders .

Main Description
A Change Request is a form that contains all the information required to support the decision making processes that are used to manage the unforeseen changes that inevitably arise during the course of all but the simplest of projects.

Key Considerations

Any stakeholder can raise a Change Request.

After a preliminary assessment of the proposal, the change control board decides whether the Change Request is to be approved for implementation (small changes only), approved for analysis (large changes only), deferred or rejected.

If an impact analysis is performed, the change control board evaluates the results and again decides if the Change Request is to be approved for implementation, deferred or rejected.

If it is decided that the proposed change will be implemented, the required Change Orders are raised.

The Change request is closed when all the corresponding Change Orders have been completed.

Note: The approach outlined here allows for one Change Request to be linked to more than one Change Order and for one Change Order to be linked to more than one Change Request. It is possible to adopt a simpler, but less flexible, approach in which one Change Request is linked to one (notional) Change Order and vice versa. In this case, there is no need for a separate Change Order work product.

Unforeseen change is being managed effectively.

Change Requests are being processed according to the rules documented in the Project Procedures Description.

Note that there may be more than one change management procedure. For example, Change Requests raised by the sponsor may be handled in a different way from those raised by a project team member or changes to the technical work products may be managed in a different way than changes to the project management plans.
