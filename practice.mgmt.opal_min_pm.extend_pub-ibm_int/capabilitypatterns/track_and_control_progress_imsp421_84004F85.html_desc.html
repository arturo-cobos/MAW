Activity: Track and control progress (IMSP421)
This activity validates progress data collected from team members, responds to variances, and reconciles the project plans for the future, as needed.

Description

This activity is based on the WWPMM work pattern M1: Track and control progress.

This activity is launched on a short-term, periodic basis (weekly, semimonthly, monthly), depending on the nature of what the project is delivering, in order to control project progress and to monitor the project's ability to meet the due dates within the financial targets .

Tracking progress is done by collecting actuals (e.g., effort spent on work items), comparing those data with the effort and duration planned to complete these work items, and then determining forecasts (e.g., estimate-to-complete or end-date for each work item). It is typically a bottom up activity, where progress on low level items of the project Work Breakdown Structure (WBS) are then rolled up to determine progress and estimates to complete of the higher activity levels of the WBS.

Note that the estimates to complete that are determined in this activity must be periodically validated by undertaking the activity "Assess estimating basis" which occurs next in this engagement model.

In a complex project Organizational Breakdown Structure (OBS) with a hierarchy of project organizational units (POUs) and subcontractors, project progress measurement needs to be rolled up from the bottom to the top of the OBS hierarchy also. Then, the tracking period and associated tracking process should be defined and documented by appropriate procedures in the project management system.

It is highly probable that, for any tracking period, the progress and estimates to complete will vary from what was planned. What is important is to focus on the critical path of the project schedule, to evaluate the project's ability to complete the work on time with the resources currently available or planned, and to keep the progress variances within limits that will not put the project at risk. See the WWPMM process "TC.30.20. Adjust project plans" for actions that may be considered.

Thus, controlling progress is done by evaluating options and making decisions to ensure progress variances stay within these limits, and then adjusting the project plans accordingly. Note that WWPMM uses the verb "adjust" if plans need to be slightly reworked in order to keep the project on track in spite of variances that have occurred, or will likely occur, and if this can be done without any impact on the commitments.

Note: The Individual Status Report, Sub-Project Team Status Report and Subcontractor Status Report (input work products to this activity) are created by project team members, sub-projects leaders and subcontractors, respectively. As such, they are external inputs to the project manager and will not be found as output work products within this PM engagement model.

The key outputs of this activity are the Operational Schedules, Milestone List and Project Management Schedule, adjusted as necessary, and Action Control Documents if appropriate.

Please refer to M1: Track and control progress on the WWPMM web site for the original work pattern steps that decompose this activity.

This activity includes the following:

  • Assess actual efforts and reconcile project plans:
    • Assess actual efforts and progress data
    • Respond to variance and create actions if necessary
    • Reconcile project plans for future periods
