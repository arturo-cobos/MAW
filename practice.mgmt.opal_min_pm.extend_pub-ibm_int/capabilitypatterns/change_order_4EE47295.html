Work Product Descriptor (Artifact): Change Order
A directive to implement a change resulting from an accepted change request.
Based on Method Work Product: Change Order
Main Description

A Change Order is a form that contains the information required to manage the implementation of all or part of one or more approved Change Requests.  It includes a specification of what must be changed and an high-level implementation schedule.

Key Considerations

Change Orders are created after Change Requests have been approved for implementation and are then tracked until completion.

Note: The approach outlined here allows for one Change Request to be linked to more than one Change Order and for one Change Order to be linked to more than one Change Request.  It is possible to adopt a simpler, but less flexible, approach in which one Change Request is linked to one (notional) Change Order and vice versa. In this case, there is no need for a separate Change Order work product.

All Change Orders are linked to Change Request that have been approved for implementation.
