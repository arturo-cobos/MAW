Main Description

The Work Product List contains the list and summary description of the work products, that are planned to be created and used by the project. The main sources for the Work Product List are:

  • Technical methods (for technical work products)
  • Product Breakdown Structure (PBS) (for components of the target solution)

The Work Product List contains intermediate work products as well as the deliverables.

To avoid ambiguities, WWPMM uses different terms for the list of the work products associated with the completion of the activities defined in the WBS (Work Product List), and the break down of the solution into components (Product Breakdown Structure). However, some constituencies use the terms “project PBS” and “solution PBS” to refer to these two work products, respectively.

Key Considerations

An initial Work Product List is created when building the planning framework. It is updated with additional items during Planning activities until the proposed Agreement is reviewed and possibly each time the Project Management Schedule is expanded.

The Work Product List is also updated with additional items, to be consistent with the related elements of the Work Breakdown Structure (WBS) and of the Product Breakdown Structure (PBS).

It is used to identify what needs to be done in change management according to the rules applicable for change control.

The Work Product List is written or at least reviewed by the project manager. Verification should be made against the Agreement and any methods (technical, project management, or others) used to plan the project.
