This task outlines the closure of the project control book and archival of project records.

Main Description

This task outlines the following:

  • Document the process, methods and tools utilized on the project in the Process, Methods and Tools (PM&T) Profile
  • Package all relevant evaluation information
  • Suggest process improvements
  • Submit the final project status report and project management schedule
  • Submit the approved project evaluation report
  • Close the project control book
  • Archive project records

Reference IGSDF - Using GBS CRM on Engagements for links to your local GBS CRM Playbook for specific guidance on CRM Step 9. Close Project for business rules on archiving projects.

Steps
Document the process, methods and tools utilized in the PM&T profile
Launch the PM&T Profile and fill out the appropriate data for the project.
Package all relevant evaluation information

Package all relevant evaluation information, in the project evaluation report, which includes:

  • Project results
  • Process evaluation
  • Lessons learned
  • Log of actions arising from previous work sessions
  • List of intellectual capital nominations
  • Final archive location of all the project records

Refer to the Organizational Measurement and Analysis Procedure (IMSP473) for collecting and analyzing project and organizational measures.
Suggest process improvements

Suggest process improvements, including:

  • Analyzing lessons learned to determine root cause of both positive and negative items
  • Identifying suggested improvements in procedures to address root causes

Prepare improvements in accordance with the Process and Technology Improvements Procedure (IMSP470).
Present the project evaluation report
Present the project evaluation report to the line of business management or quality assurance.
Update & submit the final project status report and project management schedule

Update and submit the final project status report and project management schedule to the line of business management, including project actual data and completion status.

The management approval can be provided either by e-mail or as an entry recorded in relevant meeting minutes. If the management rejects the completed project evaluation report, re-examine the evaluation, make any necessary changes to the report, and re-present it for review.

When management approval has been obtained, file the final project status report and project management schedule in the Project Control Book Repository or other location identified by the organization to store project data.

Submit the approved project evaluation report

Submit the approved project evaluation report to the organization's project data repository, along with information on accessing the project's defined project control book (PCB).

Refer to the Project Control Book Guideline for details on the PCB.

Refer to the Configuration Management Procedure (IMSP585) for managing documents and records.  
Archive project records
Close the project's defined project control book and archive project records.
