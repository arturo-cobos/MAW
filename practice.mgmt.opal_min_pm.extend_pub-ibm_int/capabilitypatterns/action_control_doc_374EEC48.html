Main Description

An action is any unanticipated work that must be assigned and tracked by the project manager.

The Action Control Document describes the actions and links them to their related issues or to other events requiring action tracking. Information in this work product should not duplicate information recorded elsewhere, such as in an Issue Document.

Key Considerations

The action is opened, assigned, and launched.

It is then followed up until completion.

When the action is completed:

  • If no effectiveness review is required, the status is changed to “closed.”
  • If effectiveness review is required, the status is changed to “completed.” Once the effectiveness has been reviewed, the status is changed to “closed.”
