Task Descriptor: Define estimating approaches and estimate tech size (IMSP380)
This task will define the estimating approaches, including the activities and resources required to perform the estimate, and will gather the information necessary to create an estimate.

Main Description

Prepare for creating an accurate estimate, using appropriate resources and available historic data.  Determine the sizing units, estimate the technical effort and its associated schedule, conduct an independent technical review of the estimate, document the estimate and the review, and place the estimate under version control.

This task includes the following steps:

  • Define estimating approaches & gather data
  • Estimate technical size and effort

Note: In the GBS PM Minimal (WWPMM) delivery process, the risk log is documented in the Event Log (under the Risk tab) .

Steps
Define estimating approaches & gather data
  • Define estimating approaches
    • Understand the scope of the project
    • Define the skills required to perform the estimate, and identify the resource required
    • Plan the activities required to perform this estimate
    • Identify any available historic estimate and actual data relevant to the client, technical environment or applications
    • Select at least two estimating approaches to use and document them in the Estimation Report
  • Gather data
    • Review the estimating work products and related actual data for information that may be of assistance to this project
    • Extract data from projects with similar characteristics
    • Advise the Estimating Practitioner on estimating process and approach, as necessary

The key output of this task is the Estimation Report.

Estimate technical size and effort
  • Estimate technical size and effort
    • Choose sizing units
    • Review the chosen Estimating approaches to ensure they are still valid
    • Size the project
    • Estimate the technical effort using the chosen estimating approaches and sizings, including effort associated with mitigating any risks identified
    • Review and rework estimates until they are acceptably close to each other
    • Document the estimates
    • Provide an indication of the accuracy of the estimate including at least the best, worst and nominal figures
    • Estimate the associated schedule for the technical effort
  • Conduct independent technical review
    • Hold a review to validate that the technical estimate is complete
    • Ensure the independent estimating practitioner chooses an approval option (approved, approved with conditions, or not approved)
    • Document the results of the review
  • Baseline technical estimate
    • Baseline the technical estimate by placing the estimation report under version control
  • Document any risks that are identified.  

The key output of this task is the Estimation Report. Additional outputs may include the Risk Definition or Risk Log.
