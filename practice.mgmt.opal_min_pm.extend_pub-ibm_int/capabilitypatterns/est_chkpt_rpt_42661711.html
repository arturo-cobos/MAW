Work Product Descriptor (Artifact): Estimation Checkpoint Report
This artifact measures the effort, cost, and schedule variance experienced on a project at each periodic estimation checkpoint. It also records the current estimate to completion during project delivery at each checkpoint.

Main Description

Periodic checkpoints during project delivery for estimation governance are required, where the status and content of the estimate to completion and the variance from the baseline are reviewed and approved. Governance confirms adherence to the procedure by performing the reviews and documenting the estimate details.

The Estimation Checkpoint Report records the values of the estimate including variance at each governance point within the estimate lifecycle. Each governance point is associated with a review of the estimate. The variance is a measure of the uncertainties and risks inherent in the baseline estimate and the estimate to completion as the project progresses. As the project progresses, the variance measured by the report will fall.

Key Considerations

The estimate variance as shown on the charts in the Estimation Checkpoint Report is expected to fall month to month as the project progresses due to successfully mitigating risks and completing work. If the estimate variance as measured in the report is not falling as the project progresses, find out why.

The current estimate at completion as shown on the charts is expected to track closely to the estimate baseline. If the estimate at completion is not tracking to the baseline, this is an early indication that some underlying assumption may have changed and you should find out why.
