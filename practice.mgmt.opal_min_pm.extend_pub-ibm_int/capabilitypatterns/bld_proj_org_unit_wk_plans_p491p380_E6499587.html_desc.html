Activity: Build project organizational unit work plans (IMSP491 and IMSP380)
This activity will refine the work units identified in the Project Definition, including stakeholder involvement, clarifying deliverables, organizing a team structure and WBS, estimating, staffing and separation of duties, understanding risks, and building a high-level schedule for the project.

Purpose
To expand upon the work done during project definition, so the project can be planned and started with an understanding of the deliverables, the schedule, the risks and the staffing requirements.

Description

This activity is launched when dates, products and services targeted for a given project organizational unit are known.

It can be launched either:

  • Several times for a project having multiple organizational units.
  • Only once for a simple project having one organizational unit. In such case, expected products and services are those expected by the sponsor as final deliverables from the project.

This activity includes the following:

  • Determine relevant stakeholder involvement
  • Clarify Deliverable Definitions and quality characteristics
  • Define deliverables components and conduct Method Adoption Workshop (MAW)
  • Organize work breakdown structure (WBS) and determine team structure
  • Define estimating approaches and gather data
  • Estimate technical size and effort
  • Document the preliminary milestone and develop staffing plan
  • Analyze assigned duties and obtain approval
  • Review the risks and build project management schedule
  • Estimate and review total project size, effort, schedule and cost
  • Finalize project organization unit (POU) planned schedule

Note that the goal here is to let each project organizational unit (POU) verify that what is expected from them, as described in the Project Definition, is realistic. In other terms, this activity consists of refining, in terms of activities, tasks, and intermediate work products, the work units that are identified in the Project Definition, to such a level that reliable estimates can be obtained, and to make sure that the work load and time frame are consistent, and that the resources will be available.

During a selling process, as described in the Client Value Method (CVM), this activity interacts with the phase: "Develop and agree to client solution" within the bid phase.

The key outputs of this activity are plans at the POU level: the Deliverable Definitions, the Product Breakdown Structure, the Work Product List, the team structure in the Organizational Breakdown Structure, the Human Resource Plan, POU estimates in the Financial Plan (will be consolidated later in task Finalize sponsor agreement) the Work Breakdown Structure, the high-level Project Management Schedule for the POU and the Project Management System Summary..

All these plans need to be detailed only to the level needed to estimate effort and costs with acceptable risk. They will be expanded when Starting the project.

Please refer to P1: Build project organizational unit work plans on the WWPMM web site for the original work pattern steps that decompose this work pattern.
