Main Description

A Deliverable Definition provides a description of a deliverable and its components, as well as the associated delivery and acceptance conditions.

A Deliverable Definition also provides the status of the deliverable.

A deliverable is any verifiable outcome defined in an agreement that must be produced by a project or a sub-project. Included are products or services provided by the delivery organization to the sponsoring organization or by a supplier to the delivery organization.

Key Considerations

The Deliverable Definition is created during project Defining and Planning activities.

It can happen, especially when customizing or creating a new solution, that some of the components of the definition are not available up front. In such case, a clear mention should indicate how and when these elements will be provided through some other deliverables that will be available prior to this being defined, according to the plan.

For example, “the acceptance criteria for the network will be determined and agreed to after a prototype has been built and a study report has been delivered and validated.”

The Deliverable Definition is progressively expanded according to the Agreement or until deliverable characteristics, description and acceptance criteria are clear enough and accepted by the sponsor (see the Introduction of DM.10.10. Define deliverables).

At the project level:

When building the Agreement, verify that the Deliverable Definition gives a clear and complete view of the deliverable from an acceptance viewpoint.

Especially if the Deliverable Definition is derived from a standard definition, verify that all necessary modifications have been made to meet the project objectives and constraints.

During the project, verify that the Deliverable Definition is completed according to the plan.

Easy access to the history must be kept to be able to check that changes to the definition have been performed under appropriate change control.
