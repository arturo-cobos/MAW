Main Description

Each project organizational unit has a Project Management Schedule. It defines the planned start and finish dates for, and the dependencies between, all the work units for which the organizational unit is responsible. These work units, which all together define the work to be accomplished, are often presented in a hierarchy made of phases, activities, and tasks.

The scope of the schedule is:

  • The technical work, if any, being carried out by the project organizational unit.
  • The project management work being carried out by the project organizational unit.
  • In summary form, the work being carried out by any reporting project organizational units; i.e., organizational units that are one level below the unit in the Organizational Breakdown Structure (OBS).

Key Considerations

The Project Management Schedules, one for each project organizational unit, are developed in two stages:

  • A draft schedule, that includes the work that is to be carried out by the organizational unit, is created for each organizational unit, more or less independently of each other.
  • A summary of the work being carried out by any reporting organizational units is added and each schedule is adjusted and finalized, from the bottom of the Organizational Breakdown Structure (OBS) up to the top, to ensure that the set of Project Management Schedules is consistent.

Individual Project Management Schedules are developed from the Work Breakdown Structure (WBS) using a five step process:

  • The activities defined in the WBS are organized into work units.
  • The dependencies between work units are identified.
  • The amount of effort or duration of each work unit is estimated.
  • Resources are assigned by category.
  • The start and finish dates of each work unit are scheduled.

The steps are repeated until an acceptable balance is achieved between duration and resource utilization. This usually takes several iterations.

In simple projects, and when it makes sense to plan and track individually each work item, the work units of the Project Management Schedule represent such work items.

When it is not appropriate to plan individually each work item in a Project Management Schedule, the work unit represents the lowest level to which dependencies must be tracked. In that case, the tracking of individual team member progress, against the work items which have been assigned to them, is usually done using an Operational Schedule. The Project Management Schedules are updated regularly, often weekly, to reflect progress.

Although they can be produced manually, Project Management Schedules are usually built and maintained using scheduler tools.

The accuracy and completeness of the Project Management Schedules is verified by inspection and by ensuring their consistency with the WBS and with each other.
