A report, usually prepared monthly, by the top-level project manager for the line of business management. It documents the status, progress/ accomplishments, and forecasts for the end of the project.

Main Description

The Project Status Report is a scheduled report that is created by the top-level project manager for the project. The Project Status Report includes:

  • Status that describes the state of the project vs. the project plans
  • Progress that describes what the project has accomplished
  • Forecasts that predict future project status and progress based on current project data.

This artifact is used to summarize the team's key accomplishments for the week, accomplishments planned for the next week, stories (and associated key tasks) completed within the iteration, and any critical blockers (that are recurring or need additional assistance from sponsors to be removed).

At the end of the project, a completion report is produced that covers these same subjects areas but at a project summary level.

Key Considerations

The Project Status Report is created by the project manager who hash responsibility for the overall project. The project manager follows the procedures and any business unit policies as described in the Project Management System Summary or the Project Procedures Description in preparing the report. The report is created by aggregating sub-project/team and subcontractor status reports into the project status report. Augmentation with graphics is helpful in showing progress or potential problems. Project Status Reports are often reviewed with the line of business management prior to being used as a source to create a sponsor review report.
