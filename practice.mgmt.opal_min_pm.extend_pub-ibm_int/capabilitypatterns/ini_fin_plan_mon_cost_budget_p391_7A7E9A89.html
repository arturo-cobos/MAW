Task Descriptor: Initiate financial plan to monitor costs against budget (IMSP391)
This task will describe how the actual costs will be monitored against the budget and how the basis of the estimate will be monitored throughout project delivery, including reporting on contractual budget obligations.

Main Description

The key outputs of this task are the Project Management Schedule, and the Financial Plan which may be part of the Project Management System Summary. 

The project may also choose to document a financial project procedure, as part of the Project Procedures Description. Note:  All plans in the GBS PM Minimal (WWPMM) delivery process are documented within the Project Management System Summary (PMSS).

This task includes the following:

  • Establish the financial tracking plan and financial tools that will be used for the project.  Note:  See the appropriate GBS CRM business process for the financial tools used by your community.
    • Register the project in the portfolio reporting tool and establish the financial measurement baseline and initial forecast
    • Describe how the actual costs will be monitored against budget, including the method of reporting on contractual budget obligations; for example, 80% budget spent, and estimate at completion. 
  • Describe how the basis of the estimate will be monitored throughout project delivery, and delegate to a team member who will:
    • Review the estimation approaches taken in the creation of the estimate and sets up the estimation factors report.
    • Determine if any sizing or Estimation Factor has altered significantly since the estimate was validated. If a significant change has occurred or a risk has been identified, investigate any possible scope changes.
    • Set up a mechanism for the collection of actual data for each Estimation Factor:
      • Sizing parameters, effort where appropriate
      • By complexity where appropriate
    • Note: Refer to the 'Monitoring Project Progress Through Estimation Factors' Guideline for further details of tracking the estimate basis.
  • Update the project management schedule and financial plan for the cost baselines.
    The project management schedule will be used to track and monitor estimated to actual start and completion dates for activities, tasks, and milestones. The financial plan is a time phased budget that will be used to measure and monitor the project's cost performance.

A Project Management Administrator (PMA) may be used by some projects to assist the Project Manager setup the financial tool and monitor financial processing.  This decision will usually be made by the Relationship Manager (e.g. Partner). This decision will usually be made by the Relationship Manager (e.g. Partner).   The following steps would be taken:

  1. PMA Assignments:  The PMA assignment process focuses on the activities associated with staffing a PMA on a project. This process includes the initial request through roll-off, once the project is complete.
  2. Pre-Contract Setup:  The tasks associated with this process are performed once a PMA is assigned to a project. This is an information gathering and preparation phase.  The data gathering and decisions mde during this phase will determine the duties performed by the PMA on the project.
  3. Initial Contract Setup:  Once you have an understanding of the scope of the PMA responsibilities, the project resource mix and project anomalies, the project will be created in the appropriate financial management tool and the initial project forecast will be validated.
