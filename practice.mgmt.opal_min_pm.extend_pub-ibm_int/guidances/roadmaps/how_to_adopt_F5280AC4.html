Roadmap: How to Adopt the Minimal Project Management (OPAL/WWPMM) Practice
This roadmap describes how to adopt the project management activities, as described in OPAL and WWPMM, for GBS small projects, as defined by the Global Project Launch Policy, regardless of any customization that might be needed to integrate activities specific to a service line or business unit.
Main Description

Getting Started

A project management schedule is usually made up of technical activities and project management activities where some project management activities are highlighted, but typically the rest are not detailed. Therefore, this minimal set of PM activities is provided more for reference than to use as a template in building a Project Management Schedule.

The Minimal PM practice can be integrated with one or more of the other delivery processes during the method tailoring process to incorporate the WWPMM and OPAL Work Products into the overall project effort. The extents to which the PM activities and tasks need to be included depend on the nature, scope, and size of the project. To make the tailoring decision process easier, many of the newer delivery processes now include automatic insertion of these materials into the proper place within the overall WBS. Many of the legacy methods already include some form of PM as part of their work breakdown structure.

The Minimal PM practice is based on OPAL and WWPMM; therefore some familiarity with OPAL and WWPMM is assumed. It is recommended that an experienced Project Manager work with the team tailoring the model to determine the appropriate level of project management tasks and work products that will be required for a project.
