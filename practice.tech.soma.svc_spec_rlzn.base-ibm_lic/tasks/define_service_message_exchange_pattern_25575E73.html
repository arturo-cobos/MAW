Task: Define Service Message Exchange Pattern
Defines the message exchange pattern (e.g. request-reply, fire and forget) applicable to the service.
Disciplines: Technical Architecture
Purpose
To define the message pattern for service operations.

        Main Description

        Messages between communicating services and components are a critical part of a SOA. Messages between consumer and provider can be exchanged using a variety of patterns, including request-reply, fire and forget, and publish-subscribe.
        This task focuses on:

        • Selection of the message exchange pattern to be supported by the service.
        • Subsequent specification of related message constructs (i.e. response message

        Outcome of this task may affect architecture decisions such as the need to support a service publish-subscribe mechanism in middleware.
