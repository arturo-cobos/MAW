Task: Define Service Interface Messaging Style
Specifies the messaging style and other message transfer related requirements, such as support for attachments, reliable messaging, etc.
Disciplines: Technical Architecture
Purpose
To establish the messaging protocol for services.
Relationships
RolesPrimary Performer:

    • Service Specification
      Main Description

      In this task, we decide what messaging styles must be supported. For many engagements, we may make these decisions at the architecture design level, across the whole portfolio. In more complex engagements, we may need to make decisions at the service level, depending on their exposure scope. service level Message styles in this context may include:

      • Message structure (e.g. SOAP, XML, fixed format)
      • Interaction model (e.g. SOAP, Representational State Transfer - REST)
      • Reliable Messaging (e.g. JMS, WS-RM)
      • Attachment support (e.g. MTOM, WS-I Attachments/MIME, etc.)
      • Security requirements and capabilities (e.g. WS-Security)
