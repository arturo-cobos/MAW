Task: Map Service Operation Message to Existing System Interfaces
This applies service operations that are implemented by leveraging existing system interfaces. The parameters of the service operations are mapped to the parameters of the existing asset interfaces.
Disciplines: Technical Architecture
Purpose
To perform a data translation to map the service operation signature to the existing asset interface signature.

      Main Description

      The service interface and its parameters (data elements) are mapped to the interfaces of the existing assets (that are planned for being leveraged). The interface-data mapping identifies areas where data transformation needs to take place. The output of a service operation should be mapped to the input of an existing asset / system interface. The output of an existing system interface should be mapped to the output of the service interface / operation.
      Key Considerations

      There can be at least three different scenarios:

      (a) You find a match of data elements and their "formats are same" meaning element A from Step 1 and Element C from Step 2 are both of same data type "character". Here no transformation is required

      (b) You find a match of data elements but their "formats are not the same" meaning element X from Step 1 is a decimal and element Y from Step 2 is a "character" i.e. different data types. This means transformation is required

      (c) You found an element M in Step 1 but no corresponding element in Step 2. Then you would have to request the appropriate back-end system to provide that data element and if possible in the required format of Step 1 - this is to avoid any transformation requirements.
