Task: Map Service Operation Parameters to Canonical Message Model
Use the entity definitions in the enterprise message format to define the service operation parameters.
Disciplines: Technical Architecture
Purpose

To align the service specifications with the enterprise message format and canonical message model.

        Main Description

        The parameters of the service operations are mapped to the industry model or to the enterprise message format (whichever is applicable). This information will be used in data mapping spreadsheet (which is used for capturing data elements required by the interface, data elements supported by the relevant backend systems, data elements supported by Industry specific schema e.g. ACORD) . If data elements identified by the Service Operation are not currently provided by the backend systems, then it is essential to notify the existing asset provider to provide these data elements. The next key aspect of this activity is that if data types of service interface are different from the data types supported by existing systems - will indicate that data transformation rules are required. These transformation rules are recovered for data transformations.
