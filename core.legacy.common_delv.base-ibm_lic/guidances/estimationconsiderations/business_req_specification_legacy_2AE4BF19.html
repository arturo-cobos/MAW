Main Description
The estimated time to assemble or package the Business Requirements Specification varies for all projects. The deliverable is created from:
  • Material provided by the customer, and/or
  • Material developed by the project team with the customer
 
As defined in the deliverable template, many possible work products can be used to create the deliverable. The work products used for an individual project will depend on the project size, project complexity, project type (new development, legacy system enhancement, maintenance, etc).
 
The packaging time varies based on the many project factors, including:
  • The number of work products to be incorporated within the deliverable for a particular project.
  • The complexity of the work products being incorporated within the deliverable
    • The number of requirements
    • The complexity of the requirements
    • The format of the requirements (either from the customer or developed with the customer). For example, text requirements are normally easy to assemble.
 
Based on the factors above, the estimated packaging time for the deliverable is between 6 and 24 hours.
