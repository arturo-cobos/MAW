Deliverable: High-Level Solution Plans (DCD 114)
This deliverable provides a consolidated collection of the recommended plans for developing and deploying the solution.
Purpose
The purpose of this deliverable is to present recommendations for moving forward with the development and deployment of the solution and gain a shared understanding among all project stakeholders of the appropriate engagement process for the remaining project phases. It accomplishes this via its documentation of the scope, approach and expectations that form the basis for estimating the release phases (Micro Design, Build Cycle, and Deployment) for the next scheduled functional segment (release) of the solution.

Main Description
This deliverable expands upon the development and deployment strategies defined and approved in earlier phases. This should be a high-level document, based on the initial design work; however, it should expand on what may have been outlined during the requirements gathering and initial scoping as conducted in the Solution Outline phase. Additional detail will be added to this document during the detailed design efforts in the Micro design phase.
 
This document contains some or all of the following plans.
  • A Release Plan describing the solution’s development approach. This should define the number of releases for the solution along with the business and data migration functions to be allocated to each. In addition, an estimate of the phases, activities and tasks required to complete each release cycle should be provided. This plan will set the scope for the design and development activities in later project phases.
  • A Deployment Plan for the solution. This document section defines the chronological sequence in which the solution will be installed in the client’s locations and the tasks, resources, and methods that will be used for the deployment. The plan will serve as the master document to guide the solution’s deployment.
  • A Usability Design & Evaluation Plan for verifying that the solution’s usability requirements and objectives are met. It defines major usability engineering design and evaluation activities, required resources, critical success factors, preliminary schedules, and key design points.
  • An Operations and Support Plan confirming the organizations responsible for operations, training, and other support responsibilities for the deployed solution.
  • A Quality Plan confirming the relative priorities of various attributes of quality to which the project team will focus.
 
Deliverable Properties
External Description

The purpose of this deliverable is to define how the solution will be implemented and deployed as well as to prepare and set expectations for the known administrative and technological resources needed to execute the plans.  The plans, developed at a summary level during the earlier phases, will be refined in preparation for moving into the release phases.  This deliverable serves to document

  • Refinements to the solution Release Plan further defining the number of releases for the solution, the business and data migration functions to be developed in each, and their relative timings. This information is used to set the scope of the solution project and divide the work into manageable units. It confirms the functionality intended for the release next to be further developed.
  • Refinements to the Usability Design and Evaluation Plan specifying how usability will be evaluated throughout the design and development of the solution and identifying the schedules and task assignments for this plan.
  • Revisions to the solution’s Deployment Plan further defining the tasks, schedules, and responsibilities for the roll out of the solution to all required client locations. It confirms the intended scope and approach for deployment for the release next to be further developed.
  • Refinements to the Operations and Support Plan confirming the organizations responsible for operations, training, and other support responsibilities of the deployed solution.
  • Refinements to the Quality Plan confirming the relative priorities of various attributes of quality to which the project team will focus.

Contents of this deliverable will be further detailed during Micro Design.  The intent of this deliverable is to provide the basis for the scope of work to be included in the release next scheduled to be further developed.  Revisions to this deliverable may be produced during the Deployment Phase to finalize details of the release to be scheduled for further development after the release completed in the Deployment Phase prior to starting its Micro Design phase.

Packaging Instructions

The final document will be delivered to the customer in a reproducible format.  The final report may also be delivered in hard copy format printed in a single, bound document.

The following work products are the inputs to the various report topics or sections.  The work products should be completed at a high level.  They will be iterated and more fully developed in later phases of the engagement, as the need for greater detail is identified.  Some work products may be extensions and/or revisions to previously generated work products while others may be separate and distinct instances specific to this project and/or phase.

For short engagements utilizing a rapid development approach, only the “Deployment Plan” report topic should be included in this deliverable.
