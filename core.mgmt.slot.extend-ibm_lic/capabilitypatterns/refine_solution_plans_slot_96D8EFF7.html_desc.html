Capability Pattern: Refine Solution Plans
DescriptionWork Breakdown StructureTeam AllocationWork Product Usage
Purpose
The purpose of Review Solution Plans is to:
  • Ensure that the release plan correctly sets the scope of the project and divides the work up into manageable units. This provides the customer and development team with an understanding of what is being delivered during each release. The customer will know when capability will be delivered, can provide feedback to the planning process, and can make business decisions based on this knowledge. The development team will have a clear understanding of what is to be developed and by when, thus establishing development goals on which the team can focus.
  • Ensure that the system test plan correctly establishes the timing, goals, hardware and software environment requirements, and resource needs for the test efforts. This provides the basis for planning future release test efforts.
  • Ensure that the acceptance test plan contains the applicable test scenarios. This allows the customer and development teams a clear understanding of precisely what function the system must implement in order to pass customer acceptance.
  • Ensure that the usability design and evaluation plan correctly reflects the current design and evaluation plans. This allows proper allocation and scheduling of resources, clear and shared understanding of how usability and user-interface design will be addressed, and ensures that the implemented system meets usability requirements. In addition, the usability design and evaluation is used to estimate how much usability will impact the project cost, project duration and required skills and resources.
  • Ensure the education training plan reflects the function in the planned releases. This allows proper training to be delivered to users at the right time.
  • Ensure the systems management plan correctly reflects the systems to be released. This provides the basis for the selection of system management tools, the selection of systems management processes, and the selection and definition of job descriptions to support the IT system.
  • Develop a top-down estimate of the next release.
Description
The release plan is reviewed by the project manager and the sponsors. Together, they ensure that the overall plan delivers an application which meets the requirements within the available time and budget.

Plans for testing the product of each development iteration, build, and release should be reviewed to ensure they match the release content.

The acceptance test is a public test, and may even be referred to contractually. It is dependent on release content and should be verified.

Usability implementation plans should be review to ensure that they reflect project realities.

Revise the deployment plan based on any new information that is available.

The deployment plan should be reviewed at each deployment, but will probably require little or no change.

The systems management plan should be reviewed to ensure it provides a detailed framework adequate to ensure the system management related nonfunctional requirements can be met.
