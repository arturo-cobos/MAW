Deliverable: System/Component Architecture (DCD 231)
The System/Component Architecture document defines the project’s architectures at both the system level and the component level.
Purpose

The document is used to define the architectures needed for project development. The project architectures, along with the requirements, and the Requirements Traceability and Verification matrix define the project’s technical baselines.

  • The project’s system baseline, which is reviewed at the System Requirements Review, may include:
    • System / Component Architecture document (at the system level)
    • Technical Solution Decisions Document
    • System Requirements Specification
    • Requirements Traceability and Verification Matrix (at the system level)
  • System / Component Architecture document (at the system level)
    • Technical Solution Decisions Document
    • System Requirements Specification
    • Requirements Traceability and Verification Matrix (at the system level)
  • The project’s component baseline, which is reviewed at the Preliminary Design Review, may include:
    • System / Component Architecture document (at the component level)
    • Technical Solution Decisions Document
    • Component Requirements Specification
    • Requirements Traceability and Verification Matrix (at the component level)

The system level architecture is defined with the system requirements to form an integrated solution that describes the system baseline. The component level architecture is defined with the component requirements to form an integrated solution that describes the component baseline.
