Deliverable: System Test Architecture (DCD 229)
The System Test Architecture document defines the architecture needed to perform all “end-to-end” system testing.
Purpose
The document describes the System Test architecture(s) to allow for verification/ validation of all requirements as defined in the project’s Requirements Traceability and Verification matrix. The System Test Architecture is reviewed at the Preliminary Design Review.
 
Examples of the test phases that can be covered by a System Test Architecture include:
  • Systems Integration Tests
  • Performance / Stress Tests
  • User Acceptance Tests
  • Operability Tests
  • Pre-Production Tests
  • Other similar test phases, which are part of your project.
 
The Systems Engineering team should develop this document in conjunction with the Test Manager and his/her team.
