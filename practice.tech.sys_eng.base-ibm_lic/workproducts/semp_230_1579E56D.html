Deliverable: Systems Engineering Management Plan (SEMP) (DCD 230)
System Engineering Management Plan (SEMP) defines the process that the Systems Engineering and Architecture team will follow for a project.
Purpose

The SEMP describes the plans, processes, and procedures required for the team to create and maintain the solution definition, development, test, integration, acceptance, and technology refreshment for project execution. It also provides the description of the process tailoring to be performed on the project to provide the project management team an understanding of the technical activities, tasks, deliverables and reviews that will be performed by the project team. The SEMP is complementary to the Project Management System Summary (PMSS), with a focus on the system development plan instead of the overall project plan.

Packaging Instructions

The SEMP Template is intended to document approved Systems Engineering changes for use when a project team is working within an established organization where the technical solution delivery approach is already defined.

The Technical Management Plan (TMP) Template is the complete description of plans, processes, and procedures required for the team to create and maintain the solution definition, development, test, integration, acceptance, and technology refreshment for project execution. 

The TMP integrates the SEMP with other technical planning aspects to provide a single deliverable with all technical plans.  Projects should develop either the SEMP or the TMP, but not both.    Which template to choose is based on whether the remaining, non-systems engineering technical plans are already in place.  For large, complex projects, including application development, package implementation, or systems integration projects, the TMP Template should always be used.

See guidance within the SEMP and TMP templates for additional notation and information.
