Deliverable: System Requirements Specification (DCD 201)
The System Requirements Specification (SRS) contains all of the system requirements for a project
Purpose
The purpose of the System Requirements Specification (SRS) is to specify the system level requirements for a project. The system level requirements will be developed from the customer / business requirements that were provided by the PDTL. The System Requirements will be reviewed at the System Requirements review (SRR). When the PDTL and the SPM approve and baseline the requirements at the SRR, then the document and the requirements will be placed under change control.

Main Description
The SRS, together with the System Architecture, and the Requirements Traceability and Verification Matrix, form the system technical baseline. This baseline will be reviewed and approved at the System Requirements Review (SRR).
