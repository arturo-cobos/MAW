Discipline: Project Definition
This discipline contains the processes for transforming a business need into a clearly defined strategy for providing a solution which meets the business need.
Relationships
Tasks
  • Adjust project organization and future sub-projects
  • Analyze project background and sponsor's expectations (IMSP395)
  • Create high-level product breakdown structure (IMSP395)
  • Create OBS, sub-projects and document decision structure (IMSP395)
  • Define work units and work products (IMSP395 and IMSP150)
  • Document approach and steps to deliver the target solution (IMSP395)
  • Document real project scope boundaries
  • Evaluate options and discuss project objectives (IMSP395)
  • Plan the defining activities (IMSP395)
  • Refine project approach and planning framework
