Work Product Descriptor (Artifact): Requirements
This artifact is used to extend the select user stories and prototypes, or a group of select user stories and prototypes, usually when the story is more complex and/or needs to be communicated to parties outside of the immediate team, or for ongoing documentation. This may include a System Context Diagram to identify interfaces that will be required (which may be embedded in the logical scope section of the Agile Project Definition and Charter)to select the use cases and use case models.
