Phase: Manage the Project
Activities which are either time driven (such as status reports) or event driven (such as change requests) are represented in this phase to accommodate level of effort planning for their accomplishment throughout the life of the project.

Description
Activities and tasks in this phase are initiated by event triggers rather than through a start or completion criteria associated with a predecessor activity or task. These triggers typically may occur at any point during the execution of other project tasks and usually involve a decision point or justification requirement that is an offshoot of the task being performed.

It is essential that some form of configuration management procedures are in place initially, and they are then used on an as-needed basis. Project status will be monitored on a time driven basis to provide status and financial information. Issues, incidents and changes will be handled as they occur.

This phase contains the following activities:
  • Monitor the Project: Keep a continuing watch on the project status and financials.
  • Handle Exceptions: deal with issues, actions, incidents, and change requests.
