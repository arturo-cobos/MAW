Task Descriptor: Perform Component Realization (R4SC)
Identify and refine classes for Custom Development and Integration Scenarios through utilization of appropriate R4SC patterns.
Based on Method Task: Perform Component Realization (R4SC)

Steps
Define Component Elements Using Scenarios

Define the component elements such as business objects, data objects, business and data mediators, and business and data mediator functions.

  1. Task: Define Component Elements Using Custom Development Scenarios
  2. Task: Define Component Elements Using Integration Development Scenarios
Perform Analysis to Design Transform

All objects/components generated during analysis can be transformed into design elements via stereotypes/profile previously defined. Prepare for analysis to design transformation and then perform the transformation.

  1. Task: Prepare Analysis To Design Transform
  2. Task: Transform Analysis To Design
Refine Component Elements Using Custom Development Scenarios

Here we will refine Business Object, Business Mediator &  Business Mediator Function

  1. Task: Refine Component Elements in Service Definition and Adaptation Layer
  2. Task: Refine Component Elements in Business Mediaton Layer
  3. Task: Refine Component Elements in Integration Mediaton Layer
  4. Task: Develop Interaction Diagrams for Custom Development Scenarios
Refine Component Elements Using Integration Development Scenarios

This activity consists of refining Data Object , Integration Mediator & Integration Mediator Function.

  1. Task: Refine Component Elements in Service Definition and Adaptation Layer
  2. Task: Refine Component Elements in Integration Mediaton Layer
  3. Task: Develop Interaction Diagrams for Integration Development Scenarios
Prepare Design to Code Transform

All objects/components generated during design can be transformed into code via stereotypes/profile previously defined. Prepare for design to code transformation.

  1. Task: Prepare Design to Code Transform
