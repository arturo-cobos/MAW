Task Descriptor: Analyze critical situations (IMSP525)
This activity will investigate items that are causing concern about a potentially adverse project situation in order to resolve them.
Based on Method Task: Analyze critical situations (IMSP525)

Steps
Investigate issue and plan for situation analysis
Investigate issues related to the situation to be analyzed and plan for the analysis. Understand the concerns about the project situation; for example recurrent conflicts, sponsor uneasiness, estimates to complete which keep increasing, etc. Collect data from the project plans and records to learn more about the project situation, and build an agenda for specific project reviews and interviews for analyzing the causes of the situation. Acquire any additional resources required to perform the situation analysis.
Analyze situation root causes and determine actions

Analyze root causes, determine corrective and/or curative actions, document the results of the analysis and share conclusions with project stakeholders. Interview selected team members to obtain their view of the situation, its causes and what should be improved. Analyze all relevant project documentation and correspondence to investigate the sequence of events which lead to the situation and determine the root causes of the issues.

Prioritize the issues and determine corrective actions to resolve the key issue(s) and return the project to a sound position. Also determine curative actions to avoid recurrence. Discuss the conclusions with the project stakeholders and team members that are concerned, and finalize the plans to return the project to a sound position. Raise Change Requests if any elements in the project baseline are affected or impacted by the corrective and/or curative actions.

Produce and distribute the Situation Analysis Documentation (usually confidential).

Report conclusions and determine actions
Discuss the situation analysis with the originator and agree on the corrective actions to address the incident.  Assemble all findings (if not completed) and update the appropriate documentation. Notify all individuals who may have an assignment for the corrective actions and distribute the situation analysis documentation to all interested parties.
Track and report status
  • Update the project status report with the approved corrective action(s) providing situation and root cause analysis measures.
  • Provide and report the results and effectiveness of the actions, using the situation analysis documentation and action control document. 
  • Track and report the improvement information.
