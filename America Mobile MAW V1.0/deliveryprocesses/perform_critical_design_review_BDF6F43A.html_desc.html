Activity: Perform the Critical Design Review
Perform the Critical Design Review to establish the fourth System Engineering standard checkpoint.

Purpose
The design baseline describes how the components will meet their requirements. The design baseline should be created from and traceable to the component baseline. The design baseline will be used to develop and test the component solution.

Parent Activities

Description

The design baseline is the fourth of six systems engineering, technical baselines. The design baseline is created by the component development and test teams. It consists of the component designs and test plans and the system test plans. The design baseline defines how each of the components will meet its requirements. The system test plans will describe how the components will be tested together in order to meet the system requirements.

The six systems engineering, technical baselines are:

  1. The customer baseline (which is defined at the Business Requirements Review),
  2. The system baseline (which is defined at the System Requirements Review),
  3. The component baseline (which is defined at the Preliminary Design Review),
  4. The design baseline (which is defined at the Critical Design Review),
  5. The test baseline (which is defined at the Test Readiness Review), and
  6. The production baseline (which is defined at the Production Readiness Review).
