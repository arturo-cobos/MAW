Activity: Perform the Test Readiness Review
Perform the Test Readiness Review to establish the fifth System Engineering standard checkpoint.

Purpose

The test baseline describes how the components will tested to verify that they meet their requirements. The test baseline should be created from and traceable to the design baseline. The test baseline will be used to test some level of the integrated solution.

Description

This activity details the actions to prepare for, conduct, and cleanup after the Test Readiness Review. This review verifies that the project test team is ready to begin testing.

The test baseline is the fifth of six systems engineering, technical baselines. The test baseline is created by the implementation and test teams. It consists of all the various implementation elements required for the desired level of integration along with the test execution plans and the system test plans. The test baseline defines how we will validate that each implementation element will meet its requirements. The system test plans will describe how the integrated components will be tested together in order to meet the system requirements.

The six systems engineering, technical baselines are:

  1. The customer baseline (which is defined at the Business Requirements Review),
  2. The system baseline (which is defined at the System Requirements Review),
  3. The component baseline (which is defined at the Preliminary Design Review),
  4. The design baseline (which is defined at the Critical Design Review),
  5. The test baseline (which is defined at the Test Readiness Review), and
  6. The production baseline (which is defined at the Production Readiness Review).
