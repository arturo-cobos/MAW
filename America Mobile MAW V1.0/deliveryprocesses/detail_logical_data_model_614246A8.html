Task Descriptor: Detail Logical Data Model
Further refine and develop the logical data design of the database based on the business requirements for the solution.
Based on Method Task: Detail Logical Data Model

Steps
Update the Model to Reflect Changes
Check that the model reflects the current requirements and design, and make appropriate updates.
Resolve Many to Many Relationships into One to Many Relationships
Add Remaining Attributes and Keys
  • Add any missing attributes. Define any additional primary and alternate keys.
  • Decompose attributes that do not represent atomic business notions.
  • Capture data attributes related to different states of an entity.
Normalize to Third Normal Form or Higher
Once the entities, attributes, and keys are finalized, the logical data design should be normalized to third normal form (3NF) or higher as input to physical database design.
Update the Data Dictionary
Ensure the data dictionary is updated to reflect changes in the logical data design. The data dictionary can be used to reconcile the logical database design with the solution's functional model.
