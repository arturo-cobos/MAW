Activity: Initiate Project Delivery Phase (IMSP391)
This activity prepares the project context and commitments, and then plan the activities necessary to get the project up and running.
DescriptionWork Breakdown StructureTeam AllocationWork Product Usage
Purpose
To ensure an appropriate understanding of the project, and plan the activities necessary to get the project up and running.
Relationships
Parent Activities
  • Start the Project
Description

This activity is based on the WWPMM work pattern S1: Initiate phase.

This activity is the first to be launched when starting the execution of a project (or of a new major technical phase or sub-project), and needs a first set of project team members (at least the project manager) who will plan the activities necessary to get the project up and running.

At a minimum, this starting team will:

  • Ensure new project is registered and material is transferred
  • Launch the project.
  • Understand scope and risk, confirm method used, and develop startup plans
  • Understand agreement and other definition and planning elements
  • Review the outcome of the seven keys based assessment, performed at the time of project launch list of questions aimed at identifying project risks and concerns
  • Refine or prepare initial risk management plan
  • Confirm the solution engineering methodology selected
  • Develop the startup plan
  • Establish the technical environment for start-up phase
