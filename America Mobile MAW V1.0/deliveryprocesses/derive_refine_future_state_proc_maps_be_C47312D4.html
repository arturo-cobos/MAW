Task Descriptor: Derive or Refine Future State Process Maps with BE Lifecycle Models
Derive a 'starter' process model or refine existing process models with BE lifecycles to produce a holistic process model

Steps
For process modeling as an entry point:
Refine the Process Definition to align with the business entity states

Process Tasks may be merged or pushed to a lower hierarchy.

  1. Map Process tasks to Entity State transitions
  2. Consider merging tasks that appear on the same state transition to a higher level task. Make the new task a sub-process, with the merged tasks as tasks of the sub-process
  3. Continue for each state transition to obtain the revised Process Definition
  4. Introduce BE typed and state-tagged global repositories in the process for new states that were introduced to support entity communication/coordination or parallelism or for queuing between tasks. For example, 'Active Folios' represent a global repository element of Folios in the Active state. This Active Folios can be used to stitch together the Check In and Check Out processes of a hotel stay process.
For information analysis as an entry point:
Derive the starter-set of process model(s) from business entity lifecycle models
  1. For each business entity state transition, add a suitably named task. The name will typically be a verb describing the exact work that needs to be done to take the BE from the source to the target state. Do this for each BE and for each transition where the BE appears in the input and output of the task with the appropriate state change.
  2. Merge all tasks with the same name (but appearing on different Business Entity transitions) to a single task with that name. This will move all the BE transitions as input/output of a single task. Semantically, this means that a single task is causing state transitions of multiple Business Entities.
  3. Create a control flow of tasks for Business Entities whose input and output states are identical for the Business Entities it processes.
  4. Consider upgrading some of the tasks and control flows into unique processes.
  5. Add ‘start’ and ‘end’ nodes to the unique processes.
