Activity: Perform User Acceptance Testing
Examination of the various elements of the solution release about to be moved to production by the users and sponsors. These elements include the UI functionality as well as any reports and user support materials associated with this release.
DescriptionWork Breakdown StructureTeam AllocationWork Product Usage
Purpose
To:
  • Verify that the "right" system has been built and that it meets the functional and technical requirements as specified
  • Execute the application in a production-like environment
  • Ensure that the internal and external interfaces will function as expected
  • Increase the confidence of the clients and the user community - business and operational
Relationships
Parent Activities
  • Deploy
Description

Acceptance tests verify that the system meets the user requirements as specified in the Use Case Models and non-Functional requirements. These tests are generally conducted after the development and system tests have been performed. This is to ensure that the environment-related problems have all been fixed and that the test environment is very stable. Acceptance testing may be done at multiple levels and they are:

  • (User) Acceptance Testing
  • Operability Testing.

The first is usually run with the participation of the business areas and business users. Finally, Operability testing is conducted quite frequently during the off-peak business hours.
