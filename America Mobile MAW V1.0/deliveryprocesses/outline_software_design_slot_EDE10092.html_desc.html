Description

Prototypes may be intended to be throw-away or reusable, depending on their purpose, the tools and environments available to the project team. Prototypes intended to be throw-away should be low-cost and quickly developed. Reusable prototypes should be coded in the same language in which the application will be built, following the coding guidelines specified by the development team. Prototypes could include any of the following as appropriate (from low to high-fidelity):

  • Sketches and drawings
  • User personas that describe details of specific user types
  • Clickable wireframes created in Visio or other similar tools
  • A software simulation built using a rapid prototyping tool appropriate for the target platform (such as iRise, Dreamweaver, or Flash for Web sites, Rational Application Developer for Eclipse Rich Client Applications, or Microsoft Visual Studio for Windows and .NET applications)
  • A proof of concept (define to fit each proposal) to demonstrate explore high risk areas (business or technology risk) or architecturally significant concerns
  • Other technology-oriented Proof of Concept endeavors (e.g., middleware integration, data access patterns, transaction management patterns, open source framework viability, 3RD party libraries, etc.)
