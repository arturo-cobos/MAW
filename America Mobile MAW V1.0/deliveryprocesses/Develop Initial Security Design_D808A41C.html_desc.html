Activity: Develop Initial Security Design
The designing of the specific solution begins, the security framework is molded and shaped.
DescriptionWork Breakdown StructureTeam AllocationWork Product Usage
Purpose
To develop a design that can be built into the solution to provide the required levels of security across all anticipated releases, and can be fitted into the specified architecture.
Relationships
Parent Activities
  • Develop Technical Architecture
Description
Now that the framework is in place, the team can begin designing the specific solution in conjunction with the rest of the project team. This activity takes place in the design phase of solution projects and is further refined in the later phases. In the previous activity, the security framework was created. Now that framework is further molded and shaped.

During this activity, the team will take the framework and start matching it against the client environment. Adjustments may need to be made, but the design should implement the requirements, complete the objectives, and fall within the principles defined previously. The team must re-assess the solution to make sure it achieves this. Also, a review must be completed to ensure the solution is consistent with the clients risk appetite and reduces the risk of the client to acceptable levels.
