Task Descriptor: Develop Deployment Unit Model
This task describes the steps involved in creating a Deployment Unit Model
Based on Method Task: Develop Deployment Unit Model

Steps
During the creation of the Logical Operational Model:
  1. Use the Node Model artifact to identify the nodes involved in the solution. This should include both those in-scope and those external to the solution that have a direct linkage to the in scope nodes.
  2. Add the nodes identified to the DU to DU Interaction Requirements template and the DU to DU Mapping template.
  3. For each application DU defined in the Deployment Unit Specification artifact, associate the DU to the application component on the Component to DU template.
  4. Once the initial set of application level DUs are defined, they are then ready for placement on the operational model. This placement may, in turn, cause a further refinement of the application level DUs (and possibly a re-factoring of the associated application component model) in order to achieve a balanced approach to the non-functional requirements. See Guideline section below.
  5. Add each Deployment Unit to the DU to DU Interaction Requirements template and to the DU to DU Mapping template, associating the DU to the node it was placed on.
    • Once the application level DUs have been placed, it is usually necessary to identify and place the underlying technical DUs that will be required to ensure the solution works and meets it’s non-functional requirements. These technical specified DUs will be associated with the technical component models identified/developed later in this process.
  7. Using the completed Deployment Unit Specification artifacts for the application component DU’s, capture the non functional requirements associated with each of the DU to DU interactions on the DU to DU interaction requirements template.
  8. Add the required technical nodes to the Component to DU Mapping template and the DU to DU Interaction Requirements template
  9. Add the DU’s associated to those technical nodes to the Component to DU Mapping template and the DU to DU Interaction Requirements template
  10. Using the completed Deployment Unit Specification artifacts for the technical component DU’s, capture the non functional requirements associated with each of the DU to DU interactions on the DU to DU Interaction Requirements template
  11. Analyze the Deployment Unit relationships. The relationships between the Deployment Units identified is largely dependent on the corresponding relationships between the components in the application’s component model. Validate that component dependencies and interactions are supported by the placement decisions made.
  12. Review the candidate DUs and their relationships. Now that you have established a first pass at the candidate deployment units and their relationships, stand back and review the information and how you have structured it.
  13. Identify "Hot-spot" DUs and inter-DU relationships.
During the creation of the Physical Operational Model:

The technologies, products and code that will implement the components represented by these specified DUs must be identified , together with a definition of the resource requirements required.

This physical level information will be developed in concert with the component selection and system configuration activities associated with the development of the physical level of the operational and component models.
