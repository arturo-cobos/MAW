Guideline: Developing User Support Specifications
Migrated development approach from existing IBM Global Services Method work product.

Related Elements
  • User Support Specifications
Main Description
  • Define the objectives for the support material. The documentation objectives should focus on usability, including aspects such as: accessibility, accuracy, clarity, readability retrievability and task supportiveness. The objectives should, when possible, be as measurable or quantitative as possible.
Examples of quantitative objectives:
  • Amount of time to find information
  • Amount of time to read the information
  • Amount of time to complete a task using the support material
  • Allowable number of task errors
  • Allowable number of documentation errors
  • Target reading grade level
  • User satisfaction ratings
  • Meets IBM checklist for Accessibility
Examples of qualitative objectives:
  • Task-oriented and task-sufficient documentation
  • Consistent format in printed documentation
  • On-line information only
  • Hypertext on-line information
  • Documentation in accessible medium 
  • Identify each support material document.
  • Map each document to the end users groups and the tasks to be performed.
  • Define the relationships between the documents.
  • Define the specific objectives for each document.
  • For the on-line help, specify how extensive it should be and what accessibility features are required. Determine what purpose it serves, by, for example, deciding the answers to which types of questions should be found in the on-line help and which language should be used.
  • For the User Guide make a sketch of the different types of scenarios to be covered, and consider the following:
  • Will the User Guide be used mainly by the new employees, or will it be used more or less daily by the "old" employees?
  • What would be the best way to organize the scenarios, so that they are easily accessible/found when needed, or should they be organized in a sequential manner to serve the purpose of the new employees?
  • Will the User Guide and Manual be delivered in hard copy, soft copy or both? (Take into account that the optimal level of detail and formatting are different for the printed page and the presentation on screen).
  • Will the User Guide be used by individuals who have vision impairments, low vision, color blindness, or dyslexia or other special needs?
  • Define the media and format specifications for each document type, based on:
  • The detailed task analysis.
  • The objectives of the user documentation.
  • The allowable costs.
  • Determine the media and formats.
  • Determine the mechanism of distribution (initial and updates).
WBS Summary Tasks:
  • Identify documentation needs, including necessary accessibility requirements
  • Determine document contents
  • Identify link to training materials
  • Specify documents to be produced
  • Plan production
