Guideline: Developing User Support Materials

Main Description
Start building the User Support Materials as an outline then fill in the outline with more specific details. Eventually, the outline will be populated with many tasks, organized by general topics further broken down into more specific tasks.
Determine what documentation to develop, and specify the content, organization, medium, and format of each. Perform the following steps:
  • Specify end-user documentation.
    • Identify each document.
    • Map each document to end users and tasks.
    • Define relationships between documents.
    • Develop specific objectives.
    • Establish design standards and guidelines, including accessibility guidelines.
  • Specify content and organization, based on specifications and detailed task analysis:
    • Develop table of contents, or topic list, for each document.
    • Define the structure of each document.
    • Develop the detailed outline of each document.
    • Select index items.
    • Determine cross-references or links.
    • Develop a list of illustrations.
    • Develop prototypes of selected information.
  • Develop the media and format specifications for each document, based on detailed task analysis, user documentation objectives, results of prototype walkthroughs, and allowable cost:
    • Determine media and format.
    • Determine the accessibility features required.
    • Determine the mechanism for distribution.
    • Determine information development environment and tools.
    • Establish guidelines (such as naming conventions for source files, and the use of common files and templates).
    • Evaluate vendors for development, publishing, and delivery.
  • When the outline is complete, divide it into the sections you plan to deliver. Assign one or more technical writers to each section.
    • Organize information for ease of access.
    • Write instructions in such a way that they are easy to follow
    • Structure the information so that it is easy to skim, yet provides sufficient information for novices.
    • Clearly differentiate between types of information such as concepts, background, purpose, feedback, and actions
    • Use graphics to support your text. This clarifies concepts and instructions and it helps reduce the number of words needed. Ensure that graphics are explained for visually impaired users, either using alt text or a description in the surrounding text.
    • Explanations should be supported by examples to help users apply new information to their context.
