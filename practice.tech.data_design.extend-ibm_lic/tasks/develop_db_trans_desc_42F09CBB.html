Task: Develop Database Transaction Descriptions
Define and analyze the atomic units of work which the database management system will have to do as part of this solution. These form the basis of transaction performance analysis from the data viewpoint.
Disciplines: Technical Design
Purpose
The purpose of the Database Transaction Descriptions is the ensure the system adequately enforces data integrity. Specifically, the work product ensures that the developers and database designers understand which database accesses must be grouped together into atomic units. this information is used:
  • To develop commit and rollback logic in detailed program specifications.
  • As performance constraints during physical database design.
  • To prioritize relative performance of the transactions.

        Steps
        Review the data and function models to determine what events affect the data
        Review the logical data model based on business events
        Review the logical data model to determine which entities and attributes are accessed and which associations are affected as a result of the business events
        Group individual accesses of a business event into transactions
        Assign a name to and describe each transaction
        Document the average and peak frequency of each transaction
        Document the performance requirements of each transaction
        Rank each transaction in terms of execution priority
