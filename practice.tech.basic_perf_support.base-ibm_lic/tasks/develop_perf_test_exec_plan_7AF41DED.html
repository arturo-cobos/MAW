Task: Develop Performance Test Execution Plan
Develop steps for performance test execution plan.
Disciplines: Technical Architecture
Purpose
To ensure that the Performance Testing activities appropriately address all operational aspects of the system. To ensure that the test environment is planned for adequately and that the Performance Test Plan clearly documents the activities to be performed.
Relationships
RolesPrimary Performer:
  • Technical Solution Architect
Additional Performers:
  • Infrastructure Architect
  • Test Specialist
InputsMandatory:
  • None
Optional:
  • Test Environment Configuration
  • Test Specification
    Outputs
    • Test Execution Plan
      Main Description

      Using artifacts developed during operational architecture development, define the system-level groupings of nodes and their associated components that must be tested to ensure that performance objectives have been met as defined in the solution's non-functional (i.e. qualitative) requirements. Determine the operational nodes and deployment units that must be performance tested. Define the Performance Testing environment and plan for its implementation. Document the overall detailed Performance Test Plan.

      Steps
      Refine performance objectives
      Refine the performance objectives that must be tested for the system being built. These were defined during the operational architecture activities. Examine the nodes and their components in the operational architecture, as well as any analysis done of service level objectives in order to identify specific performance objectives that need to be tested. These objectives must be measurable in order for the performance testing to determine the satisfaction of the system's requirements.
      Document performance test plan
      • Review performance testing scope, scenarios, approach and infrastructure requirements
      • Develop a comprehensive task list for this level/phase
      • Allocate resources for each task
      • Schedule tasks and milestones
      • Document, or point to, common tools and processes for:
        • Change management, tracking and reporting
        • Issue or problem management, tracking and reporting
        • Defect management, tracking and reporting
        • Risk management, tracking and reporting
        • Project management, tracking and reporting
      • Review the master test plan
      • Document the performance test plan
      Review system test plan
      • Review the performance test plan
      • Walk through the test focus areas
      • Verify the high priority requirements are included in the "early builds"
      • Verify that the organizational responsibilities are correctly recorded
      • Ensure performance tests to be performed are clearly defined in the plan
      • Verify all sections of the performance test plan are complete and accurate
      • Obtain the sign off from the approved signatories
