        Main Description
        Determine that the Performance Test environment has been correctly established. Execute the Performance Tests that have been planned for and prepared including any required regression testing. Review the results of the tests and identify and document variances in the expected Performance Testing results. When problems or errors with code are identified, demote the code to a lower-level development library where the code can be analyzed, debugged, Unit Tested and promoted to the Integration Test libraries. As variances are resolved, and Integration Tests execute without incident, promote the code to the Performance Test code libraries.

        The Performance Testing objectives are to ensure the system meets the stated non-functional requirements relative to solution performance objectives, and performs within the target operational environment. Part of Performance Testing includes regression testing, which ensures that corrected modules do not introduce new errors. This is all defined in the Performance Test Plan.
