Guideline: Responsibility Assignment Matrix
Main Description

1 Description

Responsibility Assignment Matrix (RAM) is a management tool which helps to align the project’s organizational structure to the Work Breakdown Structure, thereby ensuring that each element of the project’s work is assigned to a responsible individual.

This document is designed to addresses the criteria and process for creating a Responsibility Assignment Matrix (RAM) in support of a signed customer agreement on a rollout contract. The paper will focus on the documentation as well as the elements required prior to creating the matrix.

2 Context

This document specifically addressed the Responsibility Assignment Matrix associated with rollout projects.

3 Steps

Customer Signed Statement of Work

The first step on any rollout project is to ensure/validate that there is a signed statement of work.  This will be kept locally in the Contract Control Book or in the IBM Image System (contact your Contract Support Specialist for access to files on Image).

Identify Tasks to be Performed

The Work Breakdown Structure (WBS) should identify, and include, all of the tasks that are called out in the Statement of Work, all tasks required to complete the project, as well as the tasks that are internal to IBM and the management of the contract.  Using the WBS, the Project Manager can begin to group the tasks based on the skills associated/required on the rollout.

Grouping of Tasks

Using the Work Breakdown Structure (WBS) group the tasks based on the skill requirements.  Although many of the tasks will require base skills such as communication and documentation, the grouping should focus on the similarities at a high level such as:

  • Financial
  • Tracking
  • Technical
  • Reporting

Assign Estimated Time to Complete

One of the major considerations in developing an effective Responsibility Assignment Matrix is to ensure that a balanced division of labor exists.  Each of the tasks identified on the rollout project should be assigned an estimated time to complete for resource and financial planning.

Identification of Resources

The Project Manager will need to gather a listing of the team members assigned to the contract (resources can be obtained from the Resource Deployment Manager if required).  With a clear understanding of what skills are allocated to the project and the time estimates to complete all of the identified tasks, the Project Manager can begin to assign tasks.

Build The Matrix

Below is a sample shell of a Responsibility Assignment Matrix.  By filling in the matrix you can begin to visualize the project.  Make sure that all of the tasks called out in the SOW, the project tasks, and the activities internal to IBM are called out and accounted for.



GROUP  TASK TIME ESTIMATES  ASSIGNMENT
Group 1 Task 1 x hrs to complete  Perform by
Task 2 x hrs to complete  Perform by
Task 3 x hrs to complete  Perform by
Group 2 Task 1 x hrs to complete  Perform by
Task 2 x hrs to complete  Perform by
Task 3 x hrs to complete  Perform by


Check for Balance

It is essential that a degree of balance exists on the RAM.  Check to ensure that a division of labor exists over the project team.  If any one team member is overloaded with assignments, this is an indication that the matrix is not balanced and alterations are required.
