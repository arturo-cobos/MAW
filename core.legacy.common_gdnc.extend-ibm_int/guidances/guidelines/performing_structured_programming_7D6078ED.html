Guideline: Performing Structured Programming
Main Description

Structured Programming (SP) is a programming technique proposed by Professor E. W. Dijkstra of the Netherlands between the end of the 1960s and the beginning of the 1970s. "Structuring" means step-by-step decomposition of a whole into small units whose mutual relationships and objectives are clearly defined. "Programming" means to build process logic (algorithm) for data transformation by defining the structures of related input/output data according to specified restrictions.

At present, Structured Programming is used for software development by most of the programming languages, such as COBOL, PL/I, and C.

Structured Programming is based on the following two basic ideas:

  • Structure a program (Structured Theory)
    • Assumption: Decompose a program into modules that form a hierarchical structure
    • Each of the modules has one entrance and one exit
    • Define the logic of a module using combinations of three structures: Sequence, Selection, and Iteration. (In principle, do not use a GOTO statement)
  • Structure the programming process (Stepwise Refinement)
    • Make a program workable gradually, from functional specifications to the level that can be expressed by a programming language, by means of stepwise decomposition and refinemen

The objectives of Structured Programming are to:

  • Decrease the complexity of a program
  • Make a program easier to read
  • Raise the reliability of a program
  • Raise the productivity of programming
  • Improve the maintainability of a program

in order to create programs that are:

  • Correct
  • Efficient
  • Convenient for changing

Current information systems handle great many kinds of data elements. Since a program for such a system has a complicated structure, the above objectives are difficult to accomplish only by the original ideas of Structured Programming.

To accomplish these objectives, "Programming directed to data structure" (Jackson Structured Programming (JSP)), and "Information Hiding" must also be considered.
