Guideline: Gap Analysis Report (PSI)
Main Description
Introduction

The IT Infrastructure Gap Analysis conclusions identify the variance (gaps) between the IT organization’s existing technology and application portfolios, the IT processes, and the IT skill sets and organization (Current) to those required to implement and support the new environment (Future). The analysis includes the conclusions derived after a data gathering process. These conclusions quantify the impact of the transition required to migrate from the existing, “current”, IT infrastructure to the desired, “future", IT infrastructure. The Gap Analysis Report documents the work done to date, what was found during data gathering (high level requirements and key assumptions), the difference between the Current and Future infrastructure and the next steps needed to deal with the gaps.
Based on the data gathered during: Current IT Standards, Current IT Technology, Current Applications Portfolio, identify the gaps between the existing and required technologies, networks, IT processes, and IT skill sets along with the impact of the conclusions/gaps on the organization and seek agreement from the client organization. This information is an essential pre-requisite to planning the activities that need to be built and support the future IT infrastructure.

Steps

1. Gap Analysis

Purpose

Develop the differences between the current infrastructure and the future infrastructure that must be addressed through planning

Description


Create findings from facts

  • Sort facts, assumptions, questions and issues (FAQI) by Technology, Process, People (TPP) in the FAQI worksheet – Example 1 
  • Group the Technical FAQI by server and network and assign an identifier to each FAQI. Group the Process FAQI by process (Asset Management, Backup Recovery, etc.) and assign an identifier to each FAQI. Group the People FAQI by organization, staffing, training, and skills and assign an identifier to each FAQI. 
  • Sort all FAQI by the assigned group identifier within TPP 
  • Analyze FAQI data, along with data from the EUFG worksheet (Example 2), to form findings by combining facts until they produce new thought, findings, that are pertinent to the Current or Future infrastructure 
  • Result: First level analysis of facts – Example 3

Create conclusions/gaps from findings

  • Using the same process, analyze findings to form conclusions/gaps 
  • Review the conclusions/gaps using the entire client team to gain consensus 
  • Result: Second level analysis of facts
  • Product: Conclusions/gaps about the difference between the current infrastructure and the future infrastructure

2. Gap Analysis Report

Purpose

Create a report for review with the planning engagement executive sponsor to gain approval to proceed

Description


Develop a “strawman” report

  • Agenda 
  • Description of the work done to-date 
  • Description of high level requirements, key assumptions
  • Description of the gaps identified 
  • Description of the approach to follow-on implementation planning 
  • Appendix: detailed findings 
  • Result: Consultant’s view of the client situation, gaps and next steps – Example 4
    Client review/revision of “strawman” 
  • Put the entire report on the wall for client review, revision 
  • Adjust report to meet the client view 
  • Gain client consensus to the details of the report 
  • Result: Client team consensus on content of the report – Example 4

Present to sponsoring executive

  • Presentation 
  • Apply revisions 
  • Result: Executive approval to proceed
  • Product: Final Gap Analysis Report
