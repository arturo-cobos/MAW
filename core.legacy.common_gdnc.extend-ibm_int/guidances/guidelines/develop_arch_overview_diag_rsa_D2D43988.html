Guideline: Developing Architecture Overview Diagrams with RSA
Main Description
This paper provides techniques that use Rational Software Architect (RSA) to develop an Architecture Overview Diagram (AOD). These techniques may also be applied to Rational Software Modeler (RSM) to develop the same work product. This paper only focuses on the tooling to develop the work product, not the work product itself.
 
AODs depict the governing ideas and candidate building blocks of an information technology system or enterprise architecture. They provide an overview of the main conceptual elements and their relationships.
 
See GS Method work product ARC 101 for a complete description of the Architectural Overview Diagram.
 
Use the Rational Software Architect (RSA) techniques to model an overview of the main conceptual elements in an architecture.

Do not use these techniques without prior training (Refer to sections 4 and 5).
