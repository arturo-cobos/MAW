Guideline: Conducting Joint Application Development (JAD) Sessions
This technique paper outlines a sequential process for conducting Joint Application Development (JAD) sessions. It explains the process in a generic manner, in that you can use it to conduct various types of JAD sessions.
Main Description
In support of the generic process, there are also specific examples of outcomes, work products and agenda tasks that would typically result from various types of JAD sessions, that is, planning, requirements and design sessions. When applying the JAD process, you will need to tailor it to suit the purpose and type of JAD session you are running.
 
Prior to commencing a JAD session, the following should have occurred:
  • A decision is made to conduct a JAD session
  • The Facilitator role for the JAD session has been assigned external to the project
  • The purpose and scope of the JAD session is defined together with success factors
  • The Sponsor has committed to all Participants attending a briefing session on the principles and process of JAD.

This technique can be used for any Engagement Model, however it is an inherent part of the Accelerated Development Engagement Model.
