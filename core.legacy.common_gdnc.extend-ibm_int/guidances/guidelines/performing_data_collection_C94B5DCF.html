Guideline: Performing Data Collection
Main Description

1 Description

For each activity and task in the phase there will be specific information required. Any of the following techniques: non-pervasive tools, interviews, workshops, questionnaires, and data acquisition and mining; may be used to collect this information. The consultant should use their discretion as to the best means to acquire accurate and comprehensive data with as little client intervention as possible.

For each data collection attempt it is important to consider the appropriate level of personnel to contact for information required. The personnel must be at a high enough level to gain all the information required and to understand the overall enterprise view.

Non-pervasive Tools

Non-pervasive tools collect configuration level IT data across multiple platforms.  These tools are non pervasive which means that no agents have to be loaded onto the clients systems. The consultant will run the tools from their laptop; the only thing required from the client is Administrator ID and password for their network. These tools gather most of the system specific information that is required. The tools also automatically populate the eBM data store tool with the information that it gathers. For more information on the tools see the supporting documentation.

Interviews/Workshops

Individual sessions and small individual group sessions are scheduled and held throughout the data collection process. At the business or technical information gathering sessions (interviews) the team will ask detailed questions concerning each given area of responsibility. These interviews are not mandatory it is only used if the there is not enough information that was gathered form the non pervasive tools or questionnaires, or they can be used as a validation of the output, or to get a better understand as to how the clients IT and business environment works.

Conduct interviews with:

  • Key Internal Technology personnel
  • Key External Technology personnel
  • Key Internal Business Process personnel
  • Key External Business Process personnel

Conduct interviews to review:

  • e-Business environment
  • Services
  • Processes
  • Sub-Process
  • Activities
  • Non-IT resources
  • Servers
  • Applications
  • Workstations 
  • Network
  • Data
  • System recovery process
  • System, applications, and data backups

Surveys

As with the interviews the surveys are not mandatory and should be used on an as needed basis this is due to the fact that they may be subjective. The consultant can use them to get a basic understanding of the environment in order to make better use of the client’s time during interview sessions; it becomes more of a validation session rather than a learning session.

  • System Configurations
  • Application Configurations
  • Business Environment

Data Acquisition and mining

Data acquisition and mining is collected from different client enterprise data sources that currently exist in the environment in order to minimize disruptions to the clients personal. This information can also be used to get familiar with the client’s environment the problem with Data Acquisition and Mining is that the documentation is usually old and out of date. This requires a lot of validation to make sure it is current.

  • Operational Documentation
  • Business Continuity Plans
  • Project Information
  • Industry Information

  • Enterprise Backup and Restore Issues Log

  • Vendor Management
