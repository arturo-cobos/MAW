Guideline: Developing Entity-Relationship Diagram
Main Description
The Entity-Relationship Diagram (ERD) is a data model notation that was proposed by Peter P. Chen (of the US) in 1976. It is the data model notation most widely used today. An ERD can express data structures in simple form using the concepts of an "Entity" and a "Relationship between Entities". This method has the following features:
  • The data structure can be visually expressed in an easy-to-understand form.
  • The essential meaning of the data and the relationships between data can be understood.
  • The structure and characteristics peculiar to the data can be expressed independently, separate from the actual physical restrictions and hardware and software technologies involved.
The use of Entity-Relationship Diagrams in data analysis makes the following possible:
  • Gaining a structural understanding of the data to be managed by an application.
  • Finding the structure inherently possessed by the data - an ideal structure that is highly independent and has no redundancies.

The use of the Entity-Relationship Diagram is a must for data administration. Its aim is to realize the construction and maintenance of high-quality application systems and database environments with long-term stability.
