Guideline: Defining Development Processes in ADSG
Main Description

The quality and productivity of an application development project depends highly on how a system is developed. It is difficult to correct this in the middle of the project.

In a large-scale development, if the development using RAD or incremental method is repeated without keeping data consistency of the whole system, it is always that interface inconsistency of the whole system or DB deadlock emerge gradually even if it seems to be going without problems.

Misapprehension that the first success in a new method is true to the succeeding larger development unconditionally, or hasty conclusion which is led by simply comparing the productivity of small-scale project using new method with that of past large-scale development, must be avoided.

The important thing is to select the most appropriate development processes according to the characteristics of the development target. If non-waterfall type development processes are applied when waterfall type should be used, the development cost will expand beyond your imagination. In the client/server or object-oriented development, however, if the number of the members at the peak is less than six to seven, non-waterfall type has more possibility of success. When selecting the development processes, it is important not to be deluded by hypothesis or misapprehension unsupported by the data evidence.

The table below shows key points in selecting development processes.

Important Factors In Selecting Development Processes

Development Platform

  • Optimization of development environment and operation environment

Development Tools

  • Application of visual development tools and graphical user interface (GUI) tools requiring little programming

Project Size

  • The number of staff is a major factor

System Objectives / Application Characteristics

  • General rules may not apply to the scientific field or simple conversion
