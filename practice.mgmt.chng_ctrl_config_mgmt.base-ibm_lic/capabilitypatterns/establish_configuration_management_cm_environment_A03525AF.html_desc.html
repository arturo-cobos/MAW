Activity: Establish Configuration Management Environment
This activity establishes an environment where the overall product can be developed, built, and made available for stakeholders.
DescriptionWork Breakdown StructureTeam AllocationWork Product Usage

Parent Activities

Description

This work is done by making sure the essential artifacts are available to developers and integrators in the various private and public workspaces as they need them, and then are adequately baselined and stored for subsequent use. Setting up the CM environment involves creating the product directory structure, project repositories, and workspaces (developer and integration) and allocating machine resources (servers and disk space).

Staffing

To set up an appropriate environment, a person playing the Configuration Manager role needs to have a good understanding of the component structures of the overall product, and will need to work closely with the architect to ensure that adequate "place holder" CM items are established.

A person playing the Integrator role in this work needs to ensure that artifacts delivered from the developer workspaces are sufficiently tested such that they can be incorporated into a testable build. The Integrator role needs to be familiar with Project CM Policies, Build and Test Practices.
