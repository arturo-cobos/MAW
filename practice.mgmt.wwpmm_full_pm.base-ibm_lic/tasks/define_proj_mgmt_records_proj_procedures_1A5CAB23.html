Task: Define project management records and project procedures
Disciplines: Quality Management
Purpose
To identify, define, and document the project procedures and records that will be used to manage the project.
Relationships
RolesPrimary Performer:
  • Project Manager
Additional Performers:
InputsMandatory:
  • None
Optional:
  • Outline of the Project Procedures Description
    Outputs
    • Project Management System Summary
    • Project Procedures Description
      Main Description
      Review the project management system requirements (in the outline of the Project Management System Summary) in the light of the project risks that have been identified, and confirm the procedures that should be enforced to facilitate project control, communications, and exceptions handling.

      As far as possible, derive from the WWPMM procedure templates, or from your constituencies templates, the flow and decision points of each project procedure, particularly who is to do what, the types of records to be maintained, and the types of reports to be produced.

      Complete as necessary the utilization rules of the supporting project management tools, as far as possible by reference to common standard guidelines.

      When the project organization contains several organizational units (POU) or subcontractors, ensure consistency of the overall project procedures with the local procedures that may exist at POU or subcontractor level.

      Document the project procedures selected for your project in the Project Procedures Description document or in the Project Management System Summary.
