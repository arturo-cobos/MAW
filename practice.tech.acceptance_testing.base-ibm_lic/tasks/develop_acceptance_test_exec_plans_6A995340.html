Task: Develop Acceptance Test Execution Plans
User acceptance tests are carried out so that users can sign-off on the application that is built and tested. In order to do this tests in a systematic manner, execution plans need to be prepared.
Disciplines: Technical Testing
Purpose

To define how to execute the Acceptance Test. To establish entry/exit criteria for each Acceptance Test Script and prioritize them. To establish pass/fail criteria and to provide a complete plan to enable effective management of the test execution.

      Main Description

      Create a Test Execution Plan for Acceptance testing based upon the Acceptance Test Plan and Specifications taking into account the overall test environment. The plan should cover the level-specific setup to the test environment and infrastructure, the test setup and execution, and how the testing runs will be logged. Separately, it should identify all the resources needed to perform the testing activities and when they will be needed. It should also describe how the test environment and infrastructure will be torn down after the testing is completed.

      Prepare a step-by-step description of tests to be performed. This should include backup and recovery processes for testing, data selection and analysis for setting up regression test packages (test cases and data) and results evaluation and defect management processes are clearly documented and understood.

      The Acceptance Test execution plan is based upon the available resources, the planned time for performing the test case executions and the critical path through the test schedule.


       

      Steps
      Review the detailed acceptance test plan
      Consolidate acceptance test design, test environment, and test data documentation
      Assign resources
      Schedule the tests steps
      Identify milestones and checkpoints
      Document and review controls and procedures
      Review the acceptance test execution plan
