Task: Confirm Test Objectives and Report Results
After carrying out tests confirm that the objectives of testing were achieved.
Disciplines: Technical Testing
Purpose
To ensure proper testing has been conducted.
Relationships
RolesPrimary Performer:
  • Business Analyst
Additional Performers:
  • Test Architect
  • Test Specialist
InputsMandatory:
  • None
Optional:
  • Test Execution Plan
  • Test Log
  • Test Plan
    Outputs
    • Test Findings
      Main Description

      Tests are carried out to make sure the functionality or objectives are met with as desired. After carrying out all the necessary test provide confirmation that required testing has been carried out and the results meet the objectives of the testing.

      Steps
      Prepare acceptance test reports
      • Outline the scope of acceptance test
      • Report acceptance testing results:
        • What was done and not done.
        • Acceptance Test results.
        • Findings on:
          • Acceptance test components working.
          • Acceptance test components not working.
          • Acceptance test outstanding problems that need to be addressed before the next testing activities.
          • Conclusions.
          • Quality of software - actual versus planned.
          • Problems and issues outstanding, if any recommendations and action plan.
      • Review contents of the acceptance test report with all involved parties
      • Ensure all exit criteria for acceptance testing have been met
      Verify completion of acceptance tests
      Formally document agreement, and removal of all outstanding issues, regarding the completion of the exit criteria in the detailed test plan for this acceptance level of testing.
