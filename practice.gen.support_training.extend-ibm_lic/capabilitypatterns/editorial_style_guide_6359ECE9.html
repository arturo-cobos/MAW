Main Description
The style guide acts as the basis for style, tone, voice, grammar, accessibility (editing language) requirements, and usage, and contains rules applied to terms specific to the interactive media application, including Web-based or e-business types of applications. The style guide provides information about structure, page layouts and formatting specific to the presentation of the content; it should also contain accessible (editing) requirements and rules to make resulting documents accessible.

This work product triggers checks for accessibility compliance. Therefore, when the Style Guide is created, take care to incorporate metrics to evaluate this requirement.

Key Considerations

A style guide should never be static. This document is meant to change and grow over time. It will change as trends evolve, as new corporate policies are adopted, and as issues of style are debated and decided upon within an organization. A document such as this must be flexible, adaptable, and accessible. It will never be fully useful if it is not available for reference by anyone who is creating content for that organization.

The style guide should be short and flexible, steering people to write in “the spirit” of the guidelines. Nitpicking over language in, or style of a deliverable delays project completion!

A new style guide should be validated through internal proofreading checkpoints and scheduled client review sessions.
