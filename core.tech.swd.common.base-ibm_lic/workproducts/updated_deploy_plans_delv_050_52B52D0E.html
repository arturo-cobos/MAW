Deliverable: Updated Deployment Plans Deliverable (DCD 050)
This deliverable is a consolidation of the plans addressing a solution related to and supporting its deployment.
Purpose
This deliverable presents the recommended plan for conducting the solutions acceptance testing and deployment. These plans serve as the master document to guide solution acceptance and deployment for the current and future solution releases. As such, it documents the scope, approach and expectations that form the basis for estimating the release phases of the next scheduled release of the solution.
Main Description

This deliverable expands upon the strategies and plans defined and approved for a solution as may be needed for the solution release being deployed, for the solution as a whole, and as may specifically apply to the next scheduled solution release.

It is comprised of the following solution plans as each applies to the scope of a solution being deployed, as a whole, and as may pertain to the next scheduled solution release.

  • Release Plan - Confirms the functionality intended per the releases defined for a solution not yet deployed and specifically as pertains to the current and next scheduled solution releases
  • Deployment Plan - Confirms the intended scope and approach for deployment per the releases defined for a solution not yet deployed and specifically as pertains to the current and next scheduled solution releases
  • Operations and Support Plan – Confirms the organizations responsible for operations, training, and other support responsibilities for the releases defined for a solution not yet deployed and specifically as pertains to the current and next scheduled solution releases
  • Quality Plan – Confirms the relative priorities of various attributes of quality to which the project team will focus
  • Test Plan – Identifies the overall testing approach and scope for a solution and as specifically pertains to the current solution release and as may apply to the next scheduled solution release
Deliverable Properties
External Description

This deliverable serves to verify that the necessary plans are in place and approved to move forward with deploying the software developed for a specific solution release’s Build. It additionally documents the scope, approach and expectations that form the basis for estimating the release phases (Micro Design, Build Cycle, and Deployment) for the next release of the solution and as may apply to other future solution releases. It is comprised of the following solution plans as each applies to the scope of a solution being deployed, as a whole, and as may pertain to the next scheduled solution release.

  • Release Plan - Confirms the functionality intended per the releases defined for a solution not yet deployed and specifically as pertains to the current and next scheduled solution releases
  • Deployment Plan - Confirms the intended scope and approach for deployment per the releases defined for a solution not yet deployed and specifically as pertains to the current and next scheduled solution releases
  • Operations and Support Plan – Confirms the organizations responsible for operations, training, and other support responsibilities for the releases defined for a solution not yet deployed and specifically as pertains to the current and next scheduled solution releases
  • Quality Plan – Confirms the relative priorities of various attributes of quality to which the project team will focus
  • Test Plan – Identifies the overall testing approach and scope for a solution and as specifically pertains to the current solution release and as may apply to the next scheduled solution release

These plans include the following topics pertinent to solution deployment activities

  • User education
  • A detailed schedule of events to be conducted prior to and during the deployment effort, the expected duration of those events, and the event dependencies for production cut-over
  • A definition of the roles and responsibilities for both client and IBM project team resources involved in the deployment effort
  • A recommended plan for the acceptance testing to be conducted by the client defining the tests to be conducted; the test cases comprising those tests, completion criteria for the tests; resources required for executing and reviewing the tests; timing of the tests; location and equipment requirements for conducting the test; and any testing dependencies
  • Functional and technical requirements for any testing tools that must be acquired from outside vendors.
  • Refinement of the Operations and Support Plan providing a detailed framework and definition for the hardware platforms and software environments needed to implement the functionality developed in the solution release’s Build being deployed.
  • Revision to the Solution Deployment Plan, as needed, to further define the tasks, schedules, and responsibilities for the roll out of the software developed in the solution release’s Build.
  • Refinement of the specifications for acceptance testing based on feedback from the integration and system tests conducted during the Build Cycle.

The intent of this deliverable is to provide the basis for the scope of work to be included in the solution release being deployed. Additionally, revisions to the plans contained in this deliverable may be made during the Deployment Phase to finalize details for the next release of the solution prior to its Micro Design iteration. Its contents will be further detailed in support of the next and future solution releases during their respective release phases (Micro Design, Build Cycle, and Deployment).

Packaging Instructions

The following work products contribute to the various deliverable topics or sections.

The work products identified are fully defined at a detailed level for a specific release during detailed design. They may each be revised, as needed, during the build and deployment phases for a specific release. Modifications may also be made while developing one release that may apply to the solution as a whole or to its future releases.

Some work products may also contain detailed descriptions of prior solution releases and high-level descriptions for future solution releases that are discrete to those releases and not pertinent to the release undergoing the Deployment phase. Such work products will be iterated and more fully developed in later phases of the engagement as the need for greater detail is identified and required per each solution release. In this regard, work products input to this deliverable may be extensions and/or revisions to previously generated work products while others may be separate and distinct instances specific to this project and/or phase.
