Deliverable: Development Plan (DCD 057)
This document defines the functionality and components to be developed in the release, the number of builds and the development goals for each, and an estimate of the phases, activities and tasks required to complete the builds.
Purpose
The purpose of this document is to create a basis for understanding between the key stakeholders and the development team regarding what is to be developed in the release and the process of building the solution.
Main Description

This document should contain the following:

  • A definition of the functions to be developed in the release and the infrastructure components to be implemented
  • A definition of the builds to be conducted and the functionality and components that will be developed in each build
  • A high-level project plan that outlines the design and development activities in each build
Deliverable Properties
External Description

The purpose of this activity is to establish development guidelines and procedures, define goals for each Build, and update the Release Plan for the project.  The IBM team will perform the following tasks in this activity:

  • Refine the Release Plan to validate the scope of this Release, set the development goals for each Build, and to estimate the build phases, activities, and tasks to be completed.

Completion Criteria:

This activity will be considered complete when the Development Plan has been delivered to the customer or IBM has met the criteria defined in Section {XX} "Completion Criteria" of this Statement of Work.
