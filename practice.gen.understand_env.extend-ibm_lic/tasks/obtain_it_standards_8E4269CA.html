Task: Obtain IT Standards
Understand the client's current IT standards that may constrain decisions during this project
Disciplines: Environment
Purpose
IT Standards are documented to identify any pre-determined components of the future IT architecture. Many purchased applications require specific databases, operating systems or middle-ware components. By understanding the preferences of an organization, alternatives which do not conform to these standards can either be eliminated from consideration, or the work required to allow a deviation from or expansion of the specific standards can be undertaken in a timely manner.

      Main Description
      This task is performed by collecting data around several key questions:
      • What classes of technical components are required for the new application?
      • What technical components has the client made an explicit study and choice of?
      • What technical components are de-facto standards due to install base or market position?

      Collecting this data can be done by

      • Interviewing comparable IBM projects
      • Interviewing IT managers
      • Developing an inventory of current technical components
      • Collecting IT standards studies conducted recently

      This data should then be synthesized into the following sections:

      • Listing and detailing predetermined components of the information technology environment, either determined by a previous study or de-facto standards based on the installed technology in the company
      • Documenting the rationale or source of such determinations
      • Documenting the implications of deviating from these standards
      • Documenting the variance process
      • Noting any components which do not have standards established
