Artifact: Project Start (Project Launch) Letter (ART 0569)
It is a communication to the project manager of the record for a contract that the project start (project launch) procedure will need to be executed for a project.
Domains: Project Management
Work Product Kinds: Procedure
Extends: Correspondence Document
Purpose

This artifact is a communication from the delivery organization's portfolio manager to the project manager of record for a contract. It establishes a connection for the project manager to easily utilize the delivery organization representatives as resources to help with the requirements of the project management methodology. It is sent after the contract has been executed between IBM and the client and the contract information registered within IBM. It provides the project manager with key information on the project as well as instructs the project manager to commence the project start (project launch) procedure. It is also used to announce the transition of a project from the solution design phase to the solution delivery phase to the design and delivery teams.

Description
Main Description

This communication instructs the project manager of record, for a contract, that the project start (project launch) procedure is to be executed it can also contain:

  • Specific instructions for conducting project start
  • Key contract information for recording time and expenses associated with the contract
  • Key contact information; who is the Financial Advisor, Risk Manger, Partner, Solution Design Team, Portfolio Manager assigned to the contract
Brief Outline

The artifact should contain at a minimum the following:

  • Correspondence ID
  • Correspondence name (subject or short name of the correspondence)
  • Correspondence description (Details of information being communicated, including effective date for any notifications)
  • Category (for example, "notification", "approval", "informational")
  • Date sent (for outgoing communications)
  • Date received (for incoming communications
  • Sender’s name
  • Sender’s contact information (including phone, email address, street address)
  • Recipient’s name
  • Recipient’s contact information (including phone, email address, street address)
  • Other stakeholders (may include copied recipients of the communication)
  • Archival location

This artifact may also contain additional information regarding:

  • Impact of communication on recipients or expected actions to be taken
  • Whether formal acknowledgement of the communication is required

Key Considerations
Dependent on which geography, region or sector that conveys the artifact will determine the specific content within the artifact. Listed below is advice regarding typically content for the artifact:
To whom should the artifact be addressed?
  • The artifact should be addressed to the project manager of record for the contract.
  • A copy of the artifact should be sent to:
    • The solution design project team including the project manager, architect, and risk manager
    • The extended solution delivery support team including the project partner, financial manager, risk manager
    • If global delivery resource are engaged then global delivery representatives

What information should be included in the artifact?

  • A brief introduction of the delivery organization management and its responsibilities in relationship to project start
  • A brief description of what project start is and why the project manager should commence project start
  • Contract information including the Client Number, Client Name, Opportunity Number, Contract Number, Work Number, etc
  • Pricing information specific to the contract with the expectation as to how the financial performance of the project will be monitored
  • Contact information including the names and notes or internet Ids for the solution design team members and for the extended solution delivery support team (project partner, risk manager, financial manager, and global delivery resources)
  • A timeline for the duration of project start
  • If formal reviews will be conducted during project start, a brief description of the review process, such as what the project manager can expect, and an estimated timeline as to when those reviews will occur
  • A summary or quick reference to the various components of the project start (project launch) procedure
  • A pointer to where the detailed project start (project launch) procedure is located
  • Guidance regarding specific tools that should be used, where the tools can be located, and where supporting materials regarding the tools can be found
Tailoring
Impact of not having

Without the artifact, the project manager may not be aware of his/her responsibility to conduct the project start (project launch) procedure. Without conducting the project start (project launch) procedure, the project may not get off to a good start and become a troubled project.

Reasons for not needing

The artifact is not needed for internal projects where there is no contract information.

For those projects exempt from conducting the project start (project launch) procedure or those do not qualify, based on the criteria established by the Geography/IOT to conduct a project start, the artifact is not required.
