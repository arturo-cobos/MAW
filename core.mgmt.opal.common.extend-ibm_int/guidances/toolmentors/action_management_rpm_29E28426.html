Tool Mentor: Action Management using Rational PM
This tool mentor provides guidance on how to perform Action Management when using the OnDemand Process Asset Library (OPAL) and Rational Portfolio Manager (Rational PM).
Tool: Rational Portfolio Manager
Main Description

1. Purpose

This document provides guidance how Rational PM is used to support Action Management in accordance with the OnDemand Process Asset Library (OPAL). This document may serve as a template for creating a tailored procedure or guideline.
This guidance document contains three sections:

  1. Purpose
  2. How Rational PM supports the OPAL procedure
  3. Usage table for Rational PM data elements (fields)

The intent of this guidance is to provide sufficient direction on how Rational PM is used on a project to support Action Management, so that there are no ambiguities about the process and the team training can be concise.

The guidance assumes a basic understanding of how to use Rational PM and is not intended to be training material. Detailed training on the use of Rational PM can be found by selecting Education and Training from the GBS Rational Portfolio Management Enablement page.

Step 1 - Review Action Log, Cancel, or Launch Pending Actions - performed by Project Manager

Please note that Rational PM elements in BOLD text and highlights are considered required. Other fields and portlets are recommended for completeness. Refer to the table in section 3 for a complete description of all fields, standards and practices.

  • Receives Action from project stakeholder.
    • CREATE Action in Scope Management using the OPAL IMSP605 Action template and the OPAL IMSP605 Action layout. Key information that should be captured includes Rank, Description, Target Date, and Proposed By. A value for the Action Category should be captured in the Attributes portlet. Action State: OPEN
    • The OPAL IMSP605 Action template can be located by selecting Template from the Rational PM Scope Management View and expanding the Scope Templates / OPAL Scope Templates folder.





  • Review for validity:
    • If Action is determined to be valid and in scope
      • RECORD - assign Action owner in Staffing view or Resources portlet, enter Est Start, Est Finish, and Target Date. Additional information to capture includes Quantification/Impact, Order of Magnitude Cost/Benefits, Origin, and Impact portlets. Change State to ASSIGNED
    • If Action is determined to be invalid reject Action
      • RECORD - document reason for rejecting Action in the Status portlet, update Closed By portlet with person closing the Action. Change State to CLOSED




Step 2 - Monitor Progress of Actions - performed by Project Manager or designated Action Owner

  • Review progress of launched actions. If the action requires more effort than originally expected, make it a task in the Work Management view, or create a change request, or take some other appropriate action.
    • If the action activity begins or is still in progress record activity in the Status portlet. State remains ASSIGNED.
    • For those actions that are performed and are completed, update Resolution portlet to reflect what was done that resolved this Action and change State to 'TO BE VALIDATED'.
  • Use OPAL ENG313 Action Detail Report and OPAL ENG314 Action Log to assist with monitoring of actions, reviews, and provide information to non Rational PM users.





Step 3 - Review Effectiveness of Actions - performed by Project Manager or designated Action Owner

  • Determine if a effectiveness review is needed
    • If it is determined that an effectiveness review is not needed update Status portlet and Closed By portlet with person closing the action. Change State to CLOSED
    • If it is determined that an effectiveness review is needed, launch effectiveness review. Document the effectiveness review criteria and schedule in the Review portlet.
      • If the effectiveness review demonstrates the Action can be closed update Closed By portlet with person and date. Document results of any effectiveness review in Review portlet. Change State to CLOSED
      • If the effectiveness review demonstrates the Actions taken did not meet the objectives, modify the Action, and document the results of the review in the Review portlet. Change State to ASSIGNED

3. Usage table for Rational PM data elements (fields)
The table below describes the portlets and fields in the OPAL IMSP605 Action Layout in Rational PM. Portlets and fields in BOLD are required, other fields and portlets are recommended for completeness.

Note: different layouts exist and vary in the portlets they display.


Portlet/Field
Description
Standards and Practices

Identification Portlet

Rank
Rank defines the level of Severity of the Action

This field is critical to the BMR Investment Map and should be set for each Action

Select one of the following values:
900: Critical
700: High
500: Medium
300: Low

Reference Number
Repository unique identifier for Action
Rational PM assigns reference Number automatically

External Reference Number
The project may have its own numbering system. This field can be used for tracking that number
External Reference Number is the manual text assigned by the PM

Element security
Security control - project level or by invitation. Project Level is the default
For confidential and commercially sensitive information, use "by invitation"

Target Date
Target date for closing Action
Enter the target date that is expected for closure of the Action

EST/Start
Estimated start date
Enter the date that activity is expected to start on the Action

EST/Finish
Estimated finish date
Enter the date that activity is expected to finish on the Action

State
The state is related to the step in workflow
Use the following state values to identify the state of the Action during the lifecycle
States
Open: Initial state that the action has been put into Rational PM (action recorded)
Assigned: Action has been assigned to responsible project team member(s)
In Progress: Action is being resolved with approved plan
To be validated: For those actions that are performed and require a later effectiveness review
Closed: Action has been performed, validated, and accepted by the Action Owner



Free Text Portlets

Description
Action description, in sufficient detail to be understood by others on the team
Enter a complete and detailed description of the Action. Symptoms and facts that are causing concern

Status
Comments on action solution
Log of actions taken to perform Action management. Place the most current status on top marking it with the automated date and time stamp

Origin
The basis of the action item. Where was it derived from
Identifier of related issue or other event, if recorded, that this action is intended to resolve. There can be more than one identifier

Impact
Impact of this Action on the overall project
Document the impact that this Action has on the project in terms of cost, schedule, scope, or customer satisfaction. Reference any applicable Change Requests

Resolution
Action Resolution
Document the items or activities that resolved the Action

Review
Information related to Action effectiveness reviews
Enter necessity to review, schedule for review, and effectiveness criteria

Other Portlets

Attributes
Select attributes. Can be used to identify category of action. May be useful for portfolio reporting
Enter a specific value for the Action Category attribute. Other attributes are dependent upon the owning organization

Resources
Resources that are assigned to the Action
Assign resources within this portlet (or Staffing View). If restricted access is needed use the 'By Invitation' option

Proposed by
Name and date that resource opened the action
Default entry is resource that created the action. Can be modified

Approved by
Name and date that resource approved the action
For actions requiring approval this is the name of approver and date approved

Closed By
Name and date that resource closed the action
Default entry is resource that closed the action. Can be modified

Status Update
History (version) of updates made to the RPM Action record
System generated version control with ID, date, and time stamp of Action record change
