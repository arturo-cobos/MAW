Guideline: Developing the Service Request
Migrated development approach from existing IBM Global Services Method work product.
Relationships
Related Elements
  • Service Request
Main Description
Perform the following steps:
  1. The Service Request is raised and the requester completes the first section.
  2. IBM Evaluation of Effort to Investigate the Service Request is completed by the group assigned to investigate the Service Request. Once complete, the estimate is subject to approval by IBM and once approved the signature, name and date of approval is completed.
  3. The estimate is then presented to the client for approval. The Client System Manager will complete the Accepted/ Rejected/ Deferred box, the Client System Manager Name box and date box. If the decision was to Reject or Defer, then the reasons are to be documented in the appropriate box.
  4. If accepted, then the IBM Evaluation of the Effort to Implement the Service Request is completed. Once completed IBM approves it and the signature and name of the approver are entered on the form and the date of approval is completed.
  5. The estimate ID then presented to the Client System Manager for approval. As before, the Client System manager will document the decision and complete the name box and date of the decision. If the decision is to Reject or Defer then the reasons are to be documented in the appropriate box. If the decision is to accept then the priority is completed.
  6. Once accepted by the client the Implementers complete the Documentation Updates section and the Close-out section is completed with the implementer's name and the date of close-out.
