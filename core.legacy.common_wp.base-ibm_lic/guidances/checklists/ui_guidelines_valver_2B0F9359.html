Check Items
The guidelines are based on a reference to an existing UI design style guide tailored for the solution
The guidelines address the guiding usability, accessibility, and other principles
The guidelines address, at least briefly, the design process for the solution
The guidelines contain sections addressing all major user interface widget and control topic areas
This includes: windows, dialogs, message boxes, menus, push buttons, entry fields, containers, and lists.
Special user interface formatting needs for data elements are reflected in the guidelines
These special formatting needs are typically identified in the use cases, or usability/accessibility specific requirements.
The guidelines contain specific refernces pertaining to experienced versus novice users

This is only required if there are significant differences in the type and experience level of intended users of the solution.  Typically, if the solution is aimed at experienced users, the User Interface Design Guidelines contain significantly detailed guidelines for keyboard and fastpath navigation.  If there are novice users, the User Interface Design Guidelines typically provide detailed guidance on the use of performance support systems such as wizards, cue cards, and coaches.

The guidelines address any special device or environment needs of the intended user population
The guidelines address the extent to which accessibility requirements will be applied
For example, will captions be required on all multimedia, or will only transcripts be required with captions optional?
The guidelines, the documentation itself follows the accessibility documentation checklist
The guidelines contain the name of a contact person who is responsible for updates
