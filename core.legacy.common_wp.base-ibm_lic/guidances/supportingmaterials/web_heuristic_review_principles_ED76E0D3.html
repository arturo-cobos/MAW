Web Heuristic Review Principles Checklist

Users can understand how site is organized.

Users can understand which of their tasks are supported in the site.

Surfaces model to users.

Results in ease of navigation

Supports users’ tasks.

Allows users to seek information, submit information, and any other defined user tasks.

Uses metaphor to communicate model.

Uses metaphors consistently, carries them through the entire site.

Messages throughout site are consistent with other marketing materials.

Has a recognizable site identity.

Uses color, textures, labels, format (organic, industrial, etc.), etc., to communicate this identity.

Employs appropriate interaction model, including alternative interaction models if necessary for users with disabilities or alternative devices.

Enables ease of navigation through appropriate tools, including navigation using various input modalities such as keyboard and mouse.

Standard buttons (home link on each page, e-mail feedback link on appropriate pages, etc.)

Map of site

Alphabetical index

Absence of "orphan pages" (pages no links)

Uses standard home page contents.

E-mail link to Webmaster

URL

"What’s New" link

Appropriate navigation buttons

Copyright info

Uses user task language.

Meets wants, needs, tastes, etc., of clearly-defined target audience.

Uses understandable visual metaphors and semantic metaphors.

Uses metaphors consistently throughout site.

Employs readable fonts and fonts that can be customized to the users needs.

Uses real estate well.

Uses visuals judiciously.

Absence of non-communicative visuals, movies

Absence of constantly-running movies (allow users to control)

Absence of gratuitous of 3-D images

Provides alternative text for important visuals

Communicates clear targets.

Hot links discernible from background, widget targets large enough to hit, link text understandable when listed out of context (e.g. eliminate use of multiple "click here’s") etc.

Does not employ very large visuals.

Slow download times

User shouldn’t have to scroll to see an entire visual

Employs a glossary of icons.

Uses icons and labels consistently throughout site.

Uses color and shadow as effective communication tools, but not as the only method to convey important information.

Employs grouping, alignment, and balance of screen elements to communicate relationships, including semantically correct underlying coding and controls, and explicit labeling.

· Textual content

Uses appropriate writing style.

Web-appropriate (brief, to-the-point)

Consistent throughout site

Appropriate to any given task (task-oriented, action words used, etc.)

Limits amount of text per page as appropriate (info bytes vs. a research report).

Keeps number of steps in procedures to a minimum.

Presents text in logical "chunks," separated with headers, titles, swashes.

Employs glossary of unfamiliar terms, acronyms, with links to and from

Carries over information over multiple-page forms (no need to remember or repeat).

Employs appropriately sized input fields.

Employs visually appealing arrangement of fields (aligned left, etc.), including semantically correct underlying coding.

Does not use culture-specific jargon, idioms, etc.

Employs country-specific symbols for clarity (e.g. currency).

Uses and communicates formats (dates, phone numbers, addresses).

Uses icons, visuals, sounds, etc., appropriately (cultural sensitivity to meaning).

Uses simple sentence structure, vocabulary, for non-native English readers.

Employs plain-language error messages.

Employs "system busy" indicators, other feedback.

Employs help available when needed.

E.g., search engine use, form completion

Includes file size, version, plug-ins needed to view, etc., for downloads.

Uses sound judiciously.

Absence of non-communicative sounds

Chooses appropriate music

Uses voice when appropriate

Provides captions or visual indicators for important sound information

Employs clear, discernible sounds, music, and voices.

· Uses standard Web terminology

Uses navigational link names (home, previous, next, submit, reset).

· Employs fast paths for expert users (frequent visitors) such as keyboard shortcuts

· Uses frames appropriately (e.g. splitting a page into frames is confusing)

· Descriptive page names, URLs

· Keeps information up-to-date

Pays attention to "Web years."

Avoids poor performance.

· Long download times

Keeps scrolling to a minimum

· Does not employ horizontal scrolling

Uses text links for non-graphical browsers, low-end monitors, etc.

Uses appropriate keyboard input which does not interfere with standard system keyboard support (such as sticky keys)

Provides audio alternative for visual material. Visual alternative added for audio material.
