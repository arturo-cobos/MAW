Example: Increment Goals for Build Cycle
Relationships
Related Elements
  • Increment Goals
Main Description

Build cycle 1:

  • Team A builds an evolutionary prototype of the user interface only. The team gains experience using the tool set.

Build cycle 2:

  • Team A completes the object model.
  • Team B builds a first cut database, defines the persistence layer, implements a sample database, and establishes end-to-end connectivity with the first cut database design.

Build cycle 3:

  • Team A completes major functional area A.
  • Team B completes connection to printing subsystem.

Build cycle 4:

  • Team A completes major functional areas B and C. In this build, Team A may be a combination of Teams A and B from previous build cycles, or the members of Team B may be assigned to some other project.
