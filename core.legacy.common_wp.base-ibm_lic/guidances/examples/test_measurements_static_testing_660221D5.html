MEASUREMENTS SECTION

Measurements from the Testing Process (used by management to formally assess the testing process and to decide if enough testing has been performed. Given time and budget constraints, at any point in time, the amount to invest should be determined by the cost of further testing versus the degree of business risk inherent in going live into production without further testing.) Examples are:

Defect Prevention Measurements (mostly focused on Static Testing)
Defect Detection and Removal Key Measurements

  • Number of Defects per page of requirements specification (missing, wrong, extra, issue)
  • Number of Defects per page of functional specification (missing, wrong, extra, issue)
  • Number of Defects per page of non-functional requirements (missing, wrong, extra, issue)
  • Number of Defects per page of detailed specification partly, reflected in “class” work products and partly in architecture (missing, wrong, extra, issue)
  • Number of Defects per page of program specification, mostly reflected in “class” work products, interaction diagrams, and state charts (missing, wrong, extra, issue)
  • Number of Defects per page of detailed design split between architecture and “class” work products, interaction diagrams, and state charts (missing, wrong, extra, issue)
  • Number of Defects per page of user documentation, including reference manual (missing, wrong, extra, issue)
  • Detected Defects by Phase
  • Detected Defects by Type (documentation, code, other)
  • Detected Defects by Severity
  • Removed Defects by Severity
  • Defect Removal Efficiency (removed/injected)
  • Estimated Defects at delivery (injected - removed)
  • Estimated Problem Report Rates by Severity
  • Defect Removal Cost
  • Inspection/review - hours (Requirements / Design / Code Inspection)
  • Testing activities - hours (System Test)
  • Rework effort - hours
  • Number of defects found per Work Product using:
    • Prototyping
    • Desk checks 
    • Checklists 
    • Mapping 
    • Reviews 
    • Walkthroughs 
    • Inspections
  • and the effort consumed for each defect (identification vs. resolution)
  • Estimated KLOC, pages of documentation, other deliverables
  • Estimated Defect Injection Rate by Phase (defects/KLOC)
  • Application Support Defect Driven Activities
  • Defect Prevention Measurements
  • Total of Suggested Actions (number)
  • Actions Outstanding (not started) (number, estim. workload)
  • Actions being Implemented (number)
  • Actions Implemented/Closed (number)
  • Cost of Defect Prevention
  • Cost Causal Analysis Meetings - hours (CA)
  • Cost of Action Team Meetings and administration - hours (AT)
  • Cost of Action/suggestion Implementation - hours (AI)
Test Planning
  • Estimated Test Effort vs. Estimated Development Effort.
  • Estimated Test Effort by Development Phase (as a percentage of total - This is broken down by test prep and test execution).
  • Estimated Defects by phase.
  • Tolerance for Residual Defects at Delivery (Defects remaining).
  • Estimated Test Cases per KSLOC.
  • Estimated Test Cases per Function.
Test Preparation
  • Actual number of Test Cases per KSLOC.
  • Actual number of Test Cases per Function.
  • Actual Test Effort for Test Preparation by Development Phase (Note: This is the actual amount of time spent during each development phase as a percentage of estimated test effort for that development phase. This will indicate how much of testing preparation was accomplished vs. planned.) Examples are:
    • actual hours consumed in preparation of each test case or script or scenario 
    • actual hours consumed preparing test data
  • Estimated number of test cases that MAY NOT be executed as a percentage of total number of test cases.
  • Percentage of requirements coverage.
  • Percentage of statement coverage.
Test Execution
  • Number of Test Cases executed for each level of test.
  • Number of defects found for each level of test (by severity).
  • Number of defects fixed for each level of test.
  • Total number of defects outstanding at delivery (by severity).
  • Average number of runs to execute a test case successfully.
  • Actual test effort as a percentage of estimated test effort by development phase. This will indicate whether the tests took more/less time to run than planned.).
  • Number of test cases that were not completed (as a percentage of totals).


Measurements on the Testing Project (used by the Testing and Quality Management Organizations to improve the Testing Process and the effectiveness and capability of the Testing function; could also impact the Development, Infrastructure and User functions).

Summary of Section A – Measurements from the Testing Process (high-level measurements in a graphics format to be used by management):

  • Test Planning
  • Test Preparation
  • Test Execution
  • Defect Prevention Measurements
  • Defect Detection and Removal Key Measurements

Alternative Scenarios or calculations in various formats / sort sequences / representations based on the detailed measurements in order that other or new testing CSFs or problem areas may be analyzed and addressed. Some focus areas may be:

  • Test Coverage: e.g. coverage over the test phases, or code conditions
  • Test Effectiveness: e.g. thread of the test cases, ease of use of test scripts
  • Test Cycles: e.g. time and effort consumed in a test vs. number of defects removed
  • Software Reliability: e.g. continuous hours of operation before software fails with error

Test Management Measurements (may be specific to the Testing project or extracted from the Application Development Project Management metrics), e.g.:

  • Delivery on time for Customer: percentage of deliverables shipped in time
  • Delivery on Planned Gross Profit: percentage of products delivered on planned gross profit
  • Benchmarking Ranking: benchmark ranking (e.g. performance, quality, time, budget) vs. competitors
  • Utilization: percentage of productive hours / total reported hours; or percentage of billable hours / total reported hours
  • Delivered Quality: number of errors during initial production period; or defect trends relating to project size
  • Services Satisfaction: products and deliverables meet customers requirements and are shipped defect free
