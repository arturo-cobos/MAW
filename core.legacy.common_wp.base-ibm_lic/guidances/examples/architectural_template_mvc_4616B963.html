Main Description

Free format text describing the purpose of the MVC pattern

The Model View Controller (MVC) design pattern is very well suited to applications that consist mainly of interactions via the Graphical User Interface (GUI) with model objects. The design pattern aims to organize the components of the application into one of the three following distinct layers, each corresponding to a limited and precisely specified area of responsibility:

  • Model layer
  • View layer
  • Controller layer

The resulting architecture isolates the Model layer from the View layer so as to render the former independent of the latter. The communication or binding between the two is enabled and controlled by the controller. This approach lends itself naturally to a parallel development effort. The Model layer contains all the model objects with which the user interacts through the GUI. These objects need not be aware of their actual representation on screen. This approach enables developers to build up a set of business objects (i.e., classes representing objects from the business environment) that can be easily reused in different applications.
