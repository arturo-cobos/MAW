Section Two – External Data Reference Background


Data Reference
Description
Related System Comments
Volatility Comments

Customer Reference Database
This database contains the master record for all of the customers in the enterprise. This database is the system of record for customers.
Extracts from this database are distributed throughout the enterprise. System of record for all OLTP systems. Primary source for customer information in all financial reporting systems in the enterprise.
New customer records are added real time through OLTP systems. Purchase history is cycled to the Customer History file two months after the purchase date. Dormant customers are purged after three years of no activity.

Customer History
Offline flat file containing purchase history of the customer.

File is updated monthly with purchase history from Customer Reference database.

Call Center Customer Lookup
OLTP database used as reference for the call center. Customer characteristics enable client representative to classify customer upon receipt of customer inquiry.

New customers are added asynchronously from Customer Reference Database. Incremental updates are processed nightly.

Business Rules:


Type
Name (Optional)
Business Rule
Comments

Uniqueness

Each occurrence of a customer row must be unique based on the customer number.
Rule enforced through index capability of DBMS

Referential Integrity
RI-Insert
A customer row must exist in this database before any rows can be inserted in the Customer History data reference.
Rule enforced through application programming.

Referential Integrity
RI-Delete
All customer rows in the Customer History data reference must be deleted before a customer row in this database can be deleted.
Rule enforced through application programming.

Default
Default
Nulls are not allowed for any of the attributes. Blanks can be used as a default in Character attributes and zeroes can be used as the default in Numeric attributes.
Rule enforced through DBMS language definition capability
