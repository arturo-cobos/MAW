Analysis and Design Work Products
The following architecture and design work products will be produced (using the standard format, except as noted):
  • Architectural Decisions.
  • Component Model.
  • Physical Database Model.
  • Architectural Templates to handle common persistence strategies.
  • Analysis and Design Guidelines.
  • Class Diagram – including full definitions of all classes, relationships, attributes and methods.
  • Interaction Diagrams – all Interaction Diagrams will be depicted as Sequence Diagrams with a Collaboration Diagram be generated from that view if needed.

Keep all Architecture and Analysis and Design work products current. If previous architecture or design work is invalidated by subsequent work, revise any historical work products. The Component Diagram, Class Diagram and Interaction Diagram work products along with supporting documentation will be developed in RATIONAL ROSE using the latest published UML version 1.4. All bi-directional associations should be replaced with a uni-directional association and a dependency relationship. The use of aggregation should be limited to composite relationships.
