Example: Data Stores for Electronic Catalog Sales
This is an example taken from an "Electronic Catalogue Sales" business. It is a subset of all the data stores of interest to the business, but does accommodate the concerns of all users, systems and processes within the business.

In this example each data store is prefixed by "Dx.x". This is a quick reference back to the data entities from which these data stores have been derived. As can be seen, in this example it has not been considered advisable to perform any grouping, but it has been felt appropriate to split "catalogue" into three distinct data groups, each with differing values for key characteristics. In general, grouping and splitting are equally valid.


The supporting text, explaining why these characteristics are considered important, and describing the various data stores has not been included in this example.
