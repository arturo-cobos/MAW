Outcome: System Tested Solution
The System Tested Solution consists of a configured package, together with its interfaces and extensions, that has passed tests of functionality, stress, volume, performance, recovery, security, hardware configuration and operational readiness.
Work Product Kinds: Confirmation
Purpose

The System Tested Solution is used as the basis of user acceptance testing. It is, according to the project team, a configuration that is finished, has passed system testing, and is ready to be deployed and used.

Key Considerations
The tests are specified by the project team and are conducted in an environment as close to the real operating environment as possible.
Tailoring
Impact of not havingIf a System Tested Solution is not completed, there is a very high potential that the system will not perform or function as expected in the operational environment.
