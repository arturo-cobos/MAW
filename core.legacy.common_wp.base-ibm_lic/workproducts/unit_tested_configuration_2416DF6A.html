Outcome: Unit Tested Configuration
A Unit Tested Configuration is a configured package in which the business processes within each individual module have been validated against the future business process design.
Work Product Kinds: Confirmation
Purpose
The purpose of the Unit Tested Configuration is to provide the first major milestone on the road to a fully configured system that clearly supports the future business processes.
Main Description
All relevant module functions, including any common variants are tested using typical business scenarios. Unit testing is performed at the individual transaction level and is a prerequisite to the integration testing (cross module testing) which will follow.
Tailoring
Impact of not havingIf a Unit Tested Configuration is not completed, there is a very high potential for large and complicated problems during the integration testing. Potentially, this could jeopardize the cut-over date as integration testing would be extended. The configuration integrity must first be validated on a module by module basis before it can work as an integrated unit.
Reasons for not needingA Unit Tested Configuration is always the first step in developing the fully configured and tested package.
