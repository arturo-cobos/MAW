Main Description
The objectives of the Project Control Book are:
  • To define a standard way of producing, issuing and maintaining project documents.
  • To help you to control the project.
  • To provide you and all project members with up-to-date information on the progress of the project.
  • To provide a reference document of procedures against which project team performance can be measured.
  • To ensure that the project conforms to PMM standards.
 
The Project Control Book is designed for practical use within the project, and is the repository for the plans, controls, and procedures for a project. It contains:
  • The procedures used on the project.
  • The dynamic files that contain the results of the procedures, for example, the plans, risk items, change logs, etc.
 
At the start of the project, it is important to begin in an orderly manner and to ensure that all necessary infrastructure, assets and resources are in place. The Project Control Book is the focus for these activities.
 
Either the Project Manager or the Project Management Assistant (PMA) can prepare the Project Control Book. The PCB Documentation and Standards information within WSDDM provide guidance for document preparation and formatting.
 
If an automated tool such as Project Office or PCB is used as a part of the engagement, items such as Risk Management, Change Management, Issue Management, etc. will be tracked electronically, rather than on paper in the Project Control Book. In that event, there should be a single page for each of those sections referencing the location of the electronic Project Office. Periodically a print out of current information should be stored in hardcopy as well. These print outs will serve as a backup to the electronic system.
