Artifact: Requirements Traceability and Verification Matrix (APP 402)
The Requirements Traceability and Verification Matrix (RTVM) is a document that is used to verify that the system being built will satisfy the customer or business requirements. It traces the customer / business requirements through the project design, development, and test.
Domains: Requirements
Work Product Kinds: Measurement
Purpose
The Requirements Traceability and Verification Matrix (RTVM) has two purposes:
  1. Provide traceability of requirements through the project life cycle
  2. Provide a verification matrix that shows how each requirement will be tested, when each requirement will be tested, and the test acceptance criteria for each requirement.

Description
Brief Outline

The RTVM contains the following information:

  • Customer (or Business) Requirement: a requirement provided by the customer that gives the operational requirement. 
  • System Requirement: a requirement derived from the customer requirements that states what the system must do.
  • Component Requirement: a requirement that is derived from the system requirements that states what a component must do. System requirements are allocated to component such that the sum of the component requirements fully satisfies the system requirement.
  • Design Document: design documents that are affected by the appropriate requirement.
  • Build Components: build components that are affected by the appropriate requirement.
    Acceptance Criteria: the definition of the results expected from the test cases to be performed. The product must meet these criteria before implementation can be approved.
  • Test Method: method by which the requirement will be tested. See 'Advice and Guidance' section below.
  • Test Type: Testing type that will be used to verify/validate the requirement. See 'Advice and Guidance' section below.
  • Test Case: test case to be run to verify/validate the requirement.
  • Test Results: 'Pass' or 'Fail' - the results of running the test.
