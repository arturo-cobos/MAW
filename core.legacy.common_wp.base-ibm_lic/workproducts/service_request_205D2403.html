Artifact: Service Request (APP 304)
The Service Request work product is used to record the details of work received from a client that IBM is required to implement.
Domains: Maintenance
Work Product Kinds: Inventory, Requirement
Purpose
The purpose of this work product is to document the details of a request for work from a client and track the progress of that work throughout the various phases of its lifecycle. At the start of a project this work product documents the client’s request for a body of work to be done and what that work should produce. During the course of the project this work product helps guide to project to produce what the client has stated is required. At the end of the project, this work product helps establish the guidelines that will determine if the client’s original needs or desires were met.

Description
Main DescriptionThe Service Request work product is normally associated with an automatic system that allows the Service Request to be recorded and tracked throughout its lifecycle. This work product is a standard document used by the Project Office to document and track client requests from initiation through final delivery of the requested functionality. This work product is a standard element in the Project Control Book. Requests received for action under an established project or task are treated as Project Change Requests.
