Artifact: High Level Implementation Strategy (ENG 024)
The High Level Implementation Strategy provides key information to the implementation team and the client in planning the implementation of the selected solution as well as facilitates the transition to the implementation project team.
Domains: Project Management
Work Product Kinds: Strategy
Purpose

The High-Level Implementation Strategy is used as a communication tool between the selection and integration teams to facilitate a smooth transition from the selection activities to the implementation of the vendor solution. It provides the bridge for the integration team to quickly gain the necessary details for an expedient project start-up and detail planning of the software implementation. This work product:

  • Assists in the development of an implementation work plan with the integration team
  • Defines an initial implementation release strategy
  • Identifies key target dates and a high-level time-line
  • Defines the project team roles and responsibilities
  • Identifies the next steps for the project

Description
Main Description

The High-Level Implementation Strategy is produced after the final selection of the software vendor. The following information is assessed or estimated to produce this work product:

  • Application groupings for release planning
  • Preliminary application customizations
  • Initial integration and interface points between the vendor software and the existing client applications
  • Database components requiring data conversion
  • Potential organizational changes points that may require procedural changes
  • Potential role resource types and descriptions for the implementation team

Information developed to support this Work Product Description can be carried forward to assist in the development of the Statement of Work that will be needed for the follow-on phases of the engagement. This work product defines the key components needed by the implementation team, which will be used in conjunction with the Methodology adoption Workshop (MAW), to develop the detail work plan for implementation.
Brief Outline

The High-Level Implementation Strategy defines a high-level document for the implementation of a software solution. The format of this section of the work product is best described using a Gantt chart. This can best be accomplished using a spreadsheet or a project planning tool. This plan defines overall timeline and key target dates.

The release plan recommends an implementation of the software based on the priority of the applications and their dependencies. It defines the “application groups” for implementation planning and sets a priority of the applications using client criteria and other organizational and technical influences for the timing of the implementation (e.g. integration dependencies). The plan further defines the resource role types from both the client and implementation team for the software implementation.

Key Considerations
The IBM Selection team should obtain current application and technical architecture models from the client before starting this work product. This work product is dependent on the availability of these application models.
Tailoring
Impact of not havingThis work product is needed to facilitate the knowledge transfer and start-up of the design and implementation phases of the project. Not producing this work product will require additional assessment and analysis time by the implementation team to properly review the client environment and business process requirements to appropriately plan the implementation of the vendor's software. The result could be a longer start-up time and potential missed system requirements needed in the final software solution. It is, therefore, always recommended that this work product be developed.
Reasons for not needingThis work product may not be needed if the implementation partner can be secured earlier enough to allow for the software selection team to work on the detail implementation plan with the implementation team.
