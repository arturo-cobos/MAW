Artifact: Problem Resolution Report (APP 157)
The Problem Resolution Report is a free-format notepad that is used to record the history of how a problem was resolved.
Domains: Maintenance
Work Product Kinds: Record
Purpose
The purpose of the Problem Resolution Report is to act as a log for recording details of actions taken during certain activities carried out when performing an emergency fix, and during any subsequent follow-on action. This information will provide evidence of what actions were taken and who was involved, and will act as a reference point in the event of future changes to the system.

Description
Main DescriptionThe Problem Resolution Report will typically describe what the problem was, at what date/time it was raised, what investigation was performed, what items were changed in order to resolve it, who was contacted during its resolution, what time the fix was implemented, and what retrospective action was taken.
Brief Outline

This is a free-format note pad that should be used to describe actions taken during certain activities for an Emergency Fix.  There are no hard and fast rules about how this should be done; however the following information should be included:

  • Problem ID
  • Change ID
  • Date
  • Time

Key Considerations

Remember that the purpose of the Emergency Fix is to quickly resolve a problem and to get the system up and running as soon as possible.  It will not be possible to provide a thorough explanation of the problem and consequent design of a permanent fix.  The Emergency Fix is temporary and a permanent fix will be implemented afterwards in either the Corrective Maintenance or Enhancement path.

The Problem Resolution Report provides a framework for providing the minimum amount of detail necessary to perform the fix.  It allows those responsible for implementing the permanent fix to know what the problem was, how it was dealt with, and who should be contacted for further information.

Be concise when writing the report so that an adequate amount of information is recorded.  Stick to the facts and do not over elaborate, but at the same time make sure that anyone performing subsequent work is able to understand the detail.

Tailoring
Impact of not havingThe impact of not having the work product is the risk of making uninformed decisions regarding the proposed solution during any consequent corrective maintenance or enhancement work.
