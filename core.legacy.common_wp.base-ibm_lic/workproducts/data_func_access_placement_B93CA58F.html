Artifact: Data and Function Access and Placement (ARC 305)
The Data and Function Access and Placement work product provides the guidelines that must be applied when distributing data and function across the IT infrastructure implemented within the enterprise.
Domains: Enterprise
Work Product Kinds: Specification
Purpose

The Data and Function Access and Placement work product is used to:

  • Ensure any associated IT implementation will support the business specified performance, availability and operational management requirements and characteristics.
  • Ensure the IT infrastructure, which underpins the business functions, is managed efficiently and effectively.
  • Provide a framework for the placement of data and function during system design activities.

Description
Main DescriptionThe Data and Function Access and Placement guidelines are an integral part of the overall enterprise-wide IT Architecture and its component application, data and technology parts. For data, this includes guidelines for prime data, business copies used for management information, and any operational copies that may be distributed for performance, availability or other operational reasons. For functions the guidelines cover the management and distribution of business logic, presentation logic, and data access logic. The work product builds on previously defined process, data and location models and can in turn be used to construct sample data and/or function placement models, for use by development teams.
Brief Outline

The guidance for data placement will cover the following topics:

  • Checklist for Site Selection

The guidance for function placement will include guidelines for distributing:

  • Presentation logic
  • Business logic
  • Data access logic

Key Considerations

The composition of the team should include people who have the right background and experience in the management of data and function, and the issues involved in placement, across all of the major processing platforms and database technologies required.

  • A strawman set of guidelines should be produced for consideration by the team. It is not time efficient or practical to develop them from scratch. The IBM samples above can be used in conjunction with existing client documentation or practice. These should be the focus of the initial workshop and be circulated to all team members.
  • Additional advice, guidance, samples and a more detailed technique paper are available to trained EA Method practitioners in the EA method database.
Tailoring
Impact of not havingWithout a unifying set of guidelines, the placement and management of data and function across the enterprise will be difficult and expensive to manage, with an increased probability of failure to deliver the various required non-functional (availability, performance and manageability) characteristics.
Reasons for not needing

This work product is not needed when clear documented guidance already exists, or very well established de-facto rules for data and function placement and access already exist. Notice that, in some circumstances, such de facto rules may be so simple as to be not obvious (for example, any data item is only ever held once).

It is also not needed in such cases where the enterprise needs for formal data placement guidelines are negligible, such as would be the case when a business system was based on a centralized system.

Representation Options

The work product can take several forms, however not all of these would necessarily be produced on a single project.

  1. A report, detailing the placement rules to be followed, with checklists for the guidance of developers and data administrators.
  2. A summary for management purposes
  3. Sample data and/or function placement models - which will be used as future reference models
  4. Matrices of data/locations and function/locations - based
