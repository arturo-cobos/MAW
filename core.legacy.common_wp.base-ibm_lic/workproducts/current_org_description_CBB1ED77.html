The Current Organization Description is an inventory of organizational information.
Domains: Organization Design
Work Product Kinds: Model
Purpose
The purpose of the Current Organization Description is to provide a detailed blueprint of the current organization against which the redesign activity can be undertaken. It is aimed at providing a shared understanding of the in-scope part of the organization, how it is structured and behaves, and what enablers exist to motivate and support the structure and behavior.

Description
Main Description

This work product contains information on the elements of an organization structure, behaviors and enablers for the in-scope organizational units (e.g., corporate organization, strategic business unit (SBU), functions, teams, individuals).

This description is then validated by and agreed upon with the client as the basis for the remaining organization design work.

Notation

The work product is a textual document, structured according to the in-scope components of a nine-cell organization design matrix.



Structures
Behaviors
Enablers

Enterprise/Strategic Business Unit



Functions, Groups, Teams



Individuals


It will inventory the structures, behaviors, and enablers for the in-scope organizational units (e.g., corporate organization, SBU, functions, teams, individuals).

For more information on the nine-cell matrix, follow this link (ICM AssetWeb / Organization Change / Overview / Other, "Organization Change Brief: Creating a high-performance organization"). Pages 11-30 focus on the nine-cell matrix, and page 29 is a good summary view.

Key Considerations

Focus only on those elements within the scope of the project, and where the information gathering adds value to the project for concurrent or subsequent efforts.

This step is needed only at project start-up.  While the consultant's knowledge about the organization will surely continue to grow as other work products are completed, this particular work product is unlikely to undergo another iteration later in the project.

If the engagement also includes assessment of the current organization, this work product can be completed at a more cursory level.  The consultant should learn enough from the current organization description to be able to agree with the client on what is to be included in the assessment and to set up the assessment activities and schedule.

Alternatively, if the engagement does not include an assessment, the consultant should aim for a thorough description, which remains factual and does not extend into evaluative judgments.

Tailoring
Impact of not havingWithout this work product the redesign would be developed in the dark. It would be like an architect who is asked to draw up plans for changing a townhouse into three flats, without having a current blueprint to show what the building looks like now.
Reasons for not needing

Any project whose goals include major transition will also involve organization change, and therefore also organization analysis and design (OA&D). This work product is key to an effective OA&D activity. If it focuses on the agreed scope, it does not usually have to be large or complex.

It is not needed if:

  • it is not a major transition
  • the business is so new that a totally new organization has to be created

If the organization redesign is being done by the client and is therefore out of the engagement's scope, this work product is not needed
