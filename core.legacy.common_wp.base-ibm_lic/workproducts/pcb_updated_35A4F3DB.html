Outcome: Project Control Book Updated
The Project Control Book contains all the project governance documents that are critical to managing the project.
Work Product Kinds: Installation/Configuration
Main Description
The objectives of the Project Control Book are:
  • To define a standard way of producing, issuing and maintaining project documents.
  • To help you to control the project.
  • To provide you and all project members with up-to-date information on the progress of the project.
  • To provide a reference document of procedures against which project team performance can be measured.
  • To ensure that the project conforms to PMM standards.
 
The Project Control Book is designed for practical use within the project, and is the repository for the plans, controls, and procedures for a project. It contains:
 
The procedures used on the project.
 
The dynamic files that contain the results of the procedures, for example, the plans, risk items, change logs, etc.
 
Regardless of the type of Project Control Book prepared (electronic vs. paper), managing the Project Control Book is accomplished by the same set of tasks. There are two distinct aspects to managing the solution. The first is to manage the implementation activities, which include:
  • Ensuring all commitments are met (as identified in the Project Plan).
  • Reporting back to IBM management and to the client.
  • Interfacing with Quality Assurance
  • Managing of expectations, as reflected in Changes, Issues, Problems and Risks.
 
The second aspect is to manage the contract. Contract files must be maintained to ensure that all contractual obligations are met. Also that the invoices are prepared and provided to the client per the terms of the contract, and that all monies are collected per the terms of the invoices. In addition, supplier invoices must also be approved and paid.
 
There are two types of tasks that support managing the Project Control Book. Iterative tasks include:
  1. Managing the project plan, which will affect many of the ancillary plans included in the Project Control Book, such as the Organization and People Plan and the Finance Plan.
  2. Tracking project status, to support preparation of the Status Reports.
  3. Preparing reports, such as the Status Report.
  4. Participating in Quality Assuranceâ€™s Project Management Reviews.
  5. Conducting Deliverables reviews, according to the Quality Management Plan.
  6. Tasks that are triggered by events include:
  • Exception Management:
  • Change requests
  • Early closure requests
  • Critical situations identified by the customer
  • Customer payment or adjustment.
  • Deliverable status feedback or completion information.
  • Supplier invoices.
  • Discrepancy between actual progress and planned progress.
  • Identification of new project risks.
 
Both sets of tasks will result in changes or additions to the Project Control Book. Sections should never be removed from the Project Control Book since it is the record of how the project evolved.
