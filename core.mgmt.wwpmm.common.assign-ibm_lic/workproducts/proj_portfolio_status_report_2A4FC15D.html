Artifact: Project Portfolio Status Report (ART 0570)
This artifact provides summary, selected, trending and aggregated information from a selected set of projects which make up the project portfolio.
Domains: Project Management
Work Product Kinds: Record
Purpose

The purpose of this artifact is to enable the appropriate level of review of a portfolio of projects by providing relevant information to key stakeholders so they can arrive at valid portfolio assessment decisions. It supports the project portfolio reviews which occur at various levels of management in the organization.

A report supporting performance reviews of the project portfolio would enable assessments to identify which projects require additional support or attention because they are failing to meet schedule, financial or quality commitments.

A report supporting process reviews of the project portfolio would enable assessments of the compliance of projects to the organization’s process framework, and indicate the effectiveness of implementation of the organizations processes and management system.

A report supporting people reviews of the project portfolio would enable assessments of the staffing allocations to all projects in the portfolio, to identify either skills gaps or capacity gaps in resource availability, and to monitor the training programs for all staff across the portfolio.

Description
Main Description

The information in this artifact is based on the status information reported from each project, as well as information developed or collected at the portfolio organization level representing summary status across the portfolio. From these various sources the portfolio report may derive the following categories of information.
Selected or summary information about:

  • Project identification – which identify the context or groupings of the project within the portfolio.
  • General health – assessments of health as Green, Yellow or Red indicators. Statistics on percentages of project health against other project parameters.
  • Status text - describing the project or portfolio
  • Schedule - start and finish dates for projects and potentially phases, variation calculations
  • Financial information – potentially cost, revenue, margin financials from each project, variation against plan calculations and aggregated financials across the portfolio
  • Earned Value - derived from project financial and schedule information
  • Work and staffing information – work effort hours from each project, and an assessment of the staffing effectiveness across the portfolio.
  • Deliverable & Milestones: dates and status for selected focus items
  • Service Level Agreements (SLA’s) - operational outcomes based on agreed commitments with client, or imposed by delivery organization, other than deliverables. For example, responsiveness or productivity metrics
  • Scope / Exceptions - issues, risks, changes, and / or compliance incidents including selected focus items which are of value to the portfolio review, or summary statistics of items
  • Quality - process quality and solution quality. Usually managed internally not reported to clients. Material includes process assurance results, defect tracking metrics and results from independent project management reviews and process assurance reviews.
  • Actions from reviews - Actions relevant to project portfolio review meetings and requiring tracking.
Brief Outline

The report content includes:

  • Selected or summary project information from the project status report details.
  • Calculated portfolio values based on information from each project.
  • Trending portfolio values based on calculated portfolio values across successive reporting periods. (e.g. percentage of late scheduled projects)
  • Periodic portfolio values based on calculated portfolio values for all projects active within a period. (e.g. total budget portfolio financials for calendar year)
Notation

The report may take many formats. The most effective project portfolio status reports would be highly configurable and selective, so the optimized set of information is provided to each review, depending on whether the focus is process, performance or people items.

The labelling of each report should enable the set of projects included in the portfolio to be identified, and any criteria applied for including or excluding information in the report.
Key Considerations

A key requirement for meaningful information being reported in a portfolio report is consistency of data input. Consistency of project level data is achieved though consistent procedures, or guidelines across the portfolio. Data items which may need validation for consistency are:

  • Project Health indicators both overall and detailed red / yellow / green indicators for financial, schedule, risk or quality health.
  • Deliverable, Milestone status indicators
  •  Issue, Risk, Changes, Compliance Incidents - rank or priority assignments
  • Quality status indicators
  • Currency of information (when last updated)
Tailoring
Impact of not having

Lack of current and accurate information about the project portfolio makes it difficult for the management to make informed decisions and take appropriate corrective actions when individual projects or sets of projects are varying from plan or indicating risk or customer satisfaction concerns.

The intent of the process, performance or people reviews of the portfolio cannot be achieved without optimized reports. The content of the reports must be current, and span the essential categories of information to enable review assessments.
Reasons for not needingThis artifact is not needed if the organization is made up of only one project and only one layer of management as the project status report would be adequate to satisfy all communications of project status to all levels and chains of management
