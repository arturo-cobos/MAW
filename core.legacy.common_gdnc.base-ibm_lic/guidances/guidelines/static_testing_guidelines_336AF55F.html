Guideline: Static Testing Guidelines
This document describes the static testing techniques that can be used as well as provide guidelines for recommended use of each technique. The techniques can be applied to any IT project for conducting static test of work products throughout the development lifecycle.
Main Description
Static testing involves testing of the development work products before any code is developed.

The purpose of the Static Testing Plan is to ensure that any defects in work products are found and removed, as close to the defect entry point as possible and as early in the software development lifecycle as possible. Early removal of defects increases project productivity by shortening testing cycles and reducing work. It also contributes significantly to improving the quality of project deliverables. This work product provides guidance on defect detection and is aimed at producing a "defect free" work product, before it is passed on to the next phase of the development process. Static testing may be applied to many different types of product deliverables such as requirements, specifications, design, code, prototypes as well as testware including test plans and scripts.
