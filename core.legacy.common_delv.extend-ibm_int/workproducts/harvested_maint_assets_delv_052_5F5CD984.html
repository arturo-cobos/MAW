Deliverable: Harvested Maintenance Assets Deliverable (DCD 052)
This deliverable represents the culmination of the project in capturing assets from the project for re-use.
Purpose
The purpose of this deliverable is to collect the re-usable assets from the project so that future projects can be performed more effectively.
Main Description
The harvested assets can be a variety of items from the project, and the engagement templates that were used could be different for each project. However in an AMS environment it is likely that one template will be used for each type of project, depending on size, rather than one per individual project. The project estimates are the actual effort and costs recorded from monitoring the project, and can be re-used ass initial estimates for projects of the same size and type.
Deliverable Properties
Deliverable Parts
  • Assets Harvested
  • Delivery Process Archived
  • Estimation Report
Packaging Instructions

The constituent parts of the deliverable are submitted to ICM, and the actual effort and costs should form part of a database that can be re-used by the whole AMS account.

The following work products that make up the deliverable are set out in the sections below. Each work product will be complete because this deliverable is produced at the end of the project just prior to completion. The assets may be a variety of items that found use on the project and can be re-used on future projects.
