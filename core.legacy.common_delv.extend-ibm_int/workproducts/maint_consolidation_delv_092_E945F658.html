Deliverable: Maintenance Consolidation Deliverable (DCD 092)
The deliverable provides details for a problem that was corrected, either as part of an off-hours emergency fix or as part of normal maintenance.
Purpose

The purpose of this deliverable is to record in one place the whole history of a problem, its permanent solution, and the approach to be taken during the project to solve the problem.

Main Description
This document should contain the following:
 
  • Identification of the system on which the problem occurred and name of the person reporting the problem.
  • Description of the analysis that was performed.
  • Any applicable free format notes that record the history of how the problem was resolved.
  • A permanent, up-to-date, readily available project definition that summarizes the content of the engagement, and initial estimates ("Budgeting and Planning") for the project.
Deliverable Properties
Deliverable Parts
  • Emergency Fix Analysis
  • Estimation Report
  • Problem Resolution Report
  • Problem Ticket Form
Packaging Instructions

The final document will be delivered to Application Development in a reproducible format. The document may also be delivered in hard copy as a single bound document.

Each work product should now be complete because this deliverable will be used to start the project that produces a permanent solution to the problem(s) encountered.

If the Problem Incident Report is part of an electronic problem reporting system, then an attempt should be made to take a copy of the relevant page from the system and paste it into a separate document file, or obtain hardcopy printout from the system for inclusion in a single bound copy of the deliverable.

If an estimating tool is used to provide the estimate in an electronic format, then an attempt should be made to copy and paste this into a document or spreadsheet that can be concatenated with the other work products. If this is not possible, then hardcopy should be used for all the work products, and they should be bound into a single document.
