Packaging Instructions

IBM will deliver one copy of the source code, related code files, and system documentation. The actual source code and related code files can be delivered on CD-ROM, moved to a specific directory on in the client environment or be delivered by providing other access such as a URL for a website containing the actual application. System documentation will be delivered to the customer in a reproducible format. The final report may also be delivered in hard copy format printed in a single, bound document.

This deliverable is comprised of the following work products:
Application Maintenance Turnover Package – documents the actual source as well providing information on file locations and expected directory structures. A listing of any required external libraries or other supporting source files and their locations as well as version information should be included.
Build Procedures - used to outline the procedures required to consistently generate executables from the source code. Should include file locations for all components including media files and additional libraries. Media Content – provides the source for supporting files such as audio and video. The actual files can be included or a simply a pointer to the appropriate location.
Source Code – are the actual files that contain the source code for the application including any supporting files such as header files that would be needed to generate an executable version of the application.

Additional information may be included as needed from the following work products: Increment Goals, Class Diagram, Interaction Diagram, Program Module Invocation Model, Program Module Specifications, Database Transaction Descriptions.
