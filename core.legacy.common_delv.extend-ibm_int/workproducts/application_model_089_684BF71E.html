Deliverable: Application Model (DCD 089)
This deliverable describes the conceptual representations of the solution to be delivered with both textural and graphical components.
Purpose
The purpose of this document is to outline the solution definition based on user specifications, business processes and business requirements. This will provide a conceptual description of the contents and structure of the system. The conceptual description will assist with understanding the scope and complexity of the application.
Relationships
Parent Deliverables
  • System/Component Architecture
  • System Test Architecture
Main Description
Two possible approaches to work products are offered here. The work products to be included in this deliverable depend largely on whether the application will be developed with structured or object oriented techniques.
 
This document should contain an outline of the basic guidelines to be followed for the analysis and design effort, and one of the following:
  • A model that outlines the classes and their major responsibilities and collaborations for the key defined use cases, if object oriented techniques are being used. Object oriented techniques use the Component Design to determine the level of detail required to establish a reasonable level of understanding with the customer.
  • A model that outlines the automated application functions and information stores for the key defined use cases, if structured techniques are being used. Structured techniques use the DFD Application Model to determine level of detail required to establish a reasonable level of understanding with the customer. This would usually be at level 1 or 2 of the model (level 0 is produced earlier as the System Context).
