Deliverable: Requirements Matrix Deliverable (DCD 029)
The Requirements Matrix deliverable is a document detailing the functionality required by the client in its software solution.
Purpose
The Requirements Matrix is a matrix that is used to capture client requirements for software selection and to evaluate the initial functional "fit" of a vendor's software solution to the business needs of the client. It allows the vendor to highlight their ability to satisfy the client's needs and provides an initial means to compare multiple vendors against each other. In addition, vendor response results can be used as elimination criteria in determining the vendor short list.
 
The Requirements Matrix also is used to identify initial functional gaps or special software enhancements needed to enable each vendor's software to fulfill the client's desired system capabilities.
Main Description
This document is generated from the results of the business process definition task and details the system requirements for the software being evaluated. This deliverable consists of business, general, and technical software functionality requirements.

Packaging Instructions

The Requirements Matrix is developed by the entire Selection team during the business process definition.  The IBM Selection Team members refine the requirements, include appropriate best practices and complete the matrix for delivery to the vendors.

The results from defining business process definitions and associated work product components are input to creating the Requirements Matrix.

Prepare the Requirements Matrix according to the following steps:

  1. Utilize the various models and work product components produced during the business process definition task/tasks to identify the business areas to be evaluated in the Requirements Matrix.
  2. Identify business requirements for each future business process identified and group them by business area.  Include the following:
           - Business Process Functionality
           - Business Rules Support
           - System Calculation Ability
           - Workflow
           - Reporting
           - Data Collection
           - Query Capability
  3. Identify general requirements in the software and from the software vendor that will be needed by the client for their environment and resources.  Include the following:
           - Software Maintenance
           - Help Desk Support
           - Documentation
           - Training
           - General System Navigation
           - User System Interface Features
  4. Determine the technical requirements for the software.  This should be done with the assistance of the client IS staff responsible for the current and future environments. Include:
           - Technical Infrastructure and Architecture
           - Application Architecture 
           - Security Management
           - Technical Maintenance and Support
           - Software Maintenance and Control
           - Database Maintenance and Monitoring
           - Problem Investigation and Support
           - Performance Monitoring and Tuning
  5. Validate final requirements with client team members in a work session or workshop.
  6. Assemble and refine the Requirements Matrix.  Create a numbering sequence by business area for each group of requirements to allow for scoring to be assessed by business area, general and technical components.
