Deliverable: Future State Process Definition (Volume 3) (DCD 275)
The Future State Process Definition - Volume 3 is a deliverable to the client documenting the definition of the process and environment.
Purpose
The purpose of this deliverable is to provide all of the key information concerning the future environment in a clearly documented volume that can be understood by all involved in the transformation effort. The transformation business case will be based on this definition when estimating benefits and improvements.
Main Description
Sections of the Volume 3 include:
  • DesignFlow Analysis Diagram (BUS 309 Process Definition)
  • Process Narratives
  • Role Definitions (ORG 309 Job, Roles, Responsibilities and Competencies)
  • Activity Matrix
  • Data Usage Table (APP 112 Process Data Usage)
  • Process Metrics
  • IT Profile and description of strategic direction (ARC 308 Enterprise Technology Framework, APP 011 System Context)

External Description

DesignFlow “Volume 3” Document

Purpose:  Describes the Baseline (Future State) process environment at the detailed, individual level.

Content:  The “Volume 3” document includes the following:

  • DesignFlow Analysis Diagram of the Future State process
  • Future State activity narrative descriptions
  • Process metrics currently being captured
  • Future State data, data functions and data usage
  • Identification of applications that will support the Future State process
  • Planned I/T environment
  • Roles and role definitions of the people executing the future state process

Delivery:  IBM will deliver this document in softcopy (Microsoft Word and Visio formats) plus one copy in reproducible hardcopy.

Deliverable Parts
  • Enterprise Technology Framework
  • Job Roles - Responsibilities and Competencies
  • Measurement Model
  • Process Data Usage
  • Process Definition
  • System Context
Packaging InstructionsThe Volume 3 should be created at the during the process design part (Phase 2) of the engagement.  Planning for the required data and what sections to complete would be determined up front.  As the various Work Products are developed, they should be included in the Volume 3 which is delivered to the client at the end of Step 1 of the DesignFlow engagement.
