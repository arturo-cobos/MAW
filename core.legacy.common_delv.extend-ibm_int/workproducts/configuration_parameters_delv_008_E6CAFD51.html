Deliverable: Configuration Parameters Deliverable (DCD 008)
The Configuration Parameters deliverable describes the selection and setting of values and options that customize a package for a client.
Purpose
The purpose of this deliverable material is to document the package parameter settings that are a prerequisite for creating a company specific version of the package. It describes general settings that will be used by several business processes, as well as the specific settings for individual business processes.
Main Description
The customization rules and standards for configuration parameters are key elements that should be well known, agreed, and documented with the client so that the package can be can be correctly configured with the appropriate settings to support the clientâ€™s intended business processes and organization. There may have to be compromises made in the business processes and organization to accommodate the limitations of the chosen package. The selected package may provide some assistance with configuration tools and techniques.

Packaging Instructions

The Configuration Parameters deliverable is a document of approximately 10-15 pages which consists of the following, as appropriate:

  • Global settings
  • Country settings
  • Languages
  • Currencies (exchange rates, decimal places, rounding rules, etc.)
  • Units of measurement
  • Calendar
  • Chart of Accounts
  • Business Organization Structure
  • Company codes
  • Organization structure/hierarchy elements
  • Detailed rules of customizing (configuration activities):
    • Who is allowed to customize the different modules
    • How will the customization be documented, and where
    • Will the customization be done on a sand box system, a development system, or where
    • How will the different versions of the configuration be tracked and approved
