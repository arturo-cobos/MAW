Deliverable: Physical Packaging Deliverable (DCD 085)
This document will consist of - the source code developed for the Release, the executable application code for the Release, and the technical documentation for the application.
Purpose
The deliverable represents the items that will be delivered with the release application code and are required for the installation and execution of the system of the application.
Deliverable Properties
External DescriptionPrepare the physical packaging of the software and related materials developed in the build.  This will contain the source code, executables, installation instructions, user support materials, and other materials, as appropriate.
Deliverable Parts
  • Build
  • Deployment Packaging
  • User Support Materials
Packaging Instructions

The final document will be delivered to the customer in a reproducible format and/or hard copy as outlined in the actual physical packaging work product.  Typically this deliverable is delivered in a combination of formats to accommodate the actual program files and supporting materials.  The program files can be delivered on CD-ROM, moved to a specific directory on in the client environment or be delivered by providing other access such as a URL for a website containing the actual application.  Support materials can be provided in electronic format such as help files or actual bound copies of the materials or even templates and quick reference cards.
