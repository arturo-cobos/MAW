Deliverable: Current State Process Definition (Volume 1) (DCD 273)
The Volume 1 Current State Process Definition is a deliverable to the client documenting a baseline definition of the process and environment.
Purpose
The purpose of this deliverable is to provide all of the key information concerning the baseline environment in a clearly documented volume that can be understood by all involved in the transformation effort. All transformation efforts will be based on this definition when estimating benefits and improvements.
Main Description
Sections of the Volume 1 include:
  • DesignFlow Analysis Diagram (BUS 309 Process Definition)
  • Process Narratives
  • Role Definitions (ORG 309 Job, Roles, Responsibilities and Competencies)
  • Activity Matrix
  • Data Usage Table (APP 112 Process Data Usage)
  • Process Metrics
  • IT Profile and description of strategic direction (ARC 301 Current IT Environment, ARC 308 Enterprise Technology Framework, APP 011 System Context)

Deliverable Properties
External Description

DesignFlow “Volume 1” Document

Purpose: Describes the Baseline (current state) process environment at the detailed, individual level.

Content: The “Volume 1” document includes the following:

  • DesignFlow Analysis Diagram of the current state process
  • Current state activity narrative descriptions
  • Process metrics currently being captured
  • Current state data, data functions and data usage
  • Identification of applications used to support the current state process
  • Current and planned I/T environment
  • Roles and role definitions of the people executing the current process

Delivery: IBM will deliver this document in softcopy (Microsoft Word and Visio formats) plus one copy in reproducible hardcopy

Deliverable Parts
  • Enterprise Technology Framework
  • Job Roles - Responsibilities and Competencies
  • Process Assessment and Analysis
  • Process Data Usage
  • Process Definition
  • Project Definition
  • System Context
  • Technical Environment
Packaging Instructions

The Volume 1 should be created at the beginning of the engagement. Planning for the required data and what sections to complete would be determined up front. As the various Work Products are developed, they should be included in the Volume 1 which is delivered to the client at the end of Step 1 of the DesignFlow engagement.
