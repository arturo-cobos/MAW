Deliverable: Deployment Plan Deliverable (DCD 014)
This deliverable describes the organization, strategy, resources, and methods used to deploy the new application at customer sites. It identifies the responsibilities of everyone participating in the deployment and is written from the user’s perspective. It also serves as the project’s master document over the migration period.
Purpose
The purpose of this deliverable is to guide the management team through system deployment. It provides a detailed schedule of events, expected duration, persons responsible, and event dependencies that are required to support the cut-over to the new system.

Packaging Instructions

The Deployment Plan deliverable is a subset of the overall project plan.  It deals specifically with the latter part of an engagement and manages those tasks aimed at preparing the client to move forward with implementing the new packaged system.  The format for this deliverable material is the project plan with elements that may include:

  • Education and Training Schedule
  • Deployment Sequence
  • Conversion Criteria/Timing
  • Hardware Acquisition
  • Hardware/Software Installation and Configuration
  • Installation Support Requirements
  • Pre-Implementation Activities
  • Post Implementation Activities including Help Desk and Support
