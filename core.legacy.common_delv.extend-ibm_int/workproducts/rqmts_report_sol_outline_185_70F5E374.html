Deliverable: Requirements Report from Solution Outline (DCD 185)
The Requirements Report from Solution Outline overviews the scope of the solution along with its functional and non-functional (qualitative) requirements.
Purpose
The purpose of this deliverable is to document the known set of requirements that could influence the breadth of the analysis and design of the the solution’s application, architecture and approach.
Main Description
This deliverable contains the following content:
  • Solution Scope description
    • System Context
    • User Profiles
    • Business Roles and Locations
    • User Groups
    • Business Rules
    • Current Solution Evaluation
    • Highlighted out-of-scope items

  • Functional Requirements (specific business functions that the solution needs to provide)
    • Use Case Model

  • Non-Functional Requirements (qualitative attributes of the proposed solution such as performance, availability, capacity, usability)
    • Usability Requirements
    • Non-Functional Requirements
    • Change Cases

  • Security and Privacy Principles and Design Objectives
    • Security and Privacy principles provide direction for security in the enterprise. Principles may cover the entire enterprise (Generic) or be limited in scope (Design) to address a particular subset of the enterprise. Examples include Identification and Authentication, Authorization and Access Control, Message Protection, Detection and Monitoring, Auditing/ Logging, Resilience, and/or Security Management.

  • Security design objectives define what is expected of security architecture design and what the role of IT Security should be within that design. These objectives can be summarized as:
    • Trusted Credentials
    • Access Control
    • Information Flow Control
    • Resilience / Integrity
    • Logging/Monitoring

  • Requirements validation approach and results
    • Static Test Plan
    • Use Case Validation Report
    • Test Results
Deliverable Properties
External DescriptionThe Requirements Report from Solution Outline summarizes the scope of the solution along with its functional and the non-functional (qualitative) requirements.
 
The purpose of this deliverable is to document the known set of requirements that could influence the breadth of the analysis and design of the the solution’s application, architecture and approach.
 
The deliverable / report will contain the following content:
  • Solution Scope description
  • Functional Requirements (specific business and security functions that the solution needs to provide)
  • Non-Functional Requirements (qualitative attributes of the proposed solution such as performance, availability, capacity, usability, security, etc.)
  • Security Architecture Principles and Design Objectives (The principles providing direction; the security design objectives defining what is expected of security architecture)
  • Requirements validation approach and results
 
The content in this deliverable forms the baseline for the development of the Solution Outline’s Application Model, Architecture Model, Performance Model, Business Impact Assessment, and Solution Strategy Report deliverables.
 
The content in this report may be further detailed or clarified in the Requirements Report for Macro Design. The intent of this deliverable is to provide the breadth of details needed to understand the overall complexity of the solution, produce reasonable estimates for the Macro Design phase, and planning information for the subsequent Release Phases.
