Deliverable: Project Plan (DCD 311)
This deliverable documents all the activities that the project team will perform in the form of a plan.
Purpose
The purpose of this deliverable is to define all the activities and tasks that the project team must complete and to provide a basis upon which the work can be scheduled.
Main Description
The Project Plan Deliverable is a spreadsheet work plan, which should be used to:
  • Document all the activities and tasks that the project team will perform during the engagement.
  • Associate with each task the resource assignments for proper management tracking of responsibilities.
Deliverable Properties
Deliverable Parts

  • Deliverable Definition
  • Human Resource Plan

  • Project Decision Structure
  • Project Definition
  • Project Management Schedule
  • Project Management System Summary
  • Risk Definition
  • Work Breakdown Structure (WBS)
  • Work Product List
Packaging InstructionsThe final document will be delivered to the customer in a reproducible format. The actual models may have originally been created using a diagramming tool, but should be included as a finished portion of the report for final delivery. The final report may also be delivered in hard copy format printed in a single, bound document.
