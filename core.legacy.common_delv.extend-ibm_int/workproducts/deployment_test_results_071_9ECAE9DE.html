Deliverable: Deployment Test Results (DCD 071)
This deliverable provides a consolidated report of the various testing activities planned for and conducted during the Deployment phase for a solution.
Purpose
The purpose of this deliverable is to verify proper execution of the software including interfaces with other applications. It documents the results of Acceptance Testing along with Systems Integration Testing, Operability Testing, and other testing planned to be conducted in the Deployment phase for a solution release. It additionally serves to gain client acceptance for the tested solution release.
Main Description

This deliverable extends the tests defined to be performed during the Deployment phase at a high-level for a solution as a whole and in detail for the solution release being deployed along with the results obtained for each test executed and any outstanding testing issues relevant to the testing conducted for a specific solution release.

This document consists of the following major sections as apply to a solution and specifically to a given solution release:

  • For Acceptance Tests conducted
    • Definition of the acceptance tests conducted
    • The results of each test
    • A summary of the incidents reported during the Acceptance Testing and the current status of any outstanding incidents
    • Unresolved issues identified in Acceptance Testing
    • Solution release acceptance recommendation
  • For Systems Integration Tests and Operability Tests, and other testing levels planned to be  conducted during the Deployment phase beyond Acceptance Testing
    • Definition of the tests conducted
    • The results of those tests
    • Unresolved issues identified for those tests
  • For Penetration Testing
    • Definition of the conducted test
    • Results of the test
    • List of issues identified in the test and their status
Deliverable Properties
External Description

The Deployment Tests Results deliverable provides a means to verify proper execution of the software including interfaces with other applications and to gain client acceptance of the tested solution release.  Additionally, it verifies the security of the solution.

This deliverable documents the results of the following actions as pertain to the scope of a specific release for a solution.

  • Execution of the test cases defined to be conducted in the Deployment phase for a solution release’s Build verifying the solution’s system interfaces and interactions are functionally correct and operationally sound, ensure the solution created is secured, refining the goals for future release Builds based on identified errors, and documenting testing results.
  • Evaluation of the development and testing results conducted in the Build Cycle phase for the release’s completed Build along with adjustments and refinements to the goals and estimates for future solution release Builds, if needed.
  • Refinement of the specifications for acceptance testing based on feedback from the system and integration tests conducted for the release during the Build Cycle phase.
  • Assisting the client’s staff identified to perform and participate in executing the defined test cases for the test levels planned and conducted in the Deployment phase to verify that the system is functionally correct and operationally secure and sound.
  • Definition and management of the problem reporting process for acceptance testing.  This includes defining and communicating the process for reporting errors, tracking all errors reported by the users during the acceptance testing, correcting errors, and notifying the users when the errors have been corrected.
  • Definition and management of the problem reporting process for penetration testing.  This includes defining and communicating the process for reporting security holes, status tracking of fixing these holes, correcting them, and notifying the client when they have been corrected.
  • Documenting the results obtained for all tests conducted.

The tested solution release is deemed acceptable for client receipt provided Acceptance Test results fall within the criteria and tolerances specified in the Acceptance Test Plan.

Packaging Instructions

The final document may be delivered in hard copy format printed in a single, bound document.

The following work products contribute to the various deliverable topics or sections.

The testing specifications work products identified for each testing level are fully defined at a detailed level for a specific release during Micro Design. They may each be revised, as needed, during the Build Cycle and Deployment phases for a specific release. Modifications may also be made while developing one release that may apply to the solution as a whole or to its future releases.

Some work products may also contain detailed descriptions of prior solution releases and high-level descriptions for future solution releases that are discrete to those releases and not pertinent to the release undergoing the Deployment phase. Such work products will be iterated and more fully developed in later phases of the engagement as the need for greater detail is identified and required per each solution release. In this regard, work products input to this deliverable may be extensions and/or revisions to previously generated work products while others may be separate and distinct instances specific to this project and/or phase.
