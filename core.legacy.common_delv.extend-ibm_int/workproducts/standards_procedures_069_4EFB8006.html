Deliverable: Standards and Procedures (DCD 069)
This deliverable documents the development procedures and relevant standards for transforming a solution design into its physical and implementation form.
Purpose
The purpose of this document is to provide the standards, guidelines and quality assurance for data conversion and development of code and executables to ensure that a repeatable process is followed and applied for each release of a given solution. Additionally, they document the expected standards necessary to meet any additional compliance issues required by the client. This does not address standards associated with any legacy systems.
Main Description
This deliverable covers the following as apply and pertain to a given solution en total and particularly to a specific release of a solution.
  • Overall quality assurance standards.
  • Design Guidelines, updated if appropriate.
  • Coding standards for readability and maintainability.
  • Documentation standards.
  • Data standards, e.g. data declarations, maintaining data types, using floating-point numbers in intermediate calculations, etc.
  • Subroutine and functions standards, e.g. structure rules for call of subroutine of function, etc.
  • File processing guidelines including configuration management procedures.
  • Standard build processes including expected file locations.
  • Processing logic guidelines e.g. I/O processing and handling of error messages.
  • The naming conventions to which each of the development areas are to abide.
  • Process for exceptions or extensions to the agreed upon standards.
