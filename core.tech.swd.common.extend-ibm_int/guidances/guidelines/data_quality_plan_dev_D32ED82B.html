Guideline: Developing Data Quality Plan

Main Description

The development approach for the tactical level of the Data Quality Plan is outlined below.

Step 1 – Define the project level data quality requirement
The required level of data quality that is intended to be achieved by the project/customer needs to be clearly stated in the requirement.

Step 2 – Define the Goals and Objectives of Data Quality Plan
Define what needs to be achieved with respect to data quality plan and how it would be achieved.

Step 3 – Identify the critical data elements
List the critical data elements that would impact the data quality that is defined in requirement.

Step 4 – Assess the existing data quality
Measure the data quality of the existing data for which certain level of quality is being targeted in the requirements.

Step 5 – Review and analyze findings
Review the data quality that is attained after applying the data quality plan to find out the gaps between the planned and achieved quality.

Step 6 – Identify / refine the steps to attain the required quality for the data
In case gaps identified in the previous step, identify the additional steps to be added in order to achieve the planned data quality.

Step 7 – Apply the recommendation to attain data quality
Execute the steps identified in the previous step to attain the data quality as per requirement.

Step 8 – Measure data quality to assure required data quality is attained after applying recommendations
Measure the data quality to make sure the data quality is attained as per the requirements defined.
