Concept: CSI Method Blueprint Overview
This overview establishes a basic Complex Systems Integration delivery model and maps various method materials to it, providing an approach for program or project method adoption to make sure the underlying method guidances are understood and can be maintained.

Main Description

The purpose of this material is effectively twofold: firstly, to provide an overview of method more generally, but focusing on complex systems integration, and secondly, to provide a blueprint for method on complex SI engagements. The material does not contain detailed method definition.  Instead, it attempts to provide an overall picture of method basics and show how various detailed method “topics” relate to this overall picture.

This is the basic CSI delivery model and the basis for much of the material in this presentation.


Basic CSI Phases (horizonal axis)
Phase Description
Solution Outline
  • Confirming requirements and establishing outline E2E system solution in terms of breakdown into major components
High-Level Design
  • Performing E2E system integration design and high-level component solution designs within this context
Detailed Design
  • Performing detailed design of components and preparing for build
Build
  • Building and / or configuring components and performing component test
Test
  • Integrating components into an E2E solution and testing this solution
Deployment
  • Deploying the E2E solution into production
Operate / Support
  • Operating and supporting the solution once it is in production


Main CSI Activity Areas (vertical axis)
Activity Area Description
Client Relations
  • Building and maintaining relationships with key client stakeholders
Delivery Governance
  • Establishing solution and delivery method, process and tool framework
  • Delivery project management and commercial governance
  • Scope management
  • Technical solution governance
  • Includes key review checkpoints
E2E Solution Delivery
  • Requirements confirmation and management
  • Establishing solution outline and maintaining detailed integration solution definition
  • Integration and E2E system test preparation and execution
  • E2E solution operation once solution is in production
Component Solution Delivery
  • Design, build, test, fix and maintenance of solution subsystems
  • Component development method depends on nature of component, but component solution must always align with its context in E2E system solution
Delivery Environment
  • Design, build and operation of environments to support development and production
Business Readiness
  • Activities to prepare the business for the upcoming change once the solution goes into production … from re-organization to training
