Deliverable: Data Model Deliverable (DEL 0552)
The Data Model deliverable is a report that outlines the solution definition based on user specifications, business processes and business requirements.
Purpose

The purpose of this deliverable is to:

  • Complete the logical analysis of the application model for the release 
  • Provide a basis for subsequent transformation into the implementation platform specific physical application model
  • Provide sufficient technical details about the business and supporting administrative functions included in the release, such that the programming specifications can be derived.

Main Description

The report will include the following details relevant to the specific intended release of the solution:

  • Component Design model(s) showing the class structure, dependencies, and collaborations for the the components comprising the solution
  • Guidelines used in the development of the application structures
  • Physical database design depicting the persistent data storage structures to be used in the solution
  • Data migration specifications showing how data will be loaded and extracted from the database
Deliverable Properties
External Description

The Data Model deliverable is a report that outlines the solution definition based on user specifications, business processes and business requirements, for the specified release.

The purpose of this report is to provide a conceptual description of the contents and structure of the solution. This is used to complete the identification of contents and structure of the solution, and to provide a foundation for mapping the logic design to the physical implementation.

The report will include the following details relevant to the specific intended release of the solution:

  • Component designs showing the class structures, dependencies, and collaborations for the the components comprising the solution
  • Guidelines used in the development of the application structures
  • Physical database design depicting the persistent data storage structures to be used in the solution
  • Data migration specifications showing how data will be loaded and extracted from the database
