Artifact: Maintenance Asset Log (ART 0544)
This artifact tracks project assets which are required as an application migrates into the production environment and is maintained and supported.
Domains: Maintenance
Work Product Kinds: Record
Purpose
This artifact describes documents information and acceptance criteria about project assets deemed necessary by the support organization during an application turnover to maintenance and support. Project asset information and acceptance criteria are added depending on specific turnover requirements.
Description
Main Description

This artifact is used to control the development and final sign-off of project assets that impact the turnover of an application to maintenance and support in the production environment.  The log is required to manage the project assets which contain the information that is required to accomplish the turnover.  In addition, project assets which document application operation, maintenance, and support requirements are controlled in the log.

Brief Outline

The project assets in the log are a recommended list. The transition project team should determine the project assets required by the support organization. Here is a list of some of the typical information collected for each asset type:

  • Serial Number – a unique identifier for the project asset
  • Asset Name – include the UMF Artifact unique ID if appropriate
  • Asset Type – a logical grouping of the project assets. For example, Architecture documentation, Build sheets, Test Cases, Client sign-offs, and others
  • Asset Cross reference – describe the relationship between the project asset and others (if any)
  • Asset Status – Completed or Partial. If partial, describe when and how the asset will be completed
  • Asset Acceptance Criteria – Describe the established acceptance criteria for the project asset
  • Sign-off (acceptance) status – Complete or partial. If partial document when and how the acceptance will be completed
  • Asset physical media – document what the asset is created in and where it is stored

Key Considerations

The Asset Log is used during the initial stages of the project and can be updated as and when there is more information available on the project.  The log is reviewed by the Project Manager along with the relevant stakeholders to obtain commitment and sign-off of the assets based on the completion criteria.

The concept of completeness of a work product/asset depends largely on whether the work product will be completed during development, or iterated for additional work later. If the work product is to be iterated later, it will only be partially complete at the end of the development phase. Regardless of whether each work product instance is fully or partially completed at this time, the content of the work product should be fully included in the deliverable being produced.

Tailoring
Impact of not having

If this Artifact is not completed it can lead to miscommunication, lack of urgency, responsibility mix-ups, and other miss-understandings that will impact the success of the turnover.

Managing an application turnover to production requires a handoff between organizations. Information about the application and about the turnover requirements is primarily captured in various project assets. The use of this artifact provides an understanding of what is (or is not) documented and accepted about the application.

Reasons for not needingIt is recommended that all application turnovers, regardless of application complexity use this artifact.
