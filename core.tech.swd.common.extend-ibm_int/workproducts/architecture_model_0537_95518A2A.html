Deliverable: Architecture Model (DEL 0537)
This architecture model is a description of the technical solution. It provides an overview of the anticipated software and hardware platform(s) required to run the proposed business solution, as well as a consolidated view of the requirements focused on these technical elements.
Purpose

The purpose of this deliverable is to gain customer consensus and understanding of the architecture proposed to operate the business solution. This provides the basis for procurement, construction, implementation, and operational activities to be performed

Main Description

The deliverable / report will contain the following content: 

  • A specification level Operational Model presenting a specification of the functional and non-functional characteristics for all elements within this release of the target IT system: 
    1. The placement of specified components onto specified nodes, 
    2. The specified connections between specified nodes necessary to support the interactions between these specified components, 
    3. The non-functional characteristics of specified nodes and specified connections, acquired from the placed specified components and their interactions. 
  • A physical level Operational Model, and supporting Component Model, documenting the overall configuration of the technologies and products necessary to deliver the functional and non-functional requirements of the IT system specified in the specification level. It provides a fully detailed, sized configuration, suitable for use as a blueprint for the acquisition, installation, and subsequent maintenance of the solution, and details:
    1. Hardware and software technology and product selection, for computers and networks, 
    2. Hardware specifications, such as processor speed and disk configuration, or network bandwidth and latency, 
    3. Overall hardware configurations, such as "fail-over" or "scalable" arrangements, 
    4. Software product specifications (such as version) and detailed configuration, such as the need for multiple instances of a software product on a computer. 
    5. Service Level Characteristic Analysis and Viability Assessment are included if challenging architecture problems are anticipated.

External Description

The Architecture Model is a description of the technical solution to be implemented which will provide the IT operational capabilities required for the current solution releases.

The purpose of this document is to present the design for the solution architecture to be implemented in the current release(s).

The deliverable / report will contain the following content:

  • A specification level Operational Model presenting a specification of the functional and non-functional characteristics for all elements within this release of the target IT system:
    1. The placement of specified components onto specified nodes,
    2. The specified connections between specified nodes necessary to support the interactions between these specified components,
    3. The non-functional characteristics of specified nodes and specified connections, acquired from the placed specified components and their interactions.
  • A physical level Operational Model, and supporting Component Model, documenting the overall configuration of the technologies and products necessary to deliver the functional and non-functional requirements of the IT system specified in the specification level. It provides a fully detailed, sized configuration, suitable for use as a blueprint for the acquisition, installation, and subsequent maintenance of the solution, and details:
    1. Hardware and software technology and product selection, for computers and networks,
    2. Hardware specifications, such as processor speed and disk configuration, or network bandwidth and latency,
    3. Overall hardware configurations, such as “fail-over” or “scalable” arrangements,
