Deliverable: Requirements Report from Macro Design (DCD 183)
The Requirements Report from Macro Design further describes and details the scope of the solution along with its functional and non-functional (qualitative) requirements as initially defined and overviewed in the Requirements Report from Solution Outline deliverable.
Purpose
The purpose of this deliverable is to extend the detail of the requirements produced in the Solution Outline phase, to the extent needed to isolate discrete functions and services for partitioning among the various proposed solution releases.
Main Description
This deliverable contains the following content:
 
Solution Scope description
  • System Context
  • User Profiles
  • Business Roles and Locations
  • User Groups
  • Business Rules
  • Highlighted out-of-scope items
 
Functional Requirements (specific business functions that the solution needs to provide)
  • Use Case Model
  • System function model as represented in the DFD – Application Model work product
 
Non-Functional Requirements (qualitative attributes of the proposed solution such as performance, availability, capacity, usability)
  • Usability Requirements
  • Non-Functional Requirements
  • Change Cases
 
Security and Privacy Principles and Design Objectives
  • Security and Privacy principles provide direction for security in the enterprise. Principles may cover the entire enterprise (Generic) or be limited in scope (Design) to address a particular subset of the enterprise. Examples include Identification and Authentication, Authorization and Access Control, Message Protection, Detection and Monitoring, Auditing/ Logging, Resilience, and/or Security Management.
  • Security design objectives define what is expected of security architecture design and what the role of IT Security should be within that design. These objectives can be summarized as:
    • Trusted Credentials
    • Access Control
    • Information Flow Control
    • Resilience / Integrity
    • Logging/Monitoring
 
Requirements validation approach and results
  • Static Test Plan
  • Use Case Validation Report (from Solution Outline Only)
  • Test Results
Deliverable Properties
External DescriptionThe Requirements Report from Macro Design details the overall scope of the solution along with its functional and the non-functional (qualitative) requirements for all anticipated releases of the proposed solution.
 
The purpose of this deliverable is to further document and refine the detail of the requirements produced in the Solution Outline phase to the extent needed to isolate discrete functions and services for partitioning among the various proposed solution releases.
 
The deliverable / report will contain the following content:
 
Solution Scope description
 
Functional Requirements (specific business and security functions that the solution needs to provide)
 
Non-Functional Requirements (qualitative attributes of the proposed solution such as performance, availability, capacity, usability, security, etc.)
 
Security Architecture Principles and Design Objectives (The principles providing direction; the security design objectives defining what is expected of security architecture)
 
Requirements validation approach and results
 
The content in this deliverable forms the baseline for the development of the Application Model from Macro Design, Architecture Model from Macro Design, Performance Model from Macro Design, Cost Benefit Assessment, and High-Level Solution Plans deliverables.
 
The content in this report may be further detailed or clarified in the Requirements Report from Micro Design, for a specified release. The intent of this deliverable is to provide the level of detail needed to adequately isolate the content that will need to be extended or confirmed for each planned release and produce reasonable estimates for the subsequent Release Phases (Micro Design, Build, and Deploy)
