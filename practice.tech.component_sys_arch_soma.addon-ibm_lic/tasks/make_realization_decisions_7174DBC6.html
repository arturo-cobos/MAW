Task: Make Realization Decisions
Making realization decisions is an ongoing activity that takes place in each major juncture in the software development process. Realization decisions are made in order to document the key decisions and rationale behind them.
Purpose

To arrive at and document architectural decisions on how to implement services after considering various alternatives.

    Main Description
    Realization decisions are made in order to document the key decisions and rationale behind them and that can lead to those decisions in the context of the project.  These architectural decisions define how the architectural solution is being built.  At each juncture there are options to take into account and based on the current context and realities of the project combo choices are based that must be recorded so that future modifications have access to the rationale used in making those initial decisions.  The documentation of these decisions will facilitate future enhancements by identifying key variation points and key options that were available but perhaps not quite mature at that time and have possibly matured at the present time. 
    Steps
    Build vs Buy vs Integrate vs Subscribe vs Outsource
    Decide whether to realize services via build, buy, integrate, subscribe or outsourcing.
    Make service realization decisions
    Identify existing application and systems that can be leveraged to implement services by extending and customizing. Identify services that needs to be built.
    Make component realization decisions
    Identify existing application and systems that can be leveraged to implement component by extending and customizing. Identify components that needs to be built.
