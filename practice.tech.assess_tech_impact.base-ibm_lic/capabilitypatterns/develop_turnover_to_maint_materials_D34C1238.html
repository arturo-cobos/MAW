Task Descriptor: Develop Turnover to Maintenance Materials
Create summary packages of changes to the solution's previously existing set of application and service delivery documentation. This is done to facilitate an orderly turnover of this solution to a maintenance organization.

Main Description
The turnover process, from the development team to the maintenance group, involves a series of validations that the requirements of the maintenance group have provided, prior to acceptance. These requirements may have been dictated by the IT Standards or the organization, or by the requirements of the maintenance group.

Use the original initiating document for the project, along with the list of work products selected for development in this project, to identify and locate all work products which have been created or extended in this effort. These are the work products which must comprise the new turnover materials.

For new development, the Application Maintenance Turnover Definition and Service Delivery Turnover Definition artifacts are created to ensure that all required work products for turnover have been provided. A more complete understanding of these artifacts can be developed by reviewing the Application Maintenance Turnover Definition and Service Delivery Turnover Definition work product descriptions.

For enhancements, the Application Maintenance Turnover Definition and the Service Delivery Turnover Definition artifacts are created to ascertain that all updates to existing required work products have been applied. The Impact Analysis should state all work products that would need to be updated, and the nature of the updates to be provided. The Baseline Survey Report should pinpoint the artifact instances existing before the modification should any delta need to be identified.
