Main Description

An analysis of the impact of satisfying client requirements is used to identify other unstated and implied requirements that can be derived from the client stated requirements. This provides the opportunity to surface assumptions and clarify their intent and likely impact. As such, the task may have to be repeated even if partially, in the situations where the analysis surfaces issues for which information is not immediately available.

When completed, the team members should be able to use the output as an initial source of information for subsequent change and design decisions in satisfying client requirements.

The list of work products included as input to this task represents a potential list of work products that could be used (or required) as supporting documentation for maintenance and enhancement of an application.

The Baseline Survey could provide insight to the perceived quality and completeness of work products used as supporting documentation. This would be useful for estimating when an organization's policy requires that any supporting documentation be brought up to date whenever maintenance or enhancement is performed.

Note: This task begins the design process of the changes needed. Subsequent tasks will extend/update/create the content of specific Work Product instances identified.

Steps
Analyze impact on current environment
The following is a guideline for conducting an impact analysis on the current application system :
  • Obtain the client requirements package (e.g., Project Scope, Project Goals, Service Request(s), Problem Incident Report(s), etc.)
  • Determine the preferred way of viewing the current application system (e.g. sub-systems, functional, process etc.)
  • Analyze the client requirements against the segments of the current application environment (including application system)
  • Determine the affected parts of the application environment and their impact
  • Determine any additional issues or interpretations that require further clarification

Note that analysis of impact on current environment is not a design task therefore the temptation to propose solutions should be avoided. The focus should instead be on identifying issues needing further clarification and the likely effects of stated requirements on the current application environment.

Analyze impact of data migration
This step captures the design changes associated with data migration. It uses the existing work products that document the system, and creates textual narrative about what will need to change (if anything) in each of those work products.
Analyze impact of proposed solution
This step captures the design changes associated with the initial Impact Analysis Statement. This task uses the existing work products that document the system, and creates textual narrative about what will need to change (if anything) in each of those work products (from the Business, Requirements, Analysis and Design, Architecture, Training and Support, Construction and Testing Domains).
