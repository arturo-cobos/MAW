    Main Description

    The Deployment Unit (DU) is a mechanism used to represent the various aspects of components , including their Non Functional Requirement’s (NFR’s) and their placement on to nodes on an operational model. Deployment Units can represent both application and technical components.
    This artifact identifies the steps required to develop a DU Specification artifact. Detailed guidance for each step is provided in the DU Specification Guideline artifact.
    Steps
    Identify Candidate Deployment Units
    Since DUs represent the various aspects of the components involved in the application level component model, it is natural to start by a direct examination of the Component Model (CM). Therefore, make a first assessment of the need (or otherwise) for Presentation, Execution and Data DUs for each component.
    Identify DU Characteristics
    Using information documented in the System Wide Requirements artifact, determine the key non-functional characteristics for the candidate deployment units
