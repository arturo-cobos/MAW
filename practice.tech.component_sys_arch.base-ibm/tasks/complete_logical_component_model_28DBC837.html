Task: Complete Logical Component Model
Complete an initial draft of the collaborations between the logical components, define their interfaces and create detailed component specifications from which component design can be completed.
Disciplines: Technical Architecture
Purpose

The purpose of this task is to start to finalize decisions to:

  • develop a detailed Logical Component Model (by elaborating an earlier version if one exists)
  • specify logical component interfaces
  • provide the basis for release planning and work allocation

      Main Description

      Develop the component model and component specifications into sufficient detail to enable estimation, release planning, and allocation of design work for the first release. This may involve identifying lower level components and their interactions.  Confirm that the component model and component specifications can support all the use cases and business rules defined and ensure that these can be traced to the individual components and interfaces that implement them. 

      Focus on:

      • Defining component interfaces by separating out the responsibilities placed on the components.
      • For each interface specify the operation and their signatures (i.e. parameters passed and return values).
      • For each operation specify the pre- and post-conditions.
