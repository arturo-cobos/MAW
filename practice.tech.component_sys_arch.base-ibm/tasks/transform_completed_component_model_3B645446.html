Task: Transform Completed Component Model
Transform the completed Logical Component Specifications into Physical Component Specifications based on the technology chosen for their implementation.
Disciplines: Technical Architecture
Purpose

The purpose of this task is to finalize transformations to:

  • Specify the technology or technologies being used to implement logical components identified within the logical component model
  • Map the logical interfaces offered by the logical components and their operations to technical interfaces implemented by the physical components
  • Map the responsibilities identified for the logical components to physical component capabilities / functions
  • Document transformation decisions that detail the mapping between logical and physical components

      Main Description

      For each logical component defined at a detailed level:

      • If the component is to be implemented using commercial-off-the-shelf (COTS) products or packages map the components to the appropriate products or packages.
      • If the component is to be built identify the approach to implementation by defining the physical level components together with their interfaces and operation signatures.
