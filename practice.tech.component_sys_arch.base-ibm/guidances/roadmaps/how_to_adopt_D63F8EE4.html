Getting Started

Component Modeling in SOMA 

This practice boosts the existing Component System Architecture practice. One would adopt this practice when we want to introduce the discipline and rigor required to align IT subsystem with business design and business partition represented by Functional Areas and Business Domains. This practice also details how IBM SOA Reference Architecture can be leveraged for SOA and non-SOA projects/solutions and yet can make the client ready for SOA as their maturity improves. It prescribes how technical feasibility exploration can be conducted for different realization scenarios using the different layers of the architecture.
