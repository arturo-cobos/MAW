Task: Identify Common Interactions
Recognize the opportunity to create or utilize a set of templates or patterns that can provide for repeatable and consistent definition and structure throughout the solution architecture
Disciplines: Technical Architecture
Purpose

The purpose of this task is to:

  • identify common types of interaction which require architectural templates.
  • identify standard mechanisms which will increase the uniformity and maintainability of the design

      Main Description

      This task may either be done early on during requirements gathering or deferred until making global design decisions.

      This task is concerned with identifying the need for architectural templates, and provisionally deciding which pre-defined templates to use. It does not include specifying, designing or modifying architectural templates.

      Steps
      Identify characteristic issues
      These are issues which will encountered repeatedly in the design of the system (e.g. transactionality, persistence, workflow, user interface behavior)
      Identify potential architectural templates

      Decide which of these require an architectural template. Use architectural assets, literature on industry standards, and pattern libraries to identify the basis for the template.
