Task: Identify and Plan Required Proof-of-Concepts
Plan, and define set of Technical Proof-of-Concepts to exercise the architectural decisions in subsequent phases.
Purpose
To define and plan for proof-of-concept.
Relationships
RolesPrimary Performer:
  • Technical Solution Architect
Additional Performers:

  • Information Architect
  • Infrastructure Architect

    Outputs
    • Technical Proof of Concept
      Main Description
      Identify and plan for required Technical Proof of Concepts which represent an early build of some part, or all, of a proposed system design. The Technical Proof of Concept specifically addresses any new or challenging technical aspects of that proposed design. 
      Key Considerations
      The architectural decisions being tested to solve the architectural challenges may include specific technologies and/or products.
