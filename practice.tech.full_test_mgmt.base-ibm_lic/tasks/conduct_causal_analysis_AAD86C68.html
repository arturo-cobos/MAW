Task: Conduct Causal Analysis
Analyze defects in system to identify their root causes put procedures in place so that similar defects can be prevented.
Disciplines: Test Management
Purpose
To analyze defects to identify their root causes. The purpose is to identify areas where common problems occur, and to put procedures in place so that similar defects can be prevented from occurring again.
Relationships
RolesPrimary Performer:
  • Quality Analyst
Additional Performers:
InputsMandatory:
  • None
Optional:
  • Test Log
    Outputs
    • Root Cause Analysis Record
      Steps
      Examine list of variances by test procedure
      Select repetitive defects / variances
      Analyze the selected defects to determine root cause(s)
      Determine earliest point where they could have been detected
      Update procedures to prevent recurrence of similar defects
      Update lists of commonly occurring defects
