Task: Plan Operability Testing
Plan for operability testing in which the operations of the system are validated in the real or closely simulated production environment.
Disciplines: Technical Testing
Purpose
To ensure that the Operability Testing activities appropriately address the testing required to demonstrate that the system meets the operability requirements specified for it and is acceptable for production running.

    Main Description

    Define the tests required for the operational characteristics of sub-systems and the system as a whole to demonstrate that the system meets the operability requirements specified for it and is acceptable for production running. This includes verification of production installation procedures and operations procedures. Operability Testing considers such factors as performance, resource consumption, adherence to standards, etc. Operability Testing is normally performed by Operations to assess the readiness of the system for implementation in the production environment.

    Steps
    Define operability test integration approach
    Define how the different system, data and communication components will be integrated into operability testable units and ultimately into a final system/release.

    To ensure that resources and plans are in place to address the integration of:
    • Conversion
    • Full volume data
    • System components (perhaps from multiple system testing projects)
    • Specific testing facilities (e.g. model office)
    • Communication mechanisms (e.g. Networks)
    Define operability test requirements

    Define both the necessary and desired properties of the test environment. Include enough information to plan for establishing the environment but do not include a detailed specification of the environment itself - this should be provided in the test specification document, which corresponds to this plan. This specification should include:

    • Identification of the physical components, the communications, the system and middleware necessary.
    • The mode or state of usage (for example, standalone or dedicated to the discrete usage of the application alone).
    • Other software or supplies needed to support testing (e.g. specialized stationery).
    • Security and access requirements to the test area and equipment.
    • Test tools and utilities required.
    • Identify any other testing needs (publications, office space).
    • Additional software licences to cover test period usage.
    • Support resources and skills required to maintain the test environments.
    • Identify a source for any of the needs that are not currently available to the test group.
    • If any other projects are to make parallel usage of any of the above, reference it.
    Plan cutover tests and schedule

    Identify the types of cutover testing, which are to be conducted prior to installation and include these, with their associated planned schedules, as requirements in the detailed Operability Test plan. Cutover testing types to be considered are:

    • Technical rehearsal testing
    • Business cutover rehearsal
    • Final cutover test
    Document operability test plan
    • Review level test scope, scenarios, approach and infrastructure requirements
    • Develop a comprehensive task list for this level/phase
    • Allocate resources for each task
    • Schedule tasks and milestones
    • Document, or point to, common tools and processes for:
      • Change management, tracking and reporting
      • Issue or problem management, tracking and reporting
      • Defect management, tracking and reporting
      • Risk management, tracking and reporting
      • Project management, tracking and reporting
    • Review the master test plan
    • Document operability test plan
    Review operability test plan
    • Review the operability test plan
    • Walk through the test focus areas
    • Verify that the organizational responsibilities are correctly recorded
    • Ensure that the operability test environment requirements are adequate
    • Verify all sections of the operability test plan are complete and accurate
    • Obtain the sign-off from the approved signatories
