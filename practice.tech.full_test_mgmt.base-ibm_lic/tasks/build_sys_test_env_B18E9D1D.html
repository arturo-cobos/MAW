Task: Build System Test Environment
Set up the physical System test environment which mirrors, as closely as possible, the expected production environment, within the risks and constraints accepted by the project.
Disciplines: Technical Testing
Purpose
To provide the System Test environment components and infrastructure that will support the execution of the detailed System Test Plan.
Relationships
RolesPrimary Performer:
  • Test Specialist
Additional Performers:
InputsMandatory:
  • None
Optional:
  • System Test Plan
  • Test Environment Configuration
  • Test Specification
    Outputs
    • Test Environment Installed
      Main Description
      Using the detailed System Test Plan and System Test Specification, set up the physical System test environment which mirrors, as closely as possible, the expected production environment, within the risks and constraints accepted by the project.

      Identify the processes and tools required to set up, operate and maintain the infrastructure, particularly any that are unique to the System Test Environment; i.e., not part of the production environment. Document these differences from the production environment, likely impacts on testing and the risk management plan for such impacts.

      Include:
      • Hardware and software to support the System Test.
      • Network requirements for the System Test
      • Personnel staffing for establishing and maintaining the System Test Environment.
      • Tools and utilities.
      • Applications.
      • Data.
      • Security.
      • Facilities.
      Steps
      Review detailed System Test Plan
      Acquire/assemble/review Acceptance test environment components and licenses

      Components include both the new and the existing ones. 

      Develop or adopt standards, practices and processes
      Use them to build, operate and maintain the Acceptance Test Environment.
      Configure and document System Test Environment
      Test the established System Test Environment
      Document problems that prevent establishment of Acceptance Test Environment
      Also refer to issue and risk management processes.
      Sign-off and communicate environment implementation
