Task: Develop Systems Integration Test Execution Plan
System integration tests ensure that the transactions being tested can communicate with all necessary systems for successful completion. Plans need to be prepared to successfully carry out these tests.
Disciplines: Technical Testing
Purpose
  • To define how to execute the System Integration Test.
  • To establish entry/exit criteria for each System Integration Test Script and prioritize them.
  • To establish pass/fail criteria and to provide a complete plan to enable effective management of the test execution.
Relationships
RolesPrimary Performer:
  • Test Specialist
Additional Performers:
InputsMandatory:
  • None

    Outputs
    • Test Execution Plan
      Main Description

      Create a Test Execution Plan for System Integration testing based upon the Systems Integration Test Plan and Specifications taking into account the overall test environment. The plan should cover the level-specific setup to the test environment and infrastructure, the test setup and execution, and how the testing runs will be logged. Separately, it should identify all the resources needed to perform the testing activities and when they will be needed. Is should also describe how the test environment and infrastructure will be torn down after the testing is completed.

      Prepare a step-by-step description of tests to be performed. This should include backup and recovery processes for testing, data selection and analysis for setting up regression test packages (test cases and data) and results evaluation and defect management processes are clearly documented and understood.

      The System Integration Test execution plan is based upon the available resources, the planned time for performing the test case executions and the critical path through the test schedule.
      Steps
      Review the detailed system integration test plan
      Consolidate system integration test components
      Assign resources
      Schedule the tests steps
      Identify milestones and checkpoints
      Document and review controls and procedures
      Review the system integration test execution plan
      1. Review the system integration test execution plan.
      2. For each test case, verify that the tasks to be performed and resources needed to perform them are clearly documented.
        • Who will setup the tests?
        • Who will run the tests, and what are the tasks?
        • Which test cases or scenarios will run and in which order?
      4. An overall description of the system integration test environment.
      5. Ensure that the system integration test execution plan covers the major areas risk mitigation at this level.
      6. Verify that the processes for issue and change management are clearly referenced. (Note: make sure that this is only referred to in the system integration test execution plan. It is best that these are clearly defined at a project level)
      7. Finalize the system integration test execution plan.
      8. Review the system integration test plan with stakeholders
