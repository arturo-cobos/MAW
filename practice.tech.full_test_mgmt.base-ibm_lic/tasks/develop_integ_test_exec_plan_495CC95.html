Task: Develop Integration Test Execution Plan
Prepare a step-by-step description of tests to be performed.
Disciplines: Technical Testing
Purpose
  • To define how to execute the Integration Test. To establish entry/exit criteria for each Integration Test Script and prioritize them.
  • To establish pass/fail criteria and to provide a complete plan to enable effective management of the test execution.
Relationships
RolesPrimary Performer:
  • Application Developer
Additional Performers:
InputsMandatory:
  • None
Optional:
  • Integration Test Plan
  • Test Environment Configuration
  • Test Specification
    Outputs
    • Test Execution Plan
      Main Description
      Prepare a step-by-step description of tests to be performed. This should include backup and recovery processes for testing, data selection and analysis for setting up regression test packages (test cases and data) and results evaluation and defect management processes are clearly documented and understood.

      The Integration Test execution plan is based upon the available resources, the planned time for performing the test case executions and the critical path through the test schedule.
      Steps
      Review the detailed integration test plan
      Consolidate integration components
      Assign resources
      Schedule the tests steps
      Identify milestones and checkpoints
      Document and review controls and procedures
      Review the integration test execution plan
      1. For each test case, verify that the tasks to be performed and resources needed to perform them are clearly documented.
      • Who will setup the tests?
      • Who will run the tests, and what are the tasks?
      • Which test cases or scenarios will run and in which order?
      1. An overall description of the integration test environment.
      2. Ensure that the Integration test execution plan covers the major areas risk mitigation at this level.
      3. Verify that the processes for issue and change management are clearly referenced. (Note: make sure that this is only referred to in the Integration test execution plan. It is best that these are clearly defined at a project level)
      4. Finalize the integration test execution plan.
      5. Review the integration test plan with stakeholders.
