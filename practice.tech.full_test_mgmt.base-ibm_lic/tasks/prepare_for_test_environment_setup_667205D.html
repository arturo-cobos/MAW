Task: Prepare for Test Environment Setup
Examine the test environment requirements from each of the detailed test plans to determine the infrastructure build strategy.
Disciplines: Test Management
Purpose
To examine the requirements for test environments for different levels of testing, to analyze the data dependencies and to document the Test Environment for each level of testing.

        Main Description
        During this task examine the test environment requirements from each of the detailed test plans to determine the infrastructure build strategy. Rationalize how the test environments can be built and tested iteratively so as to facilitate the testing of the software build as they are delivered. The data dependencies from one level of test to the next are identified and any hardware and software incompatibilities are identified for suitable action.
        Steps
        Analyze test environment requirements for each test level

        For each level of testing included in the engagement scope:

        • Review the detailed test plan
        • Understand the requirements for test environments
        • Identify any new risks that may arise as a result of any changes to existing or new software or hardware to be used
        • Place orders for new software, hardware and tools
        • Identify any training requirements for new tools
        • Identify and document processes
        • Identify requirements for the testing of the infrastructure at each level
        • Set up the processes required to manage the test environments
        Determine data dependencies
        • Identify the applicationsâ€™ interfaces
        • Identify data owners
        • Assess any data security considerations
        • Identify the business cycles for which data is required
        • Determine data dependencies from the detailed test specification documents
        • Determine levels of data integration
          • Unit to unit
          • Function to function
          • Application to application
          • System to system (internal and external)
          • End-to-end business cycles
        • Document data dependencies
        Identify hardware, software incompatibilities

        For each level of testing included in the engagement scope:

        • Review the detailed test plan
        • Analyze the hardware and software requirements for each level of testing to see if there are any known incompatibilities
        • Run infrastructure tests at each level of software integration to help to validate that the components are compatible
        • If verification tests identify incompatibilities, advise solution development team so that alternatives can be assessed
        • Identify risks resulting from such hardware, software incompatibitlities
        • Set up the processes required to manage the test environments
        Analyze network connectivity requirements and define workloads
        • Review the requirements and understand them thoroughly
        • Define volumetrics expressed in business terms
        • Identify usage profiles with time
        • Define business scenarios and breakdown into business transactions/jobs
        • Document a realistic workload
        Document test environment
        • Analyze the information gathered. Identify environment needs not currently available, i.e., tools, DASD and any hardware and software incompatibilities identified to be actioned.
        • Define the Test Environment, address all voids, ensuring all the resources can be assembled at the proper time and place.
        • Document processes and procedures to support the Test Environment.
        • Document the data dependencies, and the workloads. Include the appropriate amount of detail for the size, complexity, and risk associated with the project.
        • Review to ensure the document is complete and accurate.
