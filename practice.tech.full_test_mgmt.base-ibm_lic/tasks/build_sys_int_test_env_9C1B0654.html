Task: Build Systems Integration Test Environment
Set up the physical Systems Integration test environment which mirrors, as closely as possible, the expected production environment, within the risks and constraints accepted by the project.
Disciplines: Technical Testing
Purpose
To provide the Systems Integration Test environment components and infrastructure that will support the execution of the detailed Systems Integration Test Plan.

    • Test Environment Installed
      Main Description
      Using the detailed Systems Integration Test Plan and Systems Integration Test Design, set up the physical Systems Integration test environment which mirrors, as closely as possible, the expected production environment, within the risks and constraints accepted by the project.

      Identify the processes and tools required to set up, operate and maintain the infrastructure, particularly any that are unique to the Systems Integration Test Environment; i.e., not part of the production environment. Document these differences from the production environment, likely impacts on testing and the risk management plan for such impacts.

      Include:
      • Hardware and software to support the Systems Integration Test.
      • Network requirements for the Systems Integration Test.
      • Personnel staffing for establishing and maintaining the Systems Integration Test Environment.
      • Tools and utilities.
      • Applications.
      • Data.
      • Security.
      • Facilities.
      Steps
      Review detailed Systems Integration Test Plan
      Acquire/assemble/review Acceptance test environment components and licenses

      Components include both the new and the existing ones. 

      Develop or adopt standards, practices and processes
      Use them to build, operate and maintain the Acceptance Test Environment.
      Configure and document Systems Integration Test Environment
      Test the established Systems Integration Test Environment
      Document problems that prevent establishment of Acceptance Test Environment
      Also refer to issue and risk management processes.
      Sign-off and communicate environment implementation
