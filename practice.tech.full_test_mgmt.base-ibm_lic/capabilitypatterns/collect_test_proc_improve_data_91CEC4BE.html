Task Descriptor: Collect Test Process Improvements Data
Identify potential improvements, incorporate those that are minor "in-flight" corrections, and feed back to the appropriate management those process improvements that should be reviewed and acted upon for the next round or level of testing.
Based on Method Task: Collect Test Process Improvements Data
Relationships
RolesPrimary:

  • None
Outputs
  • Lessons Learned
Main Description
If process improvements are implemented, they must be communicated to the appropriate personnel.
Steps
Review the testing process as the process was used
Analyze all process logs for errors that were detected
For all minor "in-flight" corrections that were implemented during the project
  • Review with appropriate team members.
  • Verify the corrective measures did help.
For corrective measures which were implemented and those that were not
  • Consolidate full and accurate information regarding the problem and the location where it was found.
  • Consolidate information on fixes that were applied and worked.
  • Collect additional information on fixes that did not work.
  • Document all the relevant information.
  • Identify the process owner corresponding to the process error.
  • Submit process error to process owner for review.
  • Set up review session with appropriate team members for process change.
