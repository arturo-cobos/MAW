Task: Assemble Release Candidate Deliverable
Bring the artifacts together representing the implementable solution from a build
Disciplines: Technical Management
Purpose
Consolidate and rationalize work products comprising the initial draft of the Release Candidate deliverable.
Relationships
RolesPrimary Performer:
  • Technical Solution Architect
Additional Performers:
  • Application Architect
  • Application Developer
  • Database Administrator
InputsMandatory:
  • None
Optional:
  • Build
  • Build Procedures
  • Data Migration Programs
  • Deployment Packaging
  • Increment Goals
  • Media Content
  • Physical Database Design
  • Release Plan
  • Source Files
  • User Support Materials
    Outputs
    • Release Candidate
      Steps
      Assemble latest revisions of required work products instances

      Note that some may require multiple instances.

      Make sure appropriate instances of each artifact are identified clearly by location, name, and version

      Coordination with configuration management procedures and tools will probably be necessary

      Review each for appropriate level of completion at this point in development
      Each artifact should be complete to the point defined in project planning documents for this build
      Perform any needed validation and verification
      Each artifact should have been tested to the level expected for inclusion in this deliverable
      Compare work products to ensure they are cohesive
      The versions of the various artifacts should be the same as the intended release build
      Assemble the Release Candidate and circulate to stakeholders for review
      The deliverable should be input to the appropriate project procedure for deliverable handling
