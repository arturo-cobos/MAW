Task: Assemble User Experience Model Deliverable
Bring the artifacts together representing the UI design intent to the level appropriate for this point in the development lifecycle
Disciplines: Technical Management
Purpose

Consolidate and rationalize work products comprising the User Experience Model deliverable. This deliverable may represent various levels of completion of the User Experience model depending where in the development lifecycle the project is currently positioned.

Relationships
RolesPrimary Performer:
  • User Experience Professional
Additional Performers:
InputsMandatory:
  • None
Optional:
  • Interactive Concept
  • User Interface Design Specifications
  • User Interface Guidelines
    Outputs
    • User Experience Model
      Steps
      Assemble latest revisions of required work products instances

      Note that some may require multiple instances.

      Make sure appropriate instances of each artifact are identified clearly by location, name, and version

      Coordination with configuration management procedures and tools will probably be necessary

      Review each for appropriate level of completion at this point in development
      Each artifact should be complete to the point defined in project planning documents for this phase
      Perform any needed validation and verification
      Each artifact should have been reviewed and tested to the level expected for inclusion in this deliverable
      Compare work products to ensure they are cohesive
      The versions of the various artifacts should be the same as the intended overall design
      Assemble draft User Experience Model deliverable
      Assemble draft User Experience Model deliverable and circulate to stakeholders for review. The deliverable should be input to the appropriate project procedure for deliverable handling
