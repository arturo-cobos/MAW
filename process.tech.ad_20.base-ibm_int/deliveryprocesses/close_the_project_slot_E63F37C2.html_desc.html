Phase: Close the Project
End of project activities are contained here that address the orderly shut down of the engagement and releasing of the staff.
Extends: Closing
DescriptionWork Breakdown StructureTeam AllocationWork Product Usage
Purpose
The purpose of this phase is to perform the activities and tasks required to formally close the project down.
Relationships
Parent Activities
  • Application Development 2.0
Description
The Close phase focuses on the activities and tasks needed to ensure that the project is successfully closed. This includes confirming customer approval, harvesting project assets, releasing staff, and preparing an evaluation report to document lessons learned.

The following activity is included in the Close phase:
  • Capture Project Results - Ensure that assets are captured on a consistent basis, and the project is terminated in a orderly process.
  • Closing The Project (or Phase) - The releasing of staff and subcontractors and managing the end of the project.

Key Considerations
When organizing staff release, the project management goal is to minimize project costs without losing team-member expertise that could still be needed on the project.
