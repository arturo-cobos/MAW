Activity: Develop Solution Software Components
This activity of the development cycle is where software components are written or modified as required to achieve the goals of the current iteration. These components can be of a variety of types and purposes, and responsibility for their creation may be divided across multiple teams.
DescriptionWork Breakdown StructureTeam AllocationWork Product Usage
Purpose
To create software components meeting the design specifications of the current release of the solution.
Relationships
Parent Activities
  • Perform Development Cycle
Description

In this activity the various components of the solution are actually written (at least those that have not been acquired through purchase of some COTS). These components may represent a variety of purposes depending on the nature of the pattern(s) used in structuring the solution, in terms of responsibilities of components and their associated operations. Over the course of all Development Cycles, all functional requirements in scope for the current release must be able to be realized against the integrated solution built from these components.

Tasks in this activity exist for a variety of technology types that may exist in the solution, only those relevant to the current release would be performed during any particular iteration.
