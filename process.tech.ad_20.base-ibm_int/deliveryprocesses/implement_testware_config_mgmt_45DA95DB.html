Task Descriptor: Implement Testware Configuration Management
Perform the identification, planning, and management of all test domain work products that may undergo changes by members of the project or by those external to it.

Steps
Define testware inventory
Defining Testware Inventory is a part of defining a project level application inventory, and is normally performed very early into project, before any workproducts are created. A Testware Inventory is a list of the applications and other related information collected in a record to support configuration management activities during testing.
Authorize baseline
All changes to items identified as controlled by a testware configuration must be approved by the Software Configuration Control Board (SCCB) for each baseline that contains the CI. This approval will be noted in the software configuration management status of its baseline entry in the application inventory. The signed document will be retained in the SCM librarian's project file.

The SCCB is established with the authority for managing the project’s baselines. This includes identification of configuration items/units. The SCCB represents the interests of the project manager and all groups that may be affected by changes to the baseline. Reviews and authorization of changes to the baseline as well as the creation of products from the baseline library are conducted by the SCCB.

Create testware configuration management
Using the software configuration management standard, the test manager/leader or delegate shall prepare an TCM documentation (part of test environment) to detail how the testing will implement, execute, and maintain its TCM activities and data. The TCM documentation is developed in the early stages of test planning. The specification must be tailored to the project’s operational environment and development methodology. This document is approved and is used as the basis for performing TCM activities.

This document should include activities, assigned responsibilities, and resources required to support the TCM. Any requirements and activities to be performed by other software development or related groups, such as developers, should be identified in the plan.

A subcontractor will use TCM. The tasks for monitoring the subcontractor’s TCM activities should be included in the TCM documentation. The subcontractor’s plans, resources, procedures and standards for TCM should be reviewed to ensure they are adequate to support the project. When a subcontractor is used, TCM activities between the client and the subcontractor should be coordinated to ensure that the subcontractor’s products can be readily incorporated into the project environment.
Approve testware configuration management
The testware configuration management (part of Test Environment) should exist as a standalone document or as a distinctive part of a project level software configuration management plan. The initial TCM document, and all changes, must at a minimum have as approvers all the project and test managers.
Perform initial baseline audit
  • Prepare adequately for the audit
  • Review TCM
  • Access test domain library baseline for integrity
  • Review structure and facilities of configuration management library control system
  • Compare library contents to TCM controlled configuration items
  • Verify completeness and correctness of the baseline library contents
  • Ensure compliance with applicable SCM standards and procedures
  • Documents and tracks to closure any action items from the audit
