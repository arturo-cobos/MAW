Task Descriptor: Develop Business Technical Event Map
Map business processes onto the technical components that support the execution of those processes and build a profile for how services (database calls, message flows etc) are used.

Steps
Identify Technical Components

In order to produce a business-technical event map for a given business process or transaction it is first necessary to identify the technical components (i.e. nodes and connections) involved in supporting the execution of that process or transaction, either background or interactive.  A number of approaches can be taken in determining this information:

  • Review the existing architectural documentation e.g. Logical Component Model, Logical Operational Model, Physical Database Design.
  • Conduct specific design walkthroughs with the architects and lead component designers.
  • Analyze trace information from tests conducted using prototype code built during the early design stages.
  • Analyze trace information extracted from existing systems having a similar profile. 
       
Analyze Function Flow

Analyze the flow of function and data that takes place between the technical components and build a list of the transactions that occur between each component.  Establish a profile of the use of services (e.g. SQL, message handling etc) for both transactions and batch programs.  For key transactions, this will need to be at the detail level of specific calls, for less significant transactions, then only a count of the number of calls (optionally by call-type) is needed.


Assign relative complexities (simple, medium, complex, etc) and message profiles (i.e. number of message pairs, and message length) to each transaction and message flow.  The complexity is typically based on an assessment of the SQL which will be required.
Document and Review Findings
Once all necessary information has been obtained it should be documented and reviewed by peers or product specialists.
