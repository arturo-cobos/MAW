Task Descriptor: Assemble Support Readiness Report Deliverable
Produce the Support Readiness deliverable to show the solution is ready for operational use.
Based on Method Task: Assemble Support Readiness Report Deliverable

Steps
Assemble latest revisions of required work products instances

Note that some may require multiple instances.

Make sure appropriate instances of each artifact are identified clearly by location, name, and version

Coordination with configuration management procedures and tools will probably be necessary

Review each for appropriate level of completion at this point in development
Each artifact should be complete to the point defined in project planning documents for this phase
Perform any needed validation and verification
Each artifact should have been reviewed and tested to the level expected for inclusion in this deliverable. This may include static and/or dynamic testing as appropriate to determine the quality level of the artifact(s)
Compare work products to ensure they are cohesive
The versions of the various artifacts should be the same as the intended overall design level
Assemble the Support Readiness Report and circulate to stakeholders for review
The deliverable should be input to the appropriate project procedure for deliverable handling
