Iteration: Perform Development Cycle
Iteratively create code for this release while continually measuring progress towards the build objectives. This is also where the goals for the individual development cycle iterations are refined and achievements recorded. Performance of the Development Cycle is split across multiple development teams and executed iteratively until the cycle objectives have been reached.

Purpose

Design, build and test the various forms of source code in a series of development cycles. A development cycle is typically a 2-3 week iteration involving a small team of 1-4 developers. It contributes to the goals of the Build Cycle, which in turn meets some of the goals of the release. The source code developed is normally part of the final application.

Description

This activity is repeated. The instances may run serially or in parallel or any combination of the two. A typical organization is for a number of development teams to work in parallel during a Build Cycle, each team performing a number of development cycles in sequence. Across development teams these sequences are typically not coordinated, but rather proceed asynchronously until each team has achieved their goals and can contribute to an integrated build.

For each development cycle:

  • Refine the goals of the development cycle, making them specific enough to be performed and evaluated.
  • Build and unit test the code for this cycle, while (possibly concurrently):
    • Developing or modifying the UI as required to support new or changed functional characteristics of the solution,
    • Creating programs to perform data extract, transform, and load of the solution's database as required to support this release,
    • Updating the design models to show new or modified implementation artifacts.
  • Reviewing the results of this development cycle and adjust or refine the goals for the next cycle.
