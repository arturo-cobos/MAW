Activity: Perform Business and System Requirements Reviews
Perform both the Business and System Requirements Reviews to establish the first two System Engineering standard checkpoints.
DescriptionWork Breakdown StructureTeam AllocationWork Product Usage
Purpose

The customer baseline must be established at the beginning of the project. This baseline will provide the project team with a set of business requirements, which can be used to drive the project cost, schedule and technical decisions.

The system baseline describes requirements to be provided by the entire system, and the supporting documentation. The system baseline should be created from and traceable to the customer baseline. The system baseline will provide the project team with a set of system requirements, which can be used to drive the entire project cost, schedule and technical objectives.

Description

The customer baseline is the first of six systems engineering, technical baselines. The customer baseline describes the business requirements. Business requirements describe the business' need in a tangible and measurable manner. It is a statement of what the system must do in order to successfully achieve the objective (mission) from a business point of view. It is not how the system will be implemented to perform this task.

The system baseline is the second of six systems engineering, technical baselines. The system baseline consists of the system requirements, the system architecture, system requirements traceability and verification, as well as technical decisions made to create the baseline. When defining the system requirements, the system is viewed as a ‘black box’; thus, the requirements define what the system will do and not what the individual components of the system will do.

The six systems engineering, technical baselines are:

  1. The customer baseline (which is defined at the Business Requirements Review),
  2. The system baseline (which is defined at the System Requirements Review),
  3. The component baseline (which is defined at the Preliminary Design Review),
  4. The design baseline (which is defined at the Critical Design Review),
  5. The test baseline (which is defined at the Test Readiness Review), and
  6. The production baseline (which is defined at the Production Readiness Review).
