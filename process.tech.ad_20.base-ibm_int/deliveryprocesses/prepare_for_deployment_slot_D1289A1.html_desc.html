Activity: Prepare for Deployment
Create detailed plans for deployment of the release, and perform the configuration management procedures neccessary for establishing production status of the release components.
Extends: Prepare for Deployment
DescriptionWork Breakdown StructureTeam AllocationWork Product Usage
Purpose
The purpose of this activity is to:
  • Complete the systems management plan to provide the basis for selection:
    • Of system management tools.
    • Of systems management processes.
  • And definition of job descriptions, based on identified roles, to support the IT system.
  • Promotion of the new release to production status in the configuration management system.
Relationships
Parent Activities
  • Build
Description

Revise the deployment plan based on any new information that is available. Use the chosen form of review process (such as a walk-through or formal inspection) to review the deployment plan to ensure that it contains the appropriate schedules, plans, procedures to deploy the system. It should also identify the responsibilities of each participating person, organization, subcontractor, and vendor.

Starting from the outlined systems management plan developed earlier, develop the plan into a detailed framework adequate to ensure the system management-related, nonfunctional requirements can be met.

Perform the previously defined configuration management procedures as neccessary to identify the various components that must be promoted to production status as a result of deploying this release, as well as the appropriate dispostion of current production elements that will be superseded or retired as a result of this production promotion.

To develop the deployment packaging document, the architect should enumerate all the deliverables required to execute and use the system. This includes components comprised by the executable work products, installation instructions, user help materials, warranty registration cards, and their actual containers.

The turnover process, from the development team to the maintenance group, involves a series of validations that the requirements of the maintenance group have provided, prior to acceptance. Use the original initiating document for the project, along with the list of work products selected for development in this project, to identify and locate all work products which have been created or extended in this effort. These are the work products which must comprise the new turnover materials.
