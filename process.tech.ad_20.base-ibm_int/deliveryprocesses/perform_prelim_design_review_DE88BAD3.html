Task Descriptor: Perform the Preliminary Design Review
This task describes how to conduct the preliminary design review meeting.
Based on Method Task: Perform the Preliminary Design Review
Relationships
RolesPrimary:
  • Application Architect
Additional: Assisting:
InputsMandatory:
  • None
Optional:
  • Architecture Model
  • Data Model
  • Test Strategy
External:
  • None
Outputs
  • Preliminary Design Review Results
Steps
Establish the architecture and component baselines.
Review system and component architectures.
Verify system and component architectures against allocated system requirements.
Review test architecture (software and hardware).
Review and approve the allocated component requirements and business scenarios.
Establish traceability from system requirements to component requirements.
Identify/review technical risks and mitigation plans.
Identify/review technical dependencies.
