Description
The focus of this task is to gather information about the existing processes and technical infrastructure supporting the solution. This is accomplished with a series of interviews, document review, and a hands-on inspection of the environment if necessary.

The project team must be looking for things that will impact the security view of the design. This would include any or all of the following:
  • User registration
  • Use of IDs and/or passwords
  • Access Controls to any portion of the design
  • Authentication mechanisms for users, processes or machines
  • Audit records generation and/or storage
  • Use of Encryption
  • Any flow control device intended to limit access to portions of the environment
  • Personnel controls (two-person policies, physical controls for machine rooms, etc)
  • Data Classifications to include data aggregation requirements
  • Information Assets and their value and owner
  • Actor profiles to include those responsible for supporting and provisioning
This is not an exhaustive list. See the various examples in the Security & Privacy ICAP section applicable to the specific solution activity being supported.

Key Considerations

On any engagement, practitioners must know and understand the atmosphere in which the client operates. From a security perspective, understanding what forces the client faces from the industry, competitors, and consumers can provide practitioners with requirements and priorities for security. For example, if a client is in the pharmaceutical industry, security practitioners should know that the client faces such challenges as security and integrity of their drug submissions, security concerns for possible test subjects and the data around those subjects, as well as the normal operational security challenges any company faces. When further knowledge of the client operations are known, security efforts can be targeted and optimized.
