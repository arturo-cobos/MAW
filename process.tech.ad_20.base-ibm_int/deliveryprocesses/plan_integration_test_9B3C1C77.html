Task Descriptor: Plan Integration Testing
Plan for integration testing in which the proper execution of application components are tested and does not require that the application under test interface with other applications.

Steps
Define unit interfaces
Using the physical design specification and program specifications, define the approach to be taken for integrating the program (modules):
  • Top-Down: the top level driver modules are tested first using stubs to imitate their subordinate modules which are not yet available.
  • Bottom-Up: individual modules are tested first and incrementally linked together to build larger and larger pieces as and when the modules become available. The modules should be called by drivers written to imitate their calling modules.
  • Critical-First: start with the system critical modules first and then integrate the rest of the modules around them.
  • Functional Groups: assemble modules together which implement a cohesive set of functions, perhaps acting on a defined set of data objects. This approach facilitates early user involvement and feedback and users are able to ‘use’ the skeleton units in this form.
  • As Available: take the modules that become available and link them together as much as possible.
  • Combination: a mixture of the above approaches may be taken. For example, bottom-up testing of input and data conversion modules may be done first to facilitate data capture to assist in testing the higher level modules top-down.
  • Big Bang: link all of the modules together at once and then test the integration.
Review build strategy
  • Review the business users' high priority requirements
  • Incorporate those high priority requirements into "early builds"
  • Determine how unit tested modules "fit" into the overall Integration test plan 
Plan handover test content and schedule
  • Identify which functions are to be tested as part of handover test
  • Decide what will be the minimum set of functions to be successfully processed
  • Prepare and agree a schedule for executing the handover testing
Document integration test plan
  • Review the master test plan
  • Prepare Integration test plan using the steps outlined in the preceding steps
  • Review the integration test plan with software quality assurance
  • Review and sign-off Integration test plan with defined approval List
Review integration test plan
  • Review the Integration test plan
  • Walk through the test focus areas
  • Verify the high priority requirements are included in the "early builds"
  • Verify that the organizational responsibilities are correctly recorded
  • Ensure that business and structural functions to be tested are within scope
  • Verify that the build strategy has been agreed to
  • Ensure that the integration test environment requirements are adequate
  • Verify all sections of the Integration test plan are complete and accurate
  • Obtain the sign off from the approved signatories
