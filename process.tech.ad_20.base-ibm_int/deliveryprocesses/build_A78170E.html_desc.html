Iteration: Build
Construct and test the programs, databases, and various user support and training materials needed to implement the current release of the solution.

Purpose
The purpose of this phase is to incrementally develop and test part of the system until the objectives of the build cycle are achieved.

Description

The key aspect to understand with regard to this phase is the incremental approach to development. The Release Plan work product outlines, as early as Solution Outline, the number of releases. In Micro Design, the Increment Goals work product breaks the release down into several build cycles, with each build cycle responsible for achieving specific objectives. Further, each build cycle will contain several development cycles - each development cycle again responsible for achieving specific objectives. In this way, the development effort iterates and with the completion of each cycle, more and more of the system is developed.

In particular, this phase focuses on the activities and tasks needed to develop support materials, plan and perform system and integration testing, program the solution through one or more development cycles, and plan for the future Build cycles in this release. A number of reviews are performed in this phase, including reviews of source code, test plans, user support materials, and deployment procedures. The physical packaging and maintenance turnover materials are also developed.

This phase contains the following major activities:

  • Perform Development Cycle: This activity represents the "inner loop" of the incremental development process and occurs many times for each build cycle. The source code is developed and unit tested and the physical design work products are updated to reflect any refinements developed to the design as a result of the actual programming. In addition, the development cycle is reviewed and the Increment Goals work product may be adjusted/refined for the remaining development cycles in the current build cycle.
  • Prepare for Testing: Define the execution sequence and contents of tests, establish pass-fail criteria for individual tests, and establish entry/exit criteria for each test.
  • Develop User Support and Training: This activity develops the End User Training Materials, User Support Materials and Deployment Procedure work products. This activity is performed once per build cycle.
  • Integrate Technical Solution: This activity represents the "inner loop" of the incremental development process and occurs many times for each build cycle. The source code is developed and unit tested and the physical design work products are updated to reflect any refinements developed to the design as a result of the actual programming. In addition, the programming cycle is reviewed and the Increment Goals work product may be adjusted/refined for the remaining programming cycles in the current build cycle.
  • Perform the Test Readiness Review: The Test Readiness review creates the test baseline describing how the various package implementation components will tested to verify that they meet their requirements. The test baseline should be created from and traceable to the design baseline.
  • Test Integrated Technical Solution: In this activity the independent test team performs system testing. System testing verifies the proper execution of the entire application components including interfaces with other applications. Both functional and structural types of tests are performed to verify that the system is functionally and operationally sound.
  • Prepare for Deployment: This activity is performed once after the completion of all the build cycles, i.e., once for the release. It prepares for the deployment of the release by refining the deployment plans, first developed in Macro Design, based on the experience of the build cycles.
