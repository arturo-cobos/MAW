Purpose
The purpose of this phase is to prepare for the build of a specific release of the system by driving the design to a release specific and implementation specific platform view.

Parent Activities
  • Application Development 2.0
Description

The key new element introduced in this phase is the binding of the application models to specific implementation platforms (creation of the physical application model). The application model represents, broadly speaking, an unconstrained (or implementation platform independent) design of the application (sometimes referred to as the logical model). The classes and methods in the application models are designed without regard to the implementation platform with which they will be implemented. This unconstrained view in these models reduces the complexity of the design process. In this phase, these models are completed for the release and then mapped to a specific implementation platform. This binding process produces the specific classes and methods needed to implement the application on the selected implementation platform.

This phase contains the following major activities:

  • Develop Detail Component Design
  • Refine Data Design:
  • Develop Test Plans
  • Develop Test Specifications
  • Design User Support and Training: This activity develops detailed training and user support specifications for the release.
  • Detail Solution Development Plans : This activity establishes development guidelines and procedures and defines goals for each development cycle in preparation for the next phase's build efforts.
