Task Descriptor: Develop Operability Test Execution Plan
Prepare a step-by-step description of tests to be performed.
Based on Method Task: Develop Operability Test Execution Plan

Steps
Review the detailed operability test plan
Consolidate operability test components
Assign resources
Schedule the tests steps
Identify milestones and checkpoints
Document and review controls and procedures
Review the operability test execution plan
  1. Review the operability test execution plan.
  2. For each test case, verify that the tasks to be performed and resources needed to perform them are clearly documented.
    • Who will setup the tests?
    • Who will run the tests, and what are the tasks?
    • Which test cases or scenarios will run and in which order?
  4. An overall description of the operability test environment.
  5. Ensure that the operability test execution plan covers the major areas risk mitigation at this level.
  6. Verify that the processes for issue and change management are clearly referenced. (Note: make sure that this is only referred to in the operability test execution plan. It is best that these are clearly defined at a project level)
  7. Finalize the operability test execution plan.
  8. Review the operability test plan with stakeholders
