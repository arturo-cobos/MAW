Phase: Start the Project
Expand the project plans to an operational level and get project execution underway.
Extends: Starting
DescriptionWork Breakdown StructureTeam AllocationWork Product Usage
Purpose

To successfully begin an iteration, phase, sub-project or project by ensuring that the various plans are at a sufficient level of detail to control the execution of the project and the necessary resources are in place.

Relationships
Parent Activities
  • Application Development 2.0
Description

On the basis of the sponsor Agreement and the project plans that have been outlined for the agreement and any amendments, expand any detailed project management plans that are needed to monitor project performance, obtain project resources, and implement the management system.

Establish the technical environment and supporting tools necessary to produce the project deliverables, and get the project teams and subcontractors underway.

Properties
Event Driven
Multiple Occurrences
Ongoing
Optional
Planned
Repeatable
Usage
Usage GuidanceSome of the tasks in this activity may be repeated as necessary when new iterations, phases, or sub-projects start or new members join the project.
Key Considerations
Note that, despite the desire to get the project productive as fast as possible, the correct execution of these tasks are paramount for project success. It may even be the case that these activities reveal a situation that had not been perceived when shaping the project, or which has changed in the meantime, and increases risk and any newly identified or changed risks must be reported.
