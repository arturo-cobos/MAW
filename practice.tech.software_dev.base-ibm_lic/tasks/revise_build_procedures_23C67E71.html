Task: Revise Build Procedures
Update the procedures for the creation and maintenance of project code and executables as required from any changes resulting from latest build
Disciplines: Technical Implementation
Purpose
Definition of the procedures for build, change control, and configuration management ensures that code product gets built in a consistent, understood, and repeatable manner. These procedures must be maintained as the builds progress to include additional components as they are completed by their development teams.

      Main Description

      Maintaining the build procedures in a timely manner ensures that code product gets built in a consistent, understood, and repeatable manner. This may include identification of such items as what files are required, what links must be in place, what libraries are accessed, the sequence of steps required to generate the system, any required passwords and so on. Details of the procedures are heavily dependent on the operating system and tools being used.

      As the Build Cycles progress, additional components need to be added to the build procedures as they are deemed ready by their respective development teams. Throughout the Build Cycle, integration of the solution typically proceeds from individual components, to subsystem builds, and finally to builds of the complete release candidate for the solution. Build procedures must evolve as these additional components are added into the build objectives.
