Walkthrough of the source code

Purpose
Conduct walkthrough of the source using the static testing technique that was indicated in the Static Test Plan

      Main Description
      Review the Source Code for against the following checklist:
      • Does the design map to the detailed design material?
      • Conform to standards:
        • Adhere to coding standards (initiation, mainline and subroutine, use of common routines, message handling) 
        • Adhere to software development basic program structure
      • Naming conventions followed
      • Program statement labels
        • Internal work fields
        • Arrays
        • Sub-routines
        • Functions
      • Comments clearly and accurately describe the function, or logic?
      • Satisfy good performance and storage considerations through using program design and coding techniques/standards and do not impair performance in any capacity
      • Has the code successfully complied?
      • Are all increment counters properly initialized?
      • Is the value of a data item used before the item is initialized?
      • Are all data areas shown referenced in the code?
      • Does the length of compared fields match?
      • Are field names defined correctly?
      • Are comments accurate and meaningful?
      • Does each internal procedure document the function it accomplishes, for the part of the overall logic it represents?
      • Do names relate to usage?
      • Are correct attributes used?
