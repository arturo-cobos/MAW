      Main Description
      A programmer highly experienced in the implementation languages should be responsible for selecting and tailoring the coding guidelines document. Depending on the language, the document can address a broad range of topics ranging from items that one would expect in UI guidelines for "visual" languages, to the more traditional naming conventions, method-signature constructs, metrics, etc., in languages such as C++ or Java. The coding guidelines should also include project policy on code quality assurance ranging from informal peer walk-throughs to formal inspections. Many such guidelines already exist; the document author is encouraged to use an existing standard and modify it to suit the project needs and staff skills currently available.
