Main Description
The Database Transaction Descriptions work product groups database operations into logically related groups, i.e., transactions, such that the successful application of a transaction does not result in a database state that violates any known data integrity or business rules.  Thus, a transaction is an atomic unit of work that causes the database to advance from one consistent state to another consistent state. Transactions are implemented as a collection of database accesses. Either all of the database accesses should complete successfully or none of them should be applied to the database. Should an unrecoverable error occur during the execution of any one of the accesses in a transaction, the database must be rolled back to its original state. The mutual dependency of these accesses, necessitating their grouping, can arise as a result of the business rules and requirements or the design of the physical database.
