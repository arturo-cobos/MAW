Getting Started for SOMA

We typically develop hybrid solutions for our clients today and these solutions are composed of part custom development, part legacy integration, and part package application integration. Therefore, the implementation of a solution in the new software development paradigm for building solutions and application based on service oriented architecture and principles and best practices of service orientation must not only include programming and code generation related business-as-usual activities but also include assembly and integration related activities. Moreover, it must also includes activities to build services and related artifacts for service development.

One would adopt this practices when exposed services are defined using web services standard and are realized by service components. Usage of  appropriate tools is very critical for model driven generation of XML schemas, Web Service Description Language (WSDL), and Business Process Execution Language (BPEL).
