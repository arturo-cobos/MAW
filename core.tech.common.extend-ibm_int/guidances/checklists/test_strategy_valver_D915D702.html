Checklist: Test Strategy Validation and Verification
Relationships
Related Elements
  • Test Strategy
Main Description

If the Test strategy is created as part of the Statement of Work, then the strategy should be reviewed as part of the Solution Assurance Review process.  Create a checklist out of the Notation section and verify the contents of the test strategy, using this as a baseline.  For example, if the Types of Testing section is missing, then the test strategy is not complete.

If the strategy is developed subsequent to the creation of Statement of Work, then it should be reviewed with the client team as one of the first work products, preferably with the solution architect and the client test manager, if one has been named, participating.  

When reviewing the Test Strategy with the client, it can be verified by asking him if the right set of testing types, etc. has been captured.  The levels of testing can also be verified against the organization standard, if there is one.  In general, after reviewing the test strategy (in generic terms), if all parties agree that following it will mitigate project risks and manage testing risks, then a good outline has been created.

If the quality plan is already written and is available, then the strategy can be tested against quality criteria.  If it can be demonstrated that by implementing the strategy, the quality goals can be achieved, then the quality criteria have been met.

In any case, it is a good practice to get the development team leaders also involved in the review process so that they are aware of the direction of testing and the rigor required, and can estimate their work effort consistently with the level of quality criteria.
