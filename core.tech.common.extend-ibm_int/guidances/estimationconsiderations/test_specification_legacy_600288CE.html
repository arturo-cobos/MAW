Main Description

Test Specifications account for the number of test cases necessary to adequately test an application. 

Techniques for estimating these costs are broken down into groups:

  • Estimates for structured code based upon the traditional function point method.
  • Object or structured based estimates that use the Lifecycle Effort and Duration (LEAD) technique.
 
Function Points
 
In Estimating Software Costs by T Capers Jones (McGraw-Hill, 1998, ISBN 0-07-913094-1), the rule of thumb is; Number of Test Cases = Function Point Count1.2.
 
LEAD Estimating Technique:
  • Calculate Logical Transactions (LTs) * 50.
  • Calculate Entities * 75.
  • Calculate total entity references (ER) as the ratio of (ER/LT) *LT, then multiply by 20.
  • Add the results from 1, 2, and 3, and divide the result by 1500.
  • Raise this to power 1.2
  • For OT, this is the number of test cases. For structured, add 15%.
As a rule of thumb, experience on client engagements suggests that one test case constructor can create 2-3 test cases per day. If automated testing tools are used to develop the test scripts, additional time per test case may be required depending on the skills of the test case constructor and the difficulty in debugging any scripting problems.
