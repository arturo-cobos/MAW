Estimation Considerations: Test Environment (Large Project)
Relationships
Related Elements

Main Description
Estimates will increase with size, technology, and complexity of the project, number of people and organizational boundaries crossed, and skill and familiarity with the project process, subject matter or business, and technology.  Effort involves the time to identify hardware/software requirements, site support facilities, test tool requirements, and support processes/tools/resources requirements. If there is an existing test environment that is used, or the infrastructure with processes and tools to support a test environment exists already, the following estimates will need to be adjusted. The technology and skill level of the people setting up the test environment will greatly impact the estimates.
 
The following table provides estimates for a 50,000-hour project. For projects of smaller size, suitable adjustments to estimates are required. Also, note that any effort spent identifying requirements during master and detailed test planning activities should be taken into consideration so that estimates are not double counted.
 
Table 1: Project Estimation
 
Test Activity Test Level Time estimate
Plan for Test Environment Set-up Unit 1 day per module
Integration 1 day
System 1 - 2 days
Acceptance 1 - 2 days
Systems Integration 2 - 4 days
Operability 2 - 4 days
Build Test Environment Unit 5 - 7 days
Integration 2 - 4 weeks
System 2 - 4 weeks
