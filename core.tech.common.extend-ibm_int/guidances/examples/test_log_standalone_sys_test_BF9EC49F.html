_________________________________________________________________________________________________
(1) The possible values for this field are unit, integration, standalone system, networked system, UAT, implementation and operability, etc.
(2) The number of the corresponding test case
(3) The number of the corresponding regression test case
(4) The name of the application being tested
(5) The type of test. It is taken from the test case.
(6) The severity level of the incident. See the master test plan for a definition of each of the severities.
(7) This field indicates where the error was introduced into the development cycle. Some of the possible values for this field are:
       - Code
       - Design and analysis
       - Documentation
       - Ease of use
       - Environment
       - New requirements
       - Process engineering
       - Scope of work
       - Testing
       - User requirement
(8) The version of build of the code where the incident was found
(9) The date assigned by the developers that this incident will be ready to be re-tested.
(10) The values for this field are accepted and rejected. This indicates whether the incident was found to have been fixed, or not, upon re-testing. If rejected, the incident report would be placed back into open status and reassigned to the developer. If accepted, the incident report would be placed into closed status and signed off by the tester.
(11) Some values for this field are:
       - Open
       - Closed
       - Ready for re-test
       - The incident has been fixed but has not yet been re-tested.
       - Pending
       - The incident cannot be completed at this time due to the circumstances outlined in the comments section
       - Deferred
       - The incident will not be run until another level, as indicated in the comments section
       - Cancelled
       - The incident will not be fixed at all as, indicated in the comments section

Note: The example described herein is a good sample of a Test Log. However, it does not necessarily follow the notation described in the artifact. It is included here only to give the reader a sense of what the contents should look like and is not intended to be an endorsement of the format that is used in the sample. The notation should be used as the recommended format where there are significant differences between the example and the formal notation.
