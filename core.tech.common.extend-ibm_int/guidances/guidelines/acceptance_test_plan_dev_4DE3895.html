Main Description

The Acceptance Test Plan is built in close communication with the accepting sponsor. Using early definitions of quality objectives and requirements, the acceptance team specifies the acceptance criteria and tests that illustrate that the criteria have been met. The Acceptance Test Plan is constructed as soon as the functional and non-functional requirements are identified, the scope is defined and the development plan is built. It should be the first detailed test plan to be developed. The Acceptance Test Plan is the measure against which all development and testing effort is gauged by the project sponsor.
   
A recommended approach is the use of facilitated workshop sessions with stakeholder representatives building and reviewing this plan to ensure that issues are raised early, consensus is developed, and ambiguities resolved. Facilitators lead the team through a structured agenda to define the plan components.   

If the acceptance criteria are defined early in the project, they can immediately be used to measure the likelihood of passing acceptance. They can be used for early deviance detection, while it is still relatively easy to correct or redirect, and there is still time to take corrective action. The sooner this information is available, the sooner it may be incorporated in other levels of testing, as appropriate.
