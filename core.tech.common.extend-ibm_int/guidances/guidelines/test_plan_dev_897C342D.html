Main Description

The development of detailed test plans should commence immediately after the required testing levels have been defined in the project lifecycle. These activities proceed in parallel with the development of the Project Plan and include the testers, the team leader, and the test manager. This approach ensures that development and testing activities are synchronized and that activities requiring long lead times, such as test environment setup, can be completed without impacting the project schedule.

Each level of testing uses the elements defined in this work product that are relevant to the testing which must be completed; i.e., a Unit Test Plan will use a different subset of elements than an Acceptance Test Plan.

The quality criteria and quality contributors are used to define the test focus areas and types of testing for a given test level. The test objectives are driven by the test focus areas and types of testing. Once the types of testing are selected, the resources and time required to run the tests are secured, the dependencies are identified and the test plan is documented. The testing lifecycle follows the development cycle, but in the reverse order. In other words what gets developed first, gets tested last. It is recommended that detailed test plans be monitored on a regular basis and at the conclusion of each testing level, to determine if adjustments are required to reflect actual project activity.
