Main Description

The production of the work product starts with an in depth review of the IT Service Design work product which is a main input. Everybody on the team needs to understand the charts and documentation to be able to proceed with the detailed design.
The main approach for this work product is to elaborate upon a sub-process from the high level design and develop the detail with regard to:

  • Sequence of the tasks within one sub-process.
  • Specific tools for each task.
  • Specific interface used for each task.
  • Specific job for each task.
  • Data and data sources needed for each task.


 
High Level Design Detailed Design

The next step after the high-level design review is to determine the detailed tasks required to perform a service with the specific tools selected and with the specific jobs defined.
When identifying the specific tool for each task a final tool selection might be necessary. Sometimes tools provide similar functions and a decision has to be made which tool to use. Criteria for this decision are:
  • Functionality included in the base tool.
  • Time requirements for customization.
  • Specialized skill requirements for customization.
For developing the workflow charts for the detailed design of the solution use the selected workflow method for documentation.
The following are sequential steps that can achieve the detailed workflow definition, but definition will likely be an iterative process.
  • Using a whiteboard, develop a workflow of the sub-process at the task level.
  • Document the workflow diagrams and description.
  • The workflow team validates the workflow diagrams and descriptions. It is critical to have full attendance at this review.
  • Optionally develop time estimates for completion of each task.
  • Gain concurrence on the activity definition from the rest of the engagement team.
