Guideline: Developing the Test Environment Configuration
Migrated development approach from existing IBM Global Services Method work product
Relationships
Related Elements
  • Test Environment Configuration
Main Description

Business and IT Planning

Identify requirements for a test environment at project start-up time. Understand the organizational context. This step is particularly important for an established test facility, but is a useful starting point for building any test environment. A detailed list of tasks is in the supporting files.

Systems Management

Perform test environment capacity planning activities. Ensure that the process is in place to keep the test environment synchronized with the other development, test, and/or production environments as required (e.g., DASD, servers, version/releases of system software). Ensure that processes are in place for change management, problem management, and software configuration management.

Test Environment Management

Understand which new and/or existing test environment policies affect this project. Review test environment staff skills. Secure new test environment skills as needed to support project. Plan, schedule, and manage test environment staff activities.

Plan for introducing hardware, software and network components. Schedule and prioritize testing in the test environment. Maintain and publish test environment testing schedule. Assemble all related test environment configuration documentation and provide to Test Environment Coordinator.

Work with Test Plan Coordinator when plan is invoked. Notify appropriate project teams which test environment testing activities are available or not, and when. Work with project and release managers to reschedule any conflicts.

Maintain test environment asset inventory.
See Supporting files for more detailed tasks for Test Facility Management.

Test Environment Involvement in Project Management

Ensure test environment resources are involved and participating in project and test project reviews. Keep them informed of changes, issues and risk management strategies.

Prepare high level estimates for test environment specific activities (project resources, costs, price, duration). Identify necessary project resources specific to test environment. Determine the availability of the identified resources for test environment.

Plan for non-application directed testing projects (e.g., environment and/or infrastructure changes that do not require application testing). Plan for introducing production equivalent hardware, software, and network components into the test environment (in sync with the other development, test and/or production environments as required).

Product Evaluation and Recommendation

Evaluate and select new/additional test environment testing and monitoring tools. Communicate selected test environment testing and monitoring tool recommendations. Evaluate new hardware, software, and network components needed to support application and non-functional testing in test environment. Communicate hardware, software, and network recommendations.

Product Acquisition and Installation

Procure new/additional test environment testing and monitoring tools according to organizational approvals, policy, and procedures. Install new/additional test environment testing and monitoring tools. Train test environment staff on new tool usage.

Procure new hardware, software, and network components needed to support application and non-functional testing in test environment. Install new hardware, software, and network components. Stabilize new hardware, software, and network components. Migrate to the other development, test and/or production environments as required. Restabilize in the other environments.

System Requirements Analysis

Review business and application requirements, such as events and scenarios, to determine additional requirements necessary for the test environment. Review non-functional requirements; e.g., performance, stress and volume testing, for the test environment. Identify impacts to current (in-place) test environment and infrastructure.

Define application and non-functional test environment test strategy. Review test types to be performed in test environment. Identify dependent applications required for testing in the test environment. Identify external dependent applications required for testing in the test environment. Define test environment test strategy for non-application directed testing projects (e.g., environment and/or infrastructure changes that do not require application testing).

Develop detailed estimates covering test environment specific activities (project resources, costs, price, duration). Identify necessary project resources specific to test environment. Determine the availability of the identified resources for test environment.

Schedule and incorporate test environment activities within the overall project plan and Work Breakdown Structure.

System Specification

Prepare application and non-functional test environment test plans and integrate within overall project test plan. Prepare test environment test plan for non-application directed testing projects (e.g., environment and/or infrastructure changes that do not require application testing) and integrate within overall project test plan. Identify and document each independent test environment configuration.

Architecture and Infrastructure Planning

Identify “new” hardware, software, and network components needed in test environment to support application and non-function requirements. Identify necessary test environment testing and monitoring tools to be used. Identify test environment testing and monitoring tools necessary for testing that must be acquired.

Technical System Design

Complete application and non-functional test environment test plan. Complete test environment test plan for non-application directed testing projects (e.g., environment and/or infrastructure changes that do not require application testing).

Produce test environment application and non-functional test design specifications. Produce test environment test design specifications for non-application directed testing projects (e.g., environment and/or infrastructure changes that do not require application testing).

Produce test case specifications. Produce test case specifications for non-application directed testing projects (e.g., environment and/or infrastructure changes that do not require application testing).

Finalize test environment application and non-functional test documentation and provide to test environment coordinator. Finalize test environment documentation for non-application directed testing projects (e.g., environment and/or infrastructure changes that do not require application testing) and provide to test environment coordinator.

Maintain all related test environment and configuration documentation.

Technical Procedure Development

Document any test environment-specific testing processes and/or procedures. Review test environment Entrance Criteria. Determine whether test environment Entrance Criteria have been met.

Testing and Supporting the Environment

Review finalized test environment test documentation. Populate databases needed for application and non-functional testing at test environment. Populate databases needed for test environment testing for non-application directed testing projects (e.g., environment and/or infrastructure changes that do not require application testing).

Establish application and non-functional test environment configuration. Establish configuration for non-application directed testing projects (e.g., environment and/or infrastructure changes that do not require application testing).

Test application and non-functional test environment configuration. Test environment configuration for non-application directed testing projects (e.g., environment and/or infrastructure changes that do not require application testing).

Migrate databases and software needed for application and non-functional testing in test environment. Migrate databases and software needed test environment testing for non-application directed testing projects (e.g., environment and/or infrastructure changes that do not require application testing).

Prepare test scripts needed for application and non-functional testing in test environment. Prepare test scripts needed for test environment testing for non-application directed testing projects (e.g., environment and/or infrastructure changes that do not require application testing)

Execute application and non-functional test environment tests. Execute test environment tests for non-application directed testing projects (e.g., environment and/or infrastructure changes that do not require application testing).

Evaluate application and non-functional test environment test results relative to test plan. Evaluate test environment test relative to test plan (for non-application directed testing projects (e.g., environment and/or infrastructure changes that do not require application testing).

Document and report results of application and non-functional test environment test. Document and report results of test environment test (for non-application directed testing projects (e.g., environment and/or infrastructure changes that do not require application testing).

Determine if Exit Criteria has been met for application and non-functional project to proceed to the next level of testing or implementation. Determine if Exit Criteria has been met for non-application project to proceed to the next level of testing or implementation.

Release hardware, software, and network components when application and non-functional test environment test concludes. Release hardware, software, and network test components when test environment test (for non-application directed testing projects, e.g., environment and/or infrastructure changes that do not require application testing) concludes.

Ensure successful migration from test environment test to next level of testing or implementation.

Backup and Restore Processes

Ensure that these processes are incorporated in the test plans before the environment is changed or torn down, to allow reuse of test assets and to facilitate repeat tests whenever necessary.

Other Management Processes

Change/issue/risk management processes should mirror those of the project wherever possible.
