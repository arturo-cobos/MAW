Guideline: Developing the Source Files
The functionality of the class, as defined in the class description and in other work products, is implemented in the coding language, source files are usually created in one of three ways.

  • All of the Source Files are is manually created with a text editor.
  • A part of the Source Files, for example, the interface declarations and the function stubs, is generated by a design tool or from other source forms, for example, the Interface Definition Language (IDL) of some Object Request Broker, such as IBMâ€™s System Object Model (SOM). The rest of the Source Code, usually the function bodies, is filled in using a text editor.
  • All of the Source Files are generated by a tool from other descriptions, whether they are a higher-level form of source code or a complete specification with design and architecture descriptions. This fully automatic code generation is not a common approach today.

Smalltalk and similar environments manage the Source Files in a repository, in which case there is no single source form.
