Checklist: Usability Test Plan

Related Elements

Check Items
Usability test cases are derived from the use case model scenarios
Test participant selection is based on one or more profiles of the users
The usability test plan identified a minimum of 10 to 12 users that will serve as participants for the testing
Sufficient numbers of usability test cases exist for each identified usability test category
For a medium size project, there will be anywhere from 8 to 15 key usability test cases.
Usability test cases are derived from those use case models executed most frequently or seen as most important by users
This selection can be verified by reviewing any use case validation data that had been collected prior in the solution development effort.
The end state descriptions in the usability test cases contain concrete data, not general descriptions of the test case
Usability requirements are reflected in the success criteria for the usability test cases
That is, there is a clear correspondence between the criteria identified for the test and the stated usability requirements.
Test roles are clearly defined and identify specific personnel for each of the roles
Usability test criteria are clearly and objectively defined

Criteria is typically defined in terms of:

  • test case completion times
  • the type and operational definition of errors
  • the specification of subjective measures of usability in the form of questionnaires
