Checklist: Software Distribution Plan Validation and Verification

Check Items
Software Distribution Plan identifies all the users with specific names and locations.
People have been clearly designated to handle problems that occur during the distribution phase
This should cover the development team, test team and operations staff.
Licenses and other dependencies are clearly identified
All security measures and validation steps are identified before distributing software

To identify and remove security defects that exist within the code or mitigate infrastructure exposures that could cause such defects:

  • Identification of security requirements based on business goals, technical constraints, and relevant standards.
  • Specific testing in the software development phase should include:
    • Specific security testing of the application similar to performance testing
    • Security Code Walkthroughs and/or line by line code reviews.
    • Regression testing and review activities
    • Virus scanning prior to inclusion in the distribution server
Change controls exist to prevent unauthorized code from being moved into the version control system or repository
Necessary system integrity controls (logical and physical) are in place

This ensures that the building and testing of each change or patch to the code is performed on trusted systems and protects the integrity of the distribution server.

e.g., read-only storage, dedicated server, physical/logical controls and configuration hardening

Network security controls are in place to protect software in-transit
e.g., use of hashing and cryptography.
