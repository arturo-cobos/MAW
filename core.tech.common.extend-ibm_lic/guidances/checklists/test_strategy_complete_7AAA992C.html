Checklist: Test Strategy Complete
Criteria for validating the completed testing strategy including the objectives, focus areas, types, levels, required resources and milestones.
Relationships
Related Elements
  • Test Strategy
Check Items
The testing focus areas and objectives were determined and prioritized
The test focus areas were used to define the test objectives
The test objectives will take into consideration each of the business and technical functions to be tested.
Each major business function and user requirement verified
Verification is done as to how each requirement would be objectively tested by outlining a test scenario.
A testing strategy was developed that addresses the test objectives
The types of tests that must be done for each test focus area were determined
The levels of testing for conducting each type of tests were identified
High level test environment requirements defined and documented
This is done so that the hardware, software, and organizational resources necessary to run the various types and levels of tests are available when required.
The test team organization was defined
Overall responsibility for the management and coordination of project testing activity assigned
The responsibility includes test planning, preparation, execution, and reporting for each major test activity.
All the skills required to conduct the selected types and levels of testing were identified
Individuals were identified who can perform (or be trained to perform) the testing activities
Testing milestones and high-level schedules were developed
The time required for the testing activities identified in the testing strategy was estimated
Critical paths that may impact testing were identified
Test activities were scheduled to ensure appropriate focus on high priority items
Training requirements for test team members were identified and included in the project plan and schedule
The Testing Strategy was documented and concurrence obtained from all interested parties
