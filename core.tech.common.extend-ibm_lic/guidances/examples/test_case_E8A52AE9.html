Phase 1 – LEC and IGF (Wireline)



Test Case #. CM20700

Test Stream: Stream 2—Customer Management/ Add Additional Accounts

Test Title:


Add Account ID’s

Scenario/Description:
This test case will check whether account ID’s may be added or overridden. This is different from the Customer ID tackled by Test case CM10900.

Prerequisites:


Customer/ Account Record already exists and all customer level testing should have already been executed successfully.

Access Path:
6, 1, 2, 2, F20 (Account ID)

Test:
1. Retrieve account record and go to Account Maintenance in order to access the account ID function key.
2. To add or change ID information from the Account ID override screen,
select an ID type, ‘P’ for passport and input the corresponding ID details
(E.g. N 682942).

Test for various ID types- type A, C, I & G.

Data Requirements / References:
Customer record, ID type

Expected Results:
1. Account maintenance screen will allow user to access Account ID function key. The customer ID details will default together with the Account ID type and description fields.
2. New ID details for selected ID type will default on the screen and on the account maintenance screen. Old ID type and description will be overridden.
