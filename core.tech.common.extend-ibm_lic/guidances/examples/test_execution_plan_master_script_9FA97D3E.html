Test Master Script:

Day 1 - N:


1) Set Holiday file (Holiday file has been set – using production copy)

STEPS 2 – 9 will begin at 8 am
2) Set Control file for day’s activity. The steps are as follows:

  • Check to see if file CICSEDUC.AOB.CTLFILE exists 
  • If not, submit job CREATCTL from MIS.Y2KTEST.BASEJCL. Ensure that the dsn for the DD statement CTLFIL of the GO step is CICSEDUC.AOB.CTLFILE 
  • Submit job UPDTCTL from MIS.Y2KTEST.BASEJCL, however, before doing so ensure the exchange dates are properly set in data set member ACTDAYxx (where xx is a number indicating the activity day) in PDS MIS.Y2KTEST.BASEINPT.

    - The dates within the ACTDAYxx member will be of the format dmm/dd/yy where d will have a value within the range of 0 – 6 (0 = Sunday, 1 = Monday, 2 = Tuesday, 3 = Wednesday, 4 = Thursday, 5 = Friday, 6 = Saturday). mm = month, dd = day, and yy = year. For example 107/26/99 is Monday the 26th of July 1999.


    - Make a copy of the create control file and update control file JCL. For example MIS.Y2KTEST.BASEOUTJ(CCFDAY1) and MIS.Y2KTEST.BASEOUTJ(UCFDAY1). Use the commands PRT DATASET, PRT, and PRT CLOSE in SDSF. See Step # 24.

3) Set Exchange date in CICSEDUC region – SEE ATTACHED DETAILS.

  • Select E Extensions from Primary Menu

Select Z Compuware Xpediter/Xchange
From Simulated Date Time Settings Panel enter CICSEDUC under Job Name along with the date to be tested, enter H in first column to set date

4) Bring up CICSEDUC – (TSO option 6, Type in CICSEDUC. A message stating CICSEDUC has been submitted along with a job number will display)

5) Sign onto the CICSEDUC region via the CESN transaction (enter user id and password) and ensure we can get to the main panel via the MISMC transaction. Note the date in the upper right hand corner. It should be the exchange date which was set in Step # 3.

6) Ensure printers IS31 and IS32 are released from CICSTEST. CEMT I TE(IS3*)

7) Check that printers IS31 and IS32 are set for PTCF and TPRT respectively, under ATI TRAN. Do this via the command DCT IS3*.

8) Acquire IS31 and IS32 in CICSEDUC. Check status via the command CEMT I TE (IS31) and CEMT I TE (IS32). CEMT I TE (IS31) and CEMT I TE (IS32).


9) Verify that all interfaces are set for MVS, IDX, and Datagate. For MVS, within the CICSEDUC region enter the transaction IMON (SEE ATTACHED DETAILS). The listener should be enabled (TCP/IP API LISTENER). For IDX, once logged on,

       - select 10.LHMY2K HPA
       - Then enter 2 for Enter Group
       - enter 13 for Select Function
       - enter 25 for Select Activity
       - enter 7 for Select Activity

       - Disregard #1, however, for #’s 2 – 6, highlight and type ‘S’, this will then display running for each

The status for #4 will be started. For Datagate we will coordinate with Susanne Beeber to ensure the interfaces are set/up between Datagate and the mainframe and Datagate and IDX.
*** For IDX when selecting 10.LHMY2K HPA, if the message “BED FILE PROBLEM” displays after entering 2 for Enter Group, do the following:
       - Go back to main menu and select 6.Bedfile Copy.
       - Then select 10.LHMY2K HPA again and continue with the process. ***

10) At 9:15 am there will be a team meeting to review and validate night batch from previous day. *** If there are problems, the following will be performed: 

  • Log problem via form in ntfs04\sysdev\yr2k\testing\meeting notes\Problem Log Sheet.DOC 
  • Enter problem(s) into PTARS db.

Results of the batch review will determine if we continue with the online activity for the day.

11) Set XCHANGE date for batch runs (see step # 3, but enter $Y2K* under Job Name and H in first column to set and hold the date).


12) Run MRT36D job for MRT in MIS.Y2KTEST.BASEJCL
Set Exchange Date to Activity Day – 1.
Save MRT36D to TSO data set: MIS.Y2KTEST.BASEOUTJ(MRT26Dx)
X= Activity Day

13) Backup of the day’s online activity, as well as, night batch updates. Run job BACKUPJ in MIS.Y2KTEST.BASEJCL, however, ensure the last qualifier of the backup data set denotes the activity day. In addition, one of the qualifiers should denote this is the backup immediately proceeding the run of the night batch. For example: DBA.Y2KTEST.BASE.HMRNI0$.DAY0.NITEBTCH.


14) Make a copy of the night batch BACKUP JCL. For example: MIS.Y2KTEST.BASEOUTJ(BBUDAY1). Use the commands PRT DATASET, PRT, and PRT CLOSE.



15) Start Test (Oversee and ensure the sequence of tasks for online activity - view newly developed, activity days spread sheet).


16) Ensure all of online is printed, each tester should get their own output, as well as, the output for fellow testers when applicable.



17) At 2:30 pm there will be a team meeting after online activity. Determine if we should proceed with the running of the night batch, based on the number of problems determined from the online activity. *** If there are problems, the following will be performed:

Log problem via form in ntfs04\sysdev\yr2k\testing\meeting notes\Problem Log Sheet.DOC
Enter problem(s) into PTARS db
Collect and file test cases from the end of day’s activity.

*** The following is performed if the go ahead is given for running the night batch:

18) Bring down CICSEDUC.

Sign on to CICSEDUC via the CESN transaction
Issue the following transaction: CEMT P SHUT IMM
*** At this point as part of the shutdown GDG’s will be created

19) Append GDG’s to Pre-allocated Y2K TSO data sets with a header indicating the activity day (i.e. Day 0).

Copy/Append CICSEDUC.Y2KTEST.BASE.LAB.RECEIVE.G0001V00 to

MIS.Y2KTEST.BASEGDG.LAB.RECEIVE

Copy/Append CICSEDUC.Y2KTEST.BASE.LAB.SEND.G0001V00 to

MIS.Y2KTEST.BASEGDG.LAB.SEND

Copy/Append CICSEDUC.Y2KTEST.BASE.ECG.RECEIVE.G0001V00 to

MIS.Y2KTEST.BASEGDG.ECG.RECEIVE

Copy/Append CICSEDUC.Y2KTEST.BASE.IDX.IDXR.GDG.G0001V00 to

MIS.Y2KTEST.BASEGDG.IDX.IDXR.GDG

Copy/Append CICSEDUC.Y2KTEST.BASE.PTH.RECEIVE.G0001V00 to

MIS.Y2KTEST.BASEGDG.PTH.RECEIVE

Copy/Append CICSEDUC.Y2KTEST.BASE.PTH.SEND.G0001V00 to

MIS.Y2KTEST.BASEGDG.PTH.SEND



20) Backup of the day’s online activity. Run job BACKUPJ in MIS.Y2KTEST.BASEJCL, however, ensure the last qualifier of the backup data set denotes the activity day. In addition, one of the qualifiers should denote this is the backup immediately proceeding the online activity. For example: DBA.Y2KTEST.BASE.HMRNI0$.DAY0.ONLINE


21) Make a copy of the online BACKUP JCL. For example: MIS.Y2KTEST.BASEOUTJ (OBUDAY1). Use the commands PRT DATASET, PRT, and PRT CLOSE in SDSF. See Step # 24.


22) Set XCHANGE date for batch runs (see step # 3, but enter $Y2K* under Job Name and H in first column to set and hold the date).


23) Run night batch job for MRT. Submit member $DAYx (where x = activity day) in MIS.Y2KTEST.BASEJCL.


24) MRT BATCH CONTINUED: The following jobs will be submitted on their current exchange date + 1 day and a time of 00:05: DAY1MOR, DAY10MO, DAY13MO, DAY4MOR, DAY8MOR, and DAY9MOR.


25) Run night batch job for OE. Submit the following members (all in MIS.Y2KTEST.BASEJCL): Change INPUT_DATE for JOES0160


- $Y2KJ150
- $Y2KJ180
- $Y2K1030
- $Y2K1050
- $Y2K9000
- $Y2KJ090
- $Y2KJ470
- $Y2KJ630
- $Y2KJ670
- $Y2KJ160
- $Y2KI700
- $Y2KJ260
- $Y2KOEBE

26) OE BATCH CONTINUED: If the activity day = 4 or = 7 then submit job $OEDAY4 or $OEDAY7 respectively. This job(s) should be submitted before the normal stream of jobs.NIGHTJOB.


27) OE BATCH CONTINUED: On the last activity day submit ENDJOB after the normal stream of jobs.


28) Save OE and MRT batch to TSO data sets (use the PRINT DATASET, PRT, and PRT CLOSE commands in SDSF) – SEE ATTACHED DETAILS. For example MIS.Y2KTEST.BASEOUTP (ONTBDY1) and MIS.Y2KTEST.BASEOUTP (MNTBDY1).


29) Print OE and MRT batch. (use the XS, PRT, and PRT CLOSE commands in SDSF) – SEE ATTACHED DETAILS


30) Batch reports for OE and MRT will be in the output bin outside of Chel and Paul’s office. These reports are for review and validation of night batch.
