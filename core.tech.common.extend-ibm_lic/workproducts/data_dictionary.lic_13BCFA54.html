Description
Brief Outline

Depending on project standards, entries in the data dictionary should document the following:

Entities
  • The entity name and aliases
  • The description of the entity
  • Additional information which, while not part of a conceptual schema, may need to be documented on particular projects:
    • The purpose of the entity
    •  What an occurrence of the entity represents
    • Special considerations for the entity relative to this project effort
    • Current volume - Current number of occurrences
    • Expected growth - Number of new occurrences per year
    • Growth percentage - Annual rate at which new occurrences are expected to increase/decrease
    • Where entity is used - Organizational unit
    • Availability - High, medium, or low
    • Security - Level of authority needed to protect an occurrence of the entity when originated or changed
    • Confidentiality - Level of authority needed for reading an occurrence of the entity
    • Approximate size - Approximate size of data for an average occurrence of the entity
Attributes
  • The purpose of the attribute
  • The definition of the attribute
  • The attribute's domain
Relationships
  • The participating entities
  • The business rule or fact that necessitates the relationship
  • The cardinality associated with the entities' participation in the relationship
Tailoring
Reasons for not needingIn some cases a simple glossary may prove sufficient if the data model is not overly complex.
Representation OptionsThe relevant information for the data dictionary is often embedded in the data modeling tools rather than being maintained as a separate stand-alone artifact.
