Description
Main Description

This artifact identifies the gaps between the IT organization's existing technology, applications, processes, skills and organization and those required to implement and support the new system.   This is comprised of the findings of a data gathering process, or a comparative design exercise. It quantifies the impact of the migration from the existing to the desired IT infrastructure.  The artifact documents the work done to date, what was found during data gathering (high level requirements and key assumptions), the difference between current and future infrastructure and the steps needed to deal with the gaps.

Brief Outline

The Infrastructure Gap Analysis is basically textual, with tables or spreadsheets showing the facts, findings and conclusions. Subject areas or headings that could be used as categorizations include:

  • Technology, Process and People – this categorizes the subject area of the analysis.
  • Facts, Assumptions, Questions and Issues – this sub-categorizes the subject areas into what is known, what is assumed, what needs to be known and what is an issue.
  • End User Function Groups – this is a categorization of the organization into functional areas (e.g. manufacturing, sales, marketing development etc) with a list of the infrastructure used. Sub-categorizations of end user function groups could include:
    • Number of Users
    • Service level objective(s)
    • Existing hardware and software
    • Required hardware and software
    • Connectivity
    • Redundancy requirements
    • Central vs. local applications
    • Interfaces

Key Considerations

The following guidance is suggested:

  • If an architecture for the new system has been created, compare this to the current IT environment of the enterprise to identify gaps.
  • Wherever possible use existing inventories and application portfolios and compare these with system documentation.


   

Tailoring
Impact of not having
Reasons for not needing

It may not be appropriate to produce this artifact if:

  • This is a green-field site or project with no existing infrastructure.
  • An existing application is being replaced with a new one and experience shows no reason for a gap analysis.
  • The existing infrastructure is being completely replaced.
