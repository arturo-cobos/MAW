Template: Work Product Inspection Form
This MS Excel template documents results and supporting information for a formal inspection of work products.

Main Description
This template captures the following details:

  • Location of the defect: This shows where the defect is found
  • Detected Process: The CMMI process area where the defect is detected
  • Severity and priority of the defect: The drop down options available are 1- Critical, 2-High, 3-Medium and 4-Low
  • Defect description: Explanation on the effect of the defect
  • Originator's name: The person who raised the defect
  • Defect type: This is the defect classification such as Data, Standards Compliance, Interface, Logic, Performance, Clarity and Requirements Compliance
  • Phase injected: The life cycle during which the defect entered. The available phases in drop down menu are Requirements, Design, Code, Test, Install and Maintain
  • Defect escaped: The technique from which the defect escaped. This is an optional field and the available options in the drop down menu are Requirement Review, Design Review, Code review and Automated (tool based) review
  • Root Cause: The technical root cause of the defect. The options provided in the drop down menu are Code, Design, Infrastructure, Methodology, Process, Rejected, Security and Test
  • Process Root Cause: This shows why the process or methodology being followed allowed the defect to be injected. The options provided in the drop down menu are Skills, Metrics, Pressure, Testing, Complexity, Requirements and Others
  • Disposition: The action taken on the defect and the options provided in the drop down menu are Accepted, Deferred and Invalid
  • Status: This is the actual status of the defect and the options provided in the drop down menu are Raised, Assigned, Bypass, Fixed and Closed
  • Date corrected: This gives the date on which the defect is corrected
