Template: Organizational PQA Non-Compliance Log
This MS Excel template is used by the Project Quality Analyst (PQA) Team to document and track the open non-compliant items that are identified in the Process Assurance reviews.

Main Description
The Non-Compliant Item (NCI) Data worksheet is used to log the non compliance items and it captures the following details:

  • Non-Compliant Item (NCI) #: These are numbered by the PQAs in sequential order as created Project ID or Application Name: Name or identifier for the project
  • Procedure : Name of the procedure against which the NCI occurred
  • Procedure Activity #: Reference ID/Procedure from the Process Assurance Review Report - Project (ENG442a) or Process Assurance Review Report - Organization (ENG 442)
  • PQA Representative: Name of the PQA for the project
  • Action Owner: Name of the person responsible for addressing the NCI issue.
  • Date of Review: Date the review was performed and the NCI was identified/opened. (mm/dd/yy format)
  • Date Planned Closure: The date by which the NCI must be closed. This is a date agreed upon by the PQA representative and the Project Manager
  • Date Escalated to Management: The date the NCI was escalated to Management because it was not resolved by the agreed upon date.
  • Date Closed: The date that the NCI was closed. (mm/dd/yy format)
  • Description of Deviation/NCI: Free form text description of the NCI
  • Actions Taken: Free form text description of Actions Taken
