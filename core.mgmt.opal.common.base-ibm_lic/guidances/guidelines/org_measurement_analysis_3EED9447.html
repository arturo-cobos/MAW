Guideline: Organizational Measurement and Analysis
Use this guideline to establish a measurement program, and ongoing organizational measurements and analysis.
Main Description

This guidance provides details of how to gather and provide data (primitive metrics) to support the organization's computed metrics, establish organization's measurement program and measure performance levels.

Note: For IBM-managed organizations, this guideline includes the following:

  • Information on the Metrics Management Framework (MMF)
  • Benchmarking and Competitive Analysis (BCA) for metrics data

Information on the Metrics Management Framework example

The MMF consists of two components:

  • A core set of intellectual capital supporting the implementation of an organization level measurement program
  • An Integrated metrics framework providing data collection, repository, analysis and reporting functions to support an organization level measurement program

 Refer to the Metrics Management Framework website for additional information on the MMF.
For new or existing clients interested in assistance to implement a measurement program, reference the measurement implementation services website.

 Refer to the KnowledgeView for MMF assets.

Submitting project data to IBM

Note: For IBM Managed Organization use BCA for submitting project data.

Requesting benchmarking data from IBM
Data is submitted from the organization to the IBM 'Organization' for use in Benchmarking and Competitive Analysis (BCA), benchmarking data may also be requested from IBM BCA. The process for requesting benchmarking data from IBM BCA, when there is a need for data from outside of the organization, is documented under this heading. An example is a need for historical data for reference or analysis, when insufficient data is available at the organization due to new technology or the recent implementation of the measurement program.
