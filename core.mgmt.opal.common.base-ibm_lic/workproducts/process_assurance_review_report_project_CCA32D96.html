Artifact: Process Assurance Review Report - Project (ENG 442a)
This artifact provides a list of items to be verified when performing reviews of a project's compliance to its processes, and provides documentation of the review results.
Domains: Quality Management
Work Product Kinds: Assessment
Purpose

Its purpose is to provide the criteria for performing a project level compliance review, to assure that processes are being followed appropriately.

Relationships
TasksInput To:
  • Participate in audit or review (IMSP421 and IMSP232)
  • Prepare for review (IMSP421)
Output From:
Description
Brief OutlineThis artifact typically contains:
  • General project and reviewer information
  • Compliance criteria or questions
  • Evidence, findings and exceptions
  • Review results
Selected Representation

For projects that must comply with the Process Metrics Reporting (PMR) requirements, a specific template exists that does not allow partial credit and provides only a binary scoring approach to support PMR.

Key Considerations
While reviewing projects that are grouped under Technology Assembly Centre for access control mechanisms (question 20), verify if there is any evidence of usage of shared desktops/ laptops. If yes, the login id used by each member is expected to be different. In cases where there is shared user id, a noncompliance is raised and GDIT (Global Delivery IT) team is informed immediately.
Tailoring
Impact of not havingConsequences of not using this artifact are:
  • Inconsistency in verifying the process compliance of projects
  • Lack of objectivity and completeness while verifying process compliance
  • Difficulty reporting metrics consistently in accordance with the organization metrics program
  • Evidence of conformance may not be recorded consistently
  • Overall compliance ratings may not be readily available leading to difficulty in reporting to senior management on the compliance status
Reasons for not needingThis artifact is not required if process assurance reviews are not conducted.
