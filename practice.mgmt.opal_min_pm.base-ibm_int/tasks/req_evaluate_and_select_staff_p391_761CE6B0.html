Task: Request, evaluate, and select staff (IMSP391)
This task will address staffing needs that have been identified for the project.
Purpose

To ensure the appropriate staff is in place to support the project.

      Main Description

      This task is based on the WWPMM work pattern S5: Obtain staff.

      Staffing requirements were documented in the Human Resource Plan when shaping the project, and sourcing organizations (such as delivery organization entities, other internal performing organizations, sponsor, and staff providers) had committed to staff the project teams accordingly. The time is now coming for these sourcing organizations to propose individuals to fill the project positions.

      This task is part of the activity that evaluates the candidates that are proposed by the sourcing organizations, and gets firm commitments on the starting date, cost, and availability of the selected candidates.

      Typically, although the staffing needs for the project startup team and the team members for early execution stages of the project may have been anticipated, many of the remaining team members still need to be determined at project startup.

      This task must also be planned in advance of starting any phase or group of activities that require new team members within the project, as forecasted in the Human Resource Plan and confirmed in the Project Management Schedule.

      The key output of this task is the Human Resource Plan which may be part of the Project Management System Summary.

      This task includes the following:

      • Request staff to fill gaps.
      • Evaluate and confirm that staff with appropriate competency and skill levels is available to execute the project.
      • Verify that all overseas resources have the correct work authorization prior to beginning work on a project.  If a resource is an IBM employee from another country, ensure that written confirmation of a valid work authorization (such as a visa) has been obtained from the appropriate manager, and that the confirmation is filed in the Project Control Book (PCB).
