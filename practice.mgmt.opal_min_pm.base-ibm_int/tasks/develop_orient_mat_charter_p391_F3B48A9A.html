Task: Develop orientation material and team charter (IMSP391)
This task will prepare to welcome team members to the project by developing orientation material and a team charter for them.
Purpose
To prepare for welcoming new team members to the project, facilitating their integration within the project team, and ensuring they are brought up to speed with the work they have to perform.

      Main Description

      This task is based on the WWPMM work pattern S6: Start staff.

      This task is part of the activity that should anticipate the date when team members are going to join the project for producing its deliverables, and should finish when it is clear that these team members are up to speed. Thus this task may be planned several times in the course of project execution.

      The first time this task is performed is at project execution startup, and orientation material must be prepared to welcome the newcomers. Subsequent introduction of new team members will likely not require the same amount of effort.

      There are a number of practical and administrative steps that must be taken, regarding office space, allocation of material, appropriate access to the technical and PM environment, most of which are usually described in full detail in the managers general guidelines for the delivery organization.

      The focus in this task is on providing the new team members with the big picture of the project, the overview of the specific part of project execution they are going to be involved in, and on ensuring they are brought up to speed with the work they have to perform.

      The key output of this task is the Team Charter.

      This task includes the following:

      • Develop orientation material and a team charter.
      • The orientation material contains information about shared performance objectives, expectations about team behavior, rules of engagement, key roles and responsibilities, the project defined process (including the policies, standards, methodology, procedures, guidance, work products and tools to be used by the project) information security and other useful information. It may include information extracts from:
        • agreement
        • operational schedule
        • project definition
        • project procedures description
        • project management system summary
        • project quality plan, which may be part of the project management system summary
        • project defined process
        • client data security and privacy plan
      • It is recommended that an overview of the following is included in the orientation material:
        • project's high-level solution outline and approach
        • processes, methodology and tools to be used by the project
        • client information classification and how to handle client sensitive information
      • Classify the information in the orientation material, deciding what information may be communicated to a broad audience and what may be communicated in a limited audience or even individually
