Task: Manage change request (IMSP610)
This task will manage each request for a change to an aspect of the project that is under change control.
Purpose

To manage each request for change, in order to keep the baselines of the project under control with full traceability, including:

  • Making sure that each request for change is assessed by appropriate project stakeholders, and that each assessed change request is accepted (or rejected or deferred) by the appropriate authority
  • Negotiating amendments to the sponsor or supplier agreements if necessary
  • Enabling the orderly implementation of each accepted change

        Main Description

        This task is based on the WWPMM work pattern HE5: Handle change request and is launched when a change, to an aspect of the project that is under change control, is requested. The request may come from the sponsor, the delivery organization, or a supplier. It may deal with a single major change or a number of small changes. It may address the schedule, resources, solution, or any aspect of the project.

        The authority for deciding to analyze, reject or approve change requests must be defined in the change management procedure of the project with respect to the delivery organization policies and the agreements.

        • Review a new change request, make an initial assessment of its impact, and determine how to handle the request.  Handling the change request includes deciding whether it should be analyzed further. If further analysis is launched, handling the change request involves revising baselined elements and dependent documents (which may include the sponsor agreement) along with communicating the change decisions to the team.
        • Assess the impact of a requested change upon the project, determine a solution, itemize the detailed costs and benefits/risks, and seek approval of the change request.
        • Obtain approval of changes, including any amendment that may be required to the Agreement, and schedule the resultant change orders.
        • When Change Requests are approved for implementation, Change Orders instruct the project team to implement the Change requests. One Change Request can give rise to more than one Change Order. Alternatively, one Change Order can result from more than one Change Request. There should be a signed amendment in place before issuing any related Change Orders.

        The key outputs of this task are the Change Request, Change Order, and Change Log, or the Event Log, plus Agreement amendments when necessary, in accordance with the change management procedure applicable to the project.

        The task includes the following:

        • Raise change request
        • Perform initial assessment, reject or accept for analysis
        • Manage contract changes

            • Local Privacy Laws
        • Determine impact and potential solutions
        • Obtain formal Agreement and schedule Change Orders
        • Implement change
        • Close change
