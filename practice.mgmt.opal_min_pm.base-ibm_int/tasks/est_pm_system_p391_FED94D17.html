Task: Establish project management system (IMSP391)
This task expands upon the summary project management system produced while planning the project and defines a detailed management system as the basis for managing the project. This includes defining or expanding upon various plans and documenting project specific procedures and standards.
Purpose
To ensure an appropriate project management system, including plans, procedures and standards, is in place to implement and support the project successfully.

        Main Description

        This task is based on the WWPMM work pattern S2: Implement project management system.

        Normally, the key features of the Project Quality Plan and of the project management system to be implemented at project startup have been defined when planning the project, particularly when these features were expected to have an impact on the project costs and commitments.

        However, these plans are typically only "outlined", being developed to a level which was sufficient to prepare the Agreement, but that is not sufficiently detailed to control project execution. It is now time to "expand" these plans to a level of detail relevant for project execution control, by doing the following:

        • Identify deliverables quality requirements: Review the Deliverable Definitions and associated acceptance criteria. Determine the quality factors and quality contributors that will likely ensure meeting the acceptance criteria, and the procedures to ensure the effectiveness of the quality contributors.  Expand the Project Quality Plan accordingly.
        • Expand the technical environment plan to identify the PM tools supporting the project management system.
        • Define the project management system, performing project tailoring to confirm the procedures which should be enforced to facilitate such things as project control, communications, and exceptions handling.  Use the project defined process to record tailoring and the project procedures description to record any project-specific procedures, and reference any applicable standards in the project quality plan.
          • Review the sponsor agreement, deliverable definitions, product breakdown structure, the outline of the project quality plan, and the Client Data Security and Privacy plan that were produced when shaping the project during the proposal effort.
        • Define project reporting in the communications management plan, defining the format and schedule for project status reviews and reports, and establishing the format, tools, schedule and frequency for status reporting,
        • Develop the project data management plan, identifying the data required to support the project in all areas.

        Complete as necessary the initialization of the project management tools, and ensure the project team is ready to apply the project management system features.

        The key outputs of this task are the Project Procedures Description, the Project Defined Process, and various plans, such as the Project Quality Plan, Technical Environment Plan, Communications Management Plan, and Project Data Management Plan, which may all be part of the Project Management System Summary.
