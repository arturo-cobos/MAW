Task: Create or Update Plans (IMSP491)
This task will identify a set of initial project plans to be outlined or built, to describe the project management system for the project and the work approach to deliver the solution.
Purpose
To identify, create and update the project plans that are needed for the project, so the project can be started with a clear understanding of the plans.

      Main Description

      This task is based on the WWPMM work pattern P4: Finalize sponsor agreement.

      The key outputs of this task are the Risk Definition which may be part of the Event Log, and the Project Management System Summary and the appropriate plans that may be part of it, such as the Project Quality Plan, Project Data Management Plan, Configuration Management Plan, Communications Management Plan, Life Sciences Organization Validation Plan, Human Resource Plan, Technical Environment Plan, Risk Management Plan, Client Data Security and Privacy Plan, Release Plan, Disaster Recovery Plan, and the Application Recovery Plan.

      Note: All plans in the GBS PM Minimal (WWPMM) delivery process are documented within the Project Management System Summary (PMSS). 

      This task includes the following:

      • Define the project management system requirements and verify that these address the project risks
      • Develop the initial project quality plan
      • Outline the project data management plan
      • Develop the initial configuration management plan
      • Define the regular meetings, reporting and other communications
      • If the project is subject to Life Sciences Organization regulations, outline the life sciences organization validation plan
      • If maintenance and support infrastructure will be developed, outline the appropriate tasks
      • Update the human resource plan
      • Outline the technical environment plan
      • Document the additional project work environment details
      • Create or update the Service level agreement management plan section of the project management system summary
      • Perform a risk assessment of the work environment established for the project
      • Create the product integration plan
      • Update the WBS and project management schedule
      • Outline any additional plans required
      • Update the project management system summary for the plans it contains or references
      • Document the process, methods and tools utilized on the project in the Process, Methods and Tools (PM&T) Profile

      Refer to Data Security and Privacy web site for the GBS Information Security Document (for contracts signed after August 2008) for guidance for security roles and responsibilities.
