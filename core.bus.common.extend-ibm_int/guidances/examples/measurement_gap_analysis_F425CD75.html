Main Description

Measurement Gap Analysis for XYZ

Introduction

This document identifies organization-level measurements that are not currently being adequately collected, analyzed and used but that are also critical to the achievement of XYZ’s business goals. It identifies new processes or changes to existing processes that need to be instituted to support such measurement data collection, analysis, and use. Furthermore, it identifies proposed changes to the organization so that it can support the data collection, analysis, and use of the selected measurements.

Purpose

The Measurement Gap Analysis serves the following purposes:

  • Identifies the gaps between the measurements that are being proposed to XYZ and the measurements that XYZ currently collects, analyzes, and uses.
  • Examines the existing organizational structure to determine if it can adequately support the measurement model. Inadequacies are identified, and requisite changes are proposed.
  • Examines the existing organizational processes to determine if they can adequately support the measurement model. Inadequacies are identified, and requisite changes are proposed.

Scope

The measurements covered in this gap analysis are limited to organization-level measurements. While project-level measurements may roll up to organization-level measurements, project-level measurements are outside the scope of this analysis.

Description

Developing the Measurement Gap Analysis consists of the following steps:

  • Examining what metrics are currently collected, analyzed, and used by XYZ.
  • Mapping those metrics to the selected metrics described in the current instance of the measurement model, noting the details for each of the metric attributes.
  • Identifying discrepancies between the selected metrics in the current instance of the measurement model and XYZ’s current metrics, as well as discrepancies in the values of the metric attributes.
  • Identifying current metrics that may serve as alternatives to the selected metrics within any given measurement category in the current instance of the measurement model.
  • Determining any additional goals not stated in the measurement model but implied by current measurement collection, analysis, and use.
  • Noting metrics that do not support a goal and marking them as candidates for elimination.
  • Identifying any necessary changes to the (proposed) organizational model.
  • Identifying any necessary changes to the (proposed) process model for the contract.
  • Identifying any other requirements critical to the successful implementation of the measurement program.
