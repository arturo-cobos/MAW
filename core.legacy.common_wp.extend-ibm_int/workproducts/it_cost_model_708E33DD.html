Artifact: IT Cost Model (BUS 430)
The IT Cost Model work product captures the IT costs associated with a particular target (e.g. organization, project or scenario).
Domains: Business Value & Financials
Work Product Kinds: Model
Purpose

The IT Cost Model is used for one or more of the following:

  • Prove or disprove hypotheses

A study is initiated following identification of key issues that lead to the formulation of a set of hypotheses, with the aim of resolving those issues by recommendations based on the result of proving or disproving those hypotheses. The key point here is that before the IT Cost Model is constructed the hypotheses are known, this focuses the selection of data items, structure, formulas and assumptions only on that information required for the defined hypotheses. The IT Cost Model in this case is most often used for comparing variations across multiple client sites or with benchmark data.

For example: Support ratios are similar across different geographical sites

  • Open analysis

A study is initiated based on the need to identify opportunities for saving money. The client may not know the size of the cost savings opportunity (e.g. "how do I compare with my peers"). Data items, structure, formulas and assumptions are based on a set known to reveal potential cost savings. The IT Cost Model in this case is most often used for comparing variations across multiple client sites, variations with benchmark data and anomalies (what looks out of line with expectations) within the data collected and calculated. There is a danger that when this is the purpose superfluous and excessive data collection and analysis takes place.

For example: Given these costs and ratios where can savings be made

  • Scenario analysis

A study is initiated based on an identified client issue - from the outset, during the progress of an engagement or as a result of recommendations from an engagement - that has a number of possible variations. These variations may be a set number (e.g. compare technologies A, B and C) or open for modeling to an optimum result (e.g. identify the optimum number of Data Centers). The IT Cost Model in this case is most often used to model the differences between the scenarios.

For example: which of these scenarios provide the optimum costs?

During the course of a single study the cost model may be used for any or all of these reasons.

Description
Main Description

The IT Cost Model is typically (but not exclusively) required when considering:

  • Total Cost of Ownership
  • Server consolidation
  • Data Center consolidation
  • Technology rollout
  • Software standardization
  • Help Desk consolidation
  • Out-sourcing
  • Technology platform comparison
  • Automated tool implementation

This is a generic list, in real engagements the objectives, focus and scope determines the details of the model required, identifying the “from” and “to be”, states that will be modeled. For example:

Server Consolidation

  • The migration of 1200 NT servers to: an appropriate mix of 400 virtual servers on a centralised z-Linux server; 400 real servers of distributed xSeries infrastructure servers; and 400 centralised xClusters running VMware and Windows applications
  • The migration of 100 SQL database servers to clustered Windows xSeries
  • The elimination of a WebSphere NT middle tier by a z-Linux WebSphere partition
    The replacement of 100 Sybase database servers running Solaris by an HACMP 2 x 6way-pSeries solution

Technology platform comparison

  • The migration of several thousand Windows clients to Citrix client and Citrix servers
  • Comparison of an existing Solaris E10K multi-domain web trading application based on WebLogic and Oracle with a similar system based on z-Linux partitions and Hipersockets connection to a backend CICS/DB2 system
  • The replacement of existing Windows/Office desktop with Linux client
  • The migration of NT/IIS web servers to z-Linux/Apache
    Like all models the IT Cost Model is a simplification of reality, in this case the model is a combination of summaries of detailed data and calculations used to help answer pertinent questions relating to the type of study being performed. The IT Cost Model is normally a spreadsheet containing:

Data elements
Individual items that have associated numeric (financial or non-financial), they may be a summary of related individual items
(E.g. servers of a particular type)

Structure
Macro grouping of related data elements, often a separate spreadsheet page
(E.g. hardware volumes and costs)

Formulas
Calculations that provide insights supporting the study objectives
(E.g. software cost divided by number of users)

Assumptions
Decisions based on available (or lack of available) pertinent information
(E.g. costs are annual)


It is often the case that worksheets (questionnaires, data forms, etc.) are used as intermediary materials between the “raw” accounting data / reports and the IT Cost Model. These worksheets are normally formatted to match the content (and sometimes structure) of the IT Cost Model to enable easy collection and transfer of information. In many cases these worksheets are given to the client (either softcopy or hardcopy) for them to supply the required information.

The Data elements do not just include standard costs of assets and resources. They will include at least some of the following:

  • Costs in local currency
  • Currency exchange rates
  • Depreciation period
  • Resource volumes (infrastructure and human)
  • Process volumes (e.g. number of Help Desk calls)

The Structure groups related data elements to facilitate ease of reading and analysis. Data items can be viewed as a multi-level hierarchy, of which only the top levels are used in the cost model:

The formulas contained in the model perform the calculations required to provide the appropriate analysis, this is typically in the form of key ratios such as hardware cost per desktop (e.g. total of hardware costs data items divided by total number of hardware client desktops).

Note:

  • Process ratings are sometimes used in conjunction with cost data, these are normally separate to the IT Cost Model work product but could be included in the Service Performance if desired.
  • If the scope of the study includes Applications Development, this may be treated as a separate group in the structure above.
  • The distinction between Availability and Productively losses above is that the former is taken from availability statistics and the latter from User surveys. Clients will frequently not allow these factors to be included.

Formulas are contained within the structure and follow standard spreadsheet notation.

Assumptions are listed separately (normally on another worksheet within the spreadsheet)as text items.

Key Considerations
  • Skeleton cost models are normally built prior to data collection, the actual numbers collected are then added into the model as they are collected. The skeleton model also normally contains an initial set of formula that is then refined as the data is populated.
  • The more the “to be” state is understood the less likely that fundamental changes to the cost model and additional, iterative, data collection will be required.
  • Process related data is often collected using the definitions and numbers based on the IBM IT Process Model, this includes high level and maturity level assessment tools.
  • Cost models are often used in organizationally sensitive situation (someone is trying to sell something and / or individuals or groups of people will be disadvantaged). Obtaining “true” figures to populate the model is often difficult and time consuming, the higher the level of experience and comparison / validation sources the greater chance of using figures that can be “defended” and accepted.
