Artifact: Infrastructure Cost Plan (ARC 008)
The Infrastructure Cost Plan work product is a budgetary costing for technical infrastructure.
Domains: Infrastructure
Work Product Kinds: Plan
Purpose
The Infrastructure Cost Plan is used to estimate the capital expenditures and on-going expenses to build the technical infrastructure.
Description
Main Description

The Infrastructure Cost Plan is a budgetary costing for:

  1. Hardware/software needed to run the package 
  2. Package software     
  3. Third party hardware/software tools   
  4. Desktop hardware and software 
  5. Communications equipment   
  6. Network carrier charges    
  7. Maintenance for items 1 through 6    
  8. Cost of additional staff   
  9. Training expenses
  10. Contractor expenses    
        

At the client’s discretion, internal recovery rate can be included to reflect cost of internal client staff that must be assigned to the project.

Key Considerations

Client consensus concerning the costs in the model is a must.  Make sure there are clearly documented assumptions, where applicable, behind costs in model.

All costs should be obtained through the client, not as an agent for the client. 

Assume an iterative approach to business gap analysis and prototyping of the systems.  

The requirements for bolt-on packages, buffer systems, interfaces and the detail transaction and data required for sizing the package evolve throughout the project life cycle. 

The costs are a point in time estimate; the baseline architecture and assumptions should be included in the work product. 

Setting the correct customer expectation about the iterative nature of package-enabled projects should be done.
Tailoring
Impact of not having

Without this work product:

  • The financial impact to the project due to IT costs are not validated or properly reflected in the customer’s business case for the selection
  • The full cost for implementing the package or system will not be identified
  • The project may fail because the overall budget is understated
  • IT management will not have a representative budget to manage
Reasons for not needing
  • The client is simply changing out one application for another and feels confident of his cost estimate.
  • The client has significant over-capacity and feels there will be no need for additional servers not network components.
  • There is an open-ended budget and cost is not a factor.
