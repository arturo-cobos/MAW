A production schedule is a listing of the client locations, including location details such as contact numbers, address, hours of operation, etc, that are sorted in an order according to a clientâ€™s deployment preferences.
Domains: Deployment
Work Product Kinds: Plan
Purpose
The primary purpose of the production schedule is to effectively communicate the sequence of the deployment to all of the parties involved with the project.
Description
Main DescriptionA production schedule indicates when segmented services are to be completed by location. This schedule is generally stored in a spreadsheet, or on a web based deployment tool such as eDeploy, which can be exported to a spreadsheet. It is used to make sure that the participants understand the schedule to deploy services.
Brief Outline

The general notation is contained in a spreadsheet with a header that allows for a long list of deployment locations. Common fields include:

  • Location number
  • Street Address
  • City
  • State
  • Zip Code
  • Geography
  • Site Number
  • Contact Name
  • Contact Phone Number
  • Fax Number
  • Hours of Operation of Site
  • IBM Customer Number
  • Site Survey performed on/reviewed on/complete date
  • Cabling begin/end date
  • Fields for type and quantity of cable drops
  • Hardware arrival/ installation begin/ installation complete dates
  • Name of installer

Key Considerations

The following should be considered: 

  • Whoever is going to manage the spreadsheet should have the skills (or should learn the skills) to use a spreadsheet. Macros and advanced spreadsheet manipulation will speed results. 
  •  Use the track changes option in the spreadsheet application, and unique file names, colors to indicate change, etc. 
  • Communicate production schedule changes quickly.
Tailoring
Impact of not having
  • Changes to the schedule become unmanageable.
  • Coordination becomes unmanageable.
Reasons for not needingIf the deployment contains a small number of locations, less than 10, then the production schedule is not as important but still could be used.
