Artifact: Phase Transition Package (ENG 013)
The Phase Transition Package provides a single location for all final versions of the work products and deliverables. In addition, each work product is assessed at this time to determine its suitability for inclusion in the Intellectual Capital Management system.
Domains: Project Management
Work Product Kinds: Inventory
Purpose

The Phase Transition Package work product is developed to facilitate a smooth, complete transition from one project phase to the next.  It supports the following tasks:

  • Consolidation of all the deliverables from the current project phase.
  • Identification of the deliverables that are candidates for submission to the Intellectual Capital Management database.
  • Confirmation of the client’s commitment to the next phase of the project.
  • Confirmation of the deliverables needed to support completion of the next phase of the project.
  • Identification of resources needed for the next phase of the project.
Description
Brief Outline

 This artifact typically has the following sections:

  • Phase - The Phase of the Project Contains the work product
  • Title - The Title of the work product
  • Work Product Number - The work product number
  • Project Plan WBS Number - The WBS Number Associated with this work product in the Project Plan
  • Primary Team Member Responsible - The "owner" of the work product
  • Deliverable Cross Reference - If this work product is a part of a contract deliverable, note the deliverable name here
  • Hard Copy Location - Where the master hard copy of the document can be found
  • Soft Copy Location - Where the soft copy of the document can be found
  • Supporting Documentation Location - Where any supporting documentation can be found
  • Candidate for ICM submission - If this is a candidate for ICM submission, mark with an X
  • Date Submitted to ICM - Date the soft copy was submitted to ICM
  • Reasons for Submission to ICM - Why it was submitted to ICM

Key Considerations

Many work products created in one phase are used as input to successive phases.  It is the Project Manager’s responsibility to ensure that all work products created in the current phase are complete and meet the Quality Assurance standards as set forth in the Project Control Book.  In addition, there may be contractual deliverables, which may be composed of multiple work products that mark the completion of the phase.

The Project Manager also verifies the client’s commitment to the next phase of the project, and confirms the set of deliverables required to complete the work associated with the next phase of the project.

At the end of each Phase, the Project Manager also must determine the resources needed to support the next Phase of the project based upon the client’s commitment and the deliverables required.

When assessing a work product for submission to the Intellectual Capital Management System, the following criteria should be used as a guideline:

  • Did the team improve upon the approach suggested in the narrative of the work product?
  • Did the team improve the templates associated with the work product?
  • Did the team create examples that are unique to the work product, such as industry specific examples?
  • Did the team encounter any unusual circumstances that affected the estimates associated with the work product?
Tailoring
Impact of not havingThere may be confusion and wasted effort trying to locate the work products needed during the subsequent phases, which will impact the project schedule and project budget. In addition, if the client is not fully committed to the next phase of the project, work on subsequent phases may commence without the appropriate financial and resource documents in place. Finally, if it is unclear what the deliverables in the next phase are, it will be impossible to accurately plan the next phase of the project.
Reasons for not needingThere are no reasons for not developing this work product on a project.
Representation OptionsThe Phase Transition Package uses a Phase Transition Worksheet, which is an electronic spreadsheet.
