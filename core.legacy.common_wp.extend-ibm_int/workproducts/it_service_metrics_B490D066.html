Artifact: IT Service Metrics (OPS 404)
The IT Service Metrics work product provides detailed documentation of the monitoring metrics associated with the IT resources required to deliver a specific IT business service.
Domains: IT Management & Monitoring
Work Product Kinds: Specification
Purpose
The IT Service Metrics is a design document. All metrics relevant to effective monitoring of a specific IT business service are documented within this work product.
Description
Main DescriptionThe IT Service Metrics work product documents all monitoring metrics associated with IT components (e.g., servers, network appliances, bandwidth providers), elements (e.g., server hardware, operating systems, middleware, applications, databases) and sub-elements (e.g., server RAM, application processes, database tables).
Brief Outline

The IT Service Metrics work product contains at minimum:

  • One or more Service Elements (SEs), each associated with an IT Component, IT Element, or IT Sub-element.
  • Each Service Element has associated with it the following attributes:
    • Service Element Name (SEName)
    • IT Component Name (ComponentName)
    • IT Element Name (ElementName)
    • IT Sub-element Name (SubElementName)
    • Service Element ID (SEID)
    • Service Element Description (SEDescription)
    • Service Element Schedule Name (SEScheduleName)
    • Service Element Measurement Source (SEMeasurementSource)
    • Service Element Metric (SEMetric)
    • Service Element Interval (SEInterval)
    • Service Element Interval Type (SEIntervalType)
    • Service Element Metric Threshold (SEMetricThreshold )
    • Service Element Event Source Name (SEEventSourceName)

Key Considerations

IT Specialists, IT Architects, and Consultants would normally use IT Service Metrics as a tool for analysis and documentation of IT service monitoring requirements and monitoring software implementation specifications.  However, the concept of service monitoring metrics may be applied to services that exist outside of the IT environment.  For example, service monitoring metrics may be used to monitor performance of business processes, independent of the IT applications that may be implemented to support these processes.  As the client’s management focus shifts from management of the IT infrastructure to management of end-to-end services and business processes service metrics may be utilized to address yet other purposes.

The metrics data as defined in the Notation section maps directly to schema supported within the Tivoli data warehousing data model.
Tailoring
Impact of not havingThis work product provides the specifications required for effective implementation of the client’s IT business service event management and monitoring solution. Without this work product the client’s systems and network management software may not be properly customized to meet the client’s business and technical monitoring objectives.
Reasons for not needingIf the client provides service offering documentation that establishes an equivalent level of documentation of proposed monitors and monitoring parameter then this work product would not be required.
