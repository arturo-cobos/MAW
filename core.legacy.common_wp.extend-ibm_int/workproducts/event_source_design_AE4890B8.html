Artifact: Event Source Design (OPS 402)
An event source is an IT component, element or sub-element upon which the delivery of a specific IT business service depends.
Domains: IT Management & Monitoring
Work Product Kinds: Specification
Purpose
The Event Source Design is a design document. Each Event Source Design work product documents the complete repertoire of events that may potentially be emitted by an event source and the detailed processing requirements for each class of event, including event filtering, thresholding, forwarding, correlation and actions. IBM Global Services personnel use these specifications when customizing IT resource implementation parameters and event management software.
Description
Main Description

The Event Source Design artifact documents for an individual event source:

  • The event source event repertoire and
  • The event filtering, forwarding, correlation, and automation specifications for each event within the event repertoire.

Examples of event sources would include (components) servers, network appliances, bandwidth providers, (elements) server hardware, operating systems, middleware, applications, databases and (sub-elements) server RAM, application processes, and database tables. Any IT technology which has the capability of logging messages or emitting standard event protocols such as Simple Network Management Protocol (SNMP) traps is an event source. Technologies which do not support this capability are not considered event sources.

Key Considerations
IT Specialists, IT Architects, and Consultants would normally use Event Source Designs as a tool for analysis and documentation of IT event processing requirements.  However, the content of Event Source Designs may be applied to event processing requirements that exist outside of the IT environment.  For example, Event Source Designs may be used to indicate processing requirements across non-IT business events issued by business processes, independent of the IT applications which may be implemented to support these processes.  As the clientâ€™s management focus shifts from management of the IT infrastructure to management of end-to-end services and business processes Event Source Designs may be utilized to address yet other classes of events and other event correlation applications.
Tailoring
Impact of not havingAll event management design and implementation projects require detailed documentation of the event repertoires associated with the event sources addressed by the projects. In addition these projects must develop detailed documentation of the event processing required for each event within the repertoire. No other standard exists for documentation of event repertoires and event processing specifications. Not utilizing this work product would result in incomplete and ineffective event processing documentation. When implemented the event processing software is unlikely to deliver the correlation results required by the client.
Reasons for not needingThe project does not include event management design and/or implementation.
