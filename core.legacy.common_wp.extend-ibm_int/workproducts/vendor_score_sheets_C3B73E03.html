Artifact: Vendor Score Sheets (BUS 072)
The Vendor Score Sheet is used by the client evaluators to score the vendor's software demonstration for each item defined in the demonstration scripts.
Domains: Selection
Work Product Kinds: Measurement
Purpose

The purpose of the Vendor Score Sheet is to provide the client evaluator with a scoring template to be used while viewing the vendor's software demonstration.   The score sheet facilitates a structured step-by-step flow through the demonstration.  The evaluator uses the template to follow along with each demonstration item being presented to understand the function being shown and then provide a score as to how well the demonstration showed the functionality.  


Description
Main Description

The score sheet lists the evaluators that are to participate in a given area. It is important to track the participation of these evaluators so that the same evaluators will score all vendors. The same evaluators must participate to maintain an objective score. The participants are listed to provide an easy way to track the required evaluators.

The time allotted for each functional area is listed on the score sheet. To obtain objective data, each vendor must have the same time to demonstrate their product. The suggested time is listed so the vendor, with the facilitator's help, can keep the demonstration on schedule.

The sequential numbering system for process area and functional area is included to help the vendor and evaluators identify scripted areas being shown. The vendor will use these numbers as a reference, so the evaluators will know what function is being demonstrated.

The item description identifies the function that is being scored. These descriptions are created from the user's requirements and the evaluator's score should reflect how the demonstrated solution supports the requirements.

Once the functional demonstrations are completed, the scores for each area are averaged to get a subtotal score for each functional area.  The functional area scores are then averaged to arrive at a total score for the demonstration.  These scores will then be used for the summary of the demonstrations and as input into the final recommendation.

The evaluation criteria are listed to educate the evaluators on the scoring metrics that will be used to evaluate the software solutions. The evaluator will record a score for each listed demonstration item using the following scale:

Brief Outline

The Vendor Score Sheet is a document that will usually have tables included to create an easy to follow scoring sheet.  Tables would be used to present the schedule and the score sheet itself. 

The components of the score sheet work product are as follows:

  • Title Page
  • Table of Contents
  • Vendor Demonstration Schedule
  • Evaluation Criteria
  • Functional Area Scoring Sheet

The functional area scoring sheet is the actual sheet that the evaluator records their score for each scripted demonstration item. The layout of the sheet is as follows:

  • Functional Area Title - Name of Functional Area
  • Attendees - List of attendees who will participate in this scoring section
  • Allocated Time - Amount of time for this demonstration area
  • Requirement Number - Sequential number of scripted item for a process area
  • Sequence Number - Sequential number of scripted item for this functional area
  • Item Description - Description of scripted item to demonstrate
  • Score - Area where evaluators records their score
  • Comments - Space for comments

The demonstration schedule is included in the work product as a reference for both the evaluator and the vendor. Its purpose is to remind both groups of the amount of time allotted for each demonstration area. Usually, there is less time than there is function to demonstrate. Consequently, the schedule is included so all are aware of the amount of time that the vendor has to demonstrate their capabilities in a functional area.

Key Considerations

It is important that the client remember that the IBM consultant is a facilitator and that the client team developed the information that is being included in the document and represents the client's requirements.

The facilitator has the responsibility of keeping the demonstration on schedule and ensuring that each vendor has the same time to demonstrate the solution.  The facilitator also plays the role of keeping the group focused on the items that are listed in the script.  If the vendor finishes in less time than allotted, then they are free to show additional function.  However, any additional function shown will not be scored.

Tailoring
Impact of not havingThe Vendor Scoring Sheet should always be used. If software demonstrations are being conducted according to scripted requirements, then the sheet is necessary as a guide to structure the demonstration and to record the evaluator's scores.
Reasons for not needingThere is no reason to not use this work product.
