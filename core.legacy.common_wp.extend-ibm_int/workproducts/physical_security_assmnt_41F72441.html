Artifact: Physical Security Assessment (OPS 312)
The Physical Security Assessment work product defines and documents gaps against best practices and client’s business capabilities found in the client’s facility security controls and processes at a single site or throughout an enterprise.
Domains: Security & Privacy
Work Product Kinds: Assessment
Purpose
For maximum protection, information security controls rely on physical security controls. This work product helps clients achieve a secure working environment for employees and other persons working at or visiting corporate facilities and helps establish processes to ensure the protection of intellectual assets and protection of all site facilities.
Description
Brief Outline

The Physical Security Assessment work product could be a text style or presentation style report.  Sections typically found are:

  • Policy
  • Sensitive Interior Area Controls
  • External Site Controls
  • Data Center Controls
  • Organization
  • Physical Asset Controls
  • Personnel
  • Contract Management
  • Incident Management
  • Emergency Planning

Key Considerations

This work should only be performed by Methodology-trained Security professionals.  Additional resources available in the IT Security ICAP Database.

This work product is validated against the client’s enterprise level security policies and security standards, established IBM procedures and guidelines and a standards-based “best practices” baseline using BS7799.
Tailoring
Impact of not having

The Physical Security Assessment reveals existing or potential security exposures in an organization’s physical environment. Insiders or outsiders who get through existing controls may exploit these exposures unnecessarily jeopardizing valuable information assets. The assessment recommendations allow an organization to reduce risks to an acceptable level based on business needs.   

This work product is most appropriate for major new or re-engineering infrastructure projects and on projects where client facilities are housing particularly sensitive applications such as Certification Authority Services, Financial Clearing, hosted Internet Application Services, Health Information Warehouse services, etc.

Reasons for not needing
  • Client has recent audit results and, thus, is aware of their physical security risks.
  • When physical security is out of scope of the engagement as is typically true on most IT engagements. For these cases, see the Information Security Assessment work product, which includes physical security considerations focused on protecting IT assets.
