Artifact: IT Service Description (OPS 403)
The IT Service Description work product documents the key business and technology attributes of an IT business service.
Domains: IT Management & Monitoring
Work Product Kinds: Inventory
Purpose
The IT Service Description is a design document. The IT Service Description work product provides the documentation base for the design and implementation phases of a systems management service monitoring project. This work product provides concise documentation of the service level objectives to be addressed by the project and the dependencies of the service upon the IT infrastructure. All IT resources that are relevant to effective monitoring of a specific IT business service are documented within this work product. This base documentation enables subsequent project phases that address detail design and implementation activities.
Description
Main DescriptionIn addition to documenting the business attributes of an IT service the IT Service Description work product provides detailed documentation of the IT resources required to deliver a specific IT business service. It documents all service dependencies upon IT components (e.g., servers, network appliances, bandwidth providers), IT elements (e.g., server hardware, operating systems, middleware, applications, and databases) and IT sub-elements (e.g., server RAM, application processes, and database tables).
Brief Outline

The business attributes within an IT Service Description include, at minimum:

  • IT Service Name
  • IT Service Description
  • IT Service Type
  • IT Service Owner Name
  • IT Service Owner Code
  • IT Service Schedule Name
  • Service Level Requirements
  • Identified Issues
  • Location of Further IT Service Documentation

The technology dependency attributes within an IT Service Description include, at minimum:

  • One or more IT Components.
  • Zero or more IT Elements, each associated with a IT Component entry.
  • And zero or more IT Sub-elements, each associated with an IT Element entry.

Key Considerations
IT Specialists, IT Architects, and Consultants will use the IT Service Description as a tool for analysis and documentation of IT service monitoring requirements.  However, the IT Service Description will be of value to any project requiring documentation of the business and technical characteristics of an IT service, such as projects addressing service level agreement development, service level management, and service-based management of IT assets, hardware and software configurations and IT resource inventories.
Tailoring
Impact of not havingThe objective of an event management and monitoring design project is to deliver to the client solutions for the effective monitoring of IT business services. Without a clear definition of the service to be monitored and the service objectives associated with the offering the engagement will not succeed in delivering an effective monitoring solution.
Reasons for not needingIf the client provides IT service information that establishes an equivalent level of documentation of service objectives, attributes and infrastructure dependencies this work product would not be required.
