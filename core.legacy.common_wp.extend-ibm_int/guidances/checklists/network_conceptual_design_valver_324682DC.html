Design validated against the network requirements
The Network Conceptual Design must be validated against primarily the network strategy and network requirements.  Key validation points are also the profiles (user, location, application, current network, industry networking, networking regulatory) and other data elements collected during the assessment or data collection phases.  Additional validation points are the client standards, principles, policies, and guidelines.  The designer will only know if the Network Conceptual Design is meeting the proper criteria if all of these elements are reviewed before the design is started, as it is being constructed, and after it is completed. 
Design is constructed in alignment with the networking strategy
It obviously does not make sense to create a Network Design that either can’t or won’t be implemented in the client’s environment.
Design reviewed by third party
It is also very helpful to have a second set of eyes both work with the creation of and review of the Network Conceptual Design.  Initially this assistance should come from the IBM network consulting team (i.e., another network specialist or architect on the team or even a non-team member). It is also appropriate to review the work with the client network design/architecture team to make sure no major gaps or flaws exist that were not initially obvious to the IBM engagement team.
