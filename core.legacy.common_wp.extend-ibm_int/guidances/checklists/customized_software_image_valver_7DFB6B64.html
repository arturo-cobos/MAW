Checklist: Customized Software Image Validation and Verification
Relationships
Related Elements
  • Customized Software Image
Main Description

The image that is delivered by the client (regardless of the medium) should be a client tested and approved image. However, in order to avoid problems during integration or rollout, IBM should test the image as well.  It is unlikely that all possible issues can be discovered and corrected, but IBM will be able to validate certain aspects of the image, which will minimize the risk associated with deploying this image.  It is customary to establish a test lab that mirrors the client site and hardware configurations.  IBM should identify the key issues, test for them, and have the customer sign off on the issues and the test.  Establishing testing criteria is a separate activity, but some basic considerations for validation are included here.
Check Items
Validated successful delivery mechanism
Are CDs bootable or is a floppy boot disk required; can server files be copied successfully; does image size, configuration, or delivery mechanism negatively impact the time planned for deployment.
Image can be deployed at the customer site as well as the implementation center
If image is server based, can equipment be configured at the site should a problem occur?
Images are included for all of the affected rollout equipment and all of the equipment configurations
Machine boots up without errors
Includes proper operating system version that includes updates and fixes
Machines have required access to the internet and client intranet
Proper versions of application software included
No adjustments are necessary after imaging for items that could have been included on the custom image
