DFDs are a useful size

An exploded process should fit on a one-page DFD. (The top-level DFD should be also on one page.)

At most seven processes can be described in one DFD.

A data element or a generic name or representative name is used to specify a data flow name

When a generic name or representative name is used, an ID and a generic name or representative name is given and data elements are separately described in data flow descriptions.

Things or actions must not be shown as data flow names.

Process names are clear expression in the form "verb object"

Example of verbs: create, extract, store, replace, record, verify

The ID is shown in the upper section of the process. 

In the case of explosion of a process box, a process identification number is a concatenation of the identification number of the high-level process box with a local identifier.

Information such as department, person in charge, or system name, is entered in the lower section of the process (in the case of a physical model).

Data flows are consistent

With respect to the I/O data flows, there must be consistency between a high-level DFD and its exploded low-level DFD at the data element level.

There must be consistency between the data elements in the data store and the data elements in the I/O data flows to the data store. 

 The data elements of an output data flow are derived from the data store or the data elements of the input data flow.

Processes and data stores must each have at least one input data flow and one output data flow
Data stores must not be connected directly to each other by a data flow
An external entity must not be directly connected by a data flow
