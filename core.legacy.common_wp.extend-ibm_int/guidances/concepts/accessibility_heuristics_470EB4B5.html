Concept: Accessibility Heuristics
Nine accessibility heuristics extracted from paper published by Claire Paddison.
Relationships
Related Elements
  • Early Usability Evaluation
Main Description
  1. Provide meaningful and relevant alternatives to non-text elements.
  2. Support consistent and correctly tagged navigation.
  3. Allow complete and efficient keyboard usage.
  4. Respect users’ browser settings.
  5. Ensure appropriate use of standard and proprietary controls.
  6. Do not rely on color alone to code and distinguish.
  7. Allow users control of potential distractions.
  8. Allow users to understand and control time restraints.
  9. Make certain the Web site‘s content is compatible with assistive technologies.

Need to make a distinction between technical accessibility and usable accessibility. The point is that a heuristic will provide an insight on usable accessibility and give an indication of the rework needed to meet technical accessibility. In this case, the technical assessment can only be conducted using assessment tools and techniques.
