Guideline: Developing Enterprise Technology Framework
Relationships
Related Elements
  • Enterprise Technology Framework
Main Description

Dependent on the time available, several different approaches can be used to define the content of this work product. These are documented in various technique papers – see the end of this section.

The approach outlined here has been used and developed over several years, based on experience in client engagements. The level of detail completed for each activity and task will depend on the scope of the engagement and the amount of time available. You should always use one or more experienced professionals, preferably trained in developing and defining technology frameworks.

1. Develop Conceptual Model – base level

  • Brief client team on activity objectives and approach
  • Define Level 0 Conceptual Model
    • Review other architecture models (using IC)
    • Review current business and IT environment summary
    • Understand future system requirements (determine IT functionality requirements and prioritize)
    • Develop conceptual model
  • Define Level 1 Conceptual Model
    • Identify sources of level 1 ABBs (using IC)
    • Define level 1 ABBs
    • Document Enterprise Technology Framework - conceptual level
  • Define Conceptual User Group Views
    • Define matrix of ABBs to User Groups
    • Walkthrough business scenarios
    • Create graphical representations of views
  • Detail Definitions of ABBs
    • Develop an ABB Documentation Template
    • Document ABBs (using IC)

2. Develop Logical Functional level

  • Define Enterprise Sharing Matrix
    • Agree scope and structure
    • Agree notation
    • Populate matrix
  • Define User Group Sharing Matrices
  • Define Logical Functional Environments (LFEs)
    • Group ABBs into LFEs
  • Define Logical Location Matrices
    • Define ABB/User Group/location sharing matrices
    • Define consolidated logical location summary
  • Develop Graphical representations of views
  • Define Additional Logical Views (optional)
  • Build overall Logical Functional level of the Conceptual Model
    • Amalgamate LFE's and logical location views
    • Document Logical Functional level of the Conceptual Model
  • Refine low level ABBs and update Conceptual Model
    • Validate ABB definitions
