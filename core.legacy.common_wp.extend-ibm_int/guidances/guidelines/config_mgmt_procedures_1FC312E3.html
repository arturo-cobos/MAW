Guideline: Developing Configuration Management Procedures
Migrated development approach from existing IBM Global Services Method work product.
Relationships
Related Elements
  • Configuration Management Plan
Main Description
The following development approach is recommended:
  • Develop a Configuration Management Procedure that satisfies the size and scope of the project.
  • Identify configuration management tools to be used.
  • Identify tracking and reporting requirements, e.g., problem reporting, corrective action (action item) tracking system.
  • Identify and implement subcontractor configuration management requirements if any subcontractor software or hardware is used.
  • Identify organizational responsibilities and required resources to accomplish the activities required by the Configuration Management Plan.
  • Develop initial schedules for tasks associated with all configuration management activities that correlate to the Project Plan.
  • Implement a plan to define a configuration management process suitable for your development project.
  • Generate the appropriate Configuration Management Plan or documentation to explain how you plan to manage configuration management.
  • Identify those configuration items on your project for which you think configuration control is necessary. Note that all customer work products should be under configuration control.
  • Identify configuration item maintenance procedures.
  • Identify change control requirements and implementation procedures for engineering changes, enhancements, and remedies.
  • Identify reporting requirements for baseline configuration items.
  • Identify audit requirements.
  • Establish a shipping/delivery methodology.
  • Establish media retention, historical archive requirements, and the physical location for storage.
