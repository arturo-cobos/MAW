Main Description
  • Capability Descriptions are best captured as a by-product of a capability model assessment or design effort.
  • Once the current or future capabilities have been identified and organized into a Strategic Capability Network, the development of the first draft of the capability descriptions can be done using the outputs of workshops or interviews where most of the capability attributes have been discussed with the client. Preferably, the Capability Description template should be used to structure the workshop or interview questions and to record the information, preferably as a table definition in a relational database, possibly in a Notes database, or, at minimum, as a word processing document.
  • Any additional attributes that have not been captured can be gathered through a follow-up interview process with client representatives.
  • The draft descriptions should be produced as a report and reviewed with the client before producing the final set of descriptions as part of a client deliverable.
