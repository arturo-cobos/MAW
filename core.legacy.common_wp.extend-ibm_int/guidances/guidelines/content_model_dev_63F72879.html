Guideline: Developing Content Model

Main Description
  1. Define the audience characteristics and tasks to be performed on the site. Prioritize the tasks.
  2. Identify the content required to complete the user’s tasks.
  3. Logically group the content (based on the user tasks) to create the hierarchy with the goal of delivering the content to complete the task all in 1 location and within 3 levels.
  4. For each major type (proposal inserts, collateral, shopping, etc.) create a page layout (map) to identify and document the content (defined in Step 2) for that page. This layout should include all content necessary to complete a given task. To ensure consistency, standard templates should be defined for all page types that include the outline of the information required. This will make it easier to extract the information from multiple sources, populate the outline and provide a consistent delivery to the user.
  5. Validate the hierarchy and maps with the given user set. Rework as necessary.
  6. Create a content matrix identifying each specific content element, its source (possibly multiple sources), special review, technology/metadata , timing requirements (daily, weekly, etc.), etc.
  7. Create the ‘system view’ of how the content will be stored and accessed.
  8. Identify the specific functional/application requirements needed to support the site. Examples include search, shopping basket, stock ticker, banner advertising, etc.
  9. Define the high level technology requirements to create and maintain the site. Examples include the use of XML, access to legacy systems, specific software required based on the client’s existing systems, etc.
