Guideline: Developing IT Management Requirements
Migrated development approach from existing IBM Global Services Method work product.
Relationships
Related Elements
  • IT Management Requirements
Main Description

The approach can be broken down into a series of activities:
  • Identify the in scope IT Services.
  • Map the IT management processes to the IT Services.
  • Identify gaps between the desired future state, and the existing IT management environment.
  • Define the process requirements.

- Produce process requirements template.

- Collect process requirements templates.
- Validate/clean up the requirements templates.

- Define process requirements.

  • Define the tool requirements.

- Identify sub-processes where requirements are element-specific:
- Produce tool requirements template.

- Collect tool requirements templates.
- Validate/clean up the requirements templates.

- Define tool requirements by sub-process and element.

- Define "boilerplate" requirements for element-independent sub-processes (A "Boilerplate" set of requirements is that set of tool requirements common across all technology).
- Define a "boilerplate" set of requirements to support performance of the systems management sub-processes on any of the managed elements.

- Validate and document the tool requirements.
- Map the requirements to generic tools.

- Review Sample Generic Tools List and Glossary.
- Customize/Document Generic Tool List and Glossary.

  • Define the data requirements

- Identify sub-processes where data requirements are element-specific:
- Produce data requirements template.


- Collect data requirements templates.
- Validate/clean up the requirements templates.

- Define data requirements by sub-process and element.

- Define "boilerplate" requirements for element-independent sub-processes (A "Boilerplate" set of requirements is that set of data requirements common across all technology).

- Validate and document the data requirements.

  • Define the organization requirements

- Identify sub-processes where organization requirements are element-specific:
- Produce organization requirements template.

- Collect organization requirements templates.
- Validate/clean up the requirements templates.

- Define organization requirements by sub-process and element.

- Review Sample Generic Roles List and Glossary.
- Customize/Document Generic Roles List and Glossary.

- Validate and document the organization requirements.

  • Document/Consolidate Final Requirements.
