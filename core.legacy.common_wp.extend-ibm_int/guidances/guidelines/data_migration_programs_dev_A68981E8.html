Guideline: Developing Data Migration Programs
Migrated development approach from existing IBM Global Services Method work product.

Main Description

Place the programs under change control as early as possible in the project’s life. Ensure that the specifications and the implementation, including test cases, remain synchronized through strict version control in the build process. Use a source code repository that supports versioning to assist with the management of the constant changes to the code files. Differences between versions of a source file can be captured without maintaining multiple physical version files of the same source file. A versioning tool that also supports release information is most desirable. A release can be frozen when the interface programs are shipped. This allows the development team to respond to customer problems by testing on a known code base.

Document the programs carefully. Place the documents under change control and make them accessible to the whole project team. This can conveniently be done within the version management control system for the project. This will assist in rectifying errors found in testing and provide a solid basis for the approval of the programs by the client.

Data Migration Programs may be created from source by using:

  • Text editors
  • Design tools or languages such as the Interface Definition Language (IDL). The output of the tools may be augmented by source written using a text editor
  • Visual programming techniques that generate source from graphic and text representations of the interface
  • DBMS tools and utilities
  • Structured Query Language (SQL) tools

Check the source system to determine if it has programs already installed that could be used to migrate the data from the source system or if other standard programs already exist. Investigate if these programs will actually migrate the data correctly. They should be treated like those that require development and should be tested in conjunction with the client’s application as a whole.

Intermediary files may be required during the transfer of information from one program to another, particularly if the programs reside on different platforms. The format of these files should conform to the specifications governing the project’s programming standards.

Perform the migration according to plans agreed with the client.
