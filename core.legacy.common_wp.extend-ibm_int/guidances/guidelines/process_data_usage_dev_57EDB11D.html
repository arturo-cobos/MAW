Main Description
At the higher level, Process/Data Usage may be created in a workshop mode, using a core team of business users and application developers â€“ people who understand the business and the current and future proposed systems. Activities are usually listed on the vertical axis and the information classes on the horizontal axis.
At the more detailed level:
  • Create a data repository, indexed by process name and entity name. The fields should minimally contain allowance for storing information on Creation, Read, Update, and Deletion of the data.
  • Typically, entries are maintained as the function and data models are developed. If this is not done, an audit of the models must be performed. In the event of an audit, entity names may suggest which processes interact with them and vice versa.
  • For each existing data and process relationship, characterize the nature of the relationship as being either Create, Read, Update, or Delete.
  • As the differences between the function and data models are reconciled, changes to both are likely to occur. Hence, iteration in function and data model development will occur.
