Guideline: Developing Report Specifications
Migrated development approach from existing IBM Global Services Method work product.

Main Description
1. Identify the purpose of each report intended to be included in the client’s application.  Determine the data and format requirements.  Write the detailed specifications for each report.  The project leader or a senior member of the development team may do these activities.  The client should be involved to assist with interpretation and clarification of each element of the intended application.  The client can also provide insight into an existing application and the data that must be used in the new application.  These items may have significant bearing in the reports to be developed.  The project manager or leader may use joint client/development team review meetings to define these details.
2. The specification of the each report may be iterative as the design of the client’s application progresses. During each iteration, the report specifications are elaborated until the application is documented sufficiently for implementation.  As each elaboration progresses, it is essential to review the client’s intended application as a whole to ensure that sum of the parts will have the intended purpose of the whole.  Design walk-throughs will assist in identifying deficiencies in the reports.  The deficiencies should be remedied with the entire application in mind.
3. Consult the client to solicit process documentation that may assist in the definition of the reports, particularly if data is to be extracted from external systems.  Application requirements documents and use cases may provide additional information.
4. Document the testing considerations so unit testing may be specified and executed.
5. Document the security requirements for each interface such as who or what program may use each report and under what conditions.  This includes but is not limited to privacy, encryption and password requirements.
6. Document each report concisely and precisely using the notation noted above as a guide.
