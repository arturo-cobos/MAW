Main Description
Upon receipt of the job, the developer must assess the impact that the problem is having on the system in terms of availability and data integrity. This should include:
  • Initial assessment of the current environment and the impact of the problem on the environment, including:
    • The infrastructure, hardware, operating systems, datastores etc
    • Datafile
    • Copybooks
    • Database views
    • Tables
    • Screens/Maps
    • Letters/Documents
    • Reports
    • Programs/Modules/Subroutines
    • Load modules
    • Sorts/Utilities
    • Batch Jobs
    • PC Menus

  • The interface between other applications or systems, including the human interface;
  • Delivery or support infrastructure, including Help Desks;

The Emergency Fix Analysis then provides the basis for the design of the emergency fix and as a measure against which it can be tested.
It is then used when performing follow-on action for reviewing emergency fix documentation to give the developer a greater insight into the nature of the problem and all the areas impacted by it.
