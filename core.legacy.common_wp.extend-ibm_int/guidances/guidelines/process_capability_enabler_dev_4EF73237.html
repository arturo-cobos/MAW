Guideline: Developing Process Capability Enabler Definition

Main Description

The following approach may be used to define and document the Process enabling requirements for the Process capabilities identified to be in scope:

  1. Start with collecting the process capability definitions that identify and describe the process capabilities and the key process design points that are necessary to support the business capabilities.
  2. Review and carefully analyze the process capability definitions and the results of prior work that defines the current status with core team to ensure common understanding of process capabilities and key process design points.
  3. In a workshop environment with the core team and key subject matter experts, identify and document the process enabling requirements for each process capability.  Each enabling requirement must be identified as to source, this means is it realized (available) or specified (needs to be developed.  This will be iterative until all process capabilities have been completed.
  4. For each process enabling requirement, the gap between realized and required must be identified, this will allow for an assessment of effort/cost required to provide the enabling capability.
  5. The priority and benefits of each enabling capability must also be identified and documented.
  6. The general comment section should be used to provide additional details about the overall assumptions and any issues or risks that may have to be dealt with to be able to provide the process capability.
