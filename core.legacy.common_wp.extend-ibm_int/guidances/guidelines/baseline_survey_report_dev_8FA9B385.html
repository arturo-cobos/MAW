Guideline: Developing Baseline Survey Report
Relationships
Related Elements
  • Baseline Survey Report
Main Description

The following is a guideline for obtaining the necessary information to be included in the Baseline Survey Report work product. Based on the scope of the engagement, only some of the steps may need to be completed.

  • Identify all existing business, process, application and technical documentation.
  • Identify all existing system components (hardware, software, network, etc.).
  • Identify gaps between business and process linkages.
  • Identify existing IT Standards (version and date) relative to required system documentation .Where appropriate, compare existing documentation to relevant requirements for its use, to requirements set by the IT Standards or an external body and highlight any gaps.
  • Compare existing process or system documentation relative to the requirements of the user and support organization and highlight any gaps.
  • Compare existing application functions to the available supporting system documentation and highlight any potential discrepancies.
  • Assess whether the documentation will be comprehensible to project team members, given their level of knowledge about the business, current application system or technical environment.
  • Identify any other relevant characteristics and assess them against the existing application system.
  • Report on all findings including areas of strength and weakness.
  • Provide recommendations for remedial actions for the weaknesses.
