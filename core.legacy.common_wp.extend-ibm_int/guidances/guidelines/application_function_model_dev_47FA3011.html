Main Description

The Application Function Model defines the major groups of business function that are required in order for the enterprise to meet its business objectives. These groups are derived by a combination of matrix analysis and clustering techniques followed by some refinement of the clusters/groups. The work breakdown structure to achieve this is typically as follows:

1. Activity: Identify Applications

  • Task: Review Business Architecture
  • Task: Identify Candidate Applications
  • Task: Refine & Complete the Application List

2. Activity: Define Applications

  • Task: Describe Application
  • Task: Define Application Dependency
