Main Description

The way in which business data is viewed will be influenced by many considerations, including the type of data, how the business needs it to be used, accessed and managed, plus volumes and volatility. The following pragmatic approach has proved successful in many client engagements.

  • Analyze existing Information Classes
    • Agree key data characteristics which are important to the business
    • Gather characteristics data against Information Classes
    • Produce matrix of Information Classes to characteristics
  • Identify baseline Data Stores
    • Identify and prioritise defined characteristics
    • Define candidate data stores
    • Document the characteristics of each data store
  • Optimize Data Stores and Information Model
    • Revise Information classes
    • Refine Data Stores
    • Produce matrix of Information Classes to Data Stores
    • Finalise documentation and review/agree with stakeholders
