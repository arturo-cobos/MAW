Guideline: Developing Content Sourcing Plan
Migrated development approach from existing IBM Global Services Method work product.
Relationships
Related Elements
  • Content Sourcing Plan
Main Description
  1. Identify the internal and external sources of content from the Content Model
  2. Classify each item by ownership, relationship between owner and user, and type
  3. Identify the content owners specific conditions of use, the management system by which the content is created, published, refreshed and distributed and any existing commercial agreements for usage.
  4. Identify gaps and issues that need to be addressed to meet the requirements laid out in the Content Management Plan.
  5. Build a list of requirements that need to be negotiated for each content provider
  6. Discuss requirements with each content provider
  7. Draft commercial frameworks for review
  8. Document all finalized requirements and agreements in the Content Sourcing Plan
