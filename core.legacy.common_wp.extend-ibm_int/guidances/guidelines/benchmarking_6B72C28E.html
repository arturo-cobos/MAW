Main Description
This technique paper is intended to describe the basics of benchmarking. Since, IBM’s core business is IT related; benchmark engagement experiences have mostly been made with IT benchmarks. Therefore the described examples are mainly IT related. Nevertheless this technique paper gives general guideline on how to conduct benchmarking engagements.
 
Benchmarking compares a company’s status quo to the average, and / or best practices, and / or some specific peers.

Benchmarks are best if they can be achieved within the same industry. However, they can be highly dynamic and are constantly being improved or changed.
