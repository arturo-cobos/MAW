Main Description
This work product can be prepared by a single person or by a small group. Either must possess a detailed:
  • Description of the client’s future business processes and organization.
  • Knowledge of the features of the package and system to be installed.

The customization rules and standards should be agreed upon with the client prior to proceeding with the configuration details. It may be necessary to brief the client on the package’s capabilities in light of the intended organization and business process changes.

Each organizational and business process entity should be paired with a package construct. Package constructs are typically available to represent different legal entities, reporting relationships, physical facilities, logical ownership of data, workflow relationships, amongst others. Configuration parameters for the constructs should be chosen to reflect the intended organization hierarchy and business processes. The decisions supporting the choice of each setting should be documented.

Once a preliminary draft of the configuration has been prepared, it is reviewed and revised by the project team, client, and other stakeholders. The client-approved results are then used as the basis for setting the configuration parameters and populating tables in the package.

The project team will continue to use this work product as a reference throughout the configuration of the package, and it should be revised as the prototyping and testing uncover the need to adjust the package parameters.
