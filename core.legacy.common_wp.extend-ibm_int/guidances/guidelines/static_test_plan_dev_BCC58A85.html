Main Description

Develop a Static Test Plan along with the development work plan. Identify the development work products for the project. Punctuate the set of tasks to develop each work product by static testing based on risks and costs. Create a simple spreadsheet identifying the technique, the participants, and the time estimate for each work product. Select the most appropriate static testing technique for each work product.

Common static testing techniques include:

  • Inspection: The most formal review method and rigorous form of static testing – a well-defined and understood procedure comprising of: planning, overview preparation, inspection meeting, rework, and follow-up.
  • Desk check: The least formal process – a product author reads his or her own work to identify defects.
  • Walk-through: Reading and/or visual inspection of a work product by one or more people in addition to the author.
  • Prototyping: Reviewing a work product model (initial version) defined without the full capability of the proposed final product; the prototype is employed and the result of the exercise is evaluated, leading to an enhanced design.
  • Review: Used extensively to provide the project with control, quality assurance, and focus on issues affecting the product, the development process, and the project. A review deals with the characteristics of what is being built, how it is built, and with the resources used to build it.

The additional techniques may be used in combination with those above or with each other:
  • Checklists: The use of a checklist to examine the work product for common errors, compliance to standards, computation errors, a guideline for the review procedure, etc.
  • Mapping: Matching of functions to corresponding specification and showing how the function directly or indirectly, aligns with the requirements.
  • Structured workshops: Using facilitation and JAD–like rules and structure for joint sessions to hold walk-throughs, reviews, or inspections.

Link static test results and reporting to project management processes, so that defects discovered during static testing are dealt with.
