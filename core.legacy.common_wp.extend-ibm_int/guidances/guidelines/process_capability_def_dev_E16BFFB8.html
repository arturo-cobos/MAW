Guideline: Developing Process Capability Definition

Main Description

The following approach may be used to define and document the Process design points and Process Capability definitions for each process identified to be in scope:

  1. Analyze the Strategic Capability Model with core team to ensure common understanding of required strategic capabilities.
  2. Analyze existing documentations of realized and specified Process capabilities with core team to ensure common understanding of already defined process capabilities.  
  3. Link Strategic Capabilities with   Process Groups or Processes.
  4. In a workshop environment, with the core team and key subject matter experts, define and create graphical documentation of the Process and Sub-process design points that will support the documented Strategic Capabilities (You can regard this as being the first step in designing the Business Processes). 
  5. In a workshop environment, with the core team and key subject matter experts, define and document required process capabilities that will drive the design of Business Process to enable the realization of the supported strategic capabilities.
  6. In a workshop environment, with the core team, identify the key process capabilities that are needed to support the overall process that may not be tied to specific sub-processes or activities.
  7. For the process as a whole and for each process capability identify the Metrics and Targets to be used to in the monitoring and assessment of the processes and the process capabilities.
  8. Review Process Capability Requirement specifications with - and obtain approval from – the Client’s sponsor(s) and key executives.
