Guideline: Developing Phase Transition Package
Migrated development approach from existing IBM Global Services Method work product.

Related Elements

Main Description

For those activities related to deliverables:

  • The Project Manager should edit the work product Phase Transition Work Sheet to reflect the actual set of work products and contract deliverables completed during this project phase.
  • For each work product, the Project Manager will note the exact location of the hard copy and soft copy versions of the work product or deliverable, along with any supporting materials.
  • A copy of the completed Deliverable Transmittal Control document will be included as a part of the Phase Transition Package, either hard copy or soft copy. Note: The signed original Deliverable Transmittal Control Document should be included in the Project Control Book, with a copy sent to NSSO.
  • The Project Team should hold a workshop to review each work product and assess its suitability for submission to the Intellectual Capital Management Database. Reasons for submittal should be documented on the Phase Transition Work Sheet.
  • The Project Manager should note the date on which any work products were submitted to the Intellectual Capital Management System.

For those activities related to client commitment and next Phase deliverables, the Project Manager should:

  • Review the Statement of Work (SOW) to determine what commitments were made to the client of a contractual nature.
  • Prepare a detailed phase project plan and hours estimate (both IBM and client) for the next phase. The project manager should also make note of any changes that may need to be made to the SOW to reflect the actual work being proposed for this phase.
  • Hold a workshop with the clientâ€™s team to confirm that the proposed project plan and hours estimates are correct and to identify any proposed changes to the SOW.
  • Finalize the project plan, deliverables list, and any changes to the SOW that reflect the work to be performed during this phase.
  • Process the changes to the SOW according to the contract (if needed).
  • Send a copy of the phase project plan, with a confirming cover memo to the client.

For those activities related to resources:

  • The Project Manager should review the IBM resource requirements reflected in the phase project plan and determine the appropriate staffing profile needed for the phase.
  • If there are current IBM team members that will be released from the project, the Project Manager should complete the Project Team Feedback work product and follow the appropriate resource deployment procedures for the geographic or practice area.
  • If additional IBM resources are needed for the next Phase, the Project Manager should follow the appropriate resource deployment procedures for the geographic or practice area.
