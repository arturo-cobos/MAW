Guideline: Developing Preliminary Cost Benefit Estimation
Migrated development approach from existing IBM Global Services Method work product.

  • Preliminary Cost-Benefit Estimation
Main Description

The most cost-effective way to develop the work product is to have it be a "by-product" of other work.  For example, work on the business processes might uncover benefit opportunities; work on IT strategy might uncover cost information for software and hardware investment.

  1. Develop data on the areas where the project expects to show benefit.  Work with process owners to determine the amount they feel is attainable.  At this point, percentage improvements are a simple way to compute benefits.  If more detail can be easily obtained, use it; however, this data should be detailed enough to justify continuing the project, as opposed to justifying the actual expenditure on the project.
  2. Develop data on the areas of cost that the project expects to incur.  Work with package experts to get a sense of what other projects of similar size and scope are investing.  Work with the package vendors to obtain early quotations on software.  Work with vendors (either software or hardware) to get preliminary hardware quotations.  Again, get enough detail to justify continuation of the project, not actual expenditure.
  3. Assemble the information in the specified formats.  Review the data with process owners for reasonableness.
  4. Publish the findings and review with the project sponsors.
