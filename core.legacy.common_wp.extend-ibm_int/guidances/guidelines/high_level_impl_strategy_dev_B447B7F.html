Main Description

The development of the High-Level Implementation Strategy is the responsibility of the IBM Selection Team.

Develop the High-Level Implementation Strategy using the following steps:

  • Review and confirm any available models or work products developed that describe the client's current application and technical environments.
  • Define the process priorities used to define the order of module implementation with the client.
  • Identify the application dependencies.
  • Define the logical application groups.  Definition of these groups varies by project.  Use best judgment to reduce the development of costly “disposable” interfaces.
  • Define the activities, milestones and key dates.
  • Create a high-level release plan (e.g., Gantt chart) to depict the timing of the implementation of the applications with an estimated level of effort.
  • Include any points of observations that would be helpful to the implementation team.  This may include:
    • Resource types
    • Training needs
    • Organizational or procedural changes
    • Data for conversion
  • Identify and document any specific issues or considerations related to the client environment, resources, package experience, etc.
