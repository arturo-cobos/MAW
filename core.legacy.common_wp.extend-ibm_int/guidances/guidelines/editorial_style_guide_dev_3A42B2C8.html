Main Description

An Editorial Style Guide should be developed prior to the creation of any new content, and requires both the input and consensus of the IBM content strategist/editor and the client’s content contact. If the client has a Style Guide, it governs. IBM Editorial Guidelines govern next; and official references (e.g., Chicago Manual of Style) should be used to cover any unanswered questions. Exceptions can be made in cases where the IBM Style Guide is outdated in regard to current usage, or when adherence to the Style Guide’s rules would be in conflict with the client’s brand image. During the analysis phase, evaluate the client’s accessibility requirements and tailor the Editorial Style Guide to those requirements.

The following resources are recommended when making decisions:

  • The client’s corporate guidelines.
  • User model to determine the characteristics of the user communities within the organization.
  • Investigation of current common usage within the delivery medium, e.g., the Web, phone, kiosk, etc.
  • Customer style guidelines.
  • Official industry and editorial guidelines (e.g., Chicago Manual of Style).
  • Web Accessibility Initiative guidelines (WAI) via http://www.w3c.org/) and IBM internal accessibility guidelines.

Here is a summary of some of the main points to consider when designing text for accessibility:

Contrast - Text should be available with the highest possible contrast. There is evidence to suggest that many older or partially sighted users may find light text on a dark black background more readable than dark letters on a light background.

Color - Very high contrasts are difficult to achieve with color combinations other than black and white so the rule for type color is to keep it simple.

Size - Do not use absolute sizes, such as points (pt) or pixels (px) for your text. Instead use relative sizes such as percent (%). In this way, you allow the text to be resized in the browser to suit the user.

Leading – Ensure enough spacing between the lines of text to easily distinguish one line from another, though be careful not to make the spacing so big as to suggest both lines are unrelated.

Font Family - Specify font faces in the format "Arial, Helvetica, sans-serif", or "Times New Roman, Times, serif", for your HTML pages. Not all operating systems have the same font families, but using the 'serif' or 'sans serif' format allows the computer to choose the nearest available font family if the specified face is not available. In this way, you can better predict the look of your pages on another machine. Decorative fonts, such as Comic Sans, should only be used for headings.

Font Style - Try to limit your use of italics, oblique, bold or condensed within the body of your text as they are not as easily readable.
