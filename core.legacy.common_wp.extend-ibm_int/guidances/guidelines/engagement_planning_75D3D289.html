Main Description
All engagements require planning, to ensure that the objectives, scope, level of detail, deliverables, responsibilities (key activities and tasks) and resourcing requirements are effectively documented and the engagement is sized and costed appropriately. This Engagement Planning technique paper provides guidance on how to define an engagement plan for a project with a typical duration of between 4 weeks and 3 or 4 months. A summary of the outputs produced using this technique paper will be attached to the Proposal and Statement of Work and form part of the contract signed by a client.
 
This technique paper would normally be used by the person responsible for negotiating, sizing and setting up an engagement of this size and scope - typical roles could include the Engagement Manager, Principal, Business Consultant, Technical Consultant, Project Manager, etc.
 
There are many types of projects/engagement models for engagements of this size and duration - the list below provides some examples, but is not exhaustive:
  • e-Business Strategy and Assessment engagement (12 - 16 weeks)
  • e-Design engagement (12 weeks)
  • e-Infrastructure engagement (6 - 12 weeks)
  • Security Assessment (4 weeks)
  • Return on Web Investment (4 - 6 weeks)
  • IT Architecture Review and Assessment (4 weeks)
  • High Level IT Architecture Definition (6 weeks)
Note: This technique would typically NOT BE USED for engagements of longer durations, especially SI projects of 6 months to 2 years - there are more detailed/rigorous WPDs and techniques which are appropriate in this environment.
