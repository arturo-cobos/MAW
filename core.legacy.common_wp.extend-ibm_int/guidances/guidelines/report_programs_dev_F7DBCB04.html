Main Description

Place the report programs under change control as early as possible in the project’s life.  Ensure that the specifications and the implementation, including test cases, remain synchronized through strict version control in the build process. Use a source code repository that supports versioning to assist with the management of the constant changes to the code files.  Differences between versions of a source file can be captured without maintaining multiple physical version files of the same code file. A versioning tool that also supports release information is most desirable.  A release can be frozen when report programs are shipped.  This allows the development team to respond to customer problems by testing on a known code base.

Document the report programs carefully.  Place the documents under change control and make them accessible to the whole project team.  This can conveniently be done within the version management control system for the project.  This will assist in rectifying errors found in testing and provide a solid basis for the approval of the report programs by the client.

Report programs may be created from source by using:

  • Text editors 
  • Design tools or languages such as the Structured Query Language (SQL) or Report Generator Program (RPG).  The output of the tools may be augmented by source written using a text editor. 
  •  Visual programming techniques that generate source from graphic and text representations of the report. 
  •  Report generators built into database management systems

However the report programs are created, they should be based on an approved specification defining the characteristics of the report.  The implementation should use approved standards and procedures, such as coding standards and build procedures.  Standard reports may already exist for a program.  They should be treated like those that require development.  They should be tested in conjunction with the client’s application as a whole.

Intermediary files may be required during the generation of a report, particularly if the data reside on different platforms. The format of these files should conform to the specifications governing the report.
