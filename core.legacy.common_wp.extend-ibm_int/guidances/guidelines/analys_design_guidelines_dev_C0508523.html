Main Description

Obtain some guidelines from another project. Interview experienced developers to check that the guidelines are reasonable and complete. Consider asking a mentor for advice on proposed Analysis Guidelines or for comments on those guidelines that you have already assembled.

Go through each work product and note any variation between project intention and the standard description and process. Document any extensions or variations, and make them known to all members of the team producing or using design work products.

Start with a minimal set of guidelines, and add to them as specific issues arise that have solutions with a general applicability.
