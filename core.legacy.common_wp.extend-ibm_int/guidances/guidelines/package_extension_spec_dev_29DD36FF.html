Main Description

Identify the purpose of each extension. Determine the connections between the package and extension. Write the detailed specifications for each extension. The project leader or a senior member of the development team may do these activities. The client should be involved to assist with interpretation and clarification of each element of the intended application. The client can also provide insight into an existing application and the data that must be used in the extension. These items may have significant bearing in the extension to be developed. The project manager or leader may use joint client/development team review meetings to define these details.

The specification of the each extension may be iterative as the design of the client’s application progresses.

During each iteration, the package extension specifications are elaborated until the extension is documented sufficiently for implementation. As each elaboration progresses, it is essential to review the client’s intended application as a whole to ensure that sum of the parts will have the intended purpose of the whole. Design walk-throughs will assist in identifying deficiencies in the extension. The deficiencies should be remedied with the entire application in mind.

Consult the client to solicit process documentation that may assist in the definition of the extensions, particularly with external systems. Application requirements documents and use cases may provide additional information.

Document the testing considerations so unit testing may be specified and executed.

Document the security requirements for each interface such as who or what program may use each extension and under what conditions. This includes but is not limited to privacy, encryption and password requirements.

Document each extension concisely and precisely using the notation noted above as a guide.
