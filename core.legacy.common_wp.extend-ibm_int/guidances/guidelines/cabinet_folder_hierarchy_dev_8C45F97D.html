Guideline: Developing Cabinet Folder Hierarchy
Migrated development approach from existing IBM Global Services Method work product.

Main Description

1. Identify your client’s business objectives for viewing documents based upon their status (i.e., Work In Progress, To Be Reviewed, Final), or organization (i.e., Cabinet/Folder hierarchy).  Include requirements for levels of nested folders and the visibility of folder contents.

2. Determine if the attributes that represent the client’s business objectives for viewing documents will be designed to exist with a type definition within the document management system or in separate database tables.

3. Perform the trade-offs of using predefined queries, access control lists, and standard cabinet/folder structures to solve the client’s business objectives.

4. Design the cabinet/folder hierarchy.

The work product is originally created in early design phases and generally contains only the hierarchy of cabinets and folders.  During detailed design phases, more detail is added, especially to the document/folder location information.

This work product is complete when the question of where a user will find every document type during each step of the document’s lifecycle has been addressed, either with cabinets and folders or queries.
