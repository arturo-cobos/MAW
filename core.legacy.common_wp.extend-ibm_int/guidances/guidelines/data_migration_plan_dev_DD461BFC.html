Guideline: Developing Data Migration Plan
Migrated development approach from existing IBM Global Services Method work product.
Relationships
Related Elements
  • Data Migration Plan
Main Description
  • To begin development of this work product, the sources and targets of each data transfer required in the system must be known.
  • For each data transfer:

- Review quality assurance / audit requirements for the data transfer.
- Review processing constraints for the source.
- Review non-functional requirements for the target.
- Develop approach for accomplishing the data transfer.
- Identify processes required to effect the data transfer.

  • Once all data transfers have been specified, review interaction of all the data transfers upon each other and resolve collisions.
  • Document a data transfer schedule showing frequency and timings.
  • Coordinate/consolidate data transfer plans into a concise and complete Data Migration Plan for the entire system.
  • Plan the staffing.
  • Identify the contingency procedures.
  • Document results.
  • Obtain client approval.
