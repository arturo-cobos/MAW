Main Description
The design of the service occurs only after process design, organization design (roles and responsibilities), and tools design (generic tools and requirements), have been completed. The basic building blocks for the services will be sub-processes previously defined during process design.
  • Conduct a review of the key I/T services defined in the first stages of the engagement with key clients. This should include a review of the IT Services Strategy.
  • Develop the service charts:

- Start with the template definitions from the SMFD-TE method.
- Review/modify/define the sub-processes including the sequence and the flow.
- Associate the generic role(s) that will execute the sub-process.
- Associate the generic tool(s) to be used to execute the sub-process.
- Associate the data products used to execute the sub-process.

  • Review and modify the service charts by role-playing scenarios.
  • Complete the Services Guide by adding descriptive text surrounding the services charts.
