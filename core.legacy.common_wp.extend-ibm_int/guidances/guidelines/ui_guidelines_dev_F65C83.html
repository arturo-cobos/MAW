Main Description

Consult the steps below in producing User Interface Design Guidelines:

  1. Determine the appropriate industry standard (e.g., IBM CUA, User Interface Style Guide for IBM Global Industries, OSF/Motif, Windows, Eclipse, Accessibility,and so forth).  In the User Interface Design Guidelines work product, refer explicitly to the documentation of this standard and how it may be accessed.
  2. Based on usability, accessibility, and other requirements and user interface related requirements as gleaned from use case modeling (or other requirements gathering techniques), determine whether any tailoring of the appropriate industry standard is necessary.
  3. Determine whether an equivalent User Interface Design Guidelines work product exists and whether the developers use it.  If it isnâ€™t being used, interview developers to find out why.  Use this feedback to restructure/update the existing User Interface Design Guidelines.
  4. After identifying user interface components that need customization to meet the specific needs of the solution, document required and recommended usage in the style guideline.
  5. Frequently update the User Interface Design Guidelines so they are always close to what is being used in practice.  Developers will lose confidence in the guidelines if they contain outdated recommendations and requirements.
  6. Ensure the guidelines are consistent.  For example, ensure the custom look-and-feel guidelines do not conflict with accessibility guidelines.
  7. Put in place a practice for recording and tracking deviations from the style guide.  Track these deviations over the course of the project.  Put in place a mechanism for ensuring that these deviations are resolved, either through the system test problem tracking mechanism or through a user interface style deviation tracking mechanism.
