This is an example showing the Credit Process Gap Analysis.

Main Description
Name: Credit Process
Identifier: Perform Customer Credit Check
Summary Results: Medium
The main issue is inaccuracies in the credit position caused by delays in processing incoming payments and the policy of updating credit status only when orders are shipped.
Detailed Results:
  • Compute available credit. (GAP)
  • Compute order value. (OK)
  • Bypass "A" rated customers. (OK)
  • Test order value vs. available credit. (OK)
  • Produce and route notification of credit rejection. (OK)
  • Release orders on credit hold. (OK)
  • Update orders-in-process value. (GAP)
Gap Analysis: Compute available credit. The package uses data on the outstanding customer receivable that is updated nightly in a batch run. This will cause available credit to be understated by the amount of the unposted payment and could cause orders to go on hold unnecessarily.
Gap Resolution Strategy: Create a user exit to check for unposted payments when computing available credit. This is estimated to take 5 man/days of programming time.
Security Implications: None
Gap Analysis: Update orders-in-process value. The package only updates the consumed available credit once the order is shipped. Since orders might take seven to ten days to process, this creates exposure to overextending of credit.
Gap Resolution Strategy: Since this is an infrequent exception, we will change our credit limit review process to account for this, and see if lower overall credit limits will handle the problem.
Security Implications: Security profiles may have to be updated to allow the credit limit change to take place.
