Program Description:
CUSTOMER 36 one of the company’s major trading partners sends Sales Orders by Fax. The company batches Sales Invoices and mails them once weekly to CUSTOMER 36.
Please attach additional page(s) as required.
Enhancement Requirements:
CUSTOMER 36 insists on being able to send Sales Orders to the company and receive invoices via EDI. The company is obliged to comply with these demands due to the magnitude of business received from CUSTOMER 36. To achieve this the following enhancement is required:
  • The company will receive Sales Orders from a trading partner and will then return Invoices.
  • The company has agreed with the trading partner to use the TRADACOMS standard.
  • The company has selected Sterling Gentran translation software.
  • The company, along with the EDI software vendor ( Sterling), will perform the technical EDI TRADACOMS message translations required by the Gentran Software.
  • The files passed, inbound and outbound, between the SAP application and the translator will be ‘flat files’ and will require the relevant directories and files to be set up by The company System Administrators.
Although SAP has standard data extraction EDI processes these may not take into account the trading.

The Command file is used to enter the path and name of the EDI system script. The Outbound file is the path to the outbound directory of the application server. The directory will be accessible from the SAP and EDI subsystem. The file name is generated by using the function module: ‘EDI_PATH_CREATE_USERNAME_DT_TM’.
The Inbound file is used to specify the input directory. This will be the same as the outbound directory. The status file is simply the path and name of the file used for status information.

Gap analysis

Split/Mixed Rate VAT Details are required by a specific trading partner. For example, two ingredients of a product may have two different VAT rates (Chocolate biscuit where the chocolate is classed as confectionery but the biscuit is not). According to the above map, it has been identified that SAP must provide the information held in the two fields AUCT & LEXC. It is therefore necessary to provide the following extensions:
The standard IDOC INVOIC01 segment E1EDP01 must be extended to include the fields AUCT & LEXC (see Sections 3.3.2 & 3.3.3).
Additional coding to be done in a user exit to fill these fields and to insert a segment for mixed VAT (see Sections 3.3.2 & 3.3.3).

Identify Fields and Segments
During the SAP-To-Be and GAP analysis processes the following requirements were identified:

  • Mixed Rate VAT is not available in standard SAP extraction.
  • The customer requires that the TRADACOMS Standard fields LEXC & AUCT are to be used to hold mixed rate VAT information.

- LEXC - Extended line value excluding VAT.
- AUCT - Unit Cost value excluding VAT.

  • The SAP tax segment is E1EDP04 so the required fields need to be added to this segment. E1EDP04 is a child segment of the E1EDP01 segment.
  • Standard SAP segments must not be changed therefore the E1EDP01 segment must be extended to include the new fields.

Code USER-EXIT To Process Mixed Rate VAT
A new segment Z1EDP04 has been added to E1EDP04 to hold two new fields:
  • ZLEXC (extended line cost excl. VAT) &
  • ZAUCT (Unit cost price excl. VAT).
User exit ZXEDFU02:
Check that the program has reached the creation of the E1EDP01 segment – item level. This check is required because the values of LEXC & AUCT need to be stored when processing a standard VAT item (A1) or a zero-rated item (A0). In these cases the Z1EDP04 extension is not used and the LEXC and AUCT fields are stored in the E1EDP01 segment:
  • E1EDP01-NETWR = LEXC.
  • E1EDP01-VPREI = AUCT.
Sterling Software will need to check the first E1EDP04 segment field E1EDP04-MWSKZ:
  • If it is = ‘A1’ or ‘A0’, the LEXC & AUCT are to be found in the E1EDP01 fields (NETWR & VPREI).
  • If E1EDP04-MWSKZ = ‘A’ (i.e. mixed), the LEXC & AUCT values are to be found in the extended segment Z1EDP04 which is a child segment of E1EDP04.
  • If the program has reached the creation of the E1EDP04 segment – tax.

Check if the item is for mixed rate VAT – check the condition table KOMV and if the key KSCHL = ‘ZM01’ then the item is mixed rate.

Note: This example is specific to the company because they identify Mixed Rate VAT items by condition type = ‘ZM01’. Other companies may identify mixed rate VAT using different criteria. If ‘ZM01’ is not found, the user exit stops and standard SAP processing creates E1EDP04 segments for either standard (A1) or zero rated (A0) items.

Workflow
When errors occur in IDoc processing, a group of people can be informed via the SAP inbox. The first stage is to set up an organizational unit, create jobs/position and then assign user id’s to the positions. The easiest way to do this is to use transaction SWLD to get to the workflow main screen and then select Definition tools -> Organizational plan -> Change (transaction PPOM). Enter the organizational unit (50000500 – SAP Team) and then select Change. Hit Staff assignments. The position EDI administration has been created and the user WALTONM assigned.

The next step involves assigning the appropriate tasks to the position. Click on the Task profile button and expand out the light blue job EDI administration. Two tasks have been assigned: Error handling for payment advice note and ORDERS input error. To view the full list of tasks that are available use transaction SWLD, hit the tasks button, enter TS in the task type and F4 the task field.

Before workflow can be used to pass the EDI errors to the appropriate inbox, it must be customized. This is done from SWLD, Utilities -> Consistency test -> Customizing (transaction SWU3).

Hitting the AutoCustomize button will carry out all the basic customizing required. Hit the Test RFC button to test the RFC connection. If this fails then select the RFC destination line (3rd one down) and then select the Create logical destination for tRFC line. This allows the user WF-BATCH to be set up. This is a mandatory user required by workflow. It is a dialog user and the password is set to ‘workflow’.

Testing the workflow configuration
This is carried out by clicking on the Verif.Workflow button. This generates a workitem and sends it to your inbox. Click on the inbox symbol. Execute the ‘First step in workflow verification’ item by double clicking on the tick and then click on the ‘Execute background step immediately’ button at the top. This will send two further notification items into your office inbox.
