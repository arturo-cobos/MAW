Identifier: 22. Evaluated Receipts Settlement (ERS)
Summary Results: Medium
For approved suppliers, no comparison of goods received to goods ordered is done at time of receipt.
Detailed Results: The goods received (quantity and stock number) are physically accepted, recorded and stored. Subsequently, an automated comparison is made between open orders with the supplier and good received. For those where valid orders exist, payment is made. For those where no valid orders exist, a document is sent to the supplier (paper or electronic), detailing the results of the comparison. The supplier can then either make disposition arrangements for goods shipped in error, or the record may be amended depending on the situation.
Gap Analysis: The package checks for a valid purchase order at time of receipt, and requires special input if one if not found. The package has no list of ERS-approved suppliers. The package does do an automated comparison, and does produce a reconciliation report.

Gap Resolution Strategy: After discussions with the receiving manager, and the I/T analyst responsible for the old application, the project team determined that the first option, a slight modification of the package, was the most prudent given the project time schedule. However, the third option, new subsystem, will be added to the second release of functionality, scheduled for third quarter next year.
The implications of this decision are:
  • The modifications specified by the first option will be added to the Inventory of Modifications, and developed before the first release of the package to the users. (Project Team)
  • The receiving procedures will be modified to match the new workflow, and exception procedures will be modified as well. (Receiving Manager)
  • The ERS suppliers will be notified of any changes in either the content or format of data they will be receiving. (Procurement Manager, Corporate Communications)
Security Implications: Security profiles may have to be modified.
