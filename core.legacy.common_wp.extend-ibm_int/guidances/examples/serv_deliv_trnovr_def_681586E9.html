Main Description

The asset checklist to be prepared shall include (any/all/more depending on the engagement context):

  • Business process documentation. 
  • Organization charts and missions. 
  • High-level technology architecture diagrams. 
  • System context - all the documentation which describe the system with respect the to client requirements. 
  • Logical design components - all the design documentation which generically describe the system. 
  • Physical design components - the design documentation which describes the actual representation of the system with respect to its generic description.
  • Hardware and software inventory – a listing of all the technologies in use and any known information regarding its selection, long-term viability, maintenance, and impact on the project objectives. 
  • IT standards - the support documentation which directs the development of the system. 
  • Operating processes and procedures - the documentation which assists users in utilizing the system and operators to keep the system available. 
  • Operating environment - all the assets which support the running of the maintenance and enhancement function. 
  • Training and support materials - all the documentation and structures that are used in the provision of the required readiness needed by both operators and users to function properly.

Typical examples of the nature of data to be captured in the asset checklist are given below:

  • Data Centre computing - utilized MIPS (or equivalent). 
  • Departmental and personal computing - numbers of users. 
  • Information capture and presentation - as a % of total IT costs. 
  • Data Network - numbers of end-user stations supported. 
  • Voice Network - numbers of end-user connections supported. 
  • Help Desk - total calls per year handled. 
  • Current IT department staffing and skills.
