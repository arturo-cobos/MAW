Main Description

Client is a new application that provides similar services to Internet users as those currently provided to America Online users. The new application will have components that reside on one of Client's Web Servers (which is protected by both inner and outer firewalls) and on the AIX systems in Client's back-end web complex (Consisting of a new Internet Client and Generic Engine, EGI). In addition, changes to support system have been made to the GateKeeper, the Sybase Security database and the Security Server (TTS) which run in production in the back-end web complex. Solution will enable customers on the Internet to securely access the same back-end legacy systems currently available through AOL, after appropriate registration and authentication information is provided. The new code is primarily written in C++. The phase one functions to be provided are: member registration, login, account status, maintenance of registration information, and award services, and customer service representative ("superuser") functions including the ability to assume a member's id.

The purpose of this project was to assess if client is "ready (sufficiently secure) to be launched" in by performing a security review of the new application components, the associated development and administrative processes, and the services (network, application operating system platforms, Security Data Base, and Security Server) used by client in providing the Phase 1 functions to be supported.

A detailed review of the security of the web complex is planned as an independent project led by Client Internal Audit. Similarly, a detailed review of the back-end production systems, processes and applications was outside the scope of this project.

A significant number of security concerns were found during the assessment. These are documented in Appendix A. Most of the concerns were not specifically related to the new components and changed components in support of client. IBM recommends that they all be addressed (see Chapter 3).
