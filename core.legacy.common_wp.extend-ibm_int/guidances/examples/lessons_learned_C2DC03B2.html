Tools Implementation (eg., Service Request Tool)

Steve N./Transition Team

The SDC's tools "push" is a great method of getting the tools onto the desktops.

Transition for ADM was impacted because there was not an end-to-end plan in place for tools rollout. It was not that it was simultaneous, it is because there was not a plan.

ADM and SDC should work together to produce a tools rollout process that can be used on future accounts.



Terry D./Delivery Project Executive

Wayne W./PDMs

(Progress on rollouts have been slowed because of what is happening on the contract).

Lawrence C./ Transition Team





In future AMS engagements we need to change the approach for Service Request Process including the tool. The proposal must include the complete cost for installing and configuring an out-of-the-box eRequester with the built in basic process. This needs to include the cost or pre-secured server space. We take the approach as is currently done with the activity for installing the base AMS Management System. Install the tool and deliver the non modified procedure. As the tool is being used at the account, then customization can occur to refine both the process and tool. We need to make the presence and availability of the tool on Day 1. The work flow of requests is an important item for the customer and can be a place to establish AMS value add.
