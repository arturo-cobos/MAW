INTRODUCTION

Purpose

The purpose of the Impact Analysis is to provide the project team with detailed information about the parts of the application system, which will be affected by changes to satisfy the release requirements. It also raises issues requiring further clarification from the client or appropriate stakeholders.

Scope

The impact assessment covers the extent to which the requirements affect the functioning of the application system as a whole.
Each requirement is assessed for impact on:

  • Inputs and Outputs 
  • Data Structures 
  • System controls and Processing Algorithms (including Security and Privacy controls) 
  • Operational controls 
  • Processing models 
  • Technology

BACKGROUND

System Context

The impact assessment document should be read in conjunction with the Baseline Survey, which also provides a summary of the system context.

Requirements Context

Refer to the Client Request Package and Use Case Model for requirements detail. Following, is a brief summary of the requirements statement.
1. Reduce the current elapse time between service activation completion and network usage recognition from two days to 2 hours.
2. Correct the display of customer credit check warnings to ‘red and flashing’ on the screen instead of the current ‘black and no flash’.
3. Introduce the following two new service products (call number display and ‘red eye’ special rates for off peak network usage) to the application’s capability.

IMPACT ASSESSMENT

Requirement 1

Input/Output Impact

There does not appear to be a need for any new input or output in order to transfer completed service activation records through the interface. There might however be a need for reporting on the elapse time to show compliance to the new transfer performance service level of 2 hours. This needs to be clarified with the client.

Data Structures Impact

The indexed sequential file (AXIS0006) which is currently created for completed service activations for overnight transmission will become redundant, because there will no longer be any batch transmissions. A new transient record structure will need to be created for on-line transmission.

System Controls and Processing Algorithms Impact

The System Description document will need to be modified to reflect the new online transfer capability.

  • All programming (AXBB0001 - 0004) currently in place to prepare the completed service activations batch file for overnight transmission will need to be removed, however the algorithms and controls for assembling each record and ensuring the accuracy of the records can be leveraged in any new programs that will be created for the online transmission. 
  • A new transmission protocol to replace the current batch mode of transfer will need to be determined for the immediate transmission of completed service activations. This should also consider alternative actions when transmission fails. This information will be documented in the Transaction Description work product. 
  • Completed service activation records will continue to be archived in the usual way.

Technological Impact

The most suitable mode of transfer will need to be determined at the design stage and documented in the Current IT Infrastructure. The batch file transfer utility will need to be replaced with an ad hoc record transfer capability.

Operational Impact

  • Release notes will need to be created to instruct operators about the changed operating procedures, to prevent any attempt to create an over night batch file.
  • Error messages and alarms will need to be included to notify online operators of transmission errors.

Requirement 2

Input/Output Impact

Investigations have revealed that 16 output screens display customer credit information. Refer to screen programs AXPS0002 - 0012, AXPS0017, AXMS0024, AXCS0014 - 0016. While the fields will not change, their appearance will and therefore need to be verified through testing.

Data Structures Impact

No changes to data structures.

System Controls and Processing Algorithms Impact

For each affected program, the algorithm, which assembles the warning message, will be modified to display with ‘red and flashing’ characteristics. Refer to screen programs AXPS0002 - 0012, AXPS0017, AXMS0024, AXCS0014 - 0016.

Technological Impact

No impact. The current programming language has adequate facilities to accommodate the request.

Operational Impact

Release notes will need to be created to inform users about: 

  • The credit warning displays. 
  • New available products.
  • Scroll capability.

Requirement 3

Input/Output Impact

The product and tariff reference tables (AXPT0525 and AXTT0460) will need to be updated with the appropriate values to support the two new services. These values have not yet been supplied.

Data Structures Impact

The two new products are being introduced because of the potential of a high immediate demand therefore client service requests are likely to increase dramatically following its launch. This points to the need to ensure adequate client service requests storage capacity. Technical support will need to be informed.

System Controls and Processing Algorithms Impact

The maximum number of available products will now exceed the screen limit of 8; therefore the program (AXPD0340) displaying the available products will need to be modified to provide a scroll capability. This new capability will also need to be reflected in the System Description.

Technological Impact

Additional data storage packs will need to be acquired in anticipation of a substantial increase in the volume of service requests for the two new products.

Operational Impact

Release notes will need to be created to inform users about: 

  • The credit warning displays.
  • New available products.
  •  Scroll capability.

General Impact

None

CONCLUSION

The document has provided an assessment of what needs to be accomplished to satisfy all the requirements in this release.

Requirement 1 however, does not indicate whether a compliance report would be required or not. If required, new functionality to record, store and report on service activation completion and receipt of notification of transfer times would need to be catered for. This would mean changes to descriptions or specifications of System Context and Process/Data Usage.

Detail values of the new proposed products are outstanding, assuming that they fit into the existing product data structures, they will only be needed at implementation time, otherwise additional work to cater for new data structures will need to be considered.

Technical support services could be of assistance in determining the most suitable online record transmission protocol as well as the appropriate data storage pack.
