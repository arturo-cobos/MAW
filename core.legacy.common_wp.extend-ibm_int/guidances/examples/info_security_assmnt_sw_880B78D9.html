Executive Summary

IBM Security and Privacy Services was engaged by client to review and analyze the security of the client system. During the period month to month IBM consultants were provided access to the source code for the client system for this security analysis.

Utilizing automated analysis tools as well as manual analysis techniques the vulnerability of the client system was analyzed. This report is the result of these activities and represents IBM Security and Privacy’s recommendations.

During this analysis 5,200 potential vulnerabilities including 1,730 high severity exceptions, 438 medium severity exceptions, and 1,198 low severity exceptions were reported. Nevertheless, the client as a whole appears to be well written with an overall Vulnerability Density of 66 across all analyzed source code. A Vulnerability Density of 200-500 is considered an average result for software undergoing its initial analysis, and client achieved a better statistical score.

While the discovered vulnerabilities and potential vulnerabilities should be addressed by client development staff as soon as possible, none of the vulnerabilities or potential vulnerabilities are, in our estimation, sufficient to warrant a change in the current client implementation project timeline.

During the code review a standard methodology is used by the consultants reviewing the software. This methodology is broken down into three distinct phases:

  1. Preparation – During the preparation phase, the consultants review application architecture with the development staff. Also during this phase, a build environment is created to run the analysis engine on.
  2. Analysis – During the analysis phase the analysis tool is used to analyze the source code for the application. Consultants review the output from the automated tools with key developers to determine the risk presented by the discovered vulnerabilities, make recommendations for corrective actions and periodically meet with the client to disclose vulnerabilities as they are discovered.
  3. Completion – During the completion phase the consultants prepare their final recommendations, discuss these recommendations with the client and produce a report on the activities and results of the code review
