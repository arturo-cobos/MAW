Main Description

Data Migration Program estimates are typically based on the number of lines of source code or function points. This is called "sizing the software". Size refers to the breadth and depth of the feature set as well as the programâ€™s difficulty and complexity. Early in the project, size is typically synonymous with function points that are often addressed using comparisons with existing programs in similar installations. The notion of size changes as the project matures, typically shifting from function points to classes or modules at design time to lines of code at construction and test time.

The effort involved in producing this work product is influenced by:

  • The number of data references (source/target) both internal and external.
  • The type and number of data interfaces both internal and external.
  • The number of unique transactions conducted across each interface.
  • The number of entity references per transaction.
  • The number of error conditions for each program/interface.

The effort is also affected to a significant extent by the technical complexity of the application.
