Main Description

Allow 1-2 weeks based on a project effort between 10,000 and 50,000 hours. More time may be required for projects larger than 50,000 hours.

The guidelines below relate to static testing effort and do not include the effort to create the static test plan itself.

Estimates will increase with size and complexity of the project, number of people, skill level and familiarity with the project process, subject matter or business, and technology. Effort involves the time to select the technique, identify the resources, estimate the effort, and document the plan.

The estimate for static testing can be as high as 20% of the total development effort for those work products that are static-tested. Twenty percent is usually applied where the inspection process technique is used, as this is the most rigorous of the static testing techniques and tends to let fewer defects through, thereby reducing back-end rework. The time spent preparing and conducting the inspection are also captured within the process steps.

The amount of material prepared in a project that can be rigorously inspected ranges between 15 to 20 pages per person month for the whole project, or about 1.25 pages per function point (Source SPR: KnowledgePlan).
A traditional structured approach would produce, per person month, about:
  • 1 page of requirements specification, reflected in use case model.
  • 2 pages of functional specification, reflected in use case model and non-functional requirements.
  • 2.5 pages of detailed specification partly, reflected in -class- work products and partly in architecture.
  • 3 pages of program specification, mostly reflected in -class- work products, interaction diagrams, and state charts.
  • 2 pages of detailed design split between architecture and -class- work products, interaction diagrams, and state charts.
  • 3 pages of user documentation, including reference manual.
