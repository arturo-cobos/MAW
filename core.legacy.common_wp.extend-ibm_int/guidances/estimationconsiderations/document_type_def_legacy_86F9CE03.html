Main Description

A good rule of thumb is 0.1 -.25 hours per Document Class Definition to be implemented. This covers the actual implementation.  Add in time for reviews with the client as well as for testing the DTDs using real sample documents.  The amount of time needed for testing is dependent on the size, complexity, and number of sample documents that are to be used for testing, but 1-3 days per document type is generally reasonable.
