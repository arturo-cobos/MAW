Estimation Considerations: Test Measurements (Legacy)
Estimate considerations for Enterprise Level and Project Level test measurements.

Main Description
Enterprise Level Test Measurements: depending on the complexity of the measurements and the requirements of the various stakeholders, about 5-10 days’ effort at the end of every development project is required to collect, collate, sort, calculate/manipulate and present the project information in a meaningful format for the different organizational functions. Use of an automated tool could reduce the time frame somewhat but manual analysis and review for accuracy of the output should be conducted before distributing this information. The assumption here is that enterprise policies, processes and guidelines exist for managing the Test Measurements, and therefore the initial program to create these is not included in the effort.
 
Project Level Test Measurements: it should take approximately 0,5-1 day’s effort per reporting period (e.g. per project phase) for each of the measurements selected as listed in Section 6 - Example. Producing all the metrics would be too time consuming, so appropriate ones need to be chosen to address identified problematic areas. An automated tool that collects the raw metrics and generates standard reports could reduce the cycle time, but it should be recognized that measuring for quality is not a quick or cheap exercise (these costs may be corporately sponsored or built into the cost of the testing project).
