Main Description
Report program estimates are typically based on the number of lines of code or SQL statements, or function points. This is called "sizing the software." Size refers to the breadth and depth of the feature set as well as the programâ€™s difficulty and complexity. Early in the project, size is typically synonymous with function points that are often addressed using comparisons with existing report programs in similar installations. The notion of size changes as the project matures, typically shifting from function points to classes or modules at design time to lines of code at construction and test time.
The effort involved in producing this work product is influenced by:
  • The number of reports to be developed.
  • The quantity of information to be presented on each report.
  • The number and complexity of the calculations and aggregations required on each report.
  • The complexity of the information presentation for each report such as text and graphics.
  • The number of methods of presenting each report such as printing, displaying, and storing as a file.
  • The number of methods by which the report can be invoked.
  • The number of test cases to be produced for each report including performance testing and error handling.
The effort is also affected to a significant extent by the technical complexity of the application.
