      Main Description
      Anti-use case or anomalous flow testing is used to identify possible weak points in the design. Although it is not normally employed in traditional testing, performing this task is vital to the security of any solution. Should these weaknesses persist and be exploited by an attacker, an unauthorized actor could fraudulently acquire legitimate credentials and wreak havoc on an enterprise. Anomalous tests should ensure users and components cannot to gain access to networks, systems and components by any inappropriate means including:
      • invoking processes within systems using false credentials
      • corrupting systems and networks through attacks
      • observing information flows
      • discovering critical information

      The project team must use intellectual capital and team knowledge to create a set of potential anti-use cases. These ‘abuse cases’ are then used in executions of the existing design. Additional countermeasures may be required to address any identified vulnerabilities. The anti-cases must then be included in the test strategy in order to ensure the effectiveness of the countermeasure. Additionally, formally testing the anti-use case guarantees no additional unexpected vulnerabilities exist.
