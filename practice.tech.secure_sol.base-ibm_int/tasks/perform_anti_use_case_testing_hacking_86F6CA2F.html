      Main Description
      Previously in the design, anti-use cases were created to identify security gaps in the solution. Now the anti-use cases must be formally tested to ensure the gaps have been addressed. Hacking the solution will test the identified anti-use cases, as well as uncover unforeseen ones and other possible modes of compromise. Security consultants with skills and tools in the area of hacking determine any potential points of weakness that could be used to exploit a solution and gain access to critical information assets. In this regard, critical information assets include both the data and any proprietary business processes utilized by the solution.

      On this occasion, this testing will take place in the test environment. Utilizing the test environment allows the team to minimize other external controls that could mask the solutionâ€™s security functions. Additionally, it allows the team to maximize the use of the limited base of skilled practitioners. Any findings should be addressed immediately by the project team. The team should ensure they have adequately dealt with the findings before moving on with the next solution task.
