Task: Identify Security Zones and Trust Levels
Disciplines: Security and Privacy
Purpose
This task segregates the architecture into areas that are indicative of the trust given to any entity performing an action within its bounds.

      Main Description
      During this task, the team of practitioners will determine the security zones. Zones are logical boundaries in the architecture, although they may have physical instantiations. Each zone represents a level of trust indicative within the zone. Typical zones are ones such as Internet/Uncontrolled, DMZ/Controlled, Restricted, Secured, etc.

      If a client currently has zones defined, these zones should be taken into consideration.If there are no zones currently defined, the project team is free to define the zones that match the client needs. Again, the different levels of trust needed should be considered when determining the appropriate number of zones and their boundaries.

      Document each zone, along with the description of the zone, and the trust level(s) assigned to the zone. Including the typical systems found within each zone is recommended. For example, the DMZ/Controlled zone may contain the clientâ€™s web server, while the Secured zone may contain the corporate financial data. Having these examples along with a good description of the zone and its trust level will provide for better communication to the client and amongst team members.
