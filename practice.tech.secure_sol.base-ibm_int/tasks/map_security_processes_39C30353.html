Task: Map Security Processes
Disciplines: Security and Privacy
Purpose
The mapping the existing security processes in the organization to a specific security model. This is done to identify what functions are already covered and what gaps may exist.

      Main Description
      In this task, the project team will select the security model appropriate to the client and the project. Possible models include ISO 17799, MASS, Cisco’s AAA, COBIT, or the 6 A’s. If the client has an existing Enterprise Security Architecture, the team should use it.

      Using the information gathered in the previous tasks, the team should identify the existing security processes and technologies within the enterprise. The team should then map these to the chosen model. It is important to note that each process or technology may map to more than one piece of any model. The results should be documented and any gaps identified. The client’s risk appetite should determine if the gaps are within reason.
