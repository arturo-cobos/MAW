      Main Description
      Security requirements are no different than any other requirement. Testing for them must first be included in the Test Strategy. Resources should be allocated for them in the Master Test Plan and the Detailed Test Plan. The testing team should plan test procedures and scripts around the security requirements just as they would for any other requirement.

      The security consultant(s) on the team should remain involved in the testing of the security requirements. Developers or testers may need assistance with the implementation of the requirements and testing that implementation. Any bugs that are detected during testing should be addressed with the security consultant(s) on the team and reworked.
