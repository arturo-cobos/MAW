      Main Description

      Different actors within a client environment need different levels of access to different things. Some will need more access than others. Some may not need access at all. During this task, the project lead will need to organize the actor profiles collected during the Current IT Environment task into groups, according to the levels of trust.

      The project team should revisit the information collected for the Actor Profiles and begin organizing the actors that have similar access needs. Each Actor Group should be assigned a level of trust, and the trust restrictions around those groups should be clearly spelled out. The trust levels assigned to the actors will be indicative of which zones the Actor Groups will be allowed access.

      Document the Actor Groups designated in a table or spreadsheet along with the description of the types of Actors in each group. Also document which trust level the Actor Group will be assigned. These groups must also be mapped to the defined zones.
