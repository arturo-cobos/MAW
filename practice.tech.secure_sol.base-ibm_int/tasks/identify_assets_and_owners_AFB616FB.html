Task: Identify Assets and Owners
Disciplines: Security and Privacy
Purpose
Identify what assets need to be protected, what is their value, where are they located and who are the decision makers for these assets. Additionally, identify roles and their accessible data.

      Main Description

      The objective for this task is to determine the appropriate level of security for the information assets involved in the solution. In order to determine this, the project team must first identify the information assets of the client, as well as the importance associated with those assets. This can be accomplished by a series of interviews with the asset owners, document review, and analysis of proposed designs. If an Enterprise Information Asset Profile exists, it should be the starting point for the project team. IT specific assets such as credentials, access control lists and audit logs must also be included.

      An important aspect of this effort is the Data Classification scheme. The level of classification of each asset helps to attach value and importance to the asset. If the client already has one, the project team should consider using it. If not, a formal data classification engagement is recommended.

      The project team should capture information about each asset in a spreadsheet or table. Items to be captured include, but are not limited to, the following:

      • Asset Owner
      • Location of the data
      • Monetary value associated to the asset
      • Known classification of the asset
      • Legislative protections of the asset 

      Further refinement and additions will be made in the Analyze Data activity.
