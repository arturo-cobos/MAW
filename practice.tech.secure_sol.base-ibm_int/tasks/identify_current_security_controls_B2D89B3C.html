      Main Description
      The enterprise may already have a number of security controls in place. These could include firewalls, intrusion detection, access control lists, logon credentials, digital certificates, or others. These types of controls indicate previous risk decisions made by the client. Most likely, the project may need to interface with or use these controls.

      The project team will need to determine the existing security products in use via interviews with key technical and security staff members, documentation review and analysis of technical diagrams. The team should look for the security support products or functions the client employs including the following:

      • Systems that support mail, including content screening
      • Interfaces between other networks - including the internet, business partners, other companies within a group. This should contain sufficient detail about architectural configuration of firewalls and routers.
      • Intrusion detection and monitoring - including network, system and application

      These controls should be cataloged and available for future use. The controls may need to be mapped to the security subsystems defined in MASS, in order to gain an understanding of where controls have been focused, and where there may be gaps.
