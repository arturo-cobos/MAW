Disciplines: Security and Privacy
Purpose

    Main Description
    With the framework for the architecture defined, the project team must begin planning the solution’s execution. Having the risk posture of the client captured and the security Zones determined, the team should begin mapping the security Zones to the current client environment.

    Information captured in the Current IT Environment, Asset Profile, and other pertinent documents should be used for the mapping process. The team should begin carving out what assets and systems need to be placed in which Zones, with respect to the security requirements of the assets, risk, and trust needed. Systems containing assets with high data classification levels or high requirements of protection should be placed in the more restricted Zones. On the other hand, systems that contain less sensitive assets can be placed in less restrictive Zones.

    The team should note each asset’s designated security Zone. For the Asset Profile, this can be accomplished by adding another column to the spreadsheet or table. If the Current IT Environment contains only diagrams, a list may need to be made of the systems and assets before assigning a security Zone. The team should make certain that there are no conflicting assignments. For assets that may be more difficult to assign, the team should consider the company’s risk posture in conjunction with the security requirements before assigning a security Zone.
