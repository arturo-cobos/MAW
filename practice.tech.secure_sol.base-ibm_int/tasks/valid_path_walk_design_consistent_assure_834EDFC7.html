Task: Validate Pathway/ Walkthrough/ Design Consistency Assurance
Disciplines: Security and Privacy
Purpose

Ensure the solution is robust, complete, and has met the needs of the customer.

      Main Description

      After the solution has been designed, a walkthrough should be performed to verify the solution is complete and robust. The project team should revisit information gathered and analyze it and ensure proper execution in the solution. The review should include the following:

      • Requirements should be traceable to their implementation in the solution.
      • Objectives should be traceable to their implementation in the solution.
      • Use Cases should be reviewed with the new design to ensure the solution upholds the necessary security for each functional process.
      • Anti-Use Cases (also known as 'Abuse Cases') should be developed to ensure the solution is robust and 'doesnt break'.

      Any points of concern or possible ‘breaking’ of the solution must be reconsidered and resolved. The team should have a common understanding and agreement of the solution before moving forward with the client.
