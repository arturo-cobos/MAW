Task: Ensure Security of Test Environment
Disciplines: Security and Privacy
Purpose
The test environment must be protected against snooping, modification of system functionality, disruption of testing activities, and other potentially harmful activities.

      Main Description

      The test environment is a likely target of hackers and others with malicious intent. It is where interactions between the various functions can be observed and documented as well as tuned for optimal performance. The security of this environment is critical to the successful protection of the production environment. If it is not secured, malicious code could be introduced, the functionality could be leaked or compromised, or it could be used as an entry point for attackers to gain a toehold on the enterprise network.

      In order to provide for the security of the test environment, basic site security functions must be performed. These should begin with system hardening to the client’s standards. Other items to be performed include:

      • Implementation of firewalls
      • Any necessary intrusion detection
      • Code signing (if required)
      • System administration policies
      • Identity management functions
      • Audit log generation and review

      Any security processes identified for the production environment should be considered for and used in this environment. Bypass of a production required security function must be an exception and only with the written approval of the project team lead and the client’s representative.
