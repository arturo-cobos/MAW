      Main Description
      During the course of any solution engagement, previous decisions are reviewed and, in some cases, remade. This is also true of the solution’s possible security components. During this period, issues regarding the high-level design may occur. These issues may require reconsideration of previous decisions. As these decisions are re-evaluated, the project team must examine the security decisions made in conjunction with the original design. If changes are required, the project team must evaluate the impact to the solution’s overall security and make the necessary adjustments.

      Various Work Products the team must pay close attention to include the Operational Model, the Security Architecture, and the Architectural Decisions. The use of identities, access to critical assets, overall solution management, and auditing of these functions must be monitored in order to ensure security is continually present throughout the many refinements that may occur.
