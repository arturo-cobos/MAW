Task: Perform Gap Analysis
Disciplines: Security and Privacy
Purpose
Performing a gap analysis ensures that the architecture is not missing any major components and therefore structurally sound.

      Main Description
      At this point, it is necessary to do a ‘double-check’ to make certain no major portions or functions of the architecture have been overlooked. In order to achieve this, it is imperative that an industry accepted standard be used as a baseline. Standards provide thought out, time-tested guidance for any undertaking. They also save time, ease design and communication, as well as provide baselines for comparison.

      The project team should begin comparing their requirements and subsystems defined against the security model selected in the Solution Outline-Analyze Data-Map Security Processes to a Model. The project team should identify gaps present and the risks associated with those gaps in a table or spreadsheet. The estimated likelihood should be noted, if known. Any gaps that may have been identified due to a client direction of using alternate Subsystems, etc should be brought to the attention of the client and appropriate sign-off or restructuring completed. Finally, the team should list appropriate countermeasures for each risk in accordance with the risk posture of the client.
