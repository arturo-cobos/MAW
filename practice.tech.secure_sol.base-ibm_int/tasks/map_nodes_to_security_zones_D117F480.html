Task: Map Nodes to Security Zones
Disciplines: Security and Privacy
Purpose
Place each Node on the network in its appropriate Zone to protect against loss.

      Main Description
      Now that the team has organized the high level architecture into the Zones and identified where in the network security Subsystems are present, the final mapping task is to designate the Zone for each Node.

      The team should retrace through the topology and consider each Node and its assets. What does the Node contain? Does it have assets that need to be highly protected? Are the Subsystem within the Node sufficient or should it be further protected? Using the mappings from the ‘Map Security Zones to Current Environment’, the team should evaluate each Node and determine if the initial Zone designated is sufficient or if the Node needs to be placed in a different Zone.

      The team should take into consideration the risk of compromise to the asset when placing its Node in a Zone. Again, there should also be a double check to make sure there are no conflicting Zone assignments.

      Additionally, the Actors need to be considered when completing this mapping. On most occasions, the mapping of the actors to the zones, and the information they need to access and its zones, will not be congruent. Where actors and assets are not mapped to the same zone, additional security controls will be needed to appropriately protect the assets. These controls will need to be incorporated into the design of the solution, and documented for future reference.
