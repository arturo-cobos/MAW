Task: Map Subsystems to Nodes
Disciplines: Security and Privacy
Purpose
Mapping the Subsystems to the Nodes identifies security services to a point in the network. Performing this activity can be used for identification, planning and/or placement of technology.

      Main Description
      Security is often a ‘weakest link’ scenario. Any point in a system or environment that is vulnerable and threatened could give rise to the compromise of the entire system. It is therefore crucial for an enterprise to know the breadth and depth of its topology. Reviewing the topology and identifying where security functions are present or are needed provides a baseline and a compass for an organization.

      The team should begin by reviewing the Current IT Environment and the network topology. The project team should have each device, its platform, applications or services, as well as any other pertinent information identified from the Current IT Environment task. The team should trace through each device and point on the topology and map what Subsystems are applicable to that Node. Most Nodes will have more than one Subsystem mapped to them. Not all Nodes on a network are specific security devices (i.e. a firewall, etc); however each Node plays a part in the security of the network.
