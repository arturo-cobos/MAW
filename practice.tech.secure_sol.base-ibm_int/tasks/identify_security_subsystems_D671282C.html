Task: Identify Security Subsystems
Disciplines: Security and Privacy
Purpose
Define the Subsystems that will be used in the architecture. These systems will be used to organize and group the architecture into necessary functions.

      Main Description
      Subsystems are components of the architecture, grouped together by the security function they perform and their behavior. Typically these systems share the same or similar business processes, properties, and Use Cases. MASS currently defines 5 security Subsystems. They are:

      • Access Control
      • Trusted Credentials
      • Information Flow Control
      • Integrity
      • Audit

      It is important to note that each of these Subsystems rely on and work with each other. One component can be mapped to more than one Subsystem, depending on its function.

      The client may already have their own security Subsystems defined. If this is the case, these should be taken into consideration when framing the design of the architecture. An evaluation may need to take place to ensure the clientâ€™s defined Subsystems provide adequate coverage for the architecture.

      Document the Subsystems by including the name of the Subsystem, a description of its purpose, as well as the scope of each Subsystem. Each Subsystem should also have a description of the components that are within it, so there is clear understanding of its coverage.
