Main Description
Before the solution can be rolled out, the project team must ensure it will reside in a secure atmosphere. Otherwise, the system(s) on which the solution resides and communicates with may give way to possible compromise of assets. The various servers, operating systems and processors that make up the deployed solution must be hardened to the client’s approved specification. These specifications should come from the client’s security controls, identified in the Data Gathering activity.

 Additional items include but are not limited to the following:

  • Nodes must be placed within the network in the proper subnets
  • System management functions such as backup and recovery capabilities need to be implemented with the proper security and access rights
  • Load balancing capabilities must be implemented and tested to ensure the appropriate levels of availability are achieved

The project team should ensure hardening has taken place and achieve client sign-off before rolling out the solution to the production environment.
