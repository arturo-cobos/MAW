Deliverable: Prototype (DEL 0518)
This deliverable is the mock-up or working piece of an application that is built early in the projectâ€™s lifecycle with the express purpose of being iteratively changed and improved throughout.
Purpose
An evolutionary prototype and / or technical proof of concept, allow the Agile team to explore user interface design approaches, clarify both functional and usability requirements, and evaluate their usability with end-users.  During an AWD project, it should be developed in parallel to requirements story writing, and architectural design activities.

Main Description

Prototypes may be intended to be throw-away or reusable, dependant on the tools and environments available to the project team during solution definition.  Prototypes intended to be throw-away should be low-cost and quickly developed.  Reusable prototypes should be coded in the same language in which the application will be built, following the coding guidelines specified by the development team.  The High Level User Interface Prototype consists of the following as appropriate based on the needs of the project (from low to high-fidelity):

  • Sketches and drawings
  • User personas that describe details of specific user types
  • Clickable wireframes created in Visio or other similar tools
  • A software simulation built using a rapid prototyping tool appropriate for the target platform (such as iRise, Dreamweaver, or Flash for Web sites, Rational Application Developer for Eclipse Rich Client Applications, or Microsoft Visual Studio for Windows and .NET applications)
  • A proof of concept (define to fit each proposal) to demonstrate a high risk, or high interest function that should be reusable and operational.
