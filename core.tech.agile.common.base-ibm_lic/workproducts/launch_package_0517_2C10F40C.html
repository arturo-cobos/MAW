Deliverable: Launch Package (DEL 0517)
This deliverable consolidates iterative development, testing and launch preparation assets.
Purpose

The build cycle physical package contains:

  • Working application software
  • Test plans, cases, data and results
  • Launch checklist
  • Transition to maintenance plan
Relationships
RolesResponsible: Modified By:
  • Project Manager
TasksInput To: Output From:
  • Crosstrain the Maintenance Team
  • Prepare for Deployment
Main Description
The team continually integrates and maintains working code, and incrementally builds the build cycle physical package throughout the project.
Deliverable Properties
External DescriptionThe Launch Package is a collection of physical artifacts that is intended to be deployed to targeted end users.

The purpose of this deliverable is to provide the client with a package comprised the release application code and installation media that will be distributed for end user installation and the operation and execution of those portions of the solution.

Deliverable Parts
  • Application Maintenance Turnover Definition
  • Build
  • Component Model
  • Data Model
  • Deployment Plan
  • External Data Reference Definition
  • Operational Model
  • Requirements Traceability and Verification Matrix
  • Test Case
  • Test Log
  • Test Script
Packaging InstructionsThis deliverable is comprised of a mix of documentation (which may be hard or soft copy) and electronic media formatted for installation into the client environment. The format of this media is dependent on the technology and tooling used to develop and implement the solution.
