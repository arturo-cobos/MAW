Checklist: Software Configuration Management
This checklist is used as a guideline to ensure that Software Configuration Management on all projects is effective.

Check Items
Applicable audit type selected and relevant audit steps have been followed

1.1 Requirements Baseline Audit

If this audit type is applicable, the Baseline Audit items need to be specified, i.e. Requirement Document, Design Specification, Test Cases, Requirement Traceability Matrix, Change Log.

Verify the following:

  • That person is authorized to make a requirement change.
  • All changes have been authorized by an approved change instrument (CR) and all approved changes have been accurately recorded.
  • Each requirement traces to at least one element in each of Design and Test case.
  • All requirement adhere to CM standards such as naming, numbering, classifications, and relationships.
  • All elements are present in the PCB and are of the appropriate size and quantity.

1.2 Software Baseline Audit

If this audit type is applicable, the Baseline Audit items need to be specified, i.e Defect Log, Change Log, Source Codes, Load Modules, Userâ€™s Guide.

Verify the following:

  • The defects and change requests (CRs) which have been approved and which have been determined as critical to the build are delivered.
  • All files and directories adhere to CM standards such as naming, numbering, coding standards, or build standards.
  • All elements are present in SCM tools and are of the appropriate size and quantity.
  • The available or delivered user documentation versions correspond to system versions.
  • All files which reside in the development baseline under the Library Management System (LMS) are employed in the build.
  • All expected objects in build are produced by build and all expected load modules in build are produced by build.

1.3 System Documentation Baseline Audit

If this audit type is applicable, the Baseline Audit items need to be specified, i.e Change Log, Deliverable Acceptance Records, <Deliverables>

Verify the following:

  • All changes are authorized and all approved changes have been accurately recorded.
  • The delivery artifacts are complete and recorded.
  • All documents adhere to CM standards such as naming, document numbering, document version and history.
  • Correct (latest approved) document version has been published in PCB.
