Task: Develop User Profiles
Determine User characteristics
Disciplines: Technical Requirements Development, User Experience
Purpose
There are many reasons why user profiles are developed. These include 1) guides many of the detailed decisions about design, usability and functionality, 2) serves as the basis for selecting end users to participate in usability tests and evaluations, 3) provides key input to infrastructure design, "data/object" placement decisions, 4) validates use case actors and 5) helps to validate completeness of requirements.

      Main Description
      Identify and describe the intended users of the new system by gathering that information from a variety of sources, including marketing groups, the human resources department, the business process model, and actual users of the intended system. All key elements including relevant demographics, educational background, prior computer experience, physical limitations and disabilities, age, gender, physical location, number, attitudes, motivation, a description of the work environment, and learning style are characterized. Each distinct segment of the target audience requires a separate definition.
