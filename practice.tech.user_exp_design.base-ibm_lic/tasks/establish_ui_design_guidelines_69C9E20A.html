Task: Establish User Interface Design Guidelines
Develop or adopt guidelines on how the user interface shall be designed based on the requirements of the project.
Disciplines: Technical Design, User Experience
Purpose
Establishing user interface guidelines:
  • Increases the Productivity of the Users. Using a similar set of system behaviors within and across solutions and systems, provides consistency and gives the user the advantage of not having to learn a new way of interacting with the system. Instead, the user can concentrate on the new functions in the system.
  • Allows for the Reuse of Common UI Design Elements.  Using a common design approach allows functionality to be reused across applications based on a consistent approach.
  • Establishes a Basis for Performing Quality Control. By having a written set of User Interface Guidelines, it becomes possible to perform an objective quality review on the user interface design that is produced.

      Main Description

      Determine the appropriate industry standard (Windows, Web-based, User Interface Style Guide for IBM Global Industries, OSF/Motif, and so forth). Based on usability requirements and user interface related requirements as gleaned from the inputs to this task, determine whether any tailoring of the appropriate industry standard is necessary.

      At this point the guidelines should focus on providing a high level overview of common design patterns to used to promote consistency within the design that will flow through to the actual implementation.
