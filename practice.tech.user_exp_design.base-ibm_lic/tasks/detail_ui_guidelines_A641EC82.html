Task: Detail User Interface Guidelines
Refine the adopted guidelines on how the user interface shall be implemented on the project.
Disciplines: Technical Design, User Experience
Purpose
Establishing user interface guidelines:
  • Increases the Productivity of the Users. Using a similar "look and feel", within and across solutions and systems, provides consistency and gives the user the advantage of not having to learn a new way of interacting with the system. Instead, the user can concentrate on the new functions in the system.
  • Increases Developer Productivity. By providing guidelines for designing user interfaces, developer productivity rises; mostly because many of design choices and options are already resolved.
  • Establishes a Basis for Performing Quality Control. By having a written set of User Interface Guidelines, it becomes possible to perform an objective quality review on the user interface design that is produced.

      Main Description

      Identify user-interface components that have been customized to meet the specific needs of the solution. Document required and recommended usage of these components by updating the style guideline. Put in place a practice for recording and tracking deviations from the style guide. Track these deviations over the course of the project.

      The steps required to complete this task are:

      • Identify user-interface components that have been customized to meet the specific needs of the solution.
      • Document required and recommended usage of these components in the style guideline.
      • Establish a mechanism for recording and tracking deviations from the style guide.
      • Track these deviations over the course of the project.
