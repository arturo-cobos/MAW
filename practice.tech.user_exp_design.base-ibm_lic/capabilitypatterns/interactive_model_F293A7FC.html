Main Description
The Interactive Mode illustrates the overall approach to a users experience with the application as it relates to navigation, content categories, grouping of functional elements, the logic of information layout, and high-level conceptual models of the application. Like architectural blueprints for a building, these work products provide a organizational framework that influences the technical architecture, and supports the appropriate media to enrich the users experience.

Key Considerations

Interactive Models are more difficult to execute without adequate user identification and requirements:

  • Audience characteristics of the application’s users found in user profiles.
  • Catalog of required content and examples of typical instances.
  • Functional requirements as captured in use cases.
  • Technical limitations of the interface.
  • Input and output requirements for key processes.

For each of the above requirements that the client is unable to specify to the senior information designer’s or the creative director’s satisfaction, add 25% to the estimated hours to allow the Information Design team to formulate assumptions that will replace the actual information. When any of the above information is missing, it is less likely the Interactive Model will meet the client’s needs, and additional iterations with the client’s feedback may be necessary provide a satisfactory work product.

If this is reviewed with user groups and clients, the abstract nature of the work products makes it necessary to carefully explain their meaning to the audience or present them in conjunction with work products that are more easily understood.
