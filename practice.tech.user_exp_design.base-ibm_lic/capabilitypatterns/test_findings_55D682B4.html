Work Product Descriptor (Artifact): Test Findings
This artifact summarizes the analysis of one or more tests, providing a relatively detailed assessment of the quality of the tested items and the status of the test effort.

Main Description

This artifact includes information on the following:

  • Scope of testing - any out-of-scope situations are also documented and refer to the risk management process as documented in the test plans
  • What was done and not done versus plan
  • Test results
  • Findings on:
    • Functions working
    • Functions not working
  • Conclusions
  • Quality of software - planned vs. actual
    • Test metrics 
    • Results
  • Issues outstanding
  • Recommendations and action plan

Additionally, this artifact includes analysis of the testing results, any statistically valid conclusions that can be drawn from these analyses are used to predict the quality level achieved by the tested product and compared to the target level established in the test plans.

Some of the factors used to judge the risk are:

  • Number of defects found
  • Severity levels
  • Status of defects - outstanding, investigated, explained, fixed, retested, accepted, etc.
  • Rate of arrival; i.e., defects found per test executed and/or per tested product component, by test period and level

Key Considerations

In order to be most effective:

  • Be very specific concerning statements of problems and the rationale behind statements. 
  • Review with the engagement team recommended solutions to problems, before they are included in the test report.
  • Write the report so report recipients can understand the test method, procedures, findings, and recommendations. 
  • Ensure that important members of the engagement team have reviewed the Test Report before it is presented to the client. 
  • Include detailed data in appendices, not the body of the report.  This will make reading easier and more focused. 
  • Prepare separate presentation graphics if the test results are to be shared with the report recipients verbally. 
  • Have client and project team sign-off the report on contentious engagements.
