Work Product Descriptor (Artifact): Interactive Concept
An Interactive Concept is the representation of design ideas for an application, presented as a visualization of the interactive processes the user follows.

Key Considerations

Different audiences require different Interactive Concept implementation styles that take into account the audience’s base knowledge and the formality of the situation.  Encourage participation from the whole team.  

The Interactive Concept can also be executed at varying levels of detail appropriate for the relevant stage of development.  Create informal, hand-drawn Interactive Concepts early in the design process to communicate macro design ideas with the entire design team. 

Show only refined Interactive Concepts to clients, making sure to set their expectations for what they will see, and set clear guidelines for the feedback you need in order to move forward.  Design the client’s Interactive Concepts to demonstrate only those aspects of the design for which input is needed.  Make sure the Interactive Concepts use language and imagery the client will understand.  The same criteria apply for Interactive Concepts presented to end users for validation or testing.
