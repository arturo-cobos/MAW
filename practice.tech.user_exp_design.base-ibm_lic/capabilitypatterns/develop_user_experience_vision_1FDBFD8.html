Task Descriptor: Develop User Experience Vision
Provides high-level direction for the UI design through development of a prototype reflecting the solution vision.

Main Description
Review and analyze the Extended Use Case Models to identify use cases that have a significant impact on end users. Build an Object and Action table based on the object-interaction diagrams and analysis object model. Create a list of 8 to 10 alternative user-interface models and their metaphors using brainstorming techniques. Refine this list of models and metaphors to three alternatives based on feasibility of implementation, consistency with existing systems and materials, and strengths and weaknesses of current or competitor solutions.

Prototype and test "model and metaphor fit" by sketching three alternatives, three to five screens deep, using low-fidelity prototyping techniques. Evaluate the prototype with eventual end users of the system with the following questions in mind: Is the prototype meaningful to users? Does it support use cases? Does it meet users' expectations for system behavior?

Based on the results, narrow this list to two alternatives. For each alternative, render three visual treatments three screens to five screens deep. Conduct preference tests with end user to determine which visual treatment is most aesthetically pleasing. Based on these results, narrow to one visual treatment.

With the visual treatment data in hand, further refine the two alternative prototypes and evaluate again with end users. Continue this iteration loop (prototype, evaluate, refine) until there is strong correspondence between the user's mental model and the conceptual model. Assess correspondence of usersâ€™ mental models and the conceptual model by collecting quantitative data on task completion time, task error data, and preference data during usability evaluations with eventual end users of the system serving as evaluation participants. As design direction becomes clearer, shift the effort to building high-fidelity prototypes.
