      Main Description

      Subsequent to fit gap Analysis, the customization of an asset may be necessary. This customization can take the form of configuration, extension or true customization which consists of intrusive changes that are less desirable to an asset. Provide a specification for how the identified gap of the existing assets with the target needs of the project are to be bridged.

      After a fit gap analysis it is often found that the required asset type, such as a business process will have to be modified or customized in order to be useful in the current context. This modification can take on several forms: customization or configuration are two ends of the spectrum in which configuration is least intrusive and customization requires most intrusive changes. Extension is an additive change made to render the asset useful in the given context.
      Key Considerations
      Whenever a new asset type is encountered, Leverage Existing Asset is used and this task is included as part of that capability pattern. Various design alternatives can exist in how to bridge a gap that has been identified.  Budgetary restrictions strategic and tactical alliances and product directions must be taken into account in a realistic fashion.
